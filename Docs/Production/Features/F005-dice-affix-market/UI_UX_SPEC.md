# F005 界面体验规格

状态：已接入待截图验收
功能：F005 骰子词缀与市场改造道具
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md
最后更新：2026-06-11

## 输入决策

- 每颗骰最多 2 前缀 + 2 后缀。
- 首批道具为 `加印石 / 剥印石 / 换印石`，均为全随机。
- 改造道具不强制购买后立即使用；购买后可持有，但只能在市场阶段使用。
- 首版市场不能刷出自带词缀的成品骰。
- 金币来源按左到右流式钱包结算，预览和真实结算必须一致。
- 金币后缀每次出手计算一次，同一条后缀在一次出手中最多触发一次。
- 当前 UI 原型仍为 OnGUI，需尊重 16:9 安全区和当前紧凑桌面工具风格。

## 体验意图

玩家在市场中应能快速判断：

- 这是什么改造道具。
- 当前持有几枚对应改造道具。
- 目标骰是否可用。
- 使用后会发生“随机添加、随机删除、随机替换”的哪类行为。
- 改造结果是什么。
- 当前骰已有哪 2 个前缀和哪 2 个后缀。

进入结算时，玩家应能理解金币为什么在同一次结算里先产生、后消费或转分。

## 信息层级

1. 骰子类型和结果点数仍是第一优先级。
2. 词缀槽作为骰子详情的第二层信息，不抢占主投骰区。
3. 市场道具卡只显示一条短规则，例如“随机加 1 条合法词缀”。
4. 完整词缀说明放在骰子详情或悬浮详情中。
5. 结算日志只显示触发了什么，不展开所有未触发词缀。

## 交互流程

### 加印石

1. 玩家在市场购买或已持有 `加印石`。
2. 玩家在市场阶段从道具栏选择使用 `加印石`。
3. 没有合法空槽的骰子置灰并显示“词缀已满”。
4. 玩家选择合法目标骰。
5. 系统消耗 1 枚 `加印石`，随机决定槽位类型和词缀，并展示结果。
6. 玩家确认后返回市场。

### 剥印石

1. 玩家在市场购买或已持有 `剥印石`。
2. 玩家在市场阶段从道具栏选择使用 `剥印石`。
3. 没有词缀的骰子置灰并显示“没有可剥词缀”。
4. 玩家选择目标骰。
5. 系统消耗 1 枚 `剥印石`，随机删除一条已有词缀，并展示删除结果。
6. 玩家确认后返回市场。

### 换印石

1. 玩家在市场购买或已持有 `换印石`。
2. 玩家在市场阶段从道具栏选择使用 `换印石`。
3. 没有词缀的骰子置灰并显示“没有可替换词缀”。
4. 玩家选择目标骰。
5. 系统消耗 1 枚 `换印石`，随机删除一条已有词缀。
6. 系统按删除后的合法空槽随机添加一条新词缀。
7. UI 同时展示旧词缀和新词缀。

## 反馈和动画

- 改造结果不需要长动画，建议用短促盖印、剥落、换印反馈。
- 新词缀出现时，目标骰卡片对应槽位短暂高亮。
- 删除词缀时，旧词缀短标签淡出或划掉。
- 换印石结果应同时显示“移除”和“新增”，避免玩家误以为只重骰数值。
- 结算中钱包变化使用 `+G` 和 `-G` 浮动反馈，并按左到右顺序出现。
- 若后续对象使用了前面产生的金币，日志应显示“读取当前钱包”或等价短句。

## 边界情况

- 非市场阶段不显示或禁用改造道具使用入口。
- 持有数量为 0 时，道具使用按钮禁用。
- 目标选择中取消使用时不消耗道具。
- 目标骰词缀满时，`加印石`不可用。
- 目标骰没有词缀时，`剥印石`和`换印石`不可用。
- 金币不足购买道具时，道具购买按钮禁用。
- 随机添加时如果抽到重复词缀家族，应重新抽取或从合法池中排除。
- 如果所有合法词缀都已存在，`加印石`应显示不可用，不消耗道具。
- 预览和真实结算出现随机差异的机制必须标记为预估；词缀本身不应出现预览随机差异。

## 适配 / 布局约束

- 维持当前 16:9 安全区。
- 商品卡和骰子详情不要出现大段说明文字。
- 四条词缀建议用两行或两列短标签：前缀区和后缀区分组。
- 移动到正式 UI 前，OnGUI 原型允许先用文本短标签和色块区分前后缀。
- 市场道具栏只服务 F005 三类改造道具，不做复杂通用背包界面。
- 主流程结算反馈不要遮挡骰面点数和当前得分。

## 实现备注

- 当前 `DiceKingDemo.cs` 已优先使用三类最终图标显示改造道具；资源缺失时仍回退到文本 / 程序化色块。
- 目标选择模式复用市场左侧骰袋列表和卖出区域，并通过标题和操作按钮与卖出模式区分。
- UI 已通过短规则和结果反馈表达随机性质，仍需 Unity 运行截图确认文字不溢出。

## 验收

- 玩家能在市场看懂三类改造道具差异。
- 玩家能在市场看到三类道具持有数量，并只能在市场阶段使用。
- 不符合条件的目标骰不可选，且说明原因。
- 取消目标选择不会消耗道具。
- 改造结果能明确显示新增、删除或替换。
- 骰子详情能同时展示 2 前缀 + 2 后缀，文字不溢出。
- 结算预览和真实结算的金币变化、分数变化一致。
- 左到右流式钱包的顺序能被玩家看见。
- 按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 完成截图验收后，才能称为界面验收通过。
