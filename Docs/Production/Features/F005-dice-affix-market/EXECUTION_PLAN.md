## 已确认默认值

- 改造道具不强制购买后立即使用；购买后可持有，但只能在市场阶段使用。
- 首版不刷带词缀成品骰。
- 首版不让词缀改变骰面概率。
- 结算顺序绑定玩家看到的锁定结果展示顺序。
- 每条金币后缀每次出手计算一次，同一条后缀一次出手最多触发一次。
- 首版启用 12 条词缀，其余采纳池后置或 0 权重。
- 猪猪骰、鎏印材质和旧账本金币源统一复查并迁移为直接进入钱包。

## 验证计划

- 市场刷新样本：至少 30 次刷新，确认道具出现率和价格可读。
- 改造样例：分别验证加印石、剥印石、换印石的合法和非法目标。
- 道具持有样例：验证购买后数量增加、保存继续后数量正确、非市场阶段不可使用、取消目标选择不消耗。
- 词缀样例：逐条验证首版 12 条词缀触发、未触发和 T 阶数值。
- 结算样例：验证左到右金币先产后用、后产不回头。
- 经济样例：验证金币后缀、猪猪、鎏印迁移后的钱包变化，并确认没有本关账本残留。
- 存档样例：新游戏、保存、继续、卖出、购买后词缀不丢失。
- UI 样例：按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 做市场和结算截图验收。
