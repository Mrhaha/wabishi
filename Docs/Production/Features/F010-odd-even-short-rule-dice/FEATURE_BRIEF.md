## 功能意图

F010 将奇偶后续改版从“奇偶专属套装”推进为一组短规则机制骰。它们不提供长期成长，不新增奇偶资源，不扩大全局牌型表，而是围绕当前手牌的奇偶关系、槽位相邻和出千结果，给玩家更短、更直接的摆位和出千判断。

本包第一阶段只准备七颗候选的规则卡和接触图需求，已完成接触图门禁。接触图通过后，七颗最终透明图标、程序类型、市场数据、预览和真实结算逻辑已接入当前原型；2026-06-16 已完成 Unity Play Mode 样例验证和截图验收。

## 玩家体验

玩家在市场看到这类骰子时，应能用一句话理解它服务什么判断：看左邻是否同奇偶、是否补全全奇或全偶、是否通过复核重摇补救、是否在出千后追求奇偶变化或保持奇偶。接触图阶段要先证明这些符号能一眼区分，再进入最终图标和程序实现。

## 已确认规则

- 本包范围锁定七颗奇偶短规则骰：`异邻骰 / 同邻骰 / 补全骰 / 复核骰 / 翻号骰 / 守号骰 / 转号骰`。
- 第一阶段只准备规则卡和接触图，不改代码、不写正式市场数据、不生成最终运行时资源；该门禁已随第二版接触图通过而完成。
- 七颗骰的玩家侧市场文案必须是一句话短规则。
- 运行时短标签不得依赖长说明；建议标签包括 `异邻 6`、`同邻 6`、`补全`、`复核`、`翻号 8`、`守号 6`、`转号`。
- 接触图要验证七颗机制是否一眼可分，不要求一次产出最终图标。
- 接触图通过后才允许进入最终透明图标生成和程序实现。

## 规则卡入口

完整规则卡见 `RULE_CARDS.md`。当前规则卡区分“用户已确认短规则”和“建议默认边界”。建议默认边界需要先经 `$wabish-packet-review` 审查，再进入程序实现。

## 建议默认值

- 左邻类骰读取当前骰袋槽位顺序中的左侧相邻实体骰；最左槽没有左邻时不触发。
- 固定分候选沿用 F001-C 的固定分处理口径：先替换本骰基础单骰分，再按有效点数接受遭遇规则修正。
- `补全骰` 只补全全奇 / 全偶副牌型识别，不改变显示点数、有效点数或长期面组。
- `复核骰` 只在自然开盅结果锁定后触发，每颗每次出手最多一次，出千确认后的强制结果不再复核。
- `转号骰` 出千重摇时只从当前面组中相反奇偶的面里抽取；没有相反奇偶面时，该商品不应生成或该效果不可用。

## 已裁决实现问题

- 固定分 `6 / 8` 保留为首版数值，后续只通过调参包调整。
- `异邻骰 / 同邻骰 / 补全骰 / 复核骰` 从第 2 章进入市场池；`翻号骰 / 守号骰 / 转号骰` 从第 3 章进入市场池。
- `复核骰` 在自然开盅后、结算预览前处理；`补全骰` 在牌型识别时处理。若同一手同时存在复核和补全，先复核，再按最终结果判断补全。

## 约束

- 不新增奇偶资源、奇偶计数器或长期成长字段。
- 不改变全奇 / 全偶作为副牌型且不与主牌型叠乘的规则。
- 不新增对子、两对、三对、葫芦、小顺等额外牌型。
- 不改变出千只重摇骰面、不刷新猪猪目标、大树命中点和赌徒阈值的现有规则。
- 不把接触图放入 `Assets/Resources/Art/DiceTypes/`。
