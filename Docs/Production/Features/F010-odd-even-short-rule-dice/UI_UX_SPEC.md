## 输入决策

- 七颗候选必须短规则化。
- 第一阶段不做 UI 实现，只准备市场卡、出千提示和结算短标签；该阶段已完成。
- 接触图已通过，最终图标和程序短标签已接入当前原型，仍需 Play Mode 截图验证。

## 体验意图

玩家不应读长说明来理解奇偶骰。市场卡用一句话说明，结果区和结算演出使用 2 到 4 字短标签加数字。出千相关骰必须让玩家在出千选择时知道“变号有收益”、“守号有收益”或“本骰会强制变号”。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 市场卡 | 判断是否购买 | 图标、骰名、一句话短规则、价格 | 购买 | 刷新、卖出 | 进入骰袋或刷新 |
| 骰袋 / 类型卡 | 识别骰子机制 | 类型图标、短规则 | 调整槽位 | 查看详情 | 投骰准备 |
| 结果预览 | 判断是否结算或出千 | 当前触发标签、预计分、牌型 | 结算 | 进入出千 | 结算或出千选择 |
| 出千选择 | 判断选哪颗重摇 | 出千相关骰的收益 / 控制提示 | 选择 1 到 3 颗 | 取消 | 确认出千或返回预览 |
| 结算演出 | 看懂触发来源 | 短标签、固定分或补全提示 | 等待 / 加速 | 无 | 入账完成 |

## 信息层级

| 骰子 | 市场短规则 | 预览 / 结算标签 | 出千选择提示 |
|---|---|---|---|
| 异邻骰 | 左邻奇偶不同，本骰 6 分 | `异邻 6` | 无特殊提示 |
| 同邻骰 | 左邻奇偶相同，本骰 6 分 | `同邻 6` | 无特殊提示 |
| 补全骰 | 只差本骰时，补全全奇/全偶 | `补全` | 若重摇会破坏或达成补全，预览更新 |
| 复核骰 | 本骰破坏全奇/全偶时，重摇一次 | `复核` | 建议出千后不复核 |
| 翻号骰 | 出千后奇偶变了，8 分 | `翻号 8` | 选中后提示“变号给分” |
| 守号骰 | 出千后奇偶没变，6 分 | `守号 6` | 选中后提示“守号给分” |
| 转号骰 | 出千重摇必变奇偶 | `转号` | 选中后提示“必变奇偶” |

## 交互流程

1. 玩家在市场看到七颗骰中的任意一颗，只读一句短规则即可判断用途。
2. 玩家在投骰前通过槽位调整影响 `异邻骰 / 同邻骰`。
3. 自然结果锁定后，预览显示邻位、补全、复核等已触发状态。
4. 进入出千选择时，`翻号骰 / 守号骰 / 转号骰` 应显示出千相关提示。
5. 确认出千后，预览按新结果更新，结算只显示短标签，不展开长说明。

## 反馈和动画

- 邻位触发：本骰和左邻之间可有短暂连线或同 / 异色闪光。
- 补全触发：全奇 / 全偶提示可显示 `补全` 后再显示最终倍率。
- 复核触发：显示一次重摇反馈，避免看起来像玩家主动出千。
- 翻号 / 守号触发：在出千确认后显示短标签。
- 转号触发：在出千选择和确认后都应明确“必变奇偶”，避免玩家以为只是加分骰。

## 边界情况

- 最左槽的 `异邻骰 / 同邻骰` 没有左邻，不显示触发标签。
- `复核骰` 自动重摇不能和出千界面混淆。
- `转号骰` 如果没有相反奇偶面，出千选择应不可用或取消选中，避免确认后无解。
- 多颗补全或复核同时存在时，先处理自然开盅后的复核，再用最终结果判断补全。

## 适配 / 布局约束

- 市场卡不放长说明。
- 结算标签优先 2 到 4 字加数字。
- 出千选择提示不应遮挡骰面点数。
- 图标和短标签需要在 1280x720 最小实用窗口下可读。

## 实现备注

当前 UI 接入沿用现有 OnGUI 市场卡、骰袋卡、结果预览、出千选择和结算短标签；未新增 UI 系统。

## 阻塞项

无 UI 硬阻塞。当前缺口是 Play Mode 截图验证七颗骰短标签和出千提示在 1280x720 / 1920x1080 下可读。

## 验收

- 七颗市场短规则均可一行展示。
- 七颗结算标签不会变成长说明。
- 三颗出千相关骰在出千界面的差异可被表达。
- 接触图通过后，UI 不需要再解释图标含义才能完成基础识别。
