## 功能验收

- [ ] 主流程不再显示剩余出手。
- [ ] 主流程不再显示剩余出千。
- [ ] 没有出千按钮、出千选择界面或出千确认流程。
- [ ] 小关只进行一次真实结算。
- [ ] 六个实体槽位仍然存在，且待机阶段可调整顺序。
- [ ] 待机阶段槽位调整采用拖拽插入排序，单击只选中，不再用两次点击交换。
- [ ] 拖拽排序有浮起骰、占位和插入标记；松手提交，右键或 `Esc` 取消。
- [ ] 非 `Ready` 阶段不能调整本次结算顺序。
- [ ] 结算按当前槽位从左到右触发和展示。
- [ ] 结算遵守 F015-R01：当前槽位引发的局部连锁队列清空后，才进入下一槽。
- [ ] 前槽连锁不会提前触发后槽自己的结算效果，除非该后槽被当前连锁明确点名为响应。
- [ ] 饲料会提高有效点数下限，且预览和真实结算一致。
- [ ] 饲料状态可保存和读取。
- [ ] 市场买入、卖出、刷新、离开都能产生统一事件。
- [ ] 市场事件每个玩家动作只触发一轮，不递归。
- [ ] 市场骰袋列表可拖拽插入排序；排序只改变骰袋顺序，不改变饲料、成长、卖价或货架商品。
- [ ] 结算演出默认速度足够看清逐槽入账、短标签、金币飞字和最终收束。
- [ ] 设置页可选择 `清晰 / 1.5x / 2.0x` 结算演出速度，且该设置只影响展示时长，不改变计分和事件顺序。
- [ ] 四家族首批骰可在市场出现、购买、保存和结算。
- [ ] 龟龟吸附后宿主显示吸附状态，卖出时卖价合计正确。
- [ ] 不生成新美术资源。

## 反套利验收

- [ ] 贸易猪通过连续买卖不能稳定无限提高饲料。
- [ ] 刷新海盗和海盗头领不能让刷新 / 购买长期无成本循环。
- [ ] 恶魔骰子和小鬼对市场商品的临时最低点不会无限追溯到新补货商品。
- [ ] 吞噬骰子不能吞噬导致负容量、空引用或重复卖出金币。
- [ ] 龟龟吸附不能重复叠加到同一宿主超出首版上限。

## 规则卡验收

- [ ] `RULE_CARDS.md` 已被程序实现、界面表达和验收样例共同引用。
- [ ] 肉猪饲料翻倍不被实现为第二次获得饲料。
- [ ] 多颗骰因同一槽位效果获得饲料时，相关响应在当前局部队列中结清。
- [ ] 三只小猪或同类汇总骰只在主游标到达自身槽位时读取当时状态。
