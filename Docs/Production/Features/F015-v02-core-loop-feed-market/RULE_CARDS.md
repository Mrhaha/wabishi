## 全局规则卡：结算主游标与局部连锁队列

| 项目 | 规则 |
|---|---|
| 规则编号 | F015-R01 |
| 适用范围 | V0.2 及后续所有新增或重做的结算期骰子效果，包括饲料、金币、吸附、临时小骰、成长、有效点、基础单骰分和触发标签。 |
| 核心原则 | 结算使用“左到右主游标 + 当前槽位局部连锁队列”。 |
| 强制性 | 已确认。后续任何结算规则卡、程序实现和验收样例不得绕过本规则。 |

### 执行流程

1. 摇骰结束后，锁定当前六槽顺序、自然点和本次可读状态。
2. 主游标从最左槽开始，按当前六槽顺序向右推进。
3. 主游标到达某槽时，先触发该槽骰子的自身结算效果。
4. 如果该槽效果引发其它响应，响应进入“当前槽位局部连锁队列”。
5. 必须先清空当前局部连锁队列，直到没有新的响应产生，主游标才可以进入下一槽。
6. 局部队列处理期间，如果又产生新的响应，继续加入当前局部队列并按规则清空。
7. 主游标进入下一槽后，才触发下一槽骰子的自身结算效果。

### 队列排序默认值

- 同一时点产生多个响应时，默认按当前六槽从左到右入队。
- 吸附、附属、临时小骰或宿主关系产生的响应，默认跟随其宿主所在槽位处理。
- 某颗骰子如果需要不同排序，必须在该骰自己的规则卡中写明，并通过审查；否则一律使用本默认值。

### 禁止行为

- 不允许先扫完整个骰袋，再统一回放本应属于某个槽位的局部连锁。
- 不允许后侧槽位自己的“结算时”效果因为前侧连锁而提前自发触发；只有被当前连锁明确点名的响应可以提前进入队列。
- 不允许预览写入长期饲料、金币、成长、吸附或存档状态。
- 不允许结算展示层重新调用真实结算、重新随机、重复写钱包或重复入档成长。

### 已确认边界案例

- 肉猪的“饲料效果翻倍”是效果计数翻倍，不视为第二次获得饲料，因此不会因为翻倍本身再次触发“获得饲料时”的响应。
- 如果某槽位的效果让多颗骰子获得饲料，所有由该槽位引发的饲料响应和后续响应都必须在当前局部队列中结清，再进入下一槽。
- 三只小猪这类读取“其它骰子总饲料”的结算骰，只在主游标到达自己槽位时读取当时已经结清的状态；它不会提前读取后续槽位尚未自发结算的收益。

### UI 表达要求

- 结算演出要让玩家感知“当前槽位触发”和“局部连锁追加”是同一组事件。
- 短标签优先显示当前槽位来源，例如 `槽2 养猪`、`连锁 贪吃`、`饲料 +1`。
- 局部连锁结束后，才切换高亮到下一槽，避免玩家误以为全部骰子同时触发。

### 程序验收要求

- 真实结算层需要有可追踪的主游标和局部队列边界。
- 测试样例必须覆盖：前槽触发多颗饲料、肉猪翻倍不二次触发、贪吃猪响应、后槽自身效果不提前触发、三只小猪读取当前已结清状态。
- 展示事件只能消费已提交的队列结果，不得再次改变真实结算。
