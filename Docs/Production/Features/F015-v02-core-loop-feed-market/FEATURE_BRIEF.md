## 功能意图

F015 是 `V0.2` 的第一个实现计划包，目标是把 `V0.1` 的多次出手、出千和结果后补救，改成更短、更容易形成构筑记忆的三段循环：

1. 待机：玩家调整六槽顺序，准备本次结算。
2. 结算：六槽固定，从左到右停转、触发、计分。
3. 市场：买入、卖出、刷新、离开成为构筑发动机，并触发骰子效果。

本包的核心体验不是“少两个按钮”，而是让玩家把注意力放到构筑机器本身：通过饲料提高骰子最低点，用市场动作喂养、献祭、吸附、降价和掠夺，让每次市场处理都改变下一次结算。

## 玩家体验

玩家进入小关时看到六个固定槽位，可以调整顺序。按下投骰后，骰子在原槽位内旋转并从左到右逐颗停下，结算只发生一次；没有出千界面，也没有本关第二、第三次出手。

通关后进入市场。市场不只是买卖补给页，而是第二个核心玩法场：买、卖、刷新、离开都会触发部分骰子。玩家能形成类似“我先卖掉低价值骰触发海盗，再买入猪猪喂养，再刷新找恶魔献祭”的构筑链条。

## 已确认规则

- `V0.2` 首版删除出千。
- `V0.2` 首版删除小关内多次出手。
- 主玩法仍保留六个实体骰槽位。
- 本次投骰结果、展示顺序和结算顺序仍按玩家当前六槽顺序，从左到右处理。
- V0.2 及后续所有新增或重做结算必须遵守 `RULE_CARDS.md` 的 F015-R01：主游标按六槽从左到右推进；每到一槽，必须先清空当前槽位引发的局部连锁队列，才进入下一槽。
- 饲料是首版成长核心，目标是提高骰子最低点，让玩家看到稳定变强。
- 市场是首版构筑发动机，买入、卖出、刷新、离开都可以成为骰子效果触发点。
- 四大家族首版不追求全量实现，每家只选择 2 到 3 颗核心骰验证机制。
- 本包不生产新美术资源，不进入 `$wabish-art-assets`，先复用现有资源和程序化回退。

## 建议默认值

### 饲料定义

- 每颗实体骰新增长期状态 `Feed`，表示该骰获得的饲料点。
- 结算时先掷出自然骰面，再计算有效点数：

```text
有效点数 = max(自然点数, 当前面组最小点数 + Feed + 本次临时最低点修正)
```

- 饲料不直接改写 `Faces` 数组，避免破坏市场面组和存档兼容。
- 饲料可以让有效点数超过 6；显示层在 `7+` 时直接显示居中数字，不再绘制密集点阵。
- V0.2 首版关闭全局牌型计分；饲料读取有效点数下限，直接影响单骰分、猪猪收益和后续家族效果，不再影响三同、顺子、全奇 / 全偶等牌型倍率。

### 市场事件队列

每个玩家市场动作只结算一轮事件队列：

1. 执行玩家动作本身：买入、卖出、刷新或离开。
2. 按当前骰袋槽位从左到右触发“市场动作读取类”骰子。
3. 按需要触发当前货架商品的临时最低点变化。
4. 购买后补货发生在本轮触发完成后；新补出的商品不回头触发本轮事件。
5. 触发效果不能自动再次触发买入、卖出或刷新，防止无限循环。

### 结算事件队列

- 结算期使用 `RULE_CARDS.md` 的 F015-R01 作为全局基本规则。
- 主游标到达某槽后，先触发该槽自身结算效果；该效果产生的所有响应进入当前槽位局部连锁队列。
- 当前局部连锁队列清空前，主游标不得进入下一槽。
- 同一时点产生多个响应时，默认按当前六槽从左到右入队；需要例外排序的骰子必须在单骰规则卡中单独写明。
- 后续槽位自己的“结算时”效果不会因为前槽连锁提前自发触发，除非被当前连锁明确点名为响应。

## 验收含义

F015 通过意味着：玩家不再进行出千和多次出手，而是通过六槽排序、一次左到右结算和市场动作构筑下一次结算；饲料能真实提高最低点；市场买卖刷新能触发清晰且不可递归套利的构筑效果；首批四家族骰能各自证明一个核心机制。

## 2026-07-17 关间市场美术后续补丁

用户在 F015 首版机制已经进入现有原型后，批准了关间市场 V2 三状态样张，并要求继续生产和接入。该批准仅覆盖市场表现层，作为原“本包不新增正式美术”的定向例外，不改写本节之前记录的首版验证背景。

- 视觉基准：午夜旧机厅后台补给柜，左六格骰袋、右三格商品柜、顶部资源窗、底部刷新 / 反馈 / 离开。
- 商品职责：未锁结果时复用家族壳 + 放大类型芯，不显示随机点数；六面与完整效果继续进入悬浮详情。
- 三状态：常态、购买后原货格留空、离场队列锁定并按骰袋顺序扫描 / 结算。
- 规则边界：不新增或修改金币、价格、刷新、购买、卖出、离场效果、随机、存档和结算规则。
- 当前结果：无字底板和 Unity 动态层已接入，双分辨率六张 Play Mode 截图通过；真实鼠标 Tooltip 和完整拖拽手势仍待人工回归。
