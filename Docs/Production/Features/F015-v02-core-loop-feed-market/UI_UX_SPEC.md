状态：已提案，待生产包审查

关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 删除出千和多次出手。
- 保留六槽和左到右结算。
- 结算采用“左到右主游标 + 当前槽位局部连锁队列”。
- 饲料提高最低点。
- 市场动作成为构筑发动机。
- 不生成新美术资源，复用当前 OnGUI、F009 和 F014 底座。

## 体验意图

界面应让玩家清楚理解三件事：

1. 待机阶段：我为什么调整这六个槽位。
2. 结算阶段：谁先触发，谁后触发，最低点如何改变结果。
3. 市场阶段：买、卖、刷新、离开如何改变下一次结算。

## 信息层级

主流程常驻信息按优先级：

1. 目标分和当前分。
2. 当前金币。
3. 六槽骰子、自然点和有效点。
4. 饲料 / 最低点短标。
5. 吸附徽标。
6. 当前槽位局部连锁短标签。
7. 家族短标签和触发飞字。

删除或隐藏：

- 剩余出手。
- 剩余出千。
- 出千按钮和出千选择提示。
- 结果后“是否出千”的二次决策界面。

## 交互流程

### 待机到结算

1. 玩家在待机阶段拖动或点击调整骰子顺序。
2. 玩家按 `Space` 开始投骰。
3. 六骰留在固定槽位内旋转。
4. 按当前 F009 规则从左到右停住。
5. 每颗骰停住后显示自然点，结算时再显示有效点和饲料修正。

### 市场动作触发

每个玩家动作都需要一个短反馈：

- 买入：显示扣金币、入袋、触发哪些骰子、是否补货。
- 卖出：显示加金币、移除骰子、触发哪些骰子。
- 刷新：显示刷新费用、费用减免来源、货架更新。
- 离开：显示离开触发，例如魔蝠或其它后置市场效果。

### 龟龟吸附

首版建议：

1. 购买龟龟家族骰时，如果有合法宿主，出现吸附选择。
2. 玩家选择一颗非龟龟宿主。
3. 确认后龟龟不占六槽，宿主显示小徽标。
4. 卖出宿主时显示宿主卖价 + 吸附龟卖价。
5. 每个宿主最多 1 个吸附龟。

## 反馈和动画

- 饲料获得：短飞字 `饲料 +1`，同时更新最低点短标。
- 最低点生效：结算时显示 `2 -> 4` 这类自然点到有效点变化。
- 局部连锁：当前槽位连锁追加时显示 `连锁` 短标签，连锁结束后才切换下一槽高亮。
- 市场触发：使用短日志行，不做长说明。
- 吸附：用小吸附动画或徽标淡入，不需要新资源。
- 反套利边界：如果效果未触发，应显示简短原因，例如 `本轮已触发` 或 `无合法目标`。

## 边界情况

- 六槽已满时购买非龟龟骰仍禁止或需要先卖出。
- 六槽已满时购买龟龟骰可以进入吸附流程，但每个宿主最多 1 个吸附龟。
- 没有合法宿主时，龟龟不能吸附；是否允许作为普通骰进入六槽由程序实现阶段确认。
- 饲料让有效点超过 6 时，必须直接显示居中数字，不能空白，也不再绘制密集点阵。
- 商品临时最低点只在当前市场货架可见，离开或刷新后失效；如果购买该商品，按规则转为该骰长期饲料或临时状态。

## 适配 / 布局约束

- 继续使用当前 `1280x720` 内部坐标和 `1920x1080` 设计映射。
- 不重做主界面大布局。
- 删除出手 / 出千后释放的左侧资源牌空间优先给金币和饲料摘要。
- 吸附徽标不能遮挡骰面点数。
- 市场触发日志最多展示最近 3 条，避免长列表挤压货架。
