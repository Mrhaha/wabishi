## 依赖关系

- F015-02 依赖 F015-01 的结算路径。
- F015-03 依赖当前市场 UI 和买卖刷新按钮。
- F015-04 依赖 F015-02 和 F015-03。
- F015-05 可和 F015-04 并行设计，但最终截图验收依赖程序实现。
- F015-06 依赖单次结算公式和首批骰强度初值。

## 建议默认值

- 饲料不改自然骰面，只改有效点数下限。
- 结算期使用左到右主游标和当前槽位局部连锁队列；局部队列清空后才进入下一槽。
- 每个宿主最多 1 个吸附龟。
- 单个市场动作只结算一轮触发，不递归。
- 购买补货在本轮触发后发生。
- 单次结算未达标即失败。
- 旧出千相关骰不进入 V0.2 市场池。

## 验证计划

- 静态检查：`git diff --check`。
- 运行验证：Unity Play Mode。
- 截图：主流程待机、结算、市场、吸附选择、失败或通关。
- 存档：新开局、市场后保存、继续游戏、旧版本存档处理。
- 反套利：连续买卖、连续刷新、海盗降价、贸易猪喂养、恶魔商品加点不能形成稳定无限收益。
- 结算队列：前槽触发多颗饲料、肉猪翻倍、贪吃类响应、三只小猪汇总读取和后槽不提前自发触发。
