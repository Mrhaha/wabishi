> 人工验收有几个问题，一个是窗外雨滴错误，放到了游玩机器上，同时窗外雨滴的吸附不符合物理规律。

### 需要修改的结论

- 雨丝和水流线段的起点、终点和旋转必须全部处于 `1280x720` 原型局部坐标，再由统一外层矩阵映射到 Game View。
- 玻璃水珠不得主动向侧方或上方追踪其它水珠；两滴只有在可见轮廓真实接触时才合并。
- 合并后保留下方水珠作为承载体，质量越大向下滑得越快；达到阈值后再转为主汇流。
- 主汇流只吸收其已经过的下游路径上、与水流轮廓真实接触的水珠。

### 不处理 / 后置反馈及原因

- 本轮不调整水珠材质、透明度、窗外雨 V3 的密度与远近速度基准；用户本次反馈只指向空间位置和吸附物理。
- CRT 故障、转场和主游戏机台细化继续后置，不并入本次修复。
