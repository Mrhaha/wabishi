状态：主流程已确认；F021-S2-R3 待回归；LED V4 用于暗色显示面；市场浅色实体键帽 V2 已接入；F021-S5-UI1 样图已通过并静态接入，待 Play Mode 回归

## 输入决策

- 样片覆盖主菜单到下一关 Ready 的一条标准成功局闭环。
- 达标越线后继续结算，直到最终分明显超过目标分。
- 金币效果在计算结束后作为奖励兑现表现触发，不绑定新增规则或指定骰子。
- 胜利后使用轻结算面板。
- 市场首版只演示一次购买和离开。

## 2026-07-16 主游戏共用运行时状态覆盖

- 共用 Run 界面冻结为顶部章节 / 小关、生命、金币；左侧目标 / 当前与积分灯条；中央可排序骰子；右侧单终端；底部唯一阶段化 `Space`。
- 删除 `1/1`、悬空六节点线、槽位结算灯和 `ResultDecision` 总分预览。结果锁定后只显示当前有效点数，确认后才进入左到右结算。
- `Ready` 使用家族壳 + 具体类型中心；`ResultDecision / Scoring` 使用同一壳 + 当前有效数字；三 / 四位数字必须缩放而非裁切。
- 当前结算骰使用轻抬 / 缩放、琥珀描边和暗色 CRT 短反馈，不使用旧浅金实心条。拖拽、悬浮详情和真实结算队列保持既有行为。
- 生命池尚未实现时显示 `生命 --`，不得用评审图中的 `3` 伪造规则。通关 / 失败动态仍不属于本轮实现。

## 2026-07-16 六槽尺寸契约（R3 当前候选）

以下尺寸使用 `1280×720` 内部设计坐标；`1920×1080` 由统一 `1.5×` 画布变换得到，不单独维护第二套值。

| 项目 | 固定规格 | 用途 |
|---|---:|---|
| 六槽中心 | `X=299+128n / Y=368`，`n=0..5` | 底图、骰子、拖拽和结算共用锚点 |
| 有效空槽 / 骰子逻辑框 | `128×128` | 骰壳绘制、命中、旋转和事件定位 |
| 中心距 | `128` | 六槽横向节拍 |
| 有效槽净距 | `0` | 物理槽按底图边框连续排布；透明壳边缘提供视觉分隔 |
| 高亮 / 选中框 | 槽内分别内缩 `1px / 2px` | 当前结算或选中态，不向相邻物理槽外扩 |
| 骰壳公共有效区 | 源图 `(32,27,448,458)` / `512×512` | 去除五张家族壳公共透明边距 |
| 裁切后中心内容框 | `18.6%, 15.3%, 62.9%, 59.2%` | 类型芯、短类型名和当前数字 |
| 单家族预计可见壳 | 宽 `123.1–127.7`、高 `127.4–128` | 贴合底图卡槽边框，同时保留少量家族轮廓差异 |

- R2 的 `112×112` 内槽观感已被用户否定；R3 直接以底图连续卡槽边框的 `128×128` 节拍作为当前正式候选。
- 单家族和双家族必须使用同一个公共 UV；双家族只在裁切后的有效区中线处分成左右两半。
- 高亮、选中、拖拽幽灵和插入标记必须内缩绘制；不允许在 `128×128` 物理槽之外额外扩边。
- 后续替换骰壳资源时，透明有效区必须满足宽高覆盖率至少 `96% / 99%`，否则重新裁切或退回美术修正。

## 体验意图

让玩家感受到一局的核心循环：开始、投骰、揭晓、构筑兑现、超额胜利、奖励收束、市场构筑、再次出发。每个状态只强调一个主焦点，避免同时闪烁、同时弹字或同时解释多个系统。

## 信息层级

1. 当前状态主焦点：投骰、停转、当前入账槽位、最终分、金币奖励或市场购买。
2. 持续可读信息：目标分、当前分、金币、骰槽位置、市场金币和骰袋容量。
3. 短反馈：达标、金币效果、购买成功、离开市场。
4. 后置说明：长规则、路线分析、详细统计、所有骰子效果列表。

## 反馈和动画

- 主菜单转场：账本翻页、印章或镜头推入，不做长剧情。
- 投骰：骰子固定槽位内旋转，不抛投、不重排。
- 停转：从左到右逐颗停住并显点。
- 结算：当前槽位唯一主焦点，其它骰子降噪。
- 越线：短促达标反馈，不立刻打开面板。
- 井喷：越线后分数继续冲高，灯泡、指针、纸带或盖章逐级升级。
- 金币效果：最终分定格后再飞金币，避免分数爽点和奖励爽点混在一起；它只是样片奖励兑现表现，不代表新增规则。
- 轻面板：信息少、停留短、可自然接市场。
- 市场：货架展开、购买骰子飞入骰袋或高亮入袋，金币和容量同步变化。

### 2026-07-14 开始界面氛围补充

- 开始界面专项已改用午夜旧机厅单机台；上方“账本翻页、印章”只保留为历史样片说明，不再作为当前开始界面导演基准。
- 常驻层只有持续窗雨、少量玻璃流滴、极弱 CRT 扫描线 / 噪点和稳定菜单焦点。
- 第一项已实现的循环层为左上吊灯局部线路瞬断：快速熄灭、短暂停黑、灯芯—近场光晕—环境反射分层恢复；不是整屏断电，也不在单次事件内追加抖闪。循环使用随机间隔，避免机械固定节拍。CRT 轻失焦和局部水平同步错位仍后置，未来接入时必须和吊灯事件互斥。
- 菜单首次出现后默认等待 `1.00s` 再触发；焦点变化后的 `0.50s`、确认 / 设置弹层和转场期间暂停异常，保证输入状态清楚。`loop_enabled=0` 时每次进入主菜单只演示一次。
- 焦点切换建议为 120 到 160 毫秒的游标平移和描边交叉淡入；确认键下压 70 到 100 毫秒，选项锁定 180 到 240 毫秒。故障不能被误读为选中、禁用或确认反馈。
- 当前设置页已提供“开始界面吊灯屏闪”开关并通过 `PlayerPrefs` 保存；关闭后吊灯保持常亮。右窗雨水已经接入，但首轮只通过 `main_menu_rain_profile.asset` 的 Inspector 总开关与参数调节，不新增玩家设置或存档字段；CRT 干扰仍后置。
- 右窗雨水不响应菜单焦点，也不作为禁用、确认或覆盖警告反馈。窗外远 / 近雨持续向下；玻璃水珠随机撞击、附着、聚合、阈值触流和淡出持续循环。离开 `MainMenu` 即停止更新，返回主菜单时按 `LockPreviewSeed` 决定复现或更换序列。
- 当前开始界面支持鼠标热区、方向键和 `Space / Enter` 共用焦点。无存档默认开始新游戏，有存档默认继续游戏；开始新游戏在有存档时保留二次覆盖确认。任意焦点输入都会触发至少 `input_protection_duration` 的吊灯保护窗口。

## 适配 / 布局约束

- 首版以 `1920x1080` 横版设计。
- 后续正式实现仍需检查 `1280x720`。
- 关键数字不能被金币、盖章、灯泡或飞字遮挡。
- 轻结算面板不能遮住仍在播放的核心胜利动作；应在最终定格之后出现。
- LED 字体清晰度必须在目标分辨率的 1:1 原始裁切中验收；任何缩略总览、聊天窗口自动缩放或非整数插值都不能作为通过依据。
- 文字核心必须先以无辉光状态可读；辉光关闭后字形不完整或明显变细，视为字体核心失败。
- 像素清晰度、字形识别、基线排版和区域装入是四个独立门禁。V2 清晰度、V3 字形身份和 V4 统一基线均已获用户通过；正式运行排版已经按真实 Rect 完成离线双分辨率装入检查。
- 已通过的 V3 字形固定保留 `16×16` 中文结构与活动灯珠，不得为修正排版重新换字形、缩字或增删笔画。
- V4 中中文、英文大写和数字共享逻辑第 `14` 行底线；短横与间隔点使用固定中线。1:1 混排必须覆盖章节数字、生命 / 金币、目标 / 当前、`SPACE 投掷` 和 `ABC 0123456789 中文混排`。
- 可读且同线的字形不能为旧框再次缩成 `7×7–10×10`。运行时按真实 Rect 从 `Display` 向 `Compact / Micro` 逐档装入，不允许连续缩放或滤波；如果 `Micro` 仍装不下，应修改排版或文案，不再降低字形结构。

## 2026-07-17 V4 全局文字交互合同

- 没有特别说明时，暗色玻璃、终端、资源窗、发光按钮、提示、结果页和旧回退 UI 的文字使用同一 V4 字形与基线；浅色实体键帽按下述 `PhysicalKeyLabel` 材质例外处理。
- 三档语义样式为 `Display`（顶部主信息、主操作、大结果）、`Compact`（常规标签、终端标题、按钮）和 `Micro`（既有小型辅助行）；样式只改变整数灯珠密度，不改变字形身份与锚点。
- 主菜单底图的旧平滑菜单文案必须由不透明动态内容层完全覆盖，不能和新点阵字重叠；焦点、禁用、覆盖确认等状态只改变颜色 / 面板，不更换字体系统。
- 骰面当前点数保留实体压印字，顶部 `DICE KING` 保留实体点阵招牌；市场浅色实体键帽使用 `PhysicalKeyLabel` 实心深墨字。三者都不是可复用的普通 V4 LED 文字样式。
- 关间市场本轮只改变文字渲染；货架布局、购买、刷新、离开和其它市场交互不随本合同调整。

## 2026-07-17 LED 语义亮度层级 V1（分级认可、同级亮度需修订）

### 状态变化

- `Idle available`：保持角色基准亮度，不再为了“未选中”压到看不见。
- `Hover / Focus`：角色升一级或切换焦点色，加边框 / 轻脉冲；不换字形、不改基线、不缩放灯珠。
- `Pressed`：`70–100ms` 实体键下压和轻微亮度收束，文字硬核心不熄灭。
- `Disabled`：同字形、同尺寸，颜色去饱和并改变面板状态；文字视觉亮度不得低于启用态约 `60%`，且必须通过 `3:1`。
- `Feedback`：青白 / 锈红只在 `160–240ms` 局部反馈或结果态使用，结束后回到原角色。

## 2026-07-22 F021-S5-UI1 按钮与提示统一规格

### 按钮家族与状态

按钮家族只决定基础材质；状态语义决定唯一表现。浅色实体动作键统一覆盖卖出、三枚购买和离开市场；暗色刷新键保留 LED 材质，但使用平行状态语义。

| 状态 | 可见表现 | 输入行为 | 后续反馈 |
|---|---|---|---|
| 可用常态 | 原浅米色纹理、正常饱和度、凸起、深墨实心字 | 可执行动作 | 成功后调用短时提示 |
| 悬停 / 焦点 | 文字与底色不变，统一增加克制暖亮外沿 | 可执行动作 | 不提前显示结果 |
| 按下 | 键帽与文字下移 `1–2 px`，材质短促压暗 | 释放时执行一次动作 | 成功或状态变化由结果回调决定 |
| 受阻常态 | 整键压暗、降饱和、取消通电边光、轻微内陷 | 不执行动作，但保留说明命中 | 等待点击解释原因 |
| 受阻点击 | 与受阻常态同外观，短促下压后小幅回弹 | 不执行交易、不扣资源、不改货架 | 在全画面中间偏上播放原因提示 |

同一状态不得因商品、位置或受阻原因更换字色、亮度、凹陷、边光或动画。三枚购买键各自读取真实资格；血契船长满袋仍可购买时必须进入统一可用态。
