# F013 界面体验规格

状态：软关闭 UI 已静态接入，待截图验收
功能：F013 骰子重平衡与词缀临时隐藏
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md、Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md

## 输入决策

- 词缀和改造道具暂时从玩家可见体验中隐藏。
- 隐藏不是教程锁定，不需要显示“暂未开放”。
- 市场和主流程仍使用当前 OnGUI 原型，不做 UI 框架迁移。

## 体验意图

减少玩家在当前平衡阶段的决策维度。市场回到“买什么骰子、卖什么骰子、要不要刷新、是否离开”的核心循环；主流程回到“点数、牌型、骰子类型效果、材质、金币和槽位”的核心阅读。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 关间市场 | 买卖骰子并准备下一关 | 金币、骰袋容量、刷新费用、3 个骰子货架、当前骰袋、卖价 | 购买骰子、卖出骰子、离开市场 | 刷新货架 | 下一小关或章节市场 |
| 章节市场 | 在 Boss 后调整构筑 | 同关间市场，但刷新费用按章节市场读取 | 购买骰子、卖出骰子、离开市场 | 刷新货架 | 下一章或胜利 |
| 骰袋列表 | 查看当前骰子 | 骰子名、类型、面组、材质或短规则 | 选择骰子查看卖价 | 无 | 继续市场操作 |
| 结果决策 | 判断是否结算或出千 | 点数、牌型、倍率、本次预览、骰子类型短反馈 | 结算 | 出千 | 结算动画 |
| 结算展示 | 理解分数和金币来源 | 左到右单骰入账、倍率、金币飞字、成长或类型触发 | 等待播放完成 | 无 | 通关或继续投骰 |

## 信息层级

1. 目标分、当前分、金币、出手和出千。
2. 当前骰子点数、类型身份和牌型倍率。
3. 市场商品的类型、面组、材质、价格和短提示。
4. 结算中已经确认生效的类型、材质、金币和成长反馈。
5. 词缀和改造道具在隐藏期间不进入信息层级。

## 交互流程

- 市场不显示改造道具货架。
- 市场不显示改造道具栏。
- 市场不进入“选择改造目标”状态。
- 点击骰袋中的骰子只用于选择卖出目标，不触发改造逻辑。
- 如果旧存档中有改造道具数量，隐藏期间不显示数量，也不能使用。
- 如果旧存档中有词缀，隐藏期间不在骰子卡、结果骰、结算日志或短标签中展示。

## 反馈和动画

- 不新增隐藏提示动画。
- 不新增“词缀关闭”弹窗。
- 刷新、购买、卖出、离开市场沿用当前反馈。
- 结算反馈中不出现“词缀 +N”“顶账 +G”等词缀文案。

## 边界情况

- 骰袋满时，只提示需要卖出骰子后再买；不再提示“仍可买改造道具”。
- 货架生成失败时应回退为骰子商品，不回退为改造道具。
- 旧存档若保存了 `activeCraftingItemKey` 等运行期状态，进入市场时应清空。
- 关闭状态下，旧存档中的词缀不应造成看不到原因的分数或金币变化。

## 适配 / 布局约束

- 保持当前 `1280x720` 内部坐标和 `1920x1080` 设计映射。
- 隐藏改造道具栏后，底部区域可以用于市场反馈或留白，但不能出现空的道具框。
- 文字不得溢出按钮或卡片。
- 主流程 UI 修改后仍按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 进行截图验收。
