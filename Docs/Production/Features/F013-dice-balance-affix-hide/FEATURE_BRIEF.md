## 功能意图

F013 先把当前原型从“持续扩张新系统”切回“可调平衡基线”。在 F008 金币收益族、F010 奇偶短规则骰和 F011 大树长期成长接口骰接入后，关卡目标、经济、市场价格、刷新权重和骰子强度需要统一重平衡。

同时，F005 词缀和改造道具暂时从玩家可见体验中隐藏。隐藏不是废弃 F005，而是把它从当前平衡变量中移出：不展示、不出售、不使用、不触发，但保留资源、配置和存档兼容，等待核心骰子数值稳定后再决定是否恢复。

## 玩家体验

玩家进入市场时，主要判断骰子类型、面组、材质、价格、当前金币和骰袋容量；不会看到改造道具货架、改造道具栏、词缀槽位或词缀短标签。进入小关后，分数和金币来自骰子类型、材质、面组、关卡规则和已确认的金币流，不再被隐藏词缀影响。

重平衡完成后，玩家应在前 1 到 3 章逐步形成路线：第 1 章能理解基础牌型和市场，第 2 章开始买入路线骰，第 3 章后再接触更高阶的成长、金币和出千联动。

## 已确认规则

- F013 先产出生产包，不直接改变代码、CSV 或资源。
- 当前实现事实仍以 `Assets/Scripts/DiceKingDemo.cs` 为准。
- 当前 F005 词缀和改造道具已经接入原型，但尚未作为已验收体验继续扩张。
- 本轮重平衡不把词缀作为核心变量。
- F013 不删除 F005 历史生产包、资源、CSV 和存档字段。
- F013 不新增骰子类型，不重做 F008、F010、F011 的机制规则。
- 词缀隐藏期间不显示也不生效：前缀不增加分数，后缀不增加金币。
- 旧存档中的词缀和三类改造道具数量保留，但隐藏期间不展示、不允许使用、不参与结算。
- F013 首轮无词缀数值候选已接入：`starting_gold = 8`，`stage_clear_base_gold = 3`，`market_test_random_refresh = 0`。
- 目标曲线已按用户要求改为平滑梯度版：前三章目标为第一章 `60 / 74 / 70 / 70`，第二章 `78 / 70 / 86 / 82`，第三章 `100 / 88 / 82 / 94`；按“6 基础公平骰 + 3 次出手 + 不出千”的单关口径，通过率从约 `95.43%` 降到 `51.62%`，每个小关仍高于 `50%`。第三章后目标平滑递增，第四章 `104 / 115 / 127 / 140`，第十章 `1141 / 1261 / 1393 / 1539`。
- 市场刷新二次调整已接入：`Basic` 在 `dice_market_config.csv` 中权重为 `0 / 0 / 0`，代码层也排除基础骰作为随机市场商品；F008 金币收益族、F010 奇偶短规则骰和 F011 新大树接口骰的开放后权重已提高；付费刷新优先避开上一批货架中的骰子类型，候选不足时回退正常池。

## 已接受默认值

- 增加或等价实现全局开关：`affix_feature_enabled = 0`。
- 开关关闭时：
  - 市场不会生成 `CraftingItem` 类型货架。
  - 市场不显示改造道具栏、目标选择面板和“骰袋已满：仍可买改造道具”提示。
  - 三类道具数量仍保存和读取，但不展示、不允许使用。
  - 骰袋、商品卡、结果骰和结算日志不显示词缀槽位或词缀短标签。
  - 前缀不增加单骰分，后缀不增加钱包金币。
  - 已有存档中的词缀和道具数量保留，等待后续恢复包决定是否重新启用。

## 建议默认值

- 重平衡默认在 `market_test_random_refresh = 0` 的正式市场规则下验证；测试随机只用于特定骰子刷出排查，不作为最终平衡样本。
- 正式起手金币候选值由 F013 执行阶段先给一版数据建议，再由用户验收，不在审查阶段阻塞程序执行。

## 职能交接

### 美术

本包不需要新资源。三类改造道具图标保留在 `Assets/Resources/Art/Items/`，隐藏期间不在运行时市场中展示，也不从 `ART_ASSETS.md` 删除。

### 界面体验

需要定义关闭词缀后的市场和骰袋显示。重点是删除玩家可见的词缀入口，而不是用“暂未开放”文案继续提醒玩家词缀存在。

### 程序

需要在当前单文件原型中做受控改动：增加或等价实现词缀开关，保护市场生成、购买、使用、展示、预览、真实结算、钱包收益和保存恢复。重平衡数据改动应集中在 CSV 和少量配置常量，不重写结算架构。

## 验收含义

F013 通过意味着：当前可玩原型中，玩家不再看到或使用词缀与改造道具，旧存档兼容不报错，隐藏词缀不会影响分数或金币；同时，核心骰子、市场和关卡目标有一套可继续 playtest 的无词缀平衡候选。
