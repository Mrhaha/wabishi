## 功能意图

F009 解决当前骰子表现仍偏 UI 图片化的问题。用户明确希望看到骰子的入场、摇晃、反转、翻滚、停转显点和结算点名，而不是只做位移、缩放、抖动这类简单形变。

本包目标是新增一层只读骰子过程表现：它把当前真实骰面和结算事件演出来，但不成为新的规则来源。首版优先建立程序可验证的序列帧式表现层，让骰子像桌面实体物件一样进入、在固定槽位内播放旋转帧、从左到右停转显点和被点名；最终美术资源和完整专属特效后置。

2026-07-21 已确认并接入的摇骰目标覆盖旧旋转资源和主动结算：第一次 `Space` 后，六个 `128×128` 物理槽继续复用 F021 家族骰壳，骰体、轮廓、投影、坐标和缩放全程不动；只有正面裁切窗内的通用 `1–6` 面值带像 `777` 滚轴一样纵向高速滚动、减速，并从左到右接入真实结果。结果格下冲 `7 px`、向上回勾 `2.5 px` 后停中，短暂读点后自动进入真实结算；加力窗口首版为 `0`。2026-07-22 用户继续确认 V6B-R5 并授权接入；当前结算层已实现原槽聚焦、单枚积分光晕、积分塔承击和隐藏五档，并补入真实来源与回环触发语义。

## 玩家体验

玩家进入小关时，六颗实体骰按当前槽位轻落到桌面投掷区，并显示各自骰子类型身份。按一次 `Space` 后，骰子在自身格内完成统一旋转并从左到右吸附到真实结果；随后自动进入结算。当前结算骰保持原位并被低幅聚焦，一枚简化积分光晕从骰面进入积分塔；贡献越大，光晕越凝实、塔体填充与回弹越强。玩家的注意力持续停留在本关得分增长，不被骰子本体的大幅动作带走。

玩家应感到“我真的掷出了骰子”，同时仍然理解结果来自当前随机、出千和骰子规则，而不是来自动画物理。

## 已确认规则

- F009 是表现层生产包，不是玩法规则生产包。
- 首版必须表现为骰子的入场、槽位内序列帧旋转、左到右停转显点和结算点名。
- 表现层只读取当前骰袋、槽位顺序、投骰阶段、真实结果和结算展示事件。
- 表现层不得调用或替代 `ScoreDice()`。
- 表现层不得重复随机骰面、重复写钱包、重复应用临时小骰或大树成长。
- 最终点数显示必须与真实结果、结果预览和结算结果一致。
- 停转后继续保持投骰前槽位顺序，不按点数重排。
- 禁用或加载失败 F009 表现层时，游戏应能回退到现有 2D 骰子结果展示。

## 阻塞项

当前没有阻塞生产包生成的问题。若用户不接受当前 OnGUI 序列帧播放器的首版观感，再转入正式帧图资源、真实 3D 相机或渲染纹理方案。

## 约束

- 不改变 `RollPhase` 的玩家可见流程语义。
- 不改变真实玩法边界：加力窗口结束后才锁定本次结果，动画只读该结果并在结算前展示。
- 不改变 `SaveVersion`、`PlayerPrefs` 和任何骰子数据序列化。
- 不把动画物理结果写回骰面。
- 不让视觉遮挡目标分、当前分、剩余出手、出千和主提示。
- 不用长教程解释动画。

## 非目标 / 后置范围

- 完整重做主流程为 3D 桌面。
- 最终生产级骰子模型、材质和类型专属动效。
- 全套音效、桌面震动、手柄震动。
- `Space` 加速结算和复杂跳过系统。
- 让物理碰撞决定结果。
- 新增玩法奖励、概率操控、额外金币或额外计分。

## 职能交接

### 美术

首版美术不产出最终运行时资源，先锁定视觉方向和资源清单。若程序验证通过，再按 `ART_BRIEF.md` 进入 `$wabish-art-production` 或 `$wabish-art-assets`。

### 界面体验

界面体验负责四个状态的焦点和遮挡边界：入场看类型和槽位、投骰看槽位内序列帧旋转、停转看从左到右显点、结算看当前点名骰和左侧积分。任何动画都不能破坏当前 F007 主流程信息层级。

### 程序

程序负责只读表现层、结果吸附、2D 回退和验证工具。程序不得重写真实计分和随机，不得把视觉状态写入存档。

## 不要改动

- 不改 `ScoreDice()` 的真实结算。
- 不改骰子类型规则、材质规则、词缀规则和市场规则。
- 不改目标分、金币、出手、出千数量。
- 不改 F006 / F007 已有结算提交态。
- 不把 F009 做成完整 UI 重构。

## 验收含义

F009 验收通过意味着：玩家能看到骰子作为实体桌面物件经历入场、固定槽位内序列帧旋转、左到右停转显点和结算点名；动画最终点数、槽位顺序和真实结算一致；普通投骰不显著拖慢节奏；禁用表现层后主流程仍可玩；表现放大真实决策而不是替代真实决策。
