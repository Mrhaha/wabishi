# F009 界面体验规格

状态：已接入统一资源首版，待运行验证
功能：F009 骰子过程动画表现
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- F009 覆盖骰子入场、槽位内序列帧旋转、左到右停转显点和结算点名。
- 动画只影响表现，不影响概率、出千、计分、金币、成长或存档。
- 停转后保持投骰前槽位顺序，不按点数重排。
- `Ready` 和结算后重置阶段优先显示统一待机骰，不套额外骰框；非基础骰叠加小类型标记。
- 结算点名沿用 F006 / F007 的左到右结算和双数字计分节奏，不再额外插入结果定格停顿。

## 体验意图

界面体验要让玩家明确感知四件事：骰子真实进入桌面、投骰过程有实体运动、结果落定可信、结算归因清楚。动画不能抢走玩法信息，也不能让玩家误以为能通过翻滚或连打操控结果。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 小关入场 | 识别本关骰子和槽位 | 六颗骰子按槽位落桌，目标分和资源保持可读 | 无 | 无 | 进入 `Ready` |
| `Ready` | 准备投骰或调整槽位 | 槽位稳定、统一待机骰直显、非基础骰小类型标记、底部短提示 | `Space` 起转 | 槽位调整 | `Shaking` |
| `Shaking` | 感受投骰加力 | 骰子固定在槽位内播放旋转序列帧、运动期不显示可读点数、提示为“敲 Space 加速” | 连打 `Space` 加强表现 | 无 | `Stopping` |
| `Stopping` / 停转 | 等待结果落定 | 骰子在原槽位内从左到右逐步停下、吸附到真实点数、提示为“停转显点” | 无；额外 `Space` 只轻提示 | 无 | `ResultDecision` |
| `ResultDecision` | 判断结算或出千 | 最终点数、类型身份、牌型提示、结算 / 出千动作 | `Space` 或按钮结算 | 进入出千 | `Scoring` 或 `CheatEdit` |
| `CheatEdit` | 选择重摇骰 | 被选骰高亮，未选骰稳定 | 选择 1 到 3 颗并确认 | 取消出千 | `Scoring` 或 `ResultDecision` |
| `Scoring` | 看懂入账来源 | 当前点名骰、短标签、基础分、倍率、当前积分跳涨 | 无 | 后续若加速，另行确认 | `StageClear` / `Ready` / `StageFailed` |

## 信息层级

1. 当前骰子结果和槽位。
2. 当前阶段提示。
3. 左侧基础分、倍率、当前积分和目标。
4. 结算点名短反馈。
5. 右侧规则和牌型表入口。
6. 日志和低价值细节。

同一时刻只突出一个主焦点。投骰期焦点在固定槽位内的序列帧旋转；停转期焦点在左到右显点；结算期焦点在当前点名骰和左侧积分跳涨。

## 交互流程

1. 小关开始后播放短入场，骰子按当前槽位落桌。
2. 玩家在 `Ready` 阶段可以按现有规则调整槽位。
3. 玩家按 `Space` 起转，F009 视觉层开始槽位内序列帧旋转过程。
4. 加力窗口内的 `Space` 只改变表现强度，不改变真实结果。
5. 停转过程中，视觉层按槽位从左到右把每颗骰子定格到真实结果。
6. 玩家在结果决策阶段结算或出千。
7. 若出千重摇纳入后续切片，被选骰播放重摇翻滚，未选骰保持稳定。
8. 结算阶段不播放额外结果定格，直接按原槽位高亮、轻抬和分数飞字点名骰子并推进左侧计分。
9. 若未达标且还有出手机会，结算完成后清掉点面结果，回到 `Ready` 的统一待机骰展示。

## 反馈和动画

| 节点 | 反馈 | 验收重点 |
|---|---|---|
| 入场 | 骰子在固定槽位轻落位，带短弹跳和接触影 | 能看出实体落桌，不遮挡目标和资源 |
| 起转 | 骰子在自身槽位内开始播放旋转序列帧 | 第一下响应明确，不离槽、不抛投 |
| 加力 | 连打增强槽位内序列帧速度、桌面暗示和提示脉冲 | 不显示收益数字，不暗示概率变化 |
| 停转 | 骰子在原槽位内从左到右逐步停下，停住的槽位立刻显示真实点数 | 等待不拖沓，未停住的槽位不能提前暴露最终点数 |
| 显点 | 骰子吸附到真实点数和槽位 | 最终点数与真实结果一致，停住后不再晃动 |
| 结果决策 | 骰子稳定可读，类型身份和状态标签清晰 | 不晃动到影响阅读，不额外定格拖节奏 |
| 结算点名 | 当前骰按原槽位轻抬或高亮，短标签出现 | 与左侧计分跳涨同步 |

## 边界情况

- 表现层关闭或加载失败时，回退到现有 2D 结果展示。
- 窗口尺寸变化时，骰子仍限制在主投掷区安全范围。
- 非基础骰、材质、成长、赌徒阈值、猪猪目标等状态标签不得被 3D 骰子遮住。
- 临时小骰仍按 F006 折叠规则处理，不因 F009 强行逐颗拖长。
- 失败、未达标继续出手和通关收束必须保持可区分。

## 适配 / 布局约束

- 以 `1920x1080`、`16:9` 为正式基准。
- 最小实用窗口按 `1280x720` 检查。
- 额外空间可用于背景延展，不能挤压主投掷区。
- F009 动画不得遮挡左侧积分塔和资源牌。
- 所有阶段提示仍保持短句，不新增教程墙。

## 实现备注

- 首版默认使用 OnGUI 序列帧播放器，不引入骰盅主运动、抛投或真实物理模拟。
- 视觉层需要与现有 `RollPhase` 对齐，但不要改变玩家可见流程。
- 若当前程序帧观感不足，后续再替换为正式帧图资源、真实 3D 相机或渲染纹理。

## 验收

- 小关入场、投骰、停转显点和结算四段焦点清楚。
- `Ready` 和结算后重置阶段显示统一待机骰，不显示额外骰框；非基础骰身份可读。
- 最终点数、槽位顺序和真实结果一致。
- 玩家能清楚区分投骰过程和结果决策。
- 结算开始后直接进入槽位点名，不出现额外结果定格停顿。
- 连打只改变表现，不暗示收益或概率。
- `1280x720` 与 `1920x1080` 下没有重要信息遮挡。
- Play Mode 录屏中普通一手不显著拖慢节奏。
