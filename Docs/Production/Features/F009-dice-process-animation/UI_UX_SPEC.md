# F009 界面体验规格

状态：摇骰 V4R6 与结算 V6B-R5 已完成 Unity 首版接入；来源 / 回环触发语义修正已通过双分辨率自动运行验证，待用户连续实机回归
功能：F009 骰子过程动画表现
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- F009 覆盖骰子入场、槽位内序列帧旋转、左到右停转显点和结算点名。
- 动画只影响表现，不影响概率、出千、计分、金币、成长或存档。
- 停转后保持投骰前槽位顺序，不按点数重排。
- `Ready` 和结算后重置阶段优先显示统一待机骰，不套额外骰框；非基础骰叠加小类型标记。
- 结算点名沿用 F006 / F007 的左到右结算和双数字计分节奏，不再额外插入结果定格停顿。

## 2026-07-21 最新覆盖规则

以下规则覆盖本文件后文仍保留的旧 `加力 / ResultDecision 主动确认 / 分数飞字`描述；旧内容只用于追溯当前已接入原型，不再代表目标体验：

- 只保留第一次 `Space` 投骰。加力窗口仍可配置，首版目标值为 `0`；结果锁定后不再等待第二次主动结算。
- `ResultDecision` 若继续保留为代码阶段，只能是短暂、不可交互的结果交接态，结束后自动进入 `Scoring`。
- 结算不画骰子之间的关系。每个当前物理槽的真实点数和本槽局部连锁聚合后，由当前骰面析出一枚小型积分光晕，直接进入左侧本关得分 / 积分塔；骰子本体保持原槽。
- 本关得分是唯一全局计分反馈锚点；右侧终端只能做低强度回声，骰台不得出现共享灯路、槽间接力或贯穿骰列的冲击。
- 精确分数保留为事实信息，但只在积分光晕被吸收时低关注度更新，不使用逐颗大号 `+N`、滚动计数、比例或档位名承担高潮。
- 五档强弱只改变积分塔的吸收、目标锁扣、回弹、余辉和暴击承击，不改变玩法收益。
- V5R1 的实体骰离槽、撞塔、空槽和回程已退回。V5R2 中六骰全程原位；当前骰只做低幅聚焦，并析出远小于骰体的点阵光晕；其它五骰全程不响应。
- 单骰贡献使用连续局部重量，整手未达 / 过关 / 超过 / 远超过 / 暴击使用独立的最终结果层；两层都不向玩家显示数值档位。

## 体验意图

界面体验要让玩家明确感知四件事：骰子真实进入桌面、投骰过程有实体运动、结果落定可信、结算归因清楚。动画不能抢走玩法信息，也不能让玩家误以为能通过翻滚或连打操控结果。

## 信息层级

1. 当前骰子结果和槽位。
2. 当前阶段提示。
3. 左侧基础分、倍率、当前积分和目标。
4. 结算点名短反馈。
5. 右侧规则和牌型表入口。
6. 日志和低价值细节。

同一时刻只突出一个主焦点。投骰期焦点在固定槽位内的序列帧旋转；停转期焦点在左到右显点；结算期焦点在当前点名骰和左侧积分跳涨。

## 交互流程

1. 小关开始后播放短入场，骰子按当前槽位落桌。
2. 玩家在 `Ready` 阶段可以按现有规则调整槽位。
3. 玩家按 `Space` 起转，F009 视觉层开始槽位内序列帧旋转过程。
4. 加力窗口内的 `Space` 只改变表现强度，不改变真实结果。
5. 停转过程中，视觉层按槽位从左到右把每颗骰子定格到真实结果。
6. 玩家在结果决策阶段结算或出千。
7. 若出千重摇纳入后续切片，被选骰播放重摇翻滚，未选骰保持稳定。
8. 结算阶段不播放额外结果定格，直接按原槽位高亮、轻抬和分数飞字点名骰子并推进左侧计分。
9. 若未达标且还有出手机会，结算完成后清掉点面结果，回到 `Ready` 的统一待机骰展示。

## 反馈和动画

| 节点 | 反馈 | 验收重点 |
|---|---|---|
| 入场 | 骰子在固定槽位轻落位，带短弹跳和接触影 | 能看出实体落桌，不遮挡目标和资源 |
| 起转 | 骰子在自身槽位内开始播放旋转序列帧 | 第一下响应明确，不离槽、不抛投 |
| 加力 | 连打增强槽位内序列帧速度、桌面暗示和提示脉冲 | 不显示收益数字，不暗示概率变化 |
| 停转 | 骰子在原槽位内从左到右逐步停下，停住的槽位立刻显示真实点数 | 等待不拖沓，未停住的槽位不能提前暴露最终点数 |
| 显点 | 骰子吸附到真实点数和槽位 | 最终点数与真实结果一致，停住后不再晃动 |
| 结果决策 | 骰子稳定可读，类型身份和状态标签清晰 | 不晃动到影响阅读，不额外定格拖节奏 |
| 结算点名 | 当前骰按原槽位轻抬或高亮，短标签出现 | 与左侧计分跳涨同步 |

## V5 单向投分交互

### 单骰一次交互

1. 当前骰停稳后 `40–60ms` 内进入原槽聚焦：紧凑琥珀边缘、骰面短亮、接触影收紧；其它骰保持原状态。
2. 本槽真实贡献用约 `50–80ms` 在骰面上方凝成一枚小型点阵光晕；骰子本体不位移、不抬升、不缩放。
3. 光晕用约 `100–140ms` 沿上方空域进入左侧积分塔，只保留贴身短残影，不画满整条路径。
4. 积分塔用约 `100–160ms` 完成吸收、灯段填入和短回弹；精确当前分只在吸收时低关注度更新。
5. 上一击余辉结束前 `60–80ms`，下一骰已经开始聚焦。连续感来自积分塔余辉与下一骰聚焦交叠，不来自骰子之间的连接。

第一颗骰投分可与后排骰的左到右机械制动重叠，不等待六骰全部静止。最终总时长以动态样片为准，目标保持连续、不形成六次完整起承转合。

### 焦点纪律

- 任意时刻只有“当前原槽骰的低幅聚焦 + 本关得分”一组主焦点，积分塔承担主要视觉重量。
- 积分光晕经过其它骰上方时，沿途骰子不亮、不震、不变色、不产生波纹。
- 当前骰的机制若真实影响其它骰，被影响对象最多做一次局部无轨迹反应；不画箭头、电弧或连线。
- 多段连锁不增加多个投射物，只提高单个贡献包密度和积分塔承击力度。
- 右侧终端不显示另一个得分动画，只保留很弱的同步波形。

### V5R1 单骰贡献重量（离槽载体已退回）

- `LocalImpactWeight01` 读取本槽聚合后的真实带符号贡献强度，并经过压缩曲线形成连续值；不直接用骰面点数，也不读取整手五档。
- V5R1 的实体飞行动势和接触压缩不再进入目标方案；`LocalImpactWeight01` 数据意图保留。
- 四个评审采样只是校准端点与中间差，游戏内不显示轻 / 中 / 重 / 极重标签，不在采样边界硬切。
- 单骰局部峰值约封顶于整手暴击完整主峰的 `70%`。若同一击首次越线或完成整手暴击，再在积分塔叠加全局层，但不增加独立停顿。

### V5R2 积分光晕重量

- `LocalImpactWeight01` 只连续改变光晕直径 / 密度、温白核心比例、`0–2` 个贴身短残影、积分塔填入速度 / 回弹和局部余辉。
- 光晕约为骰宽的 `12%–28%`，内部点阵只提供“分数 / 点数能量”质感，不应停留到让玩家数出精确值。
- 来源骰只承担归因，不承担强冲击；积分塔约占本次反馈视觉重量的 `80%`。
- 四个评审采样只校准连续曲线，游戏内不显示轻 / 中 / 重 / 极重标签，也不在采样边界硬切。

### 五档反馈位置

| 内部等级 | 本关得分区域 | 是否打断流程 |
|---|---|---|
| 未达 | 琥珀吸收，目标锁扣不闭合 | 否 |
| 过关 | 目标锁扣小面积青白闭合一次 | 否 |
| 超过 | 塔身多一层琥珀回弹与余辉 | 否 |
| 远超过 | 硬芯转温白，局部机框细振 | 否 |
| 暴击 | 白热贡献包撞塔，单次局部重击与整机短回坐 | 否 |

玩家画面不显示上述等级名、比例或阈值。暴击也不能冻结流程或横扫骰列；主击、余震和恢复都围绕积分塔发生。

## 边界情况

- 表现层关闭或加载失败时，回退到现有 2D 结果展示。
- 窗口尺寸变化时，骰子仍限制在主投掷区安全范围。
- 非基础骰、材质、成长、赌徒阈值、猪猪目标等状态标签不得被 3D 骰子遮住。
- 临时骰默认折入其来源物理骰的贡献包，不恢复临时骰托盘；确实是独立来源时才允许单独投分。
- 零贡献使用暗铜空包并被积分塔吸收；负贡献仍从当前骰单向进入积分塔并触发局部回撤，不允许从积分塔反向流回骰子。
- 零贡献使用极小暗铜点晕和干涩吸收；负贡献沿同一方向发送锈铜点晕，并由积分塔局部回撤表达符号。
- 失败、未达标继续出手和通关收束必须保持可区分。

## 适配 / 布局约束

- 以 `1920x1080`、`16:9` 为正式基准。
- 最小实用窗口按 `1280x720` 检查。
- 额外空间可用于背景延展，不能挤压主投掷区。
- F009 动画不得遮挡左侧积分塔和资源牌。
- 所有阶段提示仍保持短句，不新增教程墙。

## 实现备注

- 视觉层需要与现有 `RollPhase` 对齐，但不要改变玩家可见流程。

无硬阻塞。技术路线默认使用 OnGUI 序列帧播放器，运行验收后再决定是否升级资源或渲染方案。

## 验收

- 小关入场、投骰、停转显点和结算四段焦点清楚。
- `Ready` 和结算后重置阶段显示统一待机骰，不显示额外骰框；非基础骰身份可读。
- 最终点数、槽位顺序和真实结果一致。
- 玩家能清楚区分投骰过程和结果决策。
- 结算开始后直接进入槽位点名，不出现额外结果定格停顿。
- 连打只改变表现，不暗示收益或概率。
- 目标首版加力窗口为 `0`，结果锁定后自动结算；当前代码仍保留旧输入时，不得据此宣称目标流程已接入。
- 结算中不得出现骰子到骰子或槽位到槽位的视觉连接；每个贡献包都必须直接命中本关得分。
- 同屏贡献包数量不超过 `1`，且六次承击之间没有独立等待节点。
- 五档强度在隐藏等级切换处不得产生画面硬切；档内随真实强度连续变化。
- 单骰贡献重量的顺序必须与真实 `abs(SignedContribution)` 一致；相邻数值连续插值，不能变成新的可见分档。
- 结算期间骰子本体不得离开槽位、出现空槽或通过大幅缩放 / 抬升抢走积分塔焦点。
- `1280x720` 与 `1920x1080` 下没有重要信息遮挡。
- Play Mode 录屏中普通一手不显著拖慢节奏。
