### 本版本总体结论

### 原始反馈

- 骰盅和骰子摇晃同时存在，导致画面混乱。
- 点数贴面可能没有适配好，显得骰子不好看。
- 希望可以根据不同骰子种类做出一些旋转效果。

### 需要修改的结论

- F009 首版不能按当前画面直接进入通过验收。
- 投骰期必须收敛信息层级：首版默认取消骰盅阶段，不能让额外道具和完整翻滚骰在同一时刻同等抢焦点。
- 点数贴面需要从“跟随变形面绘制”改为“定格正面可读优先”，投骰中可以用运动感代替可读点数。
- 类型旋转效果可以先作为程序化分组旋转档案进入 F009 修正片，但不等同于最终美术级专属动效。

2026-06-15：

2026-06-16：

- 已按最新反馈将当前实现从连续程序方块形变进一步改为槽位内序列帧式旋转。
- 每次小关开始和未达标回到 `Ready` 时，各槽位显示对应骰子类型身份。
- 按 `Space` 后，每颗骰子在自身槽位内播放旋转序列帧；加力只改变表现速度和提示，不改变概率。
- 加力窗口结束后锁定本次真实结果；`Stopping` 阶段按槽位从左到右停住，停住的槽位立即显示真实点数。

### 不处理 / 后置反馈及原因

- 不做完整真实 3D 桌面重构；当前问题可先在 OnGUI 程序 2.5D 层内修正。
- 不立即生成最终模型、贴花和材质资源；先验证信息层级和类型运动语言是否成立。
- 不让物理翻滚决定最终点数；结果仍必须吸附真实骰面。
