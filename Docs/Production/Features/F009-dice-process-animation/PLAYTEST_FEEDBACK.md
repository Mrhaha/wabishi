## 版本：F009结算接入R1-2026-07-22

### 体验信息

- 日期：2026-07-22
- 版本 / 分支 / commit：当前工作区 V6B-R5 Unity 接入首轮，未提交
- 体验范围：逐骰原槽聚焦、积分光晕直入积分塔、局部连锁 / 再触发
- 体验入口：主流程小关 `Scoring`
- 是否完成目标：否
- 备注：版本名为本次反馈的临时追踪标签；不代表该接入已完成 Unity 编译或运行验收。

### 本版本总体结论

- 验收判断：需小改
- 一句话问题：光效与动画力度可以保留，但当前事件只表达贡献强弱，没有明确告诉玩家本次分数由哪颗骰子、哪项效果产生，回环中的主槽与再次触发来源也难以分辨。
- 一句话建议：继续使用单枚积分光晕和积分塔承击，在真实来源骰原槽增加短促非数值语义标签；回环时同时保留主游标上下文与来源骰“再触发”标签，不增加骰间连线、大号 `+N` 或骰子位移。

### 原始反馈

- “光效和动画可以保留，但是之前的结算能够比较好的看清楚连锁和触发效果，现在整个不清楚分数来源和触发内容。”

### 根因记录

- 新的 `SettlementContributionRecord` 没有保留触发语义，`PrepareImpactSettlementDisplayEvents()` 又把 `SettlementDisplayEvent.Label` 固定为空。
- `settlementImpactPresentationActive` 会屏蔽旧版 `DrawSettlementDieEventTag()`、逐骰分数飞字和抬升；因此旧版负责解释触发内容的唯一信息层也一并消失。
- 当前 `CursorSlotIndex / SourceSlotIndex / LocalChain` 数据已经能区分主槽和真实贡献来源，但画面只用细轨与边框表达，缺少可读语义。

### 修正边界

- 保留当前原槽骰、单枚积分光晕、玻璃直吸收、连续积分塔承击、隐藏五档和程序音效。
- 不恢复逐骰大号 `+N`、滚动计数、百分比、档位名、实体骰离槽、骰间连线或共享灯路。
- 标签只说明效果语义与因果角色，例如 `点数入塔`、`养猪`、`连锁·触发器`、`再触发·三只小猪`；不显示该次精确贡献值。
- 回环必须先显示当前主槽仍在处理，再在实际贡献骰原槽显示“再触发”；局部队列清空后才推进下一主槽。

### 修正记录

2026-07-22：

- `SettlementContributionRecord` 已补入只读 `SemanticLabel`，普通类型效果、触发器、二重唱复制、盟约、王冠、附属分与伐木扣分均可进入展示语义，不参与真实计分。
- `PrepareImpactSettlementDisplayEvents()` 不再清空逐击标签；每个真实来源骰原槽显示一条非数值短标签，右侧终端同步当前内容。
- 回环事件通过既有 `CursorSlotIndex / SourceSlotIndex / LocalChain` 区分主槽与实际来源：来源骰显示 `再触发·… / 响应·…`，主槽下方保持 `主槽·骰子类型`，没有骰间连线。
- 新表现路径继续屏蔽旧逐骰 `+N`、抬升和底部大横幅；积分光晕、玻璃直吸收、塔体承击、隐藏五档与程序音效保留。
- Unity 2019.4.33f1 隔离 BatchMode 编译 `0 error`；Play Mode 自动覆盖 `养猪·触发器 / 再触发·三只小猪 / 响应·盟约` 三种场景，并在 `1920×1080 / 1280×720` 生成六张原生截图，状态为 `PASS`。
- 已补 `0.606×` 显示压力、原生 720p 1:1 裁切和字体装入指标；新增字符缺字数为 `0`，最长来源标签正好装入 `116 px` 安全框。
- 验证证据见 `Docs/QA/20260722_f009_settlement_source_clarity.md`。自动截图通过不替代用户对完整连续一手节奏和声音的最终回归。

### 需要修改的结论

- F009 首版不能按当前画面直接进入通过验收。
- 投骰期必须收敛信息层级：首版默认取消骰盅阶段，不能让额外道具和完整翻滚骰在同一时刻同等抢焦点。
- 点数贴面需要从“跟随变形面绘制”改为“定格正面可读优先”，投骰中可以用运动感代替可读点数。
- 类型旋转效果可以先作为程序化分组旋转档案进入 F009 修正片，但不等同于最终美术级专属动效。
