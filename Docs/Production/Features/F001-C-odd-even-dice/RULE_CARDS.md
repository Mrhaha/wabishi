状态：已验收（用户确认）
功能：F001-C 奇偶骰规则细化
最后更新：2026-06-10

## 总体边界

| 项目 | 当前结论 |
|---|---|
| 二版原则 | 一眼能懂，玩深靠组合。 |
| 首版候选 | 孤证骰、盖章骰、半步骰、轨道骰。 |
| 旧版候选 | 同印骰、点名骰、合议骰、阶梯骰、补档骰已替代；列队骰后置。 |
| 全局牌型表 | 不新增全局牌型；半步骰只作为骰子专属顺子辅助。 |
| 出千边界 | 出千后预览必须刷新；孤证骰不在出千确认后的强制结果中触发。 |
| 存档边界 | 首版不新增长期状态，不新增 `DiceData` 字段。 |
| 实现状态 | 四颗候选规则已确认；代码、CSV 和资源映射已完成，并由用户确认实现验收。 |

## 公共计算约定

- 所有候选默认只统计当前实体骰，不统计龟龟系临时小骰。
- 所有候选默认不生成金币、不消费金币、不产生长期成长。
- 固定分数效果只改变本骰单骰分，不改变本骰有效点数。
- 半步骰只影响顺子判定，不影响本骰显示点数和单骰分。
- 固定分采用方案甲：先作为本骰基础单骰分，之后仍受遭遇规则修正。
- 规则正式实现前必须经过 `$wabish-packet-review`。

## 奇数候选规则卡

### 孤证骰

| 字段 | 草案 |
|---|---|
| 名称 | 孤证骰 |
| 状态 | 已确认 |
| 一句话身份 | 没有同点伙伴时，自动再找一次组织。 |
| 路线职责 | 追同稳定。 |
| 默认面组 | `1 / 1 / 3 / 3 / 5 / 5` |
| 触发时机 | 结果锁定后、结算预览前。 |
| 效果规则 | 如果本骰有效点数没有任何其它实体骰同点，本骰自动重摇自己 1 次，并必须接受新结果。 |
| 重摇次数 | 每颗孤证骰每次出手最多自动重摇 1 次。 |
| 多颗处理 | 同一结果中多颗孤证骰都满足条件时，按骰袋顺序各自重摇一次；重摇完成后不再连锁检查同一批孤证触发。 |
| 是否改变有效点数 | 是，来自本骰自己的自动重摇结果。 |
| 是否改变单骰分 | 不直接改分。 |
| 是否改变主牌型识别 | 可能间接改变，因为本骰重摇后点数变了。 |
| 出千交互 | 出千确认后的强制新结果不触发孤证骰；出千只重摇被选实体骰、刷新预览并直接进入结算。 |
| 持久状态 | 无长期状态；自动重摇使用情况只存在于当前出手。 |
| 市场定位 | `T2` 奇数稳定骰，买入 10、卖出 5，章节权重 `5 / 8 / 8`，服务追同点路线。 |
| UI 标签 | `孤证重摇`、`已重摇`。 |
| 反滥用边界 | 每次出手最多 1 次；不重摇临时小骰；不刷新其它骰子的随机目标。 |
| 验收检查 | 孤证骰结果为 5，场上没有其它 5，则自动重摇一次；如果重摇后仍没有同点，也必须接受。 |
| 出千验收 | 自然开盅后如果孤证已经触发或未触发，进入出千并确认重摇后，都不再执行新的孤证检查。 |

### 盖章骰

| 字段 | 草案 |
|---|---|
| 名称 | 盖章骰 |
| 状态 | 已确认 |
| 一句话身份 | 有同点伙伴时，本骰按固定高分盖章。 |
| 路线职责 | 同点兑现。 |
| 默认面组 | `1 / 1 / 3 / 3 / 5 / 5` |
| 触发时机 | 最终结果锁定后的预览和真实结算。 |
| 效果规则 | 如果本骰有效点数与任意其它实体骰同点，本骰基础单骰分按固定 6 分计算；否则按原点数计分。之后仍受遭遇规则修正。 |
| 是否改变有效点数 | 否。 |
| 是否改变单骰分 | 是，本骰单骰分替换为 6 分。 |
| 是否改变主牌型识别 | 否。 |
| 出千交互 | 出千后按新结果重新判断是否有同点伙伴。 |
| 持久状态 | 无。 |
| 市场定位 | `T2` 奇数收益骰，买入 11、卖出 5，章节权重 `4 / 7 / 8`，适合已有奇数骰或孤证骰的路线。 |
| UI 标签 | `盖章 6分`。 |
| 反滥用边界 | 只看实体骰；不按同点数量提高；固定分数已确认为 6。 |
| 验收检查 | 盖章骰结果为 3，场上另一颗实体骰也是 3，则本骰单骰分显示为 6；没有同点伙伴时按 3 分。 |

## 偶数候选规则卡

### 半步骰

| 字段 | 草案 |
|---|---|
| 名称 | 半步骰 |
| 状态 | 已确认 |
| 一句话身份 | 顺子判定时向低一格借位，帮助普通顺子成型。 |
| 路线职责 | 顺子辅助。 |
| 默认面组 | `2 / 2 / 4 / 4 / 6 / 6` |
| 触发时机 | 手牌识别时。 |
| 效果规则 | 只在顺子判定中，本骰可以选择使用自己的原有效点数，或向低一格借位：`2` 可当 `1`、`4` 可当 `3`、`6` 可当 `5`。 |
| 计分规则 | 本骰计分仍按原有效点数，不按借位点数计分。 |
| 是否改变有效点数 | 否；借位只存在于顺子判定。 |
| 是否改变单骰分 | 否。 |
| 是否改变主牌型识别 | 是，可能帮助形成正常顺子。 |
| 出千交互 | 出千后按新结果重新判断可借位点。 |
| 持久状态 | 无。 |
| 市场定位 | `T2` 偶数顺子辅助骰，买入 10、卖出 5，章节权重 `5 / 8 / 8`。 |
| UI 标签 | `半步成顺`、`借 4→3`。 |
| 多颗处理 | 允许多颗半步骰各自参与同一条顺子判定；每颗半步骰在一次判定中只能选择原值或低一格借位值其中之一。 |
| 反滥用边界 | 不新增“差 2 顺子”全局牌型；借位只服务顺子，不服务三同、全偶或其它牌型。 |
| 验收检查 | 实体结果包含 1、2、4、4、5，其中半步骰为 4；半步骰可借位为 3，帮助形成 1-2-3-4-5 顺子，但本骰仍按 4 分计。 |
| 多颗验收 | 多颗半步骰同时存在时，可以分别借位补齐同一条顺子；同一颗半步骰不能同时提供原值和借位值。 |

### 轨道骰

| 字段 | 草案 |
|---|---|
| 名称 | 轨道骰 |
| 状态 | 已确认 |
| 一句话身份 | `2 / 4 / 6` 齐备时，本骰进入固定高分轨道。 |
| 路线职责 | 差 2 目标。 |
| 默认面组 | `2 / 2 / 4 / 4 / 6 / 6` |
| 触发时机 | 最终结果锁定后的预览和真实结算。 |
| 效果规则 | 如果当前实体骰有效点数中同时出现 `2`、`4`、`6`，本骰基础单骰分按固定 8 分计算；否则按原点数计分。之后仍受遭遇规则修正。 |
| 是否改变有效点数 | 否。 |
| 是否改变单骰分 | 是，本骰单骰分替换为 8 分。 |
| 是否改变主牌型识别 | 否。 |
| 出千交互 | 出千后按新结果重新判断 `2 / 4 / 6` 是否齐备。 |
| 持久状态 | 无。 |
| 市场定位 | `T2` 偶数目标骰，买入 11、卖出 5，章节权重 `4 / 7 / 8`，适合多个偶数骰或半步骰构筑。 |
| UI 标签 | `轨道 8分`。 |
| 反滥用边界 | 只判断是否齐备，不按出现数量加分；固定分数已确认为 8。 |
| 验收检查 | 当前实体骰中至少有一个 2、一个 4、一个 6，则每颗轨道骰显示 `轨道 8分`；只有 2 和 4 时不触发。 |
| 多颗验收 | 多颗轨道骰同时存在时，只要实体骰结果齐备 `2 / 4 / 6`，每颗轨道骰都按固定 8 分进入后续计分。 |

## 固定分结算顺序

固定分采用方案甲：先作为本骰基础单骰分，之后仍受遭遇规则修正。

示例：轨道骰触发后，本骰基础单骰分先变为 8；若遭遇规则让偶数骰 `+1`，最终单骰分可变为 9。

界面表达要求：结果预览可以显示 `轨道 8分` 或 `盖章 6分` 作为类型来源；结算明细需要额外显示遭遇规则修正后的最终单骰分。
