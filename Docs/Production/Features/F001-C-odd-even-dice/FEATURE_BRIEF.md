## 功能意图

F001-C 把奇数路线和偶数路线从“基础控面骰”推进到第一批可执行候选。

二版原则已经收敛为：一眼能懂，玩深靠组合。新增骰不引入新的牌型表、外部资源或复杂计数器；深度来自骰袋组合、出千选择、市场取舍和关卡规则交互。

## 玩家体验

奇数路线让玩家追求“不是孤单一个点”，通过落单重摇和同点盖章鼓励同点构筑。偶数路线让玩家继续理解普通顺子，同时用半步借位和 `2 / 4 / 6` 轨道做出路线性格。

玩家不需要学习一套新公式。结果态应只出现短反馈，例如 `孤证重摇`、`盖章 6分`、`半步成顺`、`轨道 8分`。

## 已确认规则

- 奇数路线优先围绕相同点数牌型做文章。
- 偶数路线优先围绕顺子，并可探索偶数专属的差 2 连线表达。
- F001-C 二版不使用旧版“数量乘加分”的主要表达。
- 首版只推进 4 颗候选：孤证骰、盖章骰、半步骰、轨道骰。
- 当前美术最终运行资源已生成；程序已接入代码、CSV 和图标映射，并由用户确认实现验收。本机本轮未重新执行 Unity 编译、Play Mode 或 UI 截图复验。

## 建议默认值

- 孤证骰和盖章骰默认面组为 `1 / 1 / 3 / 3 / 5 / 5`。
- 半步骰和轨道骰默认面组为 `2 / 2 / 4 / 4 / 6 / 6`。
- 盖章骰固定分数确认先用 6 分。
- 轨道骰固定分数确认先用 8 分。
- 固定分采用方案甲：先作为本骰基础单骰分，之后仍受遭遇规则修正。
- 半步骰只影响顺子判定，不改变本骰有效点数、显示点数或单骰计分；多颗半步骰可以各自借位参与同一条顺子，但每颗骰在一次判定中只能选择原值或借位值其中之一。
- 轨道骰只读取实体骰结果，不读取龟龟系临时小骰。
- 孤证骰不在出千确认后的强制结果中触发。
- 所有候选正式实现前需要先过 `$wabish-packet-review`。

## 职能交接

### 策划

四颗候选规则已确认：孤证不在出千后触发，盖章固定 6 分，半步多骰可一骰一值参与顺子，轨道固定 8 分。固定分采用方案甲，先替换本骰基础单骰分，再接受遭遇规则修正。

### 美术

二版视觉重点从“统计符号”改为“行为符号”：孤证是单骰找同伴，盖章是同点盖章，半步是半格借位，轨道是 `2 / 4 / 6` 轨道。第一张接触图偏场景化，已重做为功能符号内嵌骰体的第二版接触图；最终透明运行图标已生成并登记。

### 界面体验

界面必须保持短标签，不解释公式。半步骰需要在牌型提示里说明是本骰效果帮助成顺，而不是全局顺子规则改变。固定分采用方案甲时，结算明细应能同时表达类型固定分和遭遇规则修正。

### 程序

程序已接手实现。当前已新增孤证自然开盅后自动重摇、盖章固定 6 分、半步顺子候选值、轨道固定 8 分、固定分数先替换再受遭遇规则修正、出千后预览刷新、市场 CSV、图标映射和主文档同步；仍需 Unity 编译和 Play Mode 验证旧存档兼容。
