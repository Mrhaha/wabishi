## 依赖关系

- F001-A 金币已由用户确认实现验收，不再阻塞 F001 后续切片。
- F001-B 龟龟已由用户确认验收，不再阻塞后续。

- F001-D 大树依赖成长收益范围、灌溉方案、苗圃市场边界和持久状态形状。
- 任何子切片进入程序实现前，都依赖对应图标、市场配置和验收标准明确。

## 建议默认值

- 下一步先对 `F001-D-tree-dice` 执行 `$wabish-packet-review`。
- 大树路线建议先确认“单颗核心树”和“多树规模”两条兑现收益是否同时进入首版。
- 苗圃建议默认只影响未来买入的大树，不叠加，不影响已有大树。
- 第一批新骰默认不修改全局手牌识别。
- 龟龟系临时小骰默认不进入骰袋、不保存、不触发其它实体骰长期效果。
- 最终美术资源只在单颗骰用户确认后开始。

4. 苗圃是否只影响未来买入的大树、是否不叠加、是否不影响已有大树？
