# UI_UX_SPEC

状态：界面就绪

## 体验目标

中立家族的提示重点不是解释复杂规则，而是让玩家在结算中看清三件事：

- 当前阵容有几个主体家族。
- 哪些骰子被中立效果选中或加分。
- 王冠最终读取了哪些家族结果。

## 基础显示

- 家族标识：中立使用独立颜色与图标，但不要压过四个主体家族。
- 工具提示：所有家族统计统一显示“只统计实体主体家族，不含中立与吸附壳”。
- 家族归属：可吸附龟成为壳层后不再显示为宿主的第二家族；宿主只显示自身家族徽记。
- 结算日志：中立效果需要显示目标名称、家族、获得的分数或金币。
- 浮字：接济显示金币，空袭 / 缝合显示自身加分，盟约显示左右两侧各 +2，光牙显示被成长的目标。

## 单骰表现需求

- 光牙：显示每个家族被随机选中的骰子，建议用短暂光线连接。
- 二重唱：显示“再次触发”标记、左邻来源和被复制的主体家族结算事件；明确市场、离场和中立效果不可复制。
- 触发器：每次响应显示来源骰、事件短名和 `基础分 +2`；群体事件只显示一次，复制事件不显示触发器响应。
- 王冠：回合末显示每个主体家族最高最终有效点的来源骰、点数、向下取整均值和额外分结果。
- 接济：显示不同家族数与金币数。
- 空袭：显示已有家族数、缺失家族数与加分；纯中立时显示“无主体家族，不触发”。
- 盟约：高亮左右主体家族邻居；任一侧为中立或两侧同家族时显示不触发原因。
- 缝合：显示当前不同主体家族数。

## 文案原则

- 不在主界面堆长说明。
- 规则描述优先短句：例如“每有 1 个主体家族：金币 +1”。
- 对二重唱、触发器、王冠这种复杂骰子，在 tooltip 第二行显示限制条件。
- “基础分 +N”统一解释为本次临时单骰分，不改变点数和骰面。
- 结算日志可更详细，用于验证和回放。

## 原型可接受降级

- 图标可以先用占位。
- 光牙连接线和王冠读取线可以先用日志替代。
- 二重唱复制与触发器监听必须至少有文字反馈，否则无法判断是否正确触发。
