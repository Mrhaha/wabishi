# EXECUTION_PLAN

状态：可执行  
推荐下一步：使用 `$wabish-dev-implementation` 按 F020-01、F020-02 顺序开始原型实现。

## F020-01 中立测试池与基础数据

目标：让 8 颗中立骰子进入可购买、可展示、可纸面试玩的测试池。  
产出：

- 中立家族枚举或等价标识。
- 8 颗骰子的基础配置。
- 市场测试权重或固定测试入口。
- 临时 tooltip 文案。
- 文档同步记录。

完成标准：

- Unity 原型中能看到或通过配置生成 8 颗中立骰子。
- 还未实现效果时，也不能破坏现有四族流程。

## F020-02 家族识别与触发标签

目标：统一中立效果依赖的底层查询，避免每颗骰子各算各的。  
产出：

- 主体家族集合查询。
- 缺失主体家族数查询。
- 相邻骰查询。
- 结构化主体家族结算事件。
- 最后一个可复制左邻事件读取入口。
- 各主体家族最高最终有效点读取入口。

完成标准：

- 接济、空袭、缝合可共用同一套主体家族统计。
- 可吸附龟完成吸附后不再被主体家族统计识别为龟龟。
- 二重唱和触发器只消费结构化主体家族结算事件，不读取日志文本。

## F020-03 过渡战力骰

目标：先实现风险最低的 4 颗过渡骰子。  
范围：

- 接济骰子。
- 空袭骰子。
- 盟约骰子。
- 缝合骰子。

完成标准：

- 能完成基础纸面测试。
- 结算日志显示加分或金币来源。
- 不涉及复制或递归。

## F020-04 连锁成长核心

目标：实现高风险但最能体现中立特色的 3 颗骰子。  
范围：

- 光牙骰子。
- 二重唱骰子。
- 触发器骰子。

完成标准：

- 光牙能按家族选目标并成长。
- 二重唱复制 1 次且不可递归。
- 触发器按每次真实主体家族结算事件响应，群体事件只算一次，复制事件不计入。
- 日志足够定位触发来源。

## F020-05 王冠终局

目标：实现中立家族的最高难度终局收束。  
范围：

- 王冠回合结束读取。
- 各家族目标选择。
- 均值取整。
- 终局阵容纸面试玩。

完成标准：

- 三家族、四家族阵容中能看到王冠收益随强骰结果上升。
- 单纯中立阵容不应比四族混搭更强。

## F020-06 表现、提示和验收

目标：补齐玩家可读性与验收记录。  
产出：

- 结算浮字。
- tooltip 限制条件。
- 纸面试玩记录。
- `PROJECT_CONTEXT.md`、`GAME_FLOW.md`、`DICE_ARCHETYPES.md` 的确认同步。

完成标准：

- 玩家能理解中立骰子为什么触发或未触发。
- 文档与实现行为一致。
