## 摘要

中立路线保持八颗完整名单：`接济 / 空袭 / 盟约 / 缝合` 提供过渡，`光牙 / 二重唱 / 触发器` 放大家族效果，`王冠` 读取多家族终局结果。以下底层口径已冻结：

1. 中立只统计猪猪、恶魔、龟龟、海盗四个主体家族，中立自身不计数。
2. 真实吸附龟成为壳层后不再代表龟龟家族，宿主只保留自身家族。
3. “效果触发”定义为结构化、可见、可复制或可监听的主体家族结算事件，不能通过结算日志文本反推。
4. 所有“基础分 +N / 自身 +N / 左右 +N”统一为本次临时单骰分，不改变有效点数、骰面或家族判定。

## 建议改进

### 已应用生产包补丁

- 光牙保留“每个已有主体家族各选一颗骰、所有面永久 +2”的核心，但定位为高阶成长核心；同一实体骰每次光牙结算最多获得一次光牙成长，成长不回改本次已锁定点数。
- 二重唱改为在自身槽位结算时，复制左邻本次产生的最后一个“允许复制的主体家族结算事件”一次。市场、购买、卖出、刷新、离场、纯被动、中立效果、二重唱复制事件均不可复制。
- 触发器采用用户本轮的“每触发一次其他骰子效果”口径，不再默认按全局效果键去重。建议按可见主体家族结算事件计次：一个效果同时影响多个目标仍算一次；二重唱复制出来的事件和中立效果不计入。奖励给事件源骰，本次临时基础分 +2。
- 王冠读取每个已有主体家族本回合最终有效点数最高的一颗骰，取向下取整均值，并把均值作为王冠额外单骰分；不随机选目标、不复制效果、不改变被读取骰子。
- 接济按已有主体家族数获得金币，进入左到右钱包流；每颗接济独立触发。首版不先加硬上限，但必须验证多接济和财宝海盗联动。
- 空袭只有在至少存在一个主体家族时才触发；每缺一个主体家族，自身本次临时单骰分 +2。纯中立阵容不触发，避免纯中立成为最简单答案。
- 盟约的“左右不同”统一解释为左右邻居家族不同，而不是骰子名称不同；两侧必须都存在。左右邻居本次临时单骰分各 +2，盟约自身不加分。
- 缝合按已有主体家族数获得本次临时单骰分 `+2 / 家族`，中立不计入，最高 +8。
- 所有中立放大效果默认不读取其它中立效果，避免中立骰内部形成第五家族闭环。
- 第一版接受恶魔与二重唱 / 触发器的联动较少：恶魔主要通过家族计数、光牙、王冠、接济、空袭、盟约和缝合接入。不要为了追求完全对称而允许复制购买、刷新或离场效果。

### 需要用户决策的阻塞问题

无。三项阻塞均已由用户确认，并记录在 `CONFIRMATION.md`。

### 非阻塞后置细节

- 接济、光牙、空袭和缝合的价格、阶级、章节权重先通过单家族测试池验证，不在规则审核阶段定死。
- 空袭与缝合的奖励和恒为 +8，形成一降一升的镜像关系；可以先保留为前期 / 后期两种过渡件，再通过市场阶级区分。
- 光牙四家族时一次可成长四个目标，数值上限很高；先作为高阶测试项，不在未试玩前直接削成 +1。
- 王冠、多颗接济、多颗触发器和多颗二重唱的叠加表现需要在验收样例中补齐。
- 正式图标可后置，原型优先依赖清楚的家族徽记、事件标签和结算日志。

## 已记录的用户决策

- 中立基础名单为光牙、二重唱、触发器、王冠、接济、空袭、盟约、缝合。
- 光牙按不同家族让各家族随机一颗骰子所有面 +2。
- 二重唱让左侧骰子的效果再次触发一次。
- 触发器按其他骰子效果触发，为对应骰子提供基础分 +2。
- 王冠在回合结束读取其他各家族各一颗骰子的点数均值。
- 接济按不同家族数获得金币。
- 空袭按缺失家族数给自身加分。
- 盟约读取左右两侧差异并给左右加分。
- 缝合按已有不同家族数获得分数。
- 吸附龟成为壳层后不再计为龟龟家族，只统计宿主自身家族。
- 二重唱和触发器只处理主体家族真实结算事件，排除市场、离场、中立和复制事件。
- 光牙永久改骰面但不回改本次锁定点数。
- 王冠读取各主体家族最高最终有效点，向下取整均值作为额外单骰分。
