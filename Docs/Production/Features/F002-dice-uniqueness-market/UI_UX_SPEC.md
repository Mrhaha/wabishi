## 体验意图

玩家在商店中应先读懂“这颗骰是什么类型”，再读懂“这颗骰六个面为什么特殊”，最后读懂“材质给它什么额外收益”。界面不应让玩家打开额外说明面板才能判断是否购买。

## 信息层级

商店商品卡信息优先级：

1. 类型名。
2. 材质名和材质小图标。
3. 六个面。
4. 价格和金币变化。
5. 一句购买建议。
6. 一句材质规则。

结果态信息优先级：

1. 实际掷出的点数。
2. 当前牌型和本次分数。
3. 材质触发标签。
4. 材质带来的单骰分或账本金币。

## 交互流程

1. 玩家进入市场。
2. 系统生成 3 个商品骰，每个商品骰已经确定类型、六面、材质和价格。
3. 玩家点击商品卡查看完整六面和材质短规则。
4. 玩家购买后，商品加入骰袋，货架补一颗新商品。
5. 玩家卖出骰子时，卖出区域也显示该骰材质和六面。
6. 进入小关后，未投掷时骰子身份卡显示材质角标。
7. 结果锁定后，触发材质的骰子在结果卡和日志中显示短标签。

## 反馈和动画

- 官铁触发：单骰分浮字可显示 `官铁 +1`。
- 鎏印触发：账本金币日志显示 `鎏印 +1 金`。
- 明釉触发：单骰分浮字显示 `明釉 +2`。
- 铅封触发：出千确认后显示 `铅封 +2`。
- 铜骨触发：同点骰旁显示 `铜骨 +1`。

反馈应短，不解释公式。

## 边界情况

- 骰袋已满时，购买按钮禁用，但商品信息仍可查看。
- 金币不足时，价格显示为不可购买状态。
- 商品骰六面全相同时，六面展示仍显示六个格子，不用合并成一句说明。
- 临时小骰不显示材质。
- 缺失材质图标时，使用材质名短标签回退。
- 卖出价首版按骰子类型固定回收，不随随机面组或材质上涨；卖出区域应显示“按类型回收”或等价短标签。

## 适配 / 布局约束

- 当前 OnGUI 原型仍以 `1280x720` 内部坐标映射到 `1920x1080`。
- 商品卡不能因六面展示导致按钮溢出。
- 六面展示应使用固定小格，不随点数文本改变大小。
- 1280x720 下商品卡至少能显示类型、材质、六面、价格。
- 详细规则文案过长时优先保留材质名和一句短规则，不加说明段落。

## 实现备注

首版不需要新建完整详情页。商品卡内直接显示六面和材质规则即可。后续若加入词缀系统，再考虑详情弹窗。

## 阻塞项

无界面体验阻塞。实现后需要按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 做截图验收。

## 验收

- 玩家在市场中不打开额外说明，也能区分两颗同类型骰。
- 五种材质触发时都有可见短反馈。
- 1280x720 和 1920x1080 下商品卡文字不重叠、不溢出。
- 出千选择时铅封收益可见。
- 鎏印产生的账本金币能在日志中被看见。
- 卖出区域不会暗示强面组或强材质能高价回收。
