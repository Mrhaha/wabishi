## 实现目标

将当前市场生成从按 `dice_market_config.csv` 刷出一个固定类型，扩展为生成一个具体商品骰：

```text
# 生成顺序
选择骰子类型 -> 生成类型兼容面组 -> 选择材质 -> 计算价格 -> 生成显示名和购买建议
```

购买后，该骰应以类型 key、当前六面、材质 key 和成长状态等必要数据进入骰袋，并可保存和继续游戏恢复。

## 已确认行为

- 当前骰袋上限仍为 6。
- 当前出千最多选择 3 颗实体骰，按各自当前面组重摇。
- 当前 `DiceData` 已保存类型 key、当前面组和大树成长层数。
- 当前市场价格和权重来自 `dice_market_config.csv`。
- 当前材质不改变有效点数，不改变牌型识别。
- 临时小骰不携带材质。
- 奇数路线骰必须只生成 `1 / 3 / 5` 面，偶数路线骰必须只生成 `2 / 4 / 6` 面，其它类型默认可用 `1-6`。
- 材质单骰分顺序为：类型效果后、遭遇规则前，并参与最终牌型倍率。
- 鎏印只增加本关账本金币，不进入本次分数，不被同一小关内国库、贿赂或投资读取。

## 材质结算顺序

首版采用以下顺序：

1. 确定最终有效点数，包括自然投掷、孤证自然重摇、出千重摇和其它已确认点数逻辑。
2. 根据实体骰结果识别主牌型和奇偶副牌型。
3. 计算骰子类型带来的单骰分替换或加成，例如双倍、盖章、轨道、国库、投资、壳匠。
4. 应用材质单骰分修正：官铁、明釉、铅封、铜骨。
5. 应用遭遇规则单骰修正，例如 `OddLedger`、`LowFog`、`DoubleJudge`。
6. 将实体骰单骰分、临时小骰分和规则分进入最终牌型倍率。
7. 鎏印在真实结算时写入本关账本金币，不进入本次分数，也不影响同一小关内国库、贿赂或投资。

材质边界：

- 官铁：只对本骰单骰分 `+1`。
- 明釉：本骰结果等于自身当前面组最大值时，本骰单骰分 `+2`。
- 铅封：本骰在本次确认出千中被选择并实际重摇后，本骰单骰分 `+2`。
- 铜骨：本骰与任意其它实体骰当前有效点数相同，本骰单骰分 `+1`；不读取临时小骰。
- 临时小骰不携带材质，不触发材质，也不作为铜骨同点伙伴。

- 出千重摇只在当前面组中取值。
- 官铁始终为本骰单骰分 +1。
- 鎏印只在掷出当前面组最高点时入账。
- 明釉只在掷出当前面组最高点时加分。
- 铅封只在被出千选择并重摇后加分。

- 独特骰卖出价按类型固定回收，不因强面组或强材质形成套利。
