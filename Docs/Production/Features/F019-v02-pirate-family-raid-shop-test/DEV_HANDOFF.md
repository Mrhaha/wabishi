## 输入决策

- 海盗单家族测试包包含 8 颗骰：刷新海盗、劫掠海盗、人口海盗、海盗头领、练腿海盗、财宝海盗、抢劫海盗、海贼王。
- 海盗规则优先保留爽感，只用货架、骰袋容量、金币和队列停止防真无限或卡死。
- 买入价和卖出价应来自统一骰子估值公式，这是基础通用规则，不是海盗特例。
- 海盗头领半价允许产生短期折价或倒卖收益；后续由刷新价格递增、三格货架、六槽容量、购买后货架处理和市场结束自然阻止稳定循环。
- 刷新海盗可多颗累加，允许刷新净费用为负并返还金币。
- 劫掠海盗可让相邻骰面低于 0，负点数直接扣分。
- 人口海盗生成随机售价基础骰。
- 抢劫海盗离场可递归触发新增骰的离场效果。
- 海贼王每 3 金让自身全部面永久 +1，无上限。

## 实现目标

程序实现应支持一次海盗单家族测试：

1. 通过测试开关让市场只刷新 8 颗海盗骰。
2. 玩家能买入、卖出、刷新、排序并离开市场。
3. 海盗规则能读市场动作、金币入账、离场队列和真实结算。
4. 海盗造成的正负面组变化能保存、恢复、显示并参与结算。
5. 抢劫离场递归能自然停止，不会卡死或无限复制同一货架。

## 已确认行为

- 刷新海盗：每颗让本次刷新净费用 -1；刷新基价仍递增。
- 劫掠海盗：结算时左右相邻实体骰全部面永久 -1；可变负；每个相邻目标 +3 金。
- 人口海盗：每主动卖出 2 颗骰，生成 1 颗基础骰，面组 `1 / 2 / 3 / 4 / 5 / 6`，实例售价随机，不限次数。
- 海盗头领：购买海盗骰半价，只影响买入价公式的买入倍率或临时买入修正；允许折价套利，不要求买入后立刻卖出仍亏损。
- 练腿海盗：买入后随机其它己方骰全部面永久 +1。
- 财宝海盗：每次金币入账事件后，随机己方骰全部面永久 +1。
- 抢劫海盗：离开市场时从非空货架随机获得 1 颗骰，货架置空，新增骰入队尾；新骰离场效果继续入队。
- 海贼王：结算时读取当前金币，每 3 金让自身全部面永久 +1，不消耗金币。

## 代码影响

- `DieType`：新增 8 个海盗类型。
- 价格底层：新增统一估值 helper，让市场买入价、卖出价、海盗头领折扣、人口海盗实例售价、负面组和长期成长都通过同一估值口径表达。
- `Die` 或等价实例数据：需要支持实例随机售价 / 实例估值修正，至少用于人口海盗生成的基础骰。
- 面组逻辑：允许当前面组出现 0 和负数；保存、显示和结算都不得默认夹到 1 或 0。
- 刷新费用：净费用可能为负。负费用应增加钱包金币，并记录金币入账事件，以便财宝海盗可读取。
- 买入事件：买入后触发练腿海盗；海盗头领影响海盗类型买入倍率或临时买入修正，不直接改卖出估值。
- 卖出事件：记录本市场主动卖出计数；每满 2 次尝试生成基础骰。
- 金币事件：需要结构化事件，至少让财宝海盗读取真实入账事件；避免同一金币事件被重复无限读取。
- 离场队列：抢劫海盗触发后新增骰放到队尾；若新增骰有离场效果，继续排队。
- 货架：抢劫获得后货架置空，不自动补货。
- 日志 / 短标签：记录刷新减费、返钱、劫掠、全脸正负成长、抢劫获得、队列停止原因。

## 数据影响

价格公式首版建议：

```text
骰子估值 = 类型基础值 + 面组价值 + 长期状态价值 + 实例修正
买入价 = max(0, ceil(骰子估值 * 买入倍率) + 临时买入修正)
卖出价 = max(0, floor(骰子估值 * 卖出倍率) + 实例卖出修正)
```

- 现有 `dice_market_config.csv` 可在首版继续作为类型基础值、阶级和权重来源；如程序实现需要新增字段，优先保持旧字段兼容读取。
- 面组价值应能读取普通面组、高面组、成长面组、0 或负面组。
- 长期状态价值应能覆盖猪猪饲料、真实吸附龟、海盗成长和后续类似状态。
- 实例修正用于人口海盗生成基础骰的随机售价等单颗差异。
- 海盗头领只降低买入侧，不改卖出侧；折价或倒卖收益允许存在。
