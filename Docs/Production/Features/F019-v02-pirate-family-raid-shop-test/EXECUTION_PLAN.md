## 依赖关系

- 依赖 F015 的 V0.2 市场触发和左到右结算队列。
- 依赖现有市场刷新、买入、卖出、离开市场流程。
- 依赖 F014 悬浮窗作为海盗状态展示载体。
- 抢劫离场队列可参考 F017 / F018 的离场效果处理，但不得改坏恶魔和龟龟行为。
- 价格底层应作为通用机制接入，不能只写成海盗头领的局部特例。

## 建议默认值

- 劫掠每影响 1 个相邻骰 +3 金。
- 刷新净费用允许为负，负数按返钱处理。
- 买入价和卖出价从统一骰子估值公式派生。
- 海盗头领半价允许短期折价套利，不要求买入后立刻卖出仍亏损。
- 人口海盗随机售价为 0 到 3 金。
- 抢劫获得骰后货架留空，不自动补货。
- 海盗测试池 8 颗全部开放。
