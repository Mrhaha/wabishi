## 功能意图

F019 用于验证 V0.2 海盗家族的新方向：海盗不是保守的折扣经济族，而是围绕刷新、买入、卖出、金币入账和离开市场抢掠形成高上限连锁。它的玩家记忆点应是“越刷越贪、越抢越滚、离场时战利品一串结算”，并把市场操作直接转成下一战可见点面成长。

本包按用户最新修正优先保留爽感：除非会导致真无限、队列卡死或程序不可结算，否则不为海盗规则添加低强度上限。强度主要交给刷新费用递增、三格货架、六槽容量、负面组代价、卖骰损失和金币机会成本自然约束。

## 玩家体验

玩家在市场中买入海盗骰后，市场不再只是补货页面，而会变成一场可以贪的抢掠回合：

1. 刷新海盗让多次刷新更便宜，甚至在前几次刷新中形成有限返钱爽点。
2. 劫掠海盗把左右邻骰的所有面砍低，换取金币；点面可以低于 0，负数会在下一战直接扣分。
3. 人口海盗鼓励卖出换血，每卖两颗补一颗随机售价基础骰。
4. 海盗头领让买海盗更容易，允许出现短期折价和倒卖爽感；底层买入价和卖出价应来自统一估值公式，真正阻止循环的是刷新价格递增、三格货架、六槽容量、购买后货架处理和市场结束。
5. 练腿海盗和财宝海盗把买入、金币入账转成全队面组成长。
6. 抢劫海盗在离开市场时从市场拿走随机骰，新增骰排在队尾继续触发离场效果，形成可递归的离场战利品链。
7. 海贼王把持有金币转成自身全部面成长，作为海盗家族的终局战力出口。

核心爽点是：玩家能明确说出“我这次市场为什么还要刷新 / 卖骰 / 攒钱 / 排序离场”，并在下一战看到海盗操作留下的正负点面痕迹。

## 已确认规则

### 全局口径

- 海盗家族读市场刷新、买入、卖出、金币入账、离开市场和结算等公共触发点。
- 海盗优先保留高上限爽感，不为单次强度设置低封顶。
- 买入价和卖出价使用统一估值公式派生，作为后续全项目通用底层机制：
  - `骰子估值 = 类型基础值 + 面组价值 + 长期状态价值 + 实例修正`
  - `买入价 = max(0, ceil(骰子估值 * 买入倍率) + 临时买入修正)`
  - `卖出价 = max(0, floor(骰子估值 * 卖出倍率) + 实例卖出修正)`
- 海盗家族允许通过折扣、随机售价、成长和负面组影响估值；短期折价套利是玩法收益，不作为错误处理。
- 只有以下情况作为硬停止条件：
  - 货架为空。
  - 骰袋已满。
  - 钱包不足以执行需要支付的操作。
  - 离场触发队列清空。
  - 同一真实触发已经完成，不应被同一对象反复读取导致卡死。
- V0.2 六槽、单次出手、无出千、无全局牌型计分继续成立。
- 所有永久面组改变都写入实体骰当前面组，并参与保存恢复。

### 八颗海盗规则卡

| 骰子 | 触发 | 规则 | 停止条件 / 备注 |
|---|---|---|---|
| 刷新海盗 | 付费刷新 | 每颗刷新海盗让本次刷新净费用 `-1`；多颗累加；刷新基价仍在每次刷新后 `+1`。 | 净费用允许为负并返还差额金币；该收益有限，因为刷新基价递增。 |
| 劫掠海盗 | 本骰结算 | 左右相邻实体骰全部面永久 `-1`，可低于 0；每影响 1 个相邻骰，获得 3 金。 | 没有相邻骰则不影响；负点数在后续结算中直接扣分。 |
| 人口海盗 | 主动卖出 | 本市场内每主动卖出 2 颗骰，获得 1 颗基础骰；面组固定 `1 / 2 / 3 / 4 / 5 / 6`，售价随机。 | 不限制次数；骰袋满时不再获得；卖出计数建议只读取玩家主动卖出，不读取抢劫新增。 |
| 海盗头领 | 购买海盗骰 | 购买海盗骰半价，只影响买入价公式的买入倍率或临时买入修正，不改卖出估值。 | 允许短期折价和倒卖收益；循环边界由刷新价格递增、三格货架、六槽容量、购买后货架处理和市场结束承担。 |
| 练腿海盗 | 买入骰子 | 每次买入后，随机 1 颗其它己方骰全部面永久 `+1`。 | 没有其它骰时不触发。 |
| 财宝海盗 | 钱包获得金币 | 每次金币入账事件后，随机 1 颗己方骰全部面永久 `+1`。 | 不按金币枚数逐点触发，按“金币事件”触发；不设次数上限。 |
| 抢劫海盗 | 离开市场 | 从当前市场随机获得 1 颗骰，加入骰袋最后；新骰若有离场效果，按新增顺序继续触发。 | 获得后从货架移除；货架空、骰袋满或队列清空时停止；抢到抢劫海盗可继续递归。 |
| 海贼王 | 本骰结算 | 读取当前钱包金币，每 3 金让自身全部面永久 `+1`，无上限。 | 不消耗金币；不创造金币；真实结算后入档。 |

## 建议默认值

| 项目 | 建议默认值 | 原因 | 是否阻塞 |
|---|---|---|---|
| 劫掠金币 | 每个被劫掠相邻骰 +3 金 | 永久削全部面且可变负，需要足够爽的补偿 | 否 |
| 刷新负费用 | 允许净费用为负，按差额返还金币 | 符合用户“可以累加，刷新费每次 +1”的口径 | 否 |
| 统一价格公式 | 买入价和卖出价都从同一骰子估值派生 | 让折扣、成长、负面组和随机售价都进入同一底层机制 | 否 |
| 海盗头领半价 | 对海盗骰买入倍率或临时买入修正生效，不改卖出估值；允许短期折价套利 | 保留海盗“买低、卖高、滚动市场”的爽感 | 否 |
| 人口海盗随机售价 | 随机卖价建议在 0 到 3 金之间 | 让基础战利品有波动，不抢高阶骰身份 | 否 |
| 抢劫货架处理 | 抢到后货架留空，不自动补货 | 保证递归会自然停，和恶魔贡品占货架口径一致 | 否 |
| 海盗测试池 | 8 颗全部开放 | 用户已给完整八颗草案，并要求生成包 | 否 |

- 离开市场抢劫队列：谁触发、抢了哪颗、放到哪个位置、是否继续递归、为什么停止。
