# F019 界面体验规格

状态：静态接入，仍待截图检查
功能：V0.2 海盗家族抢掠市场测试
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 海盗家族围绕刷新、买入、卖出、金币入账、离开市场和结算触发。
- 首版不压低海盗爽感，只保留防真无限和卡死的硬停止条件。
- 玩家必须看懂抢劫递归为什么继续、为什么停止、抢到了什么、放到了哪里。
- 负点数必须可见，且玩家要知道负点数会扣分。

## 体验意图

海盗市场应有“抢完再走”的仪式感。玩家点击离开市场后，如果有抢劫海盗，市场应进入短暂的自动结算状态：货架骰被拿走、新骰加入队尾、新骰离场效果继续排队、直到自然停止。这个过程要可读但不能拖太久。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 普通市场 | 买入、卖出、刷新、排序 | 金币、容量、刷新净费用、三格货架、买入价、卖出价、骰袋排序 | 买入 / 卖出 / 刷新 / 离开 | 拖拽排序、悬浮查看 | 离开市场或进入抢劫离场 |
| 刷新反馈 | 确认刷新花费或返钱 | 原刷新费、海盗减费、净费用、刷新后基价 | 自动展示 | 无 | 回到普通市场 |
| 卖出反馈 | 看懂人口海盗计数 | 本市场卖出计数、是否生成基础骰、随机售价 | 自动展示 | 无 | 回到普通市场 |
| 买入反馈 | 看懂练腿海盗触发 | 买入对象、被加面目标、`全脸 +1` 标签 | 自动展示 | 无 | 回到普通市场 |
| 离场抢劫队列 | 看懂抢劫递归 | 当前触发骰、被抢货架、加入队尾位置、队列剩余 | 自动播放 | 可后续加入加速，不在首版强求 | 队列结束进入下一小关 |

## 信息层级

1. 市场顶部优先显示金币、容量、刷新净费用。
2. 刷新按钮附近显示刷新海盗减费数量，例如 `刷新 3 - 海盗 2 = 1`，若为返钱则显示 `返 1 金`。
3. 货架卡显示是否可被抢劫，离场时被抢货架要高亮并置空。
4. 商品卡和悬浮窗显示统一估值公式派生出的当前买入价 / 卖出价；海盗头领折扣应显示为折扣后的买入价，不需要常驻展开完整公式。
5. 骰袋列表显示新增骰落点，抢来的骰用短标签 `抢来`。
6. 负面组显示必须明显，例如点面格中显示 `-1`、`-2`，不要隐藏为 0。
7. 悬浮窗显示随机售价和规则，不把所有计算细节常驻在主界面。

## 交互流程

### 刷新

1. 玩家点击刷新。
2. 计算当前刷新基价、刷新海盗数量和净费用。
3. 若净费用为正，扣金币；若净费用为负，返还金币；若为 0，则免费刷新。
4. 刷新基价按现有市场规则递增。
5. 货架整批刷新。

### 人口海盗

1. 玩家主动卖出一颗骰。
2. 本市场卖出计数 +1。
3. 每达到 2 次，生成一颗基础骰，面组为 `1 / 2 / 3 / 4 / 5 / 6`。
4. 该基础骰获得随机售价，加入当前骰袋最后。
5. 若骰袋已满，显示“袋满，未招募”，不生成。

### 抢劫离场

1. 玩家点击离开市场。
2. 如果没有离场效果，直接进入下一小关。
3. 如果有抢劫海盗，锁住买入 / 卖出 / 刷新 / 拖拽。
4. 按当前骰袋顺序建立离场队列。
5. 抢劫海盗触发时，从非空货架随机获得 1 颗骰，货架置空，新增骰加入队尾。
6. 如果新增骰有离场效果，加入队列末尾。
7. 货架空、骰袋满或队列清空时结束。
8. 进入下一小关。

## 反馈和动画

- 刷新海盗：刷新按钮附近显示短式算账，金币飞字显示扣费或返钱。
- 劫掠海盗：左右相邻骰出现刀痕 / 红色短标签 `全脸 -1`，金币飞向钱包。
- 人口海盗：卖出计数显示 `卖出 2/2`，生成基础骰时显示 `招募基础骰` 和随机售价。
- 练腿海盗：被选中的目标骰弹出 `全脸 +1`。
- 财宝海盗：金币入账后随机目标骰弹出 `财宝 +1`。
- 抢劫海盗：被抢货架卡飞向骰袋队尾，货架留空，新增骰带 `抢来` 标签。
- 海贼王：结算时显示 `金币 / 3 = 成长 xN`，然后本骰全部面变化。

## 边界情况

- 刷新净费用为负时，需要明确表现为返钱，而不是显示负费用让玩家误解。
- 骰袋满时，人口海盗和抢劫海盗都应停止新增，并给短反馈。
- 货架全空时，抢劫海盗不触发获得。
- 抢来的抢劫海盗可以继续触发，但必须从剩余货架中取骰。
- 负点数不应被低雾或其它规则错误夹到 0；如果遭遇规则有最低 0 的既有处理，需要程序明确差异。
- 离场队列播放不应允许玩家中途买入、卖出或刷新。

## 适配 / 布局约束

- 继续使用当前 OnGUI 市场和悬浮窗，不重做 UI 框架。
- 短标签应适配 1280x720 和 1920x1080。
- 负点数和 `+N` 数字必须在骰面格内可读。
- 离场队列反馈不应遮挡货架和骰袋位置关系。

## 实现备注

- 市场反馈可以先用现有文字条、短标签和日志，不要求新动画资源。
- 若队列很长，首版可以快速逐项播放，不做跳过按钮；后续根据测试再加加速。

## 阻塞项

无。

## 验收

- 玩家不读日志也能看出刷新到底扣了多少或返了多少。
- 玩家能看出哪颗骰被劫掠后变成负面组。
- 玩家能看出卖两颗后生成了哪颗基础骰和它的随机售价。
- 玩家能看出抢劫离场链条为什么继续、为什么停止。
- 玩家能在悬浮窗中确认海贼王当前按金币会成长多少。
