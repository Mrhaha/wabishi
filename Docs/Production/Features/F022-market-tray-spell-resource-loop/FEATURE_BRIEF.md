## 一句话目标

在不照搬《炉石传说：酒馆战棋》经济、不加入冻结、不改变当前三格骰子货架和差异化骰价的前提下，让玩家在每个市场中能够围绕采购、处理、施法、指定目标、转换资源、继续刷新或主动收手，驾驶一段有机会成本的资源循环。

## 已确认方向

- 保留当前三格骰子货架、章节权重、高阶保底、买走留空、付费整刷补齐三格、章节 / Boss 刷新基础价、本市场连续刷新递增、初始金币、通关收入、利息和差异化骰价。
- 不加入冻结，不把骰子统一成同一价格，不增加酒馆等级。
- 猪猪、龟龟、恶魔、海盗继续使用本项目已有资源；融合重点是家族内部协同、跨家族资源转换和由玩家控制的循环，不是逐卡搬运。
- 新增两个仅在市场存在的交易托盘格。骰子采购后先进入托盘，离店前必须清空。
- 骰袋资产不能作为基础操作免费回到托盘；只有明确写有“回收”的效果可以做到。回收不是采购，旧骰归位不重放首次装入。
- 只有少量骰子拥有每市场一次主动能力。首轮候选只改造抢劫海盗与血契船长。
- 新增一格独立法术货位和最多三张的跨市场法术架。法术不占三格骰子供给。
- 每个正常资源市场至少有两条后果不同、机会成本不同的可辩护方向；玩家主动花空并进入零利息状态后，至少仍有一个付费、合法、有结果的动作。

## 本包仍待批准的候选

- 十二张法术的卡面、价格、章节权重和 Boss 权重倍率。
- 三格法术架无常驻丢弃、满架购买时无退款原子替换。
- 贸易猪现金 / 转栏二选一、饲料进入吞吃、贡品最低二金、龟龟托盘出口、真实壳移动、人口海盗来源递归排除。
- 抢劫海盗与血契船长的主动改造，以及黑帆 / 魔蝠 / 清仓祭典共用批量目标面板。
- 软供给导演的方向计数、一次法术纠偏、最多一次非保底骰纠偏、记忆衰减和 `BridgeGap` 破同分。
- `Prepared / Committed` 原子事务、市场状态存档和异常恢复合同。

这些候选已经通过纸面连续审计，但在用户批准前不得写入 `PROJECT_CONTEXT.md`、`GAME_FLOW.md`、`DICE_ARCHETYPES.md`，也不得修改 Unity 正式行为。

## 当前问题

当前实现把“支付购买、获得骰子、直接进入骰袋、首次装入效果”压成一次 `BuyOffer()` 调用。三格市场中的处理动词也主要只有购买、卖出、刷新和离开，导致：

- 满袋时常先卖旧骰再判断新骰，缺少购买后比较、加工和转向空间。
- 吞吃、化壳、贡品和生成骰大量由系统自动选择目标，复杂规则没有稳定转化为玩家决策。
- 现有四十四枚骰的主功能集中于转化和兑现，搜索与转向稀少，三格随机货架容易同时只提供未来承诺。
- 法术、跨族弱桥和供给控制尚不存在，非猪猪路线更依赖命中特定组件。
- 当前存档只保存骰袋和少量整轮状态，不能恢复托盘、法术架、批量命令或半途市场事务。

## 目标体验

### 操作自由

市场统一使用十类动作语法：

| 动作 | 玩家决定 | 主要机会成本 |
|---|---|---|
| 采购 | 买哪一件骰子进入托盘 | 金币、货架选择、托盘容量 |
| 首次装入 | 把哪颗托盘骰写入哪个空袋位 | 六槽容量、装入顺序 |
| 出售 | 退出哪项资产，选择何种兑现 | 永久战力、饲料、龟壳、金币事件 |
| 刷新 | 是否支付本次递增价格换整柜 | 当前商品、利息、后续购买力 |
| 购买法术 | 立即使用、保存或满架替换 | 金币、三格法术架 |
| 施放法术 | 目标、额外支付、施放时点 | 一次性资源、目标状态 |
| 转化 | 把托盘骰当实体、材料、壳或贡品 | 独立身体、卖价、持续效果 |
| 激活 | 是否使用实例每市场一次主动 | 材料、目标、次数 |
| 重接 | 排序、宿主、回收与换班 | 位置、金币、离场清盘 |
| 离店 | 是否停止循环并锁定离场目标 | 最后一批货架、利息与未来计划 |

### 行动节奏

- 前期正常市场：`2–4` 个有意义动作。
- 中期正常市场：`4–8` 个有意义动作。
- 后期正常市场：`6–12` 个有意义动作。
- `12+` 动作只由清仓、换班和至少两个长期发动机共同形成，不由单骰或单法术稳定产生。
- 玩家保利息或提前离店是有效决策，不用动作数量惩罚主动收手。

### 爽点阶梯

1. 一次小操作立刻改变后续选择，例如折价后买得起、喂养后可以现金兑现或转栏。
2. 中型引擎开始把一个家族的盈余转成另一家的燃料。
3. 少数高阶市场出现连续采购、吞吃、金币、刷新与再采购。
4. 递增刷新、货架耗尽、托盘容量、资产消耗、法术消耗或玩家主动停止自然收束。
5. 下一次投骰把市场中的接线逐层兑现，玩家能复述主要因果。

## 市场结构

现行市场骨架不被替换：

- 左侧：六格长期骰袋及排序。
- 右侧：三个骰子货架，买走后留空。
- 底部：卖出、刷新、反馈、离开等现行动作。

F022 增量增加：

- 两格交易托盘：仅市场内临时持有新采购、明确生成和特殊回收骰。
- 一格独立法术货位：进店与每次付费刷新时生成，买走留空。
- 三格法术架：跨市场保存，法术只在市场施放。
- 少量骰子主动入口。
- 批量目标与原子替换弹层。
- 离店前托盘清盘门。

## 托盘规则

- 普通采购成功后扣款、清空对应货架、把完整商品实例放入一个空托盘格并发布一次采购事件。
- 托盘骰可以装入空骰袋位、主动出售、作为明确材料、龟龟直接化壳、贡品奉献或执行卡面允许的转换。
- 满六骰时仍可先采购到空托盘比较；普通装入仍需要先出售、消耗、化壳或转换一个现有资产。
- 骰袋资产不能自由拖回托盘。首批唯一通用特例是候选法术“换班凭证”。
- 离店按钮在托盘非空时不可提交；界面必须逐格显示合法出口和阻塞原因。
- 托盘不跨市场保存，不能成为第七、第八个长期骰位。

## 法术首批候选

| 层级 | 法术 | 价格 | 核心动作 |
|---|---|---:|---|
| 基础操作 | 单格换货 | `2` | 丢弃或补一个骰子货架位，不算付费刷新 |
| 基础操作 | 瑕疵折价 | `2` | 买价 `-4`、最低 `1`，采购后六面永久各 `-1` |
| 基础操作 | 饲料袋 | `3` | 对骰袋或托盘目标依次进行两次真实喂养 |
| 基础操作 | 拆解许可 | `3` | 消耗托盘骰，把其转移值的一半写入指定骰 |
| 基础操作 | 加急清算 | `4` | 清算骰袋资产，获得现金值及最多 `6` 金额外现金 |
| 资源桥 | 外壳转运 | `4` | 移动一个真实龟壳实例和内部状态 |
| 资源桥 | 货架施肥 | `5` | 本市场最低点 `+2`，影响现货与后续生成 |
| 资源桥 | 定向招募 | `5` | 按玩家选择的家族重生成一个货架位，不保证卡名 |
| 资源桥 | 黑市分账 | `5` | 本市场每次真实吞吃市场来源骰，额外 `+2` 金 |
| 资源桥 | 金币饲料 | `4 + 2/4/6` | 每额外支付 `2` 金，依次进行两次真实喂养 |
| 高阶改道 | 清仓祭典 | `8` | 预分配三格目标后，按货架顺序全部真实吞吃 |
| 高阶改道 | 换班凭证 | `6` | 原子交换托盘新骰与骰袋旧骰，旧骰成为特殊回收 |

完整事件、目标和反套利合同在 `DEV_HANDOFF.md`。

## 四家族资源循环

| 家族 | 主要来源 | 主要转换 | 兑现 | 故意保留的瓶颈 |
|---|---|---|---|---|
| 猪猪 | 饲料次数、质量、可养成资产 | 最低点、卖价、转栏、驮粮压缩 | 稳定通关、成熟资产退出 | 搜索和市场动作不足 |
| 龟龟 | 实体龟、真实壳、宿主 | 实体变附件、压缩六槽、复制真实化壳 | 持续效果、组合宿主成长 | 高价、缺现金、绑定宿主 |
| 恶魔 | 市场最低点、贡品、货架食物 | 吞吃写入面组、返金、下刷折扣 | 定向永久成长、清货架 | 消耗购买机会、持续刷新昂贵 |
| 海盗 | 出售、金币、刷新与采购次数 | 资产变金币、金币变搜索和采购 | 市场行动量、快速转向 | 缺无条件长期成长 |
| 中立 / 法术 | 搜索、一次性转换、清盘、重接 | 临时补桥 | 解决单次断供 | 不能成为无条件最优长期引擎 |

六条家族边：

- 猪猪 × 海盗：饲料资产在现金兑现与转栏之间选择，金币继续采购和刷新。
- 恶魔 × 海盗：吞旧货架换金币与下刷折扣，再寻找新食物。
- 猪猪 × 龟龟：饲料转给驮粮龟并化壳，把实体成长压进宿主。
- 恶魔 × 龟龟：采购顺序决定强化长期龟壳还是即时贡品吞吃。
- 猪猪 × 恶魔：饲料进入可吞吃面组，在现金、保留和吞吃间形成弱桥。
- 龟龟 × 海盗：化壳释放容量、金钱龟制造金币事件、组合体可保留、出售或付费保壳退出。

双阵营骰负责压缩步骤、提速和提高上限，不能成为两家能否互动的唯一开关。

## 软供给导演

原始生成始终优先：

1. 三颗骰完整执行当前章节权重、同批去重、高阶保底、差异化价格和市场最低点。
2. 法术按独立章节权重、近期出现、同功能重复、持有与拒绝记忆生成。
3. 系统基于公开状态生成可支付的完整行动链，并按首个承诺签名去重。
4. 正常资源状态少于两条方向时，法术架未满则最多重抽一次法术。
5. 仍不足时，最多替换一个可用性最低的非高阶保底骰位。
6. 到此停止；不送金币、不保证卡名、不追发完整套件、不改高阶保底。

`BridgeGap` 只在导演本来就需要纠偏且同资格候选打平时生效；它不会为已有两条方向的市场追加跨族桥。

- 一格法术货位、三格法术架、十二张法术。
- 满架购买替换。
- 两颗正式主动骰。
- 贸易猪、龟龟、贡品、吞吃、批量命令、人口招募等兼容改造。

## 不在范围内

- 冻结、酒馆等级、统一三金、三连金色、十格手牌、升阶发现。
- 新增第四个骰子货架或第二个法术货位。
- 重做四十关目标、生命、投骰、计分和左到右局部结算。
- 一次性重平衡全部四十四枚骰的价格、面组和权重。
- 让现有骰读取“购买法术 / 施放法术”事件。
- 新增大量主动能力。
- 豪华法术插画、整套新市场底板或大规模音效。
- 暗中给玩家当前流派的指定核心件。

## 纸面基线

- `270` 个连续市场决策中真实死店 `0`。
- 正常资源状态 `213 / 213` 在有限纠偏后达到两条方向。
- 主动花空后的 `57 / 57` 至少有一个付费合法动作。
- `239 / 270 = 88.5%` 完全不纠偏。
- 付费整刷 `174` 次，相对对照 `176` 次只下降 `1.1%`。
- 严格存牌两店内兑现率 `68.4%`。
- 满架替换 `4 / 270 = 1.5%`。
- 托盘龟保留实体率 `30.4%`。
- 猪猪 / 恶魔弱桥 `8 / 90`，龟龟 / 海盗弱桥 `15 / 90`，均覆盖早中晚。
- 人口招募第二代为 `0`。

这些数字只是生产行为基线，不替代可玩版本和真人选择验证。

## 成功定义

玩家在同一市场中能够指出至少两条不同的合理路线、各自牺牲什么，并通过不同动作顺序得到不同的长期状态。系统能避免频繁死店，但不会替玩家完成套牌；高阶循环可以很长，仍能自然停止、稳定保存、复现并安全离开。
