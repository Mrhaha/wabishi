# F022 界面体验规格

状态：已提案，需用户决策
功能：交易托盘、法术、主动能力与批量市场命令
视觉基线：F021 关间市场
设计分辨率：`1280×720`，同时验收 `1920×1080`

## 体验目标

玩家进入市场后应快速回答：

1. 我现在有哪些长期资产、临时组件和已保存计划？
2. 当前至少哪两条路线值得比较，各自会牺牲什么？
3. 买下骰子后还能怎样处理，而不是只能立即换入或后悔？
4. 当前循环为何还能继续，何时应该主动停止？
5. 为什么离开按钮被阻塞，怎样用最少步骤清盘？

界面不显示“系统推荐答案”，但必须让主要后果在提交前可预测、提交后可复述。

## 画面区域

现有顶部资源窗、左侧六格骰袋、右侧三格骰子货架和底部主操作带保持。新增区域采用增量模块：

| 区域 | 常驻内容 | 主要交互 |
|---|---|---|
| 骰袋区 | 六颗长期骰、顺序、饲料、壳、主动状态 | 选择、排序、出售、作为目标 |
| 交易托盘 | 两格市场临时骰、来源、处理出口 | 装入、出售、化壳、材料、奉献、归位 |
| 骰子货架 | 三件骰子、差异化价格、状态 | 采购、查看 Tooltip |
| 法术货位 | 一张法术、价格、主功能、可用性 | 购买、满架替换 |
| 法术架 | 最多三张跨市场法术 | 选择、施放、查看计划 |
| 主动区 | 当前选中骰的主动入口与次数 | 预览、选目标、提交 |
| 底部操作带 | 卖出、刷新、中央反馈、离开 | 现行动作与状态说明 |

第一轮程序灰盒可在现有底板上使用程序框承载新增区域；正式像素坐标必须在灰盒双分辨率装入后写回本文件，不能直接把未验证坐标烘焙进新底板。

## 常驻信息层级

### 顶层

- 当前金币与利息档。
- 当前刷新净价与下一次原始价格。
- 骰袋 `n/6`、托盘 `n/2`、法术架 `n/3`。
- 离店是否可提交。

### 商品与资产

- 骰子商品：名称、家族、六面摘要、买价、货架状态。
- 托盘骰：名称、来源短标、装入资格、现金卖价和当前可用处理图标。
- 法术：名称、价格、主功能、立即可执行 / 严格可保存 / 当前无完整链。
- 骰袋资产：长期状态、主动可用性、卖出模式和目标资格。

### 详情

Tooltip 负责完整规则、事件来源、预览后的面组、饲料账本、壳实例、首次装入状态和阻塞原因。常驻卡面只显示一条规则摘要，不堆完整合同。

## 核心流程

### 骰子采购

1. 玩家点击一个非空骰子货架。
2. 若金币不足，按统一短时提示显示“金币不足”，不改变任何状态。
3. 若两格托盘都满，显示“交易托盘已满”，并高亮两个待处理格。
4. 合法时预览扣款、进入哪一托盘格、采购后仍在货架的状态变化。
5. 提交后扣款、商品进入第一个空托盘、货架留空、结清采购响应。
6. 托盘格成为当前选中项，界面展开其合法出口。

采购不会自动打开装入动画，也不会把骰子直接加入骰袋。

### 托盘骰装入

1. 选择托盘骰。
2. 骰袋有空槽时，空槽高亮；玩家选择具体槽位。
3. 提交后骰子进入该槽并发布首次装入。
4. 骰袋已满时不提供普通交换；界面显示“先出售 / 消耗旧骰，或使用明确回收效果”。
5. 特殊回收骰归位时显示“归位”，不显示“首次装入”。

### 普通满袋换代

基础流程：

1. 先买新骰进入托盘。
2. 比较新旧资产。
3. 直接出售或通过其它合法效果消耗一个旧骰。
4. 把新骰首次装入腾出的空槽。

不得通过把任意骰袋骰免费拖入托盘来制造无成本替补席。

### 托盘龟

选择托盘龟时至少显示：

- 装入为实体：需要空袋位，保留龟龟身份和独立结算。
- 直接化壳：高亮骰袋或另一托盘中的合法非龟宿主。
- 主动出售。
- 作为当前明确效果的材料。

玩家悬停直接化壳目标时预览基础面组转移、持续壳、回收值、首领复制和失去的实体身份。没有合法宿主时只禁用化壳，不隐藏其它出口。

### 贡品

- 采购后进入托盘，标记“一次性，不能装入”。
- 可选择骰袋目标奉献，或 `0` 金主动出售。
- 奉献预览吞吃转移值与会响应的吞噬 / 黑市 / 共食端口。
- `0` 金出售明确显示“产生出售事件，不产生金币事件”。

### 贸易猪出售

有饲料且存在合法目标时打开一层二选一：

- 现金兑现：显示 `R + C` 金和将销毁的饲料。
- 饲料转栏：显示 `R` 金，并要求选择骰袋或托盘中的合法目标。

预览同时显示钱包、利息档、目标新饲料账本和被放弃的另一种收益。无饲料时直接使用普通出售，不打开空选择。

### 主动能力

1. 选择骰袋中标注主动的实例。
2. 可用入口显示费用、材料和目标摘要。
3. 点击后进入目标预览；取消、目标失效或支付失败不消耗次数。
4. 提交后锁定其它输入，完整结清事件队列。
5. 成功后实例显示“本市已用”；回收和归位不恢复。

首轮候选：

- 抢劫海盗：选择一个非空骰货架，移动到空托盘；不支付、不算采购。
- 血契船长：选择一枚托盘材料和骰袋目标，执行市场来源吞吃并获得 `1` 金。

### 购买法术

法术架未满：

1. 点击法术货位。
2. 预览扣款、购买后架位和利息变化。
3. 提交后法术进入第一个空架位，商店位留空。

法术架已满：

1. 点击法术货位。
2. 弹出三张旧法术与“取消”。
3. 选择旧牌后预览旧牌销毁、无退款、新牌费用、钱包和利息。
4. 原子提交后旧牌消失，新牌进入同一格，货位留空。
5. 旧牌不播放施法、不触发任何玩法事件。

满架没有常驻丢弃按钮；当前货位仍正常可见并随付费刷新变化。

### 施放法术

1. 选择法术架中的牌。
2. 显示合法目标、额外支付和完整预览。
3. 多目标或可变支付在同一准备界面完成。
4. 取消或非法不消耗。
5. 提交后先锁定法术和额外金币，再按事件顺序结清。
6. 完成后法术格清空，不自动补充商店法术位。

### 付费刷新

1. 点击刷新后先显示并锁定本次净价。
2. 若存在黑帆，打开一次批量目标面板；每颗黑帆选择唯一旧货架或跳过。
3. 取消则不扣款、不吞吃、不涨刷新价。
4. 提交后按骰袋顺序吞吃目标并结清响应。
5. 清除其它旧货架，提高本市场原始刷新价。
6. 生成三颗新骰和一张新法术。
7. 应用市场最低点、补给猪和有限供给纠偏。
8. 返回可输入状态。

本次吞吃产生的折扣只作用下一刷，不能反向降低已锁定价格。

### 离开市场

离开按钮提交前依次检查：

1. 是否有未处理托盘骰；有则阻塞并高亮每格合法出口。
2. 是否有待指定魔蝠目标；有则打开一次批量面板。
3. 是否有其它离场目标或事务仍在准备。

全部满足后进入离店锁定。锁定后不再接收购买、卖出、刷新、施法、主动或目标输入；按确定顺序完成离场队列并进入下一小关。

## 批量目标面板

同一组件支持三种命令：

| 根动作 | 左侧来源 | 右侧目标 | 跳过规则 | 提交规则 |
|---|---|---|---|---|
| 付费刷新 | 每颗黑帆 | 三格旧货架 | 每颗可跳过，同一食物不可重复 | 先锁价，再按骰袋顺序执行 |
| 离开市场 | 每颗魔蝠 | 三格货架 | 每颗可跳过，同一食物不可重复 | 托盘清空后锁定离店 |
| 清仓祭典 | 每颗非空货架骰 | 骰袋合法目标 | 每件必须分配，目标可重复 | 按货架从左到右吞吃 |

交互约束：

- 默认建议只作可见提示，不自动提交。
- 未分配黑帆 / 魔蝠视为跳过。
- 清仓有任意非空来源未分配时不能提交。
- 三格货架下最多三次分配加一次提交。
- 提交后不再逐颗弹窗；取消完整回滚。

## 十二张法术的 UI 特殊点

| 法术 | 额外交互 |
|---|---|
| 单格换货 | 空货架也可选；预览“丢弃旧货，不触发退出事件” |
| 瑕疵折价 | 商品详情同时显示折价后支付价与采购后六面 |
| 饲料袋 | 目标可在骰袋或托盘；两次喂养逐次播放 |
| 拆解许可 | 先选托盘材料，再选骰袋目标；预览来源标签 |
| 加急清算 | 只选骰袋资产；明确“不触发普通出售” |
| 外壳转运 | 先选一层真实壳，再选新宿主；预览回收值守恒 |
| 货架施肥 | 显示当前三格和本市场未来货架都会受益，价格不回算 |
| 定向招募 | 先选五个身份之一，再选货架；不预览具体卡名 |
| 黑市分账 | 施放后在市场状态栏显示剩余到离店的持续标 |
| 金币饲料 | 用 `2/4/6` 三档额外支付，不使用自由滑杆 |
| 清仓祭典 | 共用批量目标面板，全部非空货架必须分配 |
| 换班凭证 | 同时预览新骰首次装入、旧骰特殊回收和离店清盘义务 |

## 方向可读性

供给导演不向玩家展示“已纠偏”或路线评分。界面只展示客观可用信息：

- 当前能支付的完整第一步。
- 如果继续，需要的后续金币、空位或目标。
- 采购、施法、刷新和离店分别会放弃什么。
- 当前动作结束后，托盘、法术架和利息会怎样变化。

不能用“推荐”“最优”“系统为你补充”等提示替玩家做决定。

## 反馈

- 成功采购：`已采购至托盘`。
- 首次装入：`已装入第 n 槽`。
- 特殊回收：`旧骰已回收，离店前需处理`。
- 归位：`已归位，不重复首次装入`。
- 法术购买：`已加入法术架`。
- 法术替换：`旧计划已销毁，新法术已购入`。
- 托盘阻塞：`交易托盘已满` 或 `离店前请清空交易托盘`。
- 批量取消：`操作已取消，状态未改变`。
- 事务锁定：中央反馈显示当前根动作与进度，不只禁用按钮。

所有短时提示使用 F021 的 `UpperCenter` 单槽；复杂结果进入中央结构化反馈，不向上堆叠多条 Toast。

## 键鼠与取消

- 左键选择、确认和分配。
- 右键或 `Esc` 取消尚未提交的目标、替换或批量命令。
- 已提交事务不能用 `Esc` 中断，只能等队列完成。
- 拖拽可作为装入、化壳和重排的快捷方式，但所有动作必须有点击等价入口。
- Tooltip 命中不能挡住按钮与批量目标连线。

## 边界状态

- 骰袋空：不能离店，现行规则继续。
- 骰袋满、托盘空：仍可采购一颗骰进入托盘。
- 骰袋满、托盘满：不能采购，必须先处理托盘或直接卖出骰袋资产。
- 托盘有回收旧骰：不能把它当新采购重放首次装入。
- 法术架满：法术位继续刷新；空转保护暂停。
- 法术没有合法目标：可保存但不显示为立即可施放。
- 付费刷新价格为负：仍按现行返金语义锁价；黑帆响应不能反向重算本次价格。
- 批量面板中目标消失：提交失败并完整保持前态。
- 市场离场锁定：所有可变状态输入禁用，Tooltip 可继续查看。

## 适配

- 首轮必须以真实 `1280×720` 装入为最低门槛，不能先设计 1080p 再缩小。
- 长名称优先换行或扩展详情区，不把 L2 / L3 文字缩到 Micro。
- 托盘、法术架和批量面板在 `1280×720` 下必须同时可辨。
- `1920×1080` 使用原生物理字号与图标，不对 720p 字图做连续缩放。
- 非整数显示压力只检查层级生存，锐度仍以原生 1:1 图验收。

## 首轮可用性指标

- 新玩家在一次说明后能正确区分采购与首次装入。
- 满袋买新骰时，不会尝试把任意旧骰免费拖入托盘超过一次。
- 玩家能在 `10` 秒内说出托盘每格至少一个合法出口。
- 三格法术架满时，玩家能理解当前选择是放弃新牌、先施旧牌或无退款替换。
- 批量命令首次使用记录完成时长、改目标次数和取消率；熟练后目标为最多三次分配加一次提交。
- 离店被阻塞时，玩家无需打开文档即可找到未处理托盘格。
- 单个正常中期市场的有效动作目标为 `4–8`，但界面不强迫玩家达到动作数。

## UI 验收

- 三格骰子货架没有被法术稀释或挤成难读尺寸。
- 托盘中的新采购、系统生成和特殊回收有不同来源表达。
- 采购、首次装入、回收、归位、出售、吞吃和化壳不会共用含糊的一个动画。
- 法术购买与施放在视觉和事件反馈上是两个动作。
- 满架替换、清仓、黑帆刷新和魔蝠离店均支持完整取消与原子提交。
- 离场锁定后不再弹出新的目标选择。
- 无冻结入口。
- 现有市场的刷新、差异化价格、卖出、Tooltip、排序和离场演出没有回归。

## 当前角色就绪度

UI：需补充。

规则、流程、状态和输入预算已经可用于灰盒；正式布局仍需要在用户批准规则后制作一轮双分辨率灰盒，并把最终矩形与焦点顺序写回本文件。该补充不阻塞领域层和确定性测试先行，但阻塞正式美术与完整交互接入。
