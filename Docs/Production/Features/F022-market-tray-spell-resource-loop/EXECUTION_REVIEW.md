## 规则完整性

### 托盘

通过：

- 来源、容量、长期性和离店清盘均有唯一口径。
- 普通满袋换代与特殊回收明确分离。
- 采购、首次装入、回收和归位各有不同事件。
- 托盘龟、贡品、人口招募与抢劫海盗都有明确去向。

不存在“骰袋骰是否可免费放回托盘”的遗留歧义；答案为不能，除非明确回收。

### 法术

通过：

- 十二张都有价格、目标、子事件、来源和反套利边界。
- 三章节段权重和 Boss 倍率完整。
- 法术购买、保存、施放、替换和清空商店位均有唯一结果。
- 满架不提供随时丢弃，替换是无退款原子事务。
- 单格换货和定向招募不会冒充付费刷新。

### 家族资源

通过：

- 贸易猪使用 `R/C/V` 合同，现金与转栏不能全拿。
- 饲料进入吞吃有明确公式，普通龟化壳不误读该公式。
- 托盘龟实体与直接化壳互斥。
- 贡品最低二金、锁价、奉献和零售明确。
- 真实壳移动同一实例，价值守恒、内部已用状态不重置。
- 人口招募通过来源排除切断真递归，不限制正常出售收益。

### 批量与恢复

通过：

- 黑帆、魔蝠和清仓共用一次批量命令，但目标限制分开。
- 价格锁定、处理顺序、取消、提交和离店锁定明确。
- 清仓、换班、满架替换、移壳和目标出售均纳入 `Prepared / Committed`。
- 恢复只允许完整前态或完整后态。

### 供给导演

通过：

- 原始生成优先。
- 只在方向不足时纠偏。
- 先最多一次法术，再最多一颗非保底骰。
- 高阶保底不动。
- `BridgeGap` 只作已有纠偏中的破同分。
- 法术架满与玩家拒绝记忆都限制追发。
- 自然结果和导演结果要求分列。

导演没有获得暗送金币、第二颗骰替换、指定卡名或完整套件的权限。

## 审包时发现并已修正的问题

### 1. 生产包可能被误读为重做当前市场

已修正：

- `FEATURE_BRIEF.md` 明确 F015 和 F021 继续作为基线。

- 无冻结、三格骰子、差异化价格、利息和递增刷新列为回归硬门。

### 2. 采购、装入与回收可能继续共用旧买入事件

已修正：

- `DEV_HANDOFF.md` 拆分 `DiePurchased / DieFirstInstalled / DieRecovered / DieReturned`。
- 换班与归位的防重放写入事务和验收。

### 3. 多步目标操作缺少崩溃恢复

已修正：

- 将贸易猪转栏、托盘化壳、移壳、清仓、换班、黑帆、魔蝠和满架替换纳入统一事务。
- `ACCEPTANCE.md` 增加中间重载与 before / after 门。

### 4. 供给导演可能被实现成隐藏配牌器

已修正：

- 明确原始市场先完整生成。
- 方向不足才有纠偏权限。
- 法术一次、骰子一次、高阶保底零替换。
- `BridgeGap` 只能破同分，不增加预算。

### 5. UI 可能只支持拖拽，缺少可访问的等价操作

已修正：

- `UI_UX_SPEC.md` 要求所有拖拽行为有点击等价入口。
- 目标、取消、阻塞原因、法术替换和托盘出口都有显式流程。

### 6. 存档可能继续只追加分隔字段

已修正：

- `DEV_HANDOFF.md` 推荐 V13 版本化完整运行状态和事务日志。
- V12 运行存档默认清除、设置保留；不猜测半市场迁移。
