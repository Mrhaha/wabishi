## F022-S0：批准与基线冻结

### 目标

把纸面候选变成可执行规则，不在实现期间继续暗改核心合同。

### 任务

- 用户一次性批准或退回 `BG3-D` 至 `BG3-K`。
- 若批准，更新本包七份文档状态。
- 同步 `PROJECT_CONTEXT.md`、`GAME_FLOW.md`、`DICE_ARCHETYPES.md`。
- 记录当前 `DiceKingDemo.cs`、四个市场 CSV、生命与市场回归基线。
- 固定早 / 中 / 晚各十组资源状态和三个策略代理的种子。
- 明确 V12 运行存档默认清除、设置保留。

### 完成门

- 候选与已确认项不再混写。
- 三个主文档与生产包不冲突。
- 所有实现人员读取同一规则版本。

## F022-S1：确定性领域骨架

### 目标

在不改变玩家可见市场的前提下，建立后续功能共用的事件、来源、事务和测试层。

### 任务

- 增加稳定实例 ID、来源类型、首次装入和主动使用状态。
- 为饲料账本和真实壳实例补齐可迁移字段。
- 建立市场事件结构与 `RootActionId`。
- 建立 `Prepared / Committed` 事务和 before / after 摘要。
- 把钱包、商品生成、卖价、吞吃估值、壳价值做成可预览的纯函数。
- 建立确定性随机入口和市场日志。
- 先用当前购买 / 卖出 / 刷新路径跑“影子事件”，对照现行结果，不改变行为。

### 验收

- 相同种子、相同输入得到相同商品、目标顺序和状态摘要。
- 预览不写长期状态。
- 当前市场回归结果与 S1 前一致。
- 单元测试覆盖来源、价格、事件去重和事务恢复骨架。

### 停止线

若无法让现行路径通过影子事件得到一致结果，不进入托盘实现。

## F022-S2：两格交易托盘

### 目标

只验证“采购后还有处理空间”是否成立，不同时接入十二张法术和导演。

### 任务

- 三个骰子货架保持现行生成。
- 购买改为进入第一个空托盘。
- 增加空槽首次装入、普通出售、明确材料接口和离店清盘门。
- 满袋仍可采购到托盘。
- 禁止任意骰袋骰免费回托盘。
- 增加 `Purchase / Generated / Recovery / Stolen` 来源显示。
- 提供灰盒托盘 UI、阻塞原因、点击等价入口。
- 把旧血契满袋自动吞吃、抢劫离店获骰、人口直接进袋和贡品自动奉献临时关闭或迁移为明确占位处理，不能双跑。

### 验收

- 满袋采购、比较、卖旧、装新形成完整链。
- 托盘无法跨市场残留。
- 托盘满时购买不扣款。
- 首次装入只触发一次。
- 特殊回收归位不会重放首次装入。
- 当前三格骰子供给、价格和整刷没有改变。

### 决策观察

至少记录：

- 采购后改变原计划的比例。
- 两格中同时存在两种处理用途的市场数。
- 因托盘容量而主动出售、转化或停止的次数。

## F022-S3：法术货位与基础五法术

### 目标

先验证法术能否补搜索、转向和退出，而不是变成便宜加分。

### 内容

- 单格换货。
- 瑕疵折价。
- 饲料袋。
- 拆解许可。
- 加急清算。

### 任务

- 新增独立法术数据、法术货位与三格法术架。
- 进店 / 付费刷新生成，买走留空。
- 购买和施放分离。
- 三格满时无退款原子替换。
- 无常驻丢弃。
- 接入目标预览、额外状态和事务恢复。
- 仅接基础五张的早中晚权重；未实现牌权重为零。

### 验收

- 法术不占骰子权重、高阶保底或货架位置。
- 满架替换只能得到完整前态或后态。
- 五张牌都有立即使用、保存 / 机会成本或明确跳过的样本。
- 单格换货不推进付费刷新。
- 清算不触发普通出售。
- 付费整刷次数没有被局部搜索显著替代。

### 停止线

若基础法术在多数可购买样本中成为固定必买，先修目标、机会成本或分类，不扩到中高阶牌。

## F022-S4：家族资源兼容

### 目标

让当前四家族真正使用托盘与来源，而非在新系统旁保留旧自动逻辑。

### 任务组

#### 猪猪 / 海盗

- 贸易猪现金 / 转栏二选一。
- 完整饲料账本迁移。
- 人口招募进入托盘与 `CrewRecruit` 排除。
- 一笔金币事件规则。
- 抢劫海盗改为主动。

#### 恶魔 / 海盗

- 恶魔骰子读取真实采购。
- 血契船长改为主动托盘吞吃。
- 黑帆刷新前批量目标和锁价。
- 魔蝠离店前批量目标。
- 贡品进入托盘并由玩家奉献 / 零售。

#### 猪猪 / 龟龟

- 托盘龟实体 / 直接化壳。
- 驮粮龟饲料压缩。
- 组合宿主退出只算一项资产。
- 首领只读取真实化壳。

#### 弱桥

- 饲料进入可吞吃面组。
- 金钱龟金币事件与海盗读取。
- 真实壳实例和价值守恒。

### 验收

- 旧自动目标路径全部关闭。
- 贸易猪三种后续状态可反转。
- 托盘龟实体率首测不低于 `20%`。
- 单吞噬贡品至少 `25%` 合法样本延后或放弃。
- 批量根动作只开一次面板。
- 人口招募第二代为零。

## F022-S5：中高阶法术与供给导演

### 内容

- 外壳转运。
- 货架施肥。
- 定向招募。
- 黑市分账。
- 金币饲料。
- 清仓祭典。
- 换班凭证。

### 任务

- 接入真实壳移动和内部状态。
- 接入本市场持续法术。
- 接入可变额外支付。
- 接入清仓与换班事务。
- 接入完整三段权重与 Boss 倍率。
- 实现法术可用分类、严格保存、记忆和连续空转保护。
- 实现方向签名、法术优先纠偏、最多一颗非保底骰纠偏。
- 实现 `BridgeGap` 破同分。
- 自然与纠偏日志分列。

### 验收

- 高阶保底替换为零。
- 法术架满时不做法术纠偏。
- 导演不保证卡名或家族套件。
- 清仓 / 换班故障注入只有 before / after。
- 外壳转运总回收值守恒、已用状态不重置。
- 付费整刷相对对照下降不超过 `15%`。

### 停止线

若超过 `20%` 市场需要硬纠偏，先调整原始功能分布、价格或法术资格，不能扩大导演权限。

## F022-S6：正式 UI 与美术接入

### 目标

把已验证的领域动作做成可理解、可取消、可预测的市场界面。

### 任务

- 在现行旧机厅补给柜上完成双分辨率灰盒。
- 确认托盘、法术货位、法术架、主动入口和批量面板矩形。
- 更新 `UI_UX_SPEC.md` 的最终坐标、焦点顺序和热区。
- 制作两格托盘、法术模块、八类功能图标、主动图标和批量面板。
- 接入 F021 LED、实体键帽、五状态按钮和 `UpperCenter` 提示。
- Tooltip 增加来源、预览、壳实例和法术完整规则。
- 提供所有拖拽动作的点击等价入口。

### 验收

- `1280×720` 与 `1920×1080` 均无裁切和关键遮挡。
- 玩家能区分采购、首次装入、回收和归位。
- 离店阻塞可自解释。
- 满架替换与批量命令可完整取消。
- 托盘 / 法术不稀释三个骰子货架的可读性。

## F022-S7：存档恢复与行为验收

### 目标

证明系统不仅“能玩”，还可以在长循环、退出重进和异常中保持唯一状态。

### 任务

- 提升到 SaveVersion `13`。
- 保存完整市场、托盘、法术架、导演记忆、随机游标和事务日志。
- 默认清除 V12 运行存档、保留设置。
- 对所有事务动作注入重载。
- 重跑 BG3-K 的 `30` 组三市场 × 三策略。
- 建立可交互原型的批量命令时间记录。
- 做至少一轮设计人员和一轮外部玩家测试。
- 回归生命池、累计续投、通关收入、市场经济和投骰。

### 行为门

| 指标 | 首轮门 |
|---|---:|
| 正常资源市场两方向 | `100%` |
| 主动透支后至少一付费动作 | `100%` |
| 完全不纠偏 | `>=80%` |
| 严格保存两店兑现 | `>=60%` |
| 满架替换观察率 | `1%–5%` |
| 贸易猪任一路线占比 | `<80%` |
| 贡品延后 / 放弃 | `>=25%` |
| 托盘龟实体选择 | `>=20%` |
| 猪猪 / 恶魔弱桥 | `>=5%` 且覆盖早中晚 |
| 龟龟 / 海盗弱桥 | `>=10%` 且覆盖早中晚 |
| 人口招募第二代 | `0` |
| 付费整刷相对下降 | `<=15%` |
| 真死店 | `0` |

### 完成门

- 所有 P0 / P1 功能、恢复和数据问题关闭。
- 行为门达到或明确记录为下一轮数值问题。
- 用户完成一轮实际市场试玩并确认操作自由度方向成立。
- 主文档、实现与生产包再次同步。

## 角色依赖

```text
用户批准
  -> 策划规则冻结
    -> 程序领域层
      -> 托盘灰盒
        -> 法术与资源合同
```

## 推荐下一切片

用户批准后直接执行 `F022-S1`。

理由：

- 它不改变玩家可见玩法，回滚成本最低。
- 它能先暴露当前购买、吞吃、壳和存档的隐式耦合。
- 后续每个复杂动作都依赖来源、实例和事务。
- 没有 S1，托盘和法术 UI 即使看起来可用，也无法通过恢复与反重复验收。
