# F022 验收标准

状态：已提案，需用户决策
功能：交易托盘、法术与资源循环

## 验收原则

F022 的通过标准不是“十二张法术都能施放”，而是：

- 玩家在当前三格市场里获得更多可控动词。
- 每条路线有可见代价。
- 家族之间通过公共资源互相供能。
- 供给控制避免频繁死店，但不替玩家完成答案。
- 长循环可以确定性复现、自然停止和安全恢复。

纸面审计数据只作为首轮基线，不能替代 Unity 可玩版和真人行为。

## A. 基线不变

- [ ] 市场仍有三个骰子货架。
- [ ] 骰子仍使用差异化买价和独立卖价。
- [ ] 买走骰子商品后该货架留空。
- [ ] 付费刷新替换并补齐三个骰子货架。
- [ ] 章节 / Boss 基础刷新价和本市场连续递增保持。
- [ ] 初始金币、通关固定收入、每 `6` 金一档利息和利息上限保持。
- [ ] 章节权重、高阶保底和同批去重保持。
- [ ] 没有冻结入口、免费保留货架或酒馆等级。
- [ ] 投骰、生命池、累计续投和左到右局部结算不被本包改变。

## B. 两格交易托盘

- [ ] 普通骰采购后进入第一个空托盘，不直接进入骰袋。
- [ ] 骰袋已满但托盘有空位时仍能采购。
- [ ] 托盘已满时采购不扣金币、不清货架、不触发采购事件。
- [ ] 托盘骰能按规则装入、出售、化壳、奉献或作为材料。
- [ ] 普通首次装入需要骰袋空位。
- [ ] 骰袋资产不能作为基础操作免费回托盘。
- [ ] 特殊回收不算采购。
- [ ] 回收骰归位不重放首次装入。
- [ ] 离店前必须清空两个托盘格。
- [ ] 托盘不能跨市场保存，也不能生成第七个长期实体。

## C. 采购、装入和来源

- [ ] `DiePurchased` 与 `DieFirstInstalled` 是两个不同事件。
- [ ] 系统生成、抢劫和回收不会误触发采购。
- [ ] 直接化壳不会误触发首次装入。
- [ ] 每个实例有稳定 ID 和来源。
- [ ] 预览不写金币、随机、饲料、面组、货架或存档。
- [ ] 一项玩家动作的所有子事件共享根动作 ID，并可去重。

## D. 两颗主动骰

### 抢劫海盗

- [ ] 只有骰袋中的实例显示主动入口。
- [ ] 每市场每实例最多成功一次。
- [ ] 选择非空货架后把完整商品移动到空托盘。
- [ ] 不扣商品价格，不算采购。
- [ ] 托盘满、无货或取消不消耗次数。
- [ ] 不再在离店时随机获骰或递归生成托盘物。

### 血契船长

- [ ] 选择托盘材料和骰袋目标。
- [ ] 材料按市场来源真实吞吃，不获得卖价。
- [ ] 吞吃响应完整结算，随后获得 `1` 金。
- [ ] 不再以满袋采购为自动入口。
- [ ] 回收与归位不恢复本市场已用次数。

## E. 法术货位与法术架

- [ ] 所有普通和 Boss 市场只有一个独立法术货位。
- [ ] 进店与每次付费刷新生成一张法术。
- [ ] 法术不占骰子权重、三个货架或高阶保底。
- [ ] 法术买走后货位留空，不自动补牌。
- [ ] 购买与施放是两个不同事件。
- [ ] 法术架固定三格并可跨市场保存。
- [ ] 法术只能在市场施放。
- [ ] 没有常驻免费丢弃。
- [ ] 满架时法术货位仍可见并刷新。
- [ ] 满架购买可选择取消或无退款原子替换一张旧牌。
- [ ] 被替换旧牌不算施放，不触发玩法事件。
- [ ] 满架暂停法术空转保护。

## F. 十二张法术

### 单格换货

- [ ] 可选非空或空骰货架。
- [ ] 丢弃旧商品不算出售、吞吃或黑帆食物。
- [ ] 新商品按当前章节、市场最低点和合法权重生成。
- [ ] 不增加刷新价，不推进高阶保底。

### 瑕疵折价

- [ ] 目标支付价 `-4`，最低 `1`。
- [ ] 每件商品最多一个折价标记。
- [ ] 采购后六个长期面永久各 `-1`。
- [ ] 购买前可预览折价后六面。

### 饲料袋

- [ ] 可选骰袋或托盘目标。
- [ ] 两次真实喂养按顺序分别结清。
- [ ] 肉猪和贪吃猪按当前规则响应。

### 拆解许可

- [ ] 消耗一枚托盘材料且无卖价。
- [ ] 骰袋目标六面增加 `max(1,floor(materialTransfer/2))`。
- [ ] 来源标签正确决定黑市等响应。
- [ ] 龟材料不化壳、不触发首领龟。

### 加急清算

- [ ] 只选骰袋资产。
- [ ] 获得 `X + min(X,6)`。
- [ ] 只发布资产清算和一笔合并金币事件。
- [ ] 不触发贸易猪转栏、人口海盗和其它出售响应。

### 外壳转运

- [ ] 只移动一个真实壳实例。
- [ ] 类型、内部状态、卖价和已用状态完整保留。
- [ ] 旧宿主与新宿主总回收值守恒。
- [ ] 历史加面不移动、不扣除、不重放。
- [ ] 不触发真实化壳或首领龟。

### 货架施肥

- [ ] 本市场最低点 `+2`。
- [ ] 立即影响当前非空货架和之后本市场生成的商品。
- [ ] 商品锁价不回算。
- [ ] 不影响托盘和骰袋。

### 定向招募

- [ ] 玩家选择猪猪、恶魔、龟龟、海盗或中立。
- [ ] 玩家选择一个骰子货架。
- [ ] 按当前章节与所选身份权重生成。
- [ ] 可包含双阵营，不包含基础或贡品。
- [ ] 不保证具体卡名，不算付费刷新。

### 黑市分账

- [ ] 持续到离店。
- [ ] 每次市场来源真实吞吃额外 `+2` 金。
- [ ] 多张分账合并为同一次吞吃的一笔金币事件。
- [ ] 免费生成和旧资产来源不触发。

### 金币饲料

- [ ] 基础价 `4`，额外支付只允许 `2/4/6`。
- [ ] 每额外 `2` 金产生两次顺序喂养。
- [ ] 最低总价 `6`。
- [ ] 支付不触发金币入账。

### 清仓祭典

- [ ] 当前至少一格非空。
- [ ] 所有非空货架在一个面板内预分配目标。
- [ ] 按货架左到右真实吞吃。
- [ ] 不补货、不算采购或刷新。
- [ ] 取消完整回滚。

### 换班凭证

- [ ] 选择一枚托盘骰和一枚骰袋骰。
- [ ] 新骰首次装入旧槽，旧骰进入同一托盘格并标记回收。
- [ ] 不发布第二次采购。
- [ ] 旧骰归位不重放首次装入。
- [ ] 交换后仍必须清空托盘。
- [ ] 普通卖旧装新的样本中该法术应被明确跳过。

## G. 贸易猪与饲料

- [ ] 有饲料时提供现金兑现与饲料转栏二选一。
- [ ] 现金为 `R + C`，饲料销毁。
- [ ] 转栏为 `R`，完整 `C/V/质量账本` 移到合法骰袋或托盘目标。
- [ ] 转栏不发布获得饲料，不触发肉猪、贪吃猪或质量追溯。
- [ ] 两种路线各只有一次出售与一笔金币事件。
- [ ] 无饲料时不显示空二选一。
- [ ] 目标失效时整项取消，不生成半状态。

## H. 吞吃与贡品

- [ ] 可吞吃面组使用饲料下限公式。
- [ ] 不读取龟壳持续效果、临时分、随机结果或钱包。
- [ ] 来源被吞后面组与饲料一起销毁。
- [ ] 贡品基础显示价至少 `2`，生成后锁价。
- [ ] 显式折价可以降到 `1`。
- [ ] 贡品采购后进托盘且不能装入。
- [ ] 玩家选择奉献目标或 `0` 金出售。
- [ ] 奉献是市场来源真实吞吃。
- [ ] 零售有出售事件但无金币事件。

## I. 龟壳与组合体

- [ ] 托盘龟可装入为实体或直接化壳，两者互斥。
- [ ] 直接化壳目标可在骰袋或另一托盘，且必须是普通非龟实体。
- [ ] 来源实体身份、独立计分和卖价按规则转成壳。
- [ ] 首领只读取真实化壳，复制壳不递归、无基础转移、无卖价。
- [ ] 托盘宿主可先组装再装入或退出。
- [ ] 组合体出售只算一次出售与一笔金币。
- [ ] 组合体被吞 / 拆解时壳不额外结算。
- [ ] 驮粮龟只在真实化壳时把 `floor(FeedValue/2)` 写入宿主。
- [ ] 普通龟化壳不自动读取通用饲料吞吃公式。

## J. 批量目标

- [ ] 黑帆、魔蝠和清仓共用一个批量命令组件。
- [ ] 每个根动作只打开一次面板。
- [ ] 黑帆 / 魔蝠来源可跳过，同一食物不可重复。
- [ ] 清仓每个非空来源必须分配，目标可重复。
- [ ] 三格货架下最大三次分配加一次提交。
- [ ] 提交后不再请求输入。
- [ ] 付费刷新先锁价，黑市折扣只作用下一刷。
- [ ] 离店只在托盘清空后准备。
- [ ] 固定种子恢复只产生完整前态或完整后态。

## K. 人口海盗

- [ ] 每两次符合条件的主动出售生成基础骰到第一个空托盘。
- [ ] 托盘满时生成失败且无隐藏队列。
- [ ] 招募骰来源为 `CrewRecruit`。
- [ ] 招募骰出售有普通卖价、出售和金币事件。
- [ ] 招募骰出售不推进任何人口海盗计数。
- [ ] 正常资产、贡品、采购骰、回收骰和组合体出售继续计数。
- [ ] 任意数量人口海盗都不能形成第二代招募。

## L. 供给导演

- [ ] 原始三骰完整遵循现行生成。
- [ ] 原始法术独立生成并记录。
- [ ] 方向只统计当前可支付、合法、有完整结果的行动链。
- [ ] 同一关键资产的两种后续不能虚报为两个首个承诺。
- [ ] 正常资源少于两方向时，先最多纠偏一次法术。
- [ ] 仍不足时最多替换一个非高阶保底骰。
- [ ] 高阶保底永不被替换。
- [ ] 不送金币、不保证卡名、不追发家族套件。
- [ ] 法术架满时跳过法术纠偏。
- [ ] `BridgeGap` 只在已经需要纠偏的同资格候选中破同分。
- [ ] 玩家拒绝记忆、同名记忆和同功能记忆生效。
- [ ] 自然结果与导演结果分开记录。

## M. 存档与事务

- [ ] 存档版本提升为 `13`。
- [ ] 设置不因运行存档迁移而丢失。
- [ ] V12 运行存档按批准策略清除或只从稳定非市场状态迁移。
- [ ] 完整保存骰袋、货架、法术货位、托盘、法术架、刷新、持续法术、导演记忆和随机游标。
- [ ] 每个原子动作提交前持久化 `Prepared`。
- [ ] 完整后态持久化后才清事务日志。
- [ ] 在扣款、清货、第一吞吃、壳移动和替换中间强制重载，结果只能 before / after。
- [ ] 不重复扣款、金币、吞吃、分账、招募、首次装入或首领复制。

## N. 界面与可用性

- [ ] 不看文字也能区分骰袋、托盘、货架和法术架。
- [ ] 托盘来源 `Purchase / Generated / Recovery / Stolen` 可区分。
- [ ] 离店阻塞会高亮未处理托盘格及合法出口。
- [ ] 法术购买与施放有不同反馈。
- [ ] 主动取消或无目标不消耗次数。
- [ ] 满架替换预览旧牌销毁、无退款和利息变化。
- [ ] 批量目标可取消且不会逐颗弹窗。
- [ ] 所有拖拽动作有点击等价入口。
- [ ] F021 的 V4 LED、语义亮度、实体键帽和 `UpperCenter` 提示保持。
- [ ] `1280×720` 和 `1920×1080` 长名、两位数价格和多目标面板不裁切。

## O. 行为门

至少重跑 `30` 组三市场资源序列 × 保守 / 成长 / 破局三策略：

| 指标 | 通过线 | 失败后的优先处理 |
|---|---:|---|
| 正常资源市场两方向 | `100%` | 调功能权重、资格或价格 |
| 主动透支后至少一付费动作 | `100%` | 查经济死锁，不免费补第二方向 |
| 真死店 | `0` | 修基础系统 |
| 完全不纠偏 | `>=80%` | 修原始池，不扩大导演 |
| 严格保存两店兑现 | `>=60%` | 收紧保存分类 |
| 低于 `50%` 保存兑现 | 不通过 | 不用降价掩盖 |
| 满架替换率 | `1%–5%` 观察区 | 查旧计划寿命与容量 |
| 任一贸易猪路线 | `<80%` | 查估值与入口 |
| 单吞噬贡品延后 / 放弃 | `>=25%` | 查价格与返金 |
| 托盘龟实体 | `>=20%` | 查化壳是否无代价 |
| 猪猪 / 恶魔弱桥 | `>=5%` 且早中晚出现 | 查公共中段供给 |
| 龟龟 / 海盗弱桥 | `>=10%` 且早中晚出现 | 查容量与金币接口 |
| 人口招募第二代 | `0` | 阻断上线 |
| 付费整刷下降 | `<=15%` | 收紧局部搜索资格 / 调价 |
| 高阶保底替换 | `0` | 阻断上线 |

## P. 真人试玩

- [ ] 每个正常市场玩家能说出至少两条合理方向及各自机会成本。
- [ ] 记录“真正考虑的方案数”，不把纯点击计入决策。
- [ ] 每次采购至少记录一个被放弃方案；无法填写时视为决策未成立。
- [ ] 玩家能说明为何把新骰装入、化壳、吞吃、出售或保留。
- [ ] 玩家能理解满架不是错误，而是旧计划与新计划的取舍。
- [ ] 玩家能在高阶循环中主动收手，而不是只因按钮消失停止。
- [ ] 下一次投骰后能用一句话复述市场动作如何改变主要结算链。

## Q. P0 阻断项

出现任意一项不得继续扩大内容：

- 资产、金币、法术或壳重复生成。
- 第七个长期骰或托盘跨市场残留。
- 清仓 / 换班 / 刷新恢复到半状态。
- 回收归位重复首次装入。
- 人口招募形成第二代。
- 导演覆盖高阶保底或保证指定核心。
- 托盘未清仍能离店。
- 法术占用骰子货架。
- 当前经济、生命、投骰或结算基线被意外改写。
