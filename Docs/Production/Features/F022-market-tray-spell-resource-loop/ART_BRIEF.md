# F022 美术简报

状态：已提案，需补充
功能：交易托盘、法术与市场操作扩展
视觉基线：`F021-full-flow-rhythm-sample`
现行市场方向：午夜旧机厅·后台补给柜

## 美术目标

让新增的托盘、法术和主动操作看起来像同一台旧机厅补给设备上的新增模块，而不是把酒馆战棋的卡牌界面贴到当前市场上。

玩家应在一眼内区分：

- 长期骰袋：跨市场保留、下一关会上场。
- 交易托盘：本市场临时处理、离店前必须清空。
- 骰子货架：三件可采购商品。
- 法术货位：一件一次性商品。
- 法术架：最多三张跨市场计划。
- 主动能力：骰袋中少量实例每市场一次的可操作接口。

## 必须保留

- `Assets/Resources/Art/Market/arcade_market_common_base.png` 的旧机厅补给柜材质、结构和总体色温。
- F021 已确认的 V4 LED 字形、语义亮度 V2、浅色实体键帽 V2 和全局短时提示。
- 三个骰子商品井、六格骰袋和现有顶部资源窗的视觉身份。
- `1280×720` 虚拟画布与 `1920×1080` 原生适配。
- 浅色实体键帽上的深墨实心字；暗色显示面继续使用 LED。

## 不采用

- 酒馆木桌、炉石卡背、星级边框、金色随从、冻结雪花等直接参考符号。
- 让法术卡成为比骰子商品更大的主视觉。
- 用颜色 alone 区分长期资产和临时托盘。
- 为十二张首测法术逐张制作大插画。
- 在规则未批准前改写正式底板或生成运行资源。

## 新增视觉模块

### 两格交易托盘

- 形态：两枚并列、略低于骰袋的机械检修槽，材质与当前卖出托盘一致。
- 常态：空槽有清楚的插槽深度、编号和“本市清空”短标。
- 占用：完整显示骰子类型芯、名称、价格 / 来源短标和可用出口。
- 选中：青色细边，不使用大面积发光。
- 阻塞：锈橙边框加短原因，例如“需先处理另一格”或“无合法宿主”。
- 特殊回收：在槽角显示回转箭头与“回收”字样，不能被误读为新采购。

### 一格法术货位

- 形态：独立窄票券机或打孔卡槽，不占三个骰子商品井。
- 信息：法术名、价格、主功能图标、立即可用 / 可保存 / 当前无目标三态。
- 买走后：显示空卡槽和“已取走，刷新补充”，不自动补牌。
- Boss 市场：同一槽位换用危险条纹或更强扫描线，不新增第二槽。

### 三格法术架

- 形态：三枚横向票夹或磁带槽，位于同一设备的计划区。
- 每格只显示短名、功能图标和可施放状态；完整规则进入 Tooltip。
- 满架时当前法术货位仍可见；选择新法术后进入原子替换弹层。
- 不提供常驻垃圾桶、撕卡或免费丢弃按钮。

### 主动能力

- 只有规则明确的骰袋实例出现小型“主动”插孔或拨杆。
- 可用：克制青色通电。
- 已用：保持图标但降为机械锁定态，显示“本市已用”。
- 无合法目标：禁用但可命中查看原因。
- 托盘中的同类型骰不显示可发动态。

### 批量目标面板

- 共用一套面板承载黑帆刷新、魔蝠离店和清仓祭典。
- 左列显示来源，右侧显示三格货架或骰袋目标；连线与编号表达分配。
- 默认建议可以显示为虚线，但不能自动提交。
- 清楚分离“取消”和“提交”；提交后面板关闭，队列不再索取输入。
- 三格货架下最大输入为三次目标分配加一次提交，面板不能制造逐颗弹窗。

### 原子事务反馈

- `Prepared`：预览扣款、目标、销毁 / 回收结果，使用低强度琥珀。
- `Committed`：一次短促盖章或机械锁扣反馈。
- `Recovered`：只用于调试或异常恢复提示，不在正常流程中反复出现。
- 事务过程中其它按钮整体锁定，视觉上必须说明正在结算而非界面失灵。

## 功能图标首轮集合

首轮只需要八个可复用单色符号，不按十二张法术逐张画大图：

| 功能 | 建议符号 | 用途 |
|---|---|---|
| 搜索 | 放大镜 / 扫描框 | 单格换货、定向招募 |
| 点火 | 火花插头 | 饲料袋、早期入口 |
| 续航 | 循环箭头 | 分账、资源续杯 |
| 转化 | 双向漏斗 | 拆解、金币饲料 |
| 兑现 | 出币口 | 清算、清仓 |
| 转向 | 道岔 | 折价、移壳、换班 |
| 诱惑 | 警示星 | 高价未来计划 |
| 锁定事务 | 卡扣 / 印章 | 原子替换与批量提交 |

符号应采用明快、可爱但不幼稚的 2D 扁平处理，轮廓与现有骰子类型芯一致；不新增写实金属图标。

## 信息层级

| 层级 | 内容 | 现有视觉合同 |
|---|---|---|
| L3 | 当前唯一可执行主操作、选中的提交 | F021 `L3 / FocusAction` |
| L2 | 骰子名、法术名、价格、托盘状态 | F021 `L2 / PrimaryInfo` |
| L1 | 六面、规则短句、来源、目标和阻塞原因 | F021 `L1 / SecondaryInfo` |
| L0 | 插槽编号、装饰刻字 | F021 `L0 / Ambient` |

## 验收

- 不看文字也能区分骰袋、托盘、货架和法术架。
- 法术没有抢走骰子构筑的主视觉。
- 托盘满和法术架满都表达为可处理的计划压力，不像报错。
- 特殊回收不会被误读为第二次采购。
- `1280×720` 与 `1920×1080` 下长名、两位数价格和三目标面板不裁切。
- 所有可用、禁用、已用、锁定和提交状态遵循 F021 已确认的字体与亮度合同。
