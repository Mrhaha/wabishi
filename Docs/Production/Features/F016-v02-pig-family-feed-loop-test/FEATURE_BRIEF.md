## 功能意图

F016 是 F015 的第一个单家族玩法测试切片。目标不是一次性实现四大家族，而是先把猪猪家族的饲料生产、饲料放大、饲料经济和市场找件循环单独跑通，验证这条 build 是否有足够的爽感和可读性。

本包临时屏蔽旧市场池和其它 V0.2 家族骰：市场刷新只允许出现猪猪家族骰。旧骰子、旧资源和旧存档兼容不删除，只在本测试模式的随机刷新池中下架。

## 玩家体验

玩家以六槽骰袋进入 V0.2 测试局。待机阶段可以调整槽位顺序；结算阶段按 F015-R01 从左到右触发；通关后进入市场，只看到猪猪家族骰。

猪猪循环的主要体验是：

1. 通过母猪、养猪把饲料喂到关键骰上。
2. 通过肉猪把自身吃饲料的收益放大。
3. 通过贪吃猪把“别人被喂”变成自己的成长。
4. 通过三只小猪把全队饲料转成当回合临时得分。
5. 通过贸易猪卖出转移饲料，通过批发商提升未来饲料质量。
6. 通过越来越贵的刷新费用限制找件速度，避免稳定无成本循环。

## 已确认规则

- 本包是猪猪家族单独测试包，不是 V0.2 全家族最终上线包。
- 本包不改 F015 已确认的六槽和左到右结算。
- 本包不恢复出千，不恢复小关多次出手。
- 市场在本测试模式下只刷新猪猪家族骰。
- 旧骰子、旧美术和旧存档兼容不删除，只在本测试市场池中屏蔽。
- 饲料会提高骰子的最低点，并按吃过的饲料计数提高卖出价格。
- 饲料质量提高只影响未来获得的饲料，不追溯放大旧饲料。
- 肉猪的翻倍是效果计数翻倍，不产生第二次“获得饲料”事件。
- 刷新费用每刷新一次增加 1 金币，限制玩家无限找批发商、肉猪或母猪。
- 本包不生产新美术资源，先复用现有猪猪图标、统一骰面、短标签和程序化回退。

## 建议默认值

### 饲料状态

- 每颗实体骰保存两个饲料相关值：
  - `FeedCount`：吃过饲料的次数，用于卖价加成。
  - `FeedValue`：饲料带来的最低点提升总量，用于结算最低点和三只小猪读取。
- 初始 `FeedQuality = 1`。
- 获得 1 份饲料时，目标骰：
  - `FeedCount += 1`
  - `FeedValue += 当前 FeedQuality`
- 有效点数默认按 F015 饲料下限计算，不改写自然面组。
- 卖出价格默认为基础卖价 + `FeedCount`，肉猪吃 1 份饲料后因效果计数翻倍，`FeedCount` 增加 2。

### 结算队列

- 本包继续使用 F015-R01。
- 单槽自身效果引起多个投喂时，所有投喂和响应都属于当前槽位局部连锁队列。
- 同一时点多个响应按当前六槽从左到右入队。
- 三只小猪只在主游标到达自身槽位时读取当时已经结清的其它骰状态，不提前读取未来槽位。
- 贪吃猪响应是防递归边界，不是次数上限：一个市场或结算动作可以触发多次贪吃猪，但贪吃猪响应产生的饲料不再作为新的贪吃猪触发源。

### 市场

- 新增临时测试开关，例如 `v02_pig_family_market_only = 1`。
- 开关开启时，市场候选类型池只包含：养猪、肉猪、贸易猪、母猪、三只小猪、贪吃猪、批发商。
- 购买后补货、手动刷新、章节市场、倾向槽位和测试随机池都必须走同一个过滤规则，避免旧骰从旁路刷出。
- 刷新费用本市场内每刷新一次增加 1；离开市场后下一市场按基础刷新费用重新开始，除非 F015 或后续数值包另行确认。
- 首版不新增“饲料换金币”的新转换按钮，经济只来自卖价、通关金币、既有金币规则和后续市场价格。

## 验收含义

F016 通过意味着：玩家可以在只刷猪猪家族骰的市场里测试饲料内循环；旧骰不会从市场旁路刷出；饲料能同时影响最低点和卖价；肉猪、母猪、养猪、贪吃猪、三只小猪、贸易猪和批发商能形成可观察的成长与经济循环；循环不会因为贪吃猪递归或刷新套利直接失控。
