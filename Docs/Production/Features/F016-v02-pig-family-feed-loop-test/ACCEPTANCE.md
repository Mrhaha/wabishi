## 功能验收

- [ ] 开启猪猪测试模式后，市场只刷新猪猪家族骰。
- [ ] 购买后补货只补猪猪家族骰。
- [ ] 手动刷新只刷猪猪家族骰。
- [ ] 章节市场和隐藏倾向槽位不会绕过过滤刷出旧骰。
- [ ] 旧骰子、旧数据、旧资源没有被删除。
- [ ] 饲料提高有效点数下限。
- [ ] 饲料计数提高卖出价格。
- [ ] 批发商只影响后续饲料质量，不追溯已有饲料。
- [ ] 刷新费用每次刷新后 +1。
- [ ] `Ready` 阶段和市场骰袋列表都可拖拽插入排序，用于测试不同猪猪槽位链路。
- [ ] 拖拽排序只改变从左到右扫描顺序，不改变饲料计数、饲料价值、卖价或骰子 ID。

## 猪猪骰验收

- [ ] 养猪结算时给左右相邻实体骰各 +2 份饲料。
- [ ] 肉猪吃饲料时效果计数翻倍。
- [ ] 肉猪翻倍不生成第二次获得饲料事件。
- [ ] 贸易猪卖出时按规则转移自身饲料。
- [ ] 母猪结算时给所有实体骰各 +1 份饲料。
- [ ] 三只小猪结算时获得其它骰总饲料点数作为本次临时分。
- [ ] 贪吃猪在其它骰获得饲料时获得 1 份饲料。
- [ ] 多颗贪吃猪不会互相递归到无限队列。
- [ ] 批发商买入时饲料质量 +1。

## 队列验收

- [ ] 结算遵守 F015-R01：当前槽位局部连锁清空后才进入下一槽。
- [ ] 后槽位自身结算效果不会因为前槽投喂连锁提前触发。
- [ ] 同一时点多个响应按当前六槽左到右入队。
- [ ] 三只小猪只读取自身槽位触发时已经结清的状态。
- [ ] 买入动作在新骰入袋后，按当前骰袋槽位从左到右扫描一轮市场触发。
- [ ] 卖出动作先结算卖价并移除骰子；被卖骰如有卖出效果，按卖出前原槽位参与本次虚拟扫描。
- [ ] 购买后补货和卖出后的 UI 选择变化不会回头触发新的买入 / 卖出事件。
- [ ] 市场骰袋拖拽排序后，后续买入 / 卖出扫描按新顺序执行。
- [ ] 结算演出默认速度能看清猪猪投喂、肉猪翻倍、贪吃响应和三只小猪得分链条；`1.5x / 2.0x` 只加速展示，不改变队列顺序。
