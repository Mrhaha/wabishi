# F016 界面体验规格

状态：已提案，待界面审查
功能：V0.2 猪猪家族饲料内循环测试
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 本包是猪猪家族单独测试模式。
- 市场只刷新猪猪家族骰。
- 饲料影响最低点和卖价。
- 批发商提高全局饲料质量。
- 结算按 F015-R01 左到右主游标和当前槽位局部连锁队列。

## 体验意图

玩家需要不读日志也能判断三件事：

1. 当前市场为什么只出现猪猪骰。
2. 哪颗骰吃了多少饲料，最低点和卖价变成多少。
3. 本次结算里哪颗猪猪骰触发了投喂、放大、汇总或贪吃响应。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 待机 | 调整猪猪结算顺序 | 六槽、骰名短标签、饲料计数、最低点 | 调整槽位、开始投骰 | 查看悬浮窗 | 进入结算 |
| 结算 | 看懂猪猪链条 | 当前槽位高亮、投喂飞字、贪吃响应、三只小猪临时加分 | 等待结算 | 快进如已有通用能力 | 达标进市场，未达标失败 |
| 市场 | 找猪猪组件和管理经济 | 猪猪测试池标记、金币、刷新费用、当前饲料质量、卖价 | 买入、卖出、刷新、离开 | 查看悬浮窗 | 下一小关 |
| 悬浮窗 | 理解单骰规则 | 名称、短效果、FeedCount、FeedValue、最低点、卖价 | 悬停阅读 | 无 | 鼠标离开 |

## 信息层级

主投掷区优先级：

1. 当前骰结果和结算高亮。
2. 类型短标签。
3. 饲料计数和最低点短标，例如 `饲 4 / 底 5`。
4. 触发短反馈，例如 `母猪 +1 全体`、`肉猪 x2`、`贪吃 +1`。

市场优先级：

1. 金币和刷新费用。
2. 当前饲料质量，例如 `饲料质量 2`。
3. 商品名称、价格、短效果。
4. 卖出区显示卖价构成，例如 `卖 5 = 基础 2 + 饲料 3`。
5. 轻量状态标记：`猪猪测试池`。

## 交互流程

- 待机排序仍按现有六槽调整方式，不新增单独编辑页。
- 市场刷新按钮显示当前刷新费用；每次刷新后费用 +1。
- 买入批发商后，当前饲料质量立即更新。
- 卖出吃过饲料的骰时，卖价显示必须包含饲料加成。
- 卖出贸易猪时，若触发饲料转移，至少用短反馈指出饲料去向。

## 反馈和动画

- 投喂反馈：目标骰旁显示 `+饲料` 和本次质量，例如 `+2 饲料` 或 `+1x2`。
- 肉猪反馈：在自身旁显示 `肉猪 x2`，不额外播放第二次获得事件。
- 贪吃猪反馈：在贪吃猪旁显示 `贪吃 +1`；多次响应可以合并为 `贪吃 +N`。
- 三只小猪反馈：显示读取总量，例如 `三只小猪 +18`。
- 批发商反馈：市场顶部饲料质量跳涨。
- 刷新反馈：刷新费用数字跳涨，避免玩家误以为刷新免费。

## 边界情况

- 多颗贪吃猪响应时，普通触发可合并显示，底层仍按队列结算。
- 如果贸易猪卖出时没有其它实体骰，显示饲料消失或不触发转移。
- 如果商品补货因过滤池较小出现重复猪猪骰，允许重复；不需要为了去重而刷出旧骰。
- 如果饲料让结果超过 6，直接在骰面底图上显示居中数字，不再绘制密集点阵。

## 适配 / 布局约束

- 不新增大面板。
- 不把饲料详情常驻堆满主投掷区，详细构成放悬浮窗。
- 主投掷区短标最多两段：类型短标签 + 饲料 / 底线短标。
- 市场测试标记应轻量，不遮挡商品和操作按钮。

## 实现备注

- 可以先用 OnGUI 文本短标完成测试表达。
- 不要求新增 UI 图片。
- 若 F014 悬浮窗字段不足，优先补动态文本，不改整体布局。

## 阻塞项

无。界面实现后仍需截图验收。

## 验收

- 玩家能在市场一眼看出当前是猪猪测试池。
- 玩家能看出刷新费用每次递增。
- 玩家能看出当前饲料质量。
- 玩家能看出吃过饲料的骰卖价上涨。
- 玩家能在结算中追踪母猪、养猪、肉猪、贪吃猪和三只小猪的主要触发。
