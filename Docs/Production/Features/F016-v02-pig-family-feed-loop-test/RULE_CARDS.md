## F016-R01 猪猪测试市场池

本包开启猪猪单家族测试模式后，所有市场随机刷新入口只允许从猪猪家族池抽取商品骰。

允许类型：

- 养猪
- 肉猪
- 贸易猪
- 母猪
- 三只小猪
- 贪吃猪
- 批发商

屏蔽范围：

- V0.1 旧骰子。
- F008 金币收益族。
- F010 奇偶短规则骰。
- F011 大树接口骰。
- F015 中尚未进入本包的恶魔、龟龟、海盗骰。

屏蔽只影响本测试模式的市场刷新池，不删除代码、数据、资源或存档兼容。

验收：

- 新市场、购买后补货、手动刷新、章节市场、隐藏倾向槽位和测试全池随机都不能刷出非猪猪骰。
- 关闭测试开关后，旧市场池可以按后续版本规则恢复。

## F016-R02 饲料质量和饲料状态

首版饲料拆为计数和价值：

- `FeedCount`：吃过饲料的计数，用于卖价。
- `FeedValue`：饲料带来的最低点提升总量，用于有效点数下限和三只小猪读取。
- `FeedQuality`：全局饲料质量，初始值为 1。

获得 1 份饲料时：

```text
FeedCount += 1
FeedValue += 当前 FeedQuality
```

卖出价格：

```text
SellPrice = BaseSellPrice + FeedCount
```

有效点数下限沿用 F015 建议默认值：饲料不改写自然面组，只提高结算时有效点数下限。

验收：

- 批发商买入后新增的饲料使用新质量。
- 批发商买入前已经获得的 `FeedValue` 不被追溯放大。
- 肉猪吃 1 份饲料时可以让 `FeedCount` 和 `FeedValue` 按效果计数翻倍，但不能生成第二次“获得饲料”事件。

## F016-R03 结算队列

本包所有结算期效果必须遵守 F015-R01：

1. 主游标按当前六槽从左到右推进。
2. 到达当前槽位后，先触发当前槽位自身效果。
3. 当前槽位效果产生的所有响应进入当前槽位局部连锁队列。
4. 局部队列清空前，不推进下一槽。
5. 同一时点多个响应按当前六槽从左到右入队。

猪猪特化边界：

- 三只小猪只在主游标到达自身槽位时读取当时已经结清的其它骰饲料状态。
- 养猪、母猪产生的投喂响应在当前局部队列内处理。
- 肉猪翻倍不产生新的获得饲料事件。
- 贪吃猪响应获得饲料不再触发其它贪吃猪响应。

验收：

- 槽位顺序改变后，同一组猪猪骰结算结果和触发归因应随顺序改变。
- 后槽位自己的结算效果不会因为前槽位投喂连锁提前自发触发。
- 多颗贪吃猪不会因互相响应进入无限队列。

## F016-R04 猪猪单骰规则

| 骰子 | 触发 | 效果 | 边界 |
|---|---|---|---|
| 养猪 | 自身结算触发 | 左右相邻实体骰各获得 2 份饲料 | 不存在的相邻槽不触发 |
| 肉猪 | 自身获得饲料时 | 自身获得饲料的效果计数翻倍 | 不算第二次获得饲料 |
| 贸易猪 | 卖出时 | 自身饲料随机转移给其它实体骰 | 建议默认：卖价先按自身饲料计数结算 |
| 母猪 | 自身结算触发 | 所有实体骰各获得 1 份饲料 | 包含自身 |
| 三只小猪 | 自身结算触发 | 本次临时得分增加其它骰总 `FeedValue` | 不写入长期饲料 |
| 贪吃猪 | 其它骰由原始投喂源获得饲料时 | 自身获得 1 份饲料 | 贪吃猪响应源不再触发贪吃猪 |
| 批发商 | 买入时 | `FeedQuality += 1` | 只影响后续饲料 |

## F016-R05 市场反套利边界

本包不新增饲料换金币按钮，也不新增自动买卖转换。

市场动作队列：

1. 每次玩家完成一次买入或卖出，生成一个市场动作事件。
2. 该事件只结算一轮，从当前骰袋槽位左到右扫描触发。
3. 买入事件在新骰进入骰袋后扫描，因此新买入骰按其当前槽位参与本次触发。
4. 卖出事件先按卖出骰当前卖价结算金币并从骰袋移除；如果被卖骰自身有卖出效果，它按卖出前原槽位插入本次虚拟扫描队列，只参与这一次事件。
5. 购买后的补货、手动刷新出的新货架、卖出后的选择变化不反向生成新的买入或卖出事件。
6. 同一市场动作中多个响应按当前六槽从左到右入队；首版每个动作只扫一轮，避免无限买卖或无限刷新连锁。

已确认限制：

- 刷新费用本市场内每刷新一次增加 1。
- 找批发商、肉猪、母猪和贸易猪依赖市场随机刷新。
- 卖出高饲料骰会失去一个已成长实体骰或转移其成长，不是无损复制。

建议默认限制：

- 购买后补货不回头触发当前买入事件。
- 批发商买入只影响后续投喂。
- 贸易猪卖出后的饲料转移只分配给剩余实体骰；如果没有剩余实体骰，则饲料消失。

验收：

- 连续刷新需要支付递增金币，不能稳定用肉猪卖价覆盖无限刷新。
- 批发商不能追溯放大已经吃过的饲料。
- 贸易猪卖出不会把同一份饲料无限复制。
