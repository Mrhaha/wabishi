# F016 程序交接

状态：已静态接入，待 Unity 验证
功能：V0.2 猪猪家族饲料内循环测试
关联主文档：PROJECT_CONTEXT.md、GAME_FLOW.md、DICE_ARCHETYPES.md
实现事实来源：Assets/Scripts/DiceKingDemo.cs

## 输入决策

- 只做猪猪家族测试切片。
- 测试模式下市场只刷新猪猪家族骰。
- 旧骰和其它家族骰只屏蔽市场刷新，不删除兼容。
- 保留六槽和左到右结算。
- 饲料提高最低点，并按饲料计数提高卖价。
- 肉猪翻倍不产生第二次获得饲料事件。
- 批发商只影响后续饲料质量。
- 刷新费用每次刷新 +1。
- 不生成新美术资源。

## 实现目标

程序实现应能让用户单独试玩猪猪家族内循环：

1. 开启 V0.2 猪猪测试模式。
2. 市场只刷 7 颗猪猪骰。
3. 饲料状态可保存、可显示、可参与计分和卖价。
4. 猪猪 7 颗骰能按 F016 规则触发。
5. 结算队列遵守 F015-R01。
6. 贪吃猪不会递归到无限队列。
7. 连续刷新成本递增，不能稳定无成本找件。

## 已确认行为

- 测试市场池只包含养猪、肉猪、贸易猪、母猪、三只小猪、贪吃猪、批发商。
- `FeedQuality` 初始为 1。
- 批发商买入时 `FeedQuality += 1`。
- 已获得的 `FeedValue` 不受后续批发商影响。
- 肉猪获得 1 份饲料时，饲料效果计数翻倍，但不生成第二个获得事件。
- 卖价按 `BaseSellPrice + FeedCount` 计算。
- 刷新费用在当前市场内每次刷新后 +1。
- 买入和卖出作为市场动作事件处理，每次动作按当前骰袋槽位从左到右扫描一轮触发。
- 买入事件在新骰入袋后扫描；卖出事件保留被卖骰的原槽位作为本次虚拟扫描位置。
- `Ready` 阶段和市场骰袋列表都可拖拽插入排序；排序只改变猪猪链路的从左到右扫描顺序，不改变骰子 ID、饲料计数、饲料价值或卖价。
- 没有新美术资源时必须可玩。

## 代码影响

- `DieType`：新增 7 个猪猪测试类型，或使用可配置 subtype；优先选符合当前代码结构的低风险方案。
- 市场生成：增加测试模式过滤池，覆盖商品刷新、购买后补货、章节市场和隐藏倾向槽位。
- 计分：增加饲料下限计算，并接入 F015-R01 队列。
- 经济：卖价读取 `FeedCount`；刷新费用在当前市场内递增。
- 存档：增加饲料状态；若 V0.2 测试和 V0.1 存档结构冲突，优先清空旧运行存档，保留设置。
- UI：补短标签和悬浮窗动态字段。
- UI：骰袋顺序调整使用拖拽插入排序，单击只选中；结算中、失败后和结果锁定后不允许改顺序。
- UI：猪猪链条测试默认用清晰慢速结算演出，设置页可选 `1.5x / 2.0x` 加速；加速只影响展示时长，不改变投喂、响应、得分或市场扫描顺序。

- `FeedValue`

当前实现：运行存档版本已升至 `SaveVersion = 6`，`DiceData` 追加保存 `FeedCount` 和 `FeedValue`，运行状态保存 `FeedQuality`；同版本也清理 V0.2 单次出手、无出千、无牌型计分的旧运行存档兼容边界。
