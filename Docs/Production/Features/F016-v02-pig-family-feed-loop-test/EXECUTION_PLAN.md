## 执行顺序

1. 使用 `$wabish-packet-review` 审查本包，重点看贪吃猪防递归、贸易猪卖出双收益、三只小猪读取口径和测试市场过滤。
2. 使用 `$wabish-dev-implementation` 实现测试开关和市场过滤。
3. 接入饲料状态、饲料质量、卖价和基础显示。
4. 实现 7 颗猪猪骰效果和 F015-R01 队列。
5. 做一轮猪猪内循环试玩，打印或记录过程。
6. 根据试玩决定是否缩减、增强或进入第二家族。

## 依赖关系

- F016 依赖 F015 的 V0.2 三态循环、饲料下限和结算队列规则。
- 如果 F015 基础还没实现，F016 程序可以作为临时测试路径实现，但必须保持代码边界清楚，不能把测试逻辑散落到 V0.1 正式流程。
- 本包不依赖新美术。

## 建议默认值

- 初始测试局保留 6 基础骰，市场只刷猪猪。
- 贪吃猪不响应贪吃猪响应源。
- 三只小猪读取其它骰 `FeedValue`。
- 贸易猪卖出先计算包含饲料的卖价，再转移饲料给剩余骰。
- 刷新费用每个市场内从基础费用开始，每刷新一次 +1。

## 阻塞问题

无。上述建议默认值足够执行首轮测试；若审查认为贸易猪双收益过强，可先改为卖出后饲料消失或只转移不计卖价。

## 验证计划

- 市场过滤样例：连续刷新和购买后补货都只出现猪猪。
- 结算队列样例：`母猪 -> 肉猪 -> 养猪 -> 贪吃猪 -> 三只小猪`，观察饲料和分数。
- 经济样例：买批发商，提高饲料质量，喂肉猪，卖出验证卖价上涨。
- 反递归样例：两颗贪吃猪同时在场，母猪投喂后队列正常结束。
- 反套利样例：连续刷新找批发商，刷新费用递增后不能稳定免费循环。

## 交接备注

本包的目标是获得试玩判断，不是把猪猪家族定成最终全量。程序实现应保留开关，便于后续切回 F015 四家族混合市场或拆掉过强的猪猪组件。
