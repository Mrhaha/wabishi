# F006 美术需求

状态：已确认
功能：F006 结算演出与槽位顺序
关联美术文档：PROJECT_CONTEXT.md、ART_ASSETS.md

## 输入决策

- F006 首版重点是表现节奏，不强制新增运行时美术资源。
- 当前项目稳定视觉方向为明亮账本桌游风。
- 结算演出可使用槽位编号、账本行、高亮框、金币入账、印章落点和目标收束表达顺序。
- 不要让点数摘要、装饰边框或说明面板喧宾夺主。
- F007 已确认左端积分塔、放大投掷区和连续留白，F006 不推翻该布局。

## 视觉意图

F006 的视觉目标不是做大量新图标，而是让当前结算更像“王室账本过账”。每一颗骰子被点名时，画面像账本逐行入账；倍率成立时像盖章；通关时像账本被正式批复；未达标时则保持克制，只显示还差多少，不做胜利式爆点。

## 风格关键词

- 明亮账本桌游风。
- 王室账本、蜡封、印章、金币袋、纸页入账。
- 轻快、清楚、克制。
- 读数优先，装饰退后。

## 必须保留

- F007 左端积分塔的主结构。
- 中央投掷区的连续留白。
- 现有骰子类型图标和运行时骰面点数可读性。
- 基础分数字、倍率章和目标数字的阅读层级。

## 避免

- 不要加入大面积遮罩块或独立表演面板。
- 不要在待投骰或结果决策静态阶段预画大量结算轨迹。
- 不要用生成文字替代 Unity 文本。
- 不要做赌场霓虹、写实筹码、夸张老虎机风。
- 不要让金币、印章或光效遮住骰面点数。
- 不要生成幼态吉祥物或大角色表演。

## 生成 / 来源备注

首版可以完全不生成新资源。若审查后决定需要接触图，应使用 `$wabish-art-production` 或 `$wabish-art-assets`，先做一张 1920x1080 结算态接触图，验证倍率盖章、金币飞行和通关收束的视觉层级。接触图应放在 `Assets/ArtSource/Production/F006/`，不直接进入 `Assets/Resources/Art/`。

## 放置 / 运行时用途

- 当前槽位高亮：围绕正在结算的骰子绘制，不改变骰子位置。
- 金币入账：从当前骰子或短标签附近飞向左侧金币资源牌。
- 倍率盖章：围绕左端倍率章发生，不遮挡基础分数字。
- 目标收束：围绕左端目标数字或当前积分数字发生。
- 成长入档：在当前大树骰附近短暂显示，随后消失。

## 实现备注

程序首版可以使用 `DrawRect`、`DrawBorder`、缩放、透明度、短浮字和现有图标完成。美术不应要求程序一次性接入复杂粒子、骨骼动画或外部动画系统。

## 阻塞项

无硬阻塞。首版不新增资源即可执行。

## 验收

- 结算期当前槽位焦点明确。
- 金币、倍率、成长和通关反馈能被区分。
- 动效不遮挡骰面点数、基础分数字、倍率章和目标数字。
- 通关收束比未达标收束更有仪式感。
- 整体仍符合明亮账本桌游风，不偏赌场或幼态风。
