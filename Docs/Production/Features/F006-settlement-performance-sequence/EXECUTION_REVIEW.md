## 摘要

F006 的首版方向成立：在现有 F007 双数字计分器和左到右跳涨基础上，把结算升级为结果定格、逐槽点名、路线小爆点、倍率盖章、目标收束。该方案不需要新增运行时资源，不需要改数据表、存档、骰面概率、计分公式或市场规则。

审查后的关键要求已写入生产包：动画层只能消费 `ScoreDice()` 后冻结出的提交态和由提交态派生的展示事件。播放、加速、重绘或跳过都不得再次调用会影响结果的随机、计分、钱包扣加、贿赂判断或成长入档逻辑。

1. 已批量应用本审查建议到生产包文档，未改运行时代码。
