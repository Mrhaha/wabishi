## 功能意图

F006 解决结算演出仍然偏素、分数来源归因不够清楚的问题。当前 F007 已经接入左端双数字计分器和左到右跳涨，但玩家看到的主要还是数字变化，缺少开场定格、当前槽位焦点、路线触发、倍率重击和通关收束这些情绪层级。

本包目标是把结算从“逐颗跳字”升级为“五拍结算舞台”：结果定格、逐槽点名、路线小爆点、倍率盖章、目标冲刺与收束。该升级只改变展示和节奏，不改变真实计分公式、骰面概率、出千规则、目标曲线、经济、存档或骰子效果。

2026-07-23 V7-R3 增量把最终目标收束扩展为隐藏五档导演层：未达到反向泄压、过关单拍闭环、超过双拍继电、远超过可控整机过载、最高暴击真空击穿。最高暴击独占“锁死—吸光—近静默真空—单次击穿—余震”，前四档不能只做亮度缩放，也不能借用最高档的近黑与全屏击穿。该增量继续只消费冻结后的展示事件。

F006-V7R3-FB-004 已进一步把运行时第 2 / 3 / 4 档拆成三种主轮廓：Pass 只做目标线局部单扣，Exceed 使用空间分离的上轨去程 / 下轨回程双继电，FarExceed 使用整机持续负载、12 格热芯保持与一次回坐卸压；Critical 的近黑真空和全屏击穿保持独占。

F006-V7R3-FB-005 只重构第 5 档的受力反差：爆发前机柜内容先下沉并纵向压缩，周边流动在积分塔小爆心处锁死；击穿不再提前泄漏，也不再以横向铺满为主，而是从塔芯向上顶穿、上冲过冲、反向回落后进入余震。前四档、五档阈值、总时长、计分、奖励、事件与存档保持不变。

## 玩家体验

玩家确认结算后，骰子不再重新排序，仍按投骰前的固定槽位从左到右过账。每一颗骰子被点名时，投掷区只突出当前槽位、当前分数变化和最重要的触发标签。普通骰快速入账，路线关键效果慢半拍，倍率成立时以更重的倍率章反馈把分数推高，最后目标数字滚动到真实结果。

玩家应该能在结算结束后说出本次高分主要来自哪几个节点，例如“第三颗盖章触发”“第五颗形成顺子倍率”“龟龟补了几颗小骰”“最后贿赂补到过关”。演出要让构筑贡献被看见，而不是让所有触发都挤成日志。

## 已确认规则

- 投骰前的 `Ready` 阶段允许玩家调整骰子槽位。
- 开盅、孤证重摇、出千重摇和结算开始后，不按点数重排骰子。
- 左到右结算指当前骰袋槽位顺序。
- 结算演出必须读取真实结算后的提交态或由提交态派生的展示事件，不得在动画播放中改变随机、计分或钱包结果。
- 五拍结算舞台的顺序为：结果定格、逐槽点名、路线小爆点、倍率盖章、目标冲刺与收束。
- 当前槽位是结算期唯一主焦点，同一时刻不要让所有骰子、日志和目标数字一起闪。
- 每个槽位旁只显示最重要的短反馈，例如 `+6`、`盖章 6`、`金币 +1`、`成长待入档`。
- 通关收束和未达标收束必须有视觉差异；通关可以盖章，未达标保持安静。
- 大树成长应在分数落定后表现为“成长入档”，避免玩家误以为成长已经影响本次基础点数。

## 建议默认值

- 首版使用现有 OnGUI、现有骰子图标、现有投掷区和左端计分器，不新增运行时贴图。
- 普通槽位入账节奏约 0.18 到 0.25 秒，路线高光额外停顿约 0.15 到 0.25 秒，最终收束约 0.35 秒。
- 逐骰阶段沿用当前事件时长；五档终局分别为 `1.70s / 1.55s / 1.65s / 1.85s / 2.20s`，完整整手时长以用户实机回归为准。
- `Space` 加速不进入首轮；若后续执行 F006-04，按住或再次按下后可以缩短剩余演出，但必须直接落到同一最终提交态。
- V7-R3 已接程序生成的蓄压、击穿和余震音色，不新增外部音频资源；独立桌面震动继续后置。
- 点数摘要首版沿用现有牌型提示和牌型表入口，不新增常驻摘要条。

## 职能交接

### 美术

首版不强制新增运行时资源。美术只需要约束视觉语言：明亮账本桌游风、账本盖章、金币飞行、倍率章压印、成长入档。若后续需要新贴图，应先由 `$wabish-art-production` 生成资源清单和接触图，再写入 `ART_ASSETS.md`。

### 界面体验

界面体验负责把五拍拆成可读状态。当前槽位、基础分、倍率和目标数字要按同一节拍脉冲。路线触发只能短提示，不要把日志和说明面板塞回主流程。

### 程序

程序应在 `Assets/Scripts/DiceKingDemo.cs` 中沿用当前 `scoringDice`、`RunScoreCounterStep`、`BeginSettle()` 和 `UpdateScoreReveal()` 的提交态基础，新增展示事件或扩展现有展示快照。动画播放、加速和最终落点必须消费同一份提交态。

## 不要改动

- 不恢复按点数排序结算。
- 不改变投骰概率和骰面随机。
- 不改变真实分数、金币、成长或贿赂结果。
- 不改变 F007 左端积分塔的主结构。
- 不新增常驻第三个主数字。
- 不把临时小骰折叠规则改成全量拖长播放。

## 验收含义

F006 验收通过意味着：结算从当前单纯逐颗跳字升级为有明确节奏的五拍舞台；玩家能看到当前槽位、路线触发、倍率成立和目标达成的先后关系；最终显示和真实结算结果一致；普通结算不拖慢，爆发结算有清晰爽点；主流程没有新增规则负担。
