# F006 界面体验规格

状态：V7-R3 与 FB-001 至 FB-005 五项小修已实现并完成双分辨率、方向数值与降低特效自动验证，待用户实机体验验收
功能：F006 结算演出与槽位顺序
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md、Docs/Production/Features/F007-main-run-ui-score-counter/UI_UX_SPEC.md

## 输入决策

- 结算顺序按当前骰袋槽位从左到右。
- 结果锁定后不按点数重排。
- F007 左端积分塔、双数字计分器和放大投掷区是当前基础。
- F006 使用五拍结算舞台强化结算表现。
- 动画只展示真实提交态，不改变结果。
- V7-R3 使用程序音效和程序化终局，不新增外部音频 / 贴图；独立桌面震动仍后置。
- 首轮 F006-02 / F006-03 默认不实现 `Space` 加速；F006-04 作为后置增强切片。

## 体验意图

玩家确认结算后，界面要把注意力从“我投出了什么”自然引到“这些骰子如何入账”。每一拍只服务一个焦点：先停住结果，再点名槽位，再显示路线触发，再重击倍率，最后让目标数字落地。

V7-R3 在最后一拍增加隐藏五档导演层。结果不再一进入 `TargetSettle` 就直接揭示，而是在对应档位的动作释放后出现；最高暴击先主动吸走常亮层并留出近黑真空，再只击穿一次，随后进入余震回亮。

## V7-R3 五档终局

| 档位 | 终局时长 | 动作语义 | 结果揭示 | 专属边界 |
|---|---:|---|---|---|
| 未达到 | `1.70s` | 能量向回泄压、局部回撤 | 反向释放后 | 不做庆祝式闭环 |
| 过关 | `1.55s` | 目标线左右夹合的一次局部单扣 | 单拍落稳后 | 不绘制整机外框、托盘扫光或 CRT 切片 |
| 超过 | `1.65s` | 上轨左至右去程、下轨右至左回程的双继电 | 第二拍后 | 不保持全机柜常亮，不扩展为持续过载 |
| 远超过 | `1.85s` | 12 格热芯与整机柜持续持压，峰值后一次回坐 / 卸压 | 负载峰值后 | 不使用近黑真空、全屏击穿、射线或碎片 |
| 最高暴击 | `2.20s` | 下蹲压缩、锁死小爆心、向上顶穿、上冲过冲、反向回落、余震 | 击穿后 | 独占近黑、近静默、纵向白金顶穿、射线、碎片和强后坐 |

统一 `1.0x / 1.5x / 2.0x` 演出倍率同时压缩视觉和程序音效。开启降闪 / 降镜头运动后，白闪、CRT、射线和后坐按配置缩减，但结果、档位动作方向和声音身份必须保留。

第 2 / 3 / 4 档的主识别顺序固定为“点 → 线 → 面”：Pass 的峰值只占目标线局部，Exceed 必须能分别看到去程与回程两条空间路径，FarExceed 必须让积分塔、骰盘、终端与机柜外框共同保持负载并在末段回坐。颜色和亮度只辅助动作，不承担档位身份。

第 5 档的主识别不是“面再放大”，而是受力方向反转。真空前机柜内容要持续向下沉并纵向压缩，横向流动在积分塔中央小热核处停止；击穿前不得出现释放射线、全屏波或持续向右扫光。爆点后先从塔芯形成窄而快的向上通道，再展开上半屏扇形冲击；机柜内容必须越过原位形成向上顶点，随后反向回落并以更小余震收束。横向冲击只能是短促次级脉冲。

普通通关的 `TargetSettle` 是独立表现边界：进入时不得携带上一笔贡献的达标十字或积分塔扫描。Pass 只在积分塔目标线附近形成一次暖色、硬边、可追溯的机械接触，不在目标图标上增加青白漫射圆光，不绘制贯穿积分塔的全高亮柱；完成后若保留边框回声，最长为 `0.18s`，不得继续填充整块塔内区域。

除 Pass 的目标线短接点外，结算阶段的持续塔内能量必须服从积分塔 12 格实体灯结构：外晕、主体与温白热芯均逐格绘制并保留格间断口，移动继电头只能在单格内形成短横火花。Exceed 使用规则继电，FarExceed 可在降效设置约束下轻微错位，Critical 可在瞬时击穿时扩展为分叉放电；任何档位、BribeFinal 或终局残留都不得重新用贯穿塔灯的实心长矩形承担能量感。

普通 `SlotScore / BribeFinal` 的入账余辉属于逐次因果反馈，不是终局档位。它应在约 `0.46s` 内让暖色电流头沿积分塔左右边沿移动：正贡献自下向上，负贡献反向向下；每侧允许短外晕、温白热核和分段尾迹，但不得持续点亮整条边框、填满塔内、扩大到机柜其它区域或抢占 `TargetSettle` 的爆发焦点。进入 `TargetSettle` 时仍必须立即收口该余辉。

## 信息层级

1. 当前正在结算的骰子槽位。
2. 左端基础分数字和倍率章。
3. 当前积分和目标积分。
4. 路线短反馈。
5. 左侧资源牌。
6. 右侧规则与牌型表入口。
7. 日志。

日志只作为兜底，不作为结算演出的主说明。路线短反馈最多保留一到两个词或一个数字变化。

## 交互流程

1. 玩家在结果决策阶段按 `Space`。
2. `BeginSettle()` 或等效入口冻结真实提交态。
3. 界面进入短暂停格，所有骰子保持槽位顺序。
4. 演出按槽位从左到右推进。
5. 当前槽位高亮并轻抬，短浮字显示本槽最重要变化。
6. 基础分数字跳涨；如当前节点触发倍率，倍率章随后重击。
7. 若有金币、龟龟、成长或贿赂等路线事件，插入短高光。
8. 所有槽位播放完后，目标数字滚动到最终提交态。
9. 通关进入通关收入流程；未达标且还有出手回到等待投骰；失败进入失败流程。

首轮实现期间，结算播放中不新增加速输入。若后续执行 F006-04，加速只允许缩短展示时间，并且必须落到同一最终提交态。

## 反馈和动画

- 结果定格：约 0.25 到 0.35 秒，不遮挡骰子。
- 当前槽位：描边、轻抬、短促缩放或纸页入账闪光。
- 普通入账：快速浮字和基础分脉冲。
- 金币入账：小金币或金币图标从骰子附近飞向资源牌。
- 龟龟临时小骰：按现有展示上限入账；超过展示上限时折叠为汇总反馈。
- 大树成长：分数落定后出现“成长入档”短反馈。
- 倍率盖章：倍率章放大、压印、短暂停顿。
- 通关收束：目标数字越线后盖章或账本批复。
- 未达标收束：保持克制，显示最终累计，不使用通关式盖章。

## 边界情况

- 无牌型时，倍率保持 `×1`，五拍流程仍应成立但更快。
- 多个普通触发连续出现时，不逐条拖慢；可合并低价值反馈。
- 临时小骰超过展示上限时，折叠行入账，不逐颗拖长。
- 贿赂补分不进入倍率章，只作为最终补分短反馈。
- 通关后不允许继续追分，沿用当前流程。
- 未达标回到等待投骰时，可以继续保留上一手基础分和倍率，直到下一次起摇清零。
- 若后续执行 F006-04，加速或跳过必须直接落到最终提交态。

## 适配 / 布局约束

- 继续遵守 F007 G 版左端积分塔和投掷区布局。
- 设计基准为 `1920x1080`，当前 OnGUI 仍按 `1280x720` 内部坐标映射。
- 动画不得造成文本超框、骰面遮挡或按钮遮挡。
- 不新增常驻大面板，不挤压中央投掷区。
- 浮字不能长时间停留，也不能堆叠成阅读负担。
- 最高档允许极短全屏覆盖，但击穿只能是单次主峰；覆盖结束后必须恢复既有信息层级。

## 实现备注

界面体验建议以展示事件为单位，而不是直接在绘制函数里散落判断。每个事件至少需要事件类型、槽位、计分索引、数值变化、是否高光、持续时间、短反馈文本和目标区域。程序可以先用硬编码默认时长，后续再决定是否数据化。

## 阻塞项

无硬阻塞。是否支持 `Space` 加速属于增强切片；当前只待用户回归第 5 档正常速“下蹲—锁死—顶穿—回落”爽感，以及第 2 / 3 / 4 档动态、整手时长、实际声音、重复耐受和降闪设置。

## 验收

- 结果决策阶段不提前把本次分数演成已入账。
- 结算开始后有可感知的结果定格。
- 当前槽位高亮清楚，顺序与骰袋槽位一致。
- 普通骰入账快，路线关键触发能被看见。
- 普通入账后的边沿电流在低到中等贡献权重下仍可追踪，但不读成终局爆发。
- 倍率成立时倍率章有明显重击。
- 通关和未达标收束不同。
- 最终数字与真实结算结果一致。
- 桌面和最小目标窗口截图没有重叠、遮挡和超框。
- 五档不用外部档位名也能靠动作方向、拍数、范围和峰值区分；最高暴击与远超过保持“可控过载 → 真空击穿”的质变。
- 最高暴击结果在击穿后出现，真空期不提前显示“本关通过”。
- 最高暴击在爆点前没有射线或扩张预泄漏；爆点后主方向向上，机柜位移完成向下蓄力、向上过冲和反向回落。
- 降闪 / 降镜头运动开启后明显减弱峰值，但不会造成结果丢失。
