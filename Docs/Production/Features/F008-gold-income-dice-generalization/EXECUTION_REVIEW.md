1. 先执行 F008-02：悬赏骰和顶金骰。原因是这两颗只依赖本骰点数，能最小成本验证新金币类型、左到右钱包流、预览复用和市场配置。
2. 再执行 F008-03：牌税骰、铅票骰和壳税骰。原因是这批覆盖牌型、出千和临时小骰三类构筑事件。
3. 最后执行 F008-04：收账骰、复利骰、柜台骰和伐木骰。原因是这批最容易影响经济膨胀、槽位理解和分数代价，需要更多样例测试。
4. F008-05 美术接触图可以在 F008-02 程序执行时并行启动；最终资源在方向确认后落地。
5. F008-06 界面体验随程序切片接入，不需要等待九颗全部实现后一次性做完。

## 风险

| 风险 | 影响 | 缓解方式 | 负责人 |
|---|---|---|---|
| 稳定金币过强 | 收账骰和复利骰可能成为所有构筑必买。 | 每小关限次、复利总上限、章节后置和价格控制。 | 策划 / 程序 |
| 牌税文本过重 | 玩家可能需要同时记牌型倍率和金币档位。 | 只分 `+1 / +2` 两档，不新增牌型。 | 策划 / 界面体验 |
| 槽位读法不清 | 柜台骰和伐木骰可能让玩家误解读全部前序。 | 只读左邻，并用短线或邻接高亮表达。 | 界面体验 |
| 伐木变控点工具 | 如果削有效点数，会改变牌型概率。 | 首版只削本次单骰分，不改变有效点数或牌型识别。 | 策划 / 程序 |
| 复利递归膨胀 | 复利收益如果再计息，会放大钱包雪球。 | 复利在基础利息后追加，不回头计入本关基础利息。 | 程序 |
