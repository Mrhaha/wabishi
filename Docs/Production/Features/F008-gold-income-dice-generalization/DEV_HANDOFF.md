## 输入决策

- 九颗金币收益族骰子进入后续范围。
- 所有金币来源默认按当前骰袋槽位从左到右进入钱包。
- 临时小骰默认不触发实体骰效果；壳税骰只读取真实结算生成的小骰数量。
- 出千只重摇骰面，不刷新本回合目标、阈值或已固定快照。
- F008-01 已确认首版规则卡；程序实现不得自行改成固定悬赏目标、提高利息上限、全前序柜台或削有效点数版本。

## 实现目标

在当前 `DiceKingDemo.cs` 原型中新增九种金币收益骰的类型、市场生成、运行时状态、预览和真实结算。实现应保持现有小关流程、牌型表、出千规则、存档策略和左到右结算顺序。

## 已确认行为

- 新增类型必须进入市场配置，而不是开局初始骰袋。
- 新增金币入账要进入钱包，不进入本次分数。
- 新增金币入账可参与通关结算利息基数。
- 较早金币来源可以被较晚贿赂等对象读取；较晚金币不回头。
- 失败清空本轮，不能保留失败局金币。

## 规则实现表

| 骰子 | 建议代码键 | 触发 | 收益 | 实现边界 |
|---|---|---|---:|---|
| 悬赏骰 | `BountyGold` | 出手开始从自身去重面组生成临时悬赏点，结算时本骰有效点数命中 | `+3` | 目标不写入存档；出千不刷新目标 |
| 顶金骰 | `TopGold` | 本骰有效点数等于当前面组最高点 | `+1` | 成长后按当前面组最高点；可与材质独立触发 |
| 牌税骰 | `HandTax` | 最终实体骰牌型不是无牌型 | `+1` 或 `+2` | 只按最高档触发；不新增牌型 |
| 收账骰 | `Collection` | 本小关第一次真实结算经过本骰槽位 | `+1` | 每颗每小关最多 1 次；需要小关内临时标记 |
| 复利骰 | `CompoundInterest` | 通关结算基础利息完成后 | `min(2, floor(基础利息 / 2))` | 全部复利骰合计最多 `+4`；不递归，不提高利息上限 |
| 铅票骰 | `LeadTicket` | 本骰被确认出千并实际重摇 | `+2` | 取消出千或未选本骰不触发 |
| 壳税骰 | `ShellTax` | 真实生成临时小骰数量达到 `3` | `+2` | 不按每 3 颗重复触发；预览标记估算 |
| 柜台骰 | `CounterGold` | 左侧相邻实体骰本次出手结算阶段有正向钱包入账 | `+1` | 只读左邻，不读全部前序、通关固定金币、利息或市场收入 |
| 伐木骰 | `LumberGold` | 左侧相邻实体骰本次单骰分大于 0 且未被伐木 | `+2` | 目标本次单骰分 `-1`，不改有效点数、牌型或长期面组 |

## 代码影响

- 可能新增或扩展骰子类型枚举、显示名、短规则、市场推荐文案。
- 需要复用已有左到右结算钱包流，避免再引入本关账本。
- 需要为预览层和真实结算层共享触发判断，避免重复实现导致不一致。
- 需要为 `悬赏骰` 保存本次出手目标点的临时状态，不持久化。
- 需要为 `收账骰` 保存本小关是否已触发的临时状态，不持久化。
- 需要为 `铅票骰` 读取本骰是否在本小关确认出千中实际重摇过。
- `伐木骰` 只削本次单骰分，不改变有效点数和牌型识别。

## 数据影响

## 测试 / 验证计划

- 悬赏样例：目标在自身面组中，命中和未命中各一次。
- 顶金样例：成长或高偏面组下最高点判断正确。
- 牌税样例：三同、顺子、全奇 / 全偶、四同以上分别触发预期金币。
- 收账样例：每颗收账骰每小关最多触发 1 次，已触发后后续出手不再入账。
- 复利样例：无利息、有利息、利息封顶时收益正确。
- 铅票样例：出千确认重摇触发，取消出千不触发。
- 壳税样例：临时小骰不足和达到门槛分别验证。
- 柜台样例：左侧相邻不存在、存在但未触发、存在且触发金币。
- 伐木样例：被削对象分数大于 0 和等于 0 两种边界。
- 流式钱包样例：前置金币能被后置贿赂读取，后置金币不回头。
- 市场样例：九种骰出现、价格、卖价、章节开放符合配置。
- 存档样例：购买九种骰后保存继续，类型和面组不丢失。
