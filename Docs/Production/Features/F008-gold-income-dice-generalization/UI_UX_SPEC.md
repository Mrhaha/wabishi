# F008 界面体验规格

状态：已实现待截图
功能：F008 金币收益族泛用化
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md
最后更新：2026-06-15

## 输入决策

- 九颗金币收益族骰子都进入后续范围。
- F008-01 已确认九颗骰首版规则卡、收益、上限、出千边界和槽位对象。
- 金币来源默认按左到右槽位顺序进入钱包。
- 预览和真实结算必须口径一致；含随机结果时预览必须标记估算。
- 主流程界面当前以短标签、角标和结算动画为主，不增加教程墙。

## 体验意图

玩家需要在不读长说明的情况下理解两件事：本骰读什么触发条件，以及本次有没有给金币。F008 的界面核心不是展示九条复杂规则，而是在市场、预览和结算时把“触发条件 -> 金币入账”做成短而可追踪的反馈。

## 信息层级

1. 当前钱包金币总数。
2. 本次新增金币来源和数值。
3. 本骰触发短标签。
4. 完整规则详情。
5. 调参或概率说明。

## 状态契约

| 骰子 | 市场短规则 | 预览状态 | 真实结算反馈 | 灰态原因 |
|---|---|---|---|---|
| 悬赏骰 | 掷中悬赏点，钱包 +3 | 显示 `悬赏点 N` 和命中 / 未命中 | `悬赏 +3` | 未命中悬赏点 |
| 顶金骰 | 掷出最高点，钱包 +1 | 显示 `最高点` 和命中 / 未命中 | `顶金 +1` | 不是当前最高点 |
| 牌税骰 | 成牌型时钱包 +1/+2 | 显示本次牌型和档位 | `牌税 +1` 或 `牌税 +2` | 无牌型 |
| 收账骰 | 每小关首次结算钱包 +1 | 未收账时显示可收；已收账后灰态 | `收账 +1` | 本小关已收账 |
| 复利骰 | 通关后按利息追加金币 | 仅达标预览显示估算 | 通关结算显示 `复利 +G` | 未达标或基础利息不足 |
| 铅票骰 | 本骰出千重摇后钱包 +2 | 出千选择时选中本骰显示可兑现 | `铅票 +2` | 未确认出千或未选本骰 |
| 壳税骰 | 临时小骰达 3 颗，钱包 +2 | 显示 `小骰税 N/3`，含随机时标记估算 | `壳税 +2` | 临时小骰少于 3 颗 |
| 柜台骰 | 左邻入账金币时钱包 +1 | 左邻来源确定时显示可收；随机时标记估算 | `柜台 +1`，并短高亮左邻 | 无左邻或左邻未入账金币 |
| 伐木骰 | 左邻单骰分 -1，钱包 +2 | 显示被削左邻和收益 | `伐木 -1 / +2` | 无左邻、左邻分为 0、目标已被伐木 |

## 交互流程

1. 市场中玩家看到九颗骰的短规则，不显示长解释。
2. 玩家购买后，骰子进入骰袋并按现有槽位系统可调整顺序。
3. 投骰结果锁定后，预览层计算可确定的金币触发；壳税骰等依赖真实临时小骰的对象显示估算。
4. 出千选择时，只对可能受出千改变的金币触发显示提示，例如悬赏、顶金、铅票。
5. 确认结算后，金币按槽位顺序逐颗写入钱包；每次入账显示短标签。
6. 通关结算阶段再显示复利骰收益，不提前混入出手内金币，也不让复利收益回头增加本关利息。

## 反馈和动画

- 金币入账使用轻量跳字和钱包数字变化，不打断结算节奏。
- 槽位相关触发需要在本骰与读取对象之间显示短线或邻接高亮。
- 伐木触发需要同时显示被削对象和获得金币，避免玩家以为是免费金币。
- 复利骰只在通关结算面板出现，不在普通出手结算中刷存在感。

## 边界情况

- 未触发时只显示静默或灰色短标签，不弹长提示。
- 预览估算与真实结算不同的对象必须标明“估算”。
- 出千取消不显示铅票入账。
- 临时小骰数量不足时，壳税骰显示当前数量和门槛。
- 柜台骰没有合法读取对象时，显示未触发，不改变槽位。
- 伐木骰被削对象单骰分为 0 时，不应继续削成负数。
- 同一目标本次已被伐木时，后续伐木骰显示未触发。
- 复利骰基础利息为 0 或 1 且公式结果为 0 时，显示未触发，不显示失败感强的扣除文案。

## 适配 / 布局约束

- 仍遵守 1920x1080 设计基准和 1280x720 最低实用目标。
- 不新增大面板常驻解释九颗骰。
- 短标签必须适配当前 OnGUI 原型的骰子行空间。
- 主流程界面改动后需要按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 做截图验收。

## 实现备注

界面体验需要和程序共同确认每颗骰的预览字段，避免界面重新推导随机或结算逻辑。F008-06 可直接基于本规格展开。

## 阻塞项

无。

## 验收

- 市场卡能在一屏内展示九颗骰的核心短规则。
- 结果预览能准确区分确定入账和估算入账。
- 出千界面能清楚表达铅票触发条件。
- 左到右结算能看出金币来自哪颗骰。
- 复利骰只在通关结算阶段反馈。
