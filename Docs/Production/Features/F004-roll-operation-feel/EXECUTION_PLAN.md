## 验证计划

- 静态检查：`git diff --check`。
- 文案检查：搜索“延迟停止”，确认主流程旧文案已替换。
- 默认配置检查：`Assets/Resources/Data/roll_feedback_config.csv` 存在，字段可解析，并有代码安全默认值。
- 非法配置检查：缺失字段、非法值和极端值会被钳制或回退，不会阻断投骰。
- 外部覆盖检查：`Application.persistentDataPath/roll_feedback_config.csv` 可覆盖默认配置，并在日志中显示读取路径。
- 重载检查：修改外部覆盖配置并触发重载，下一次投骰使用新参数。
- 当前投骰快照检查：投骰过程中重载配置，当前投骰不跳变，新配置从下一次投骰生效。
- 状态机检查：起摇、加力窗口、回落、结果决策顺序正确。
- 随机隔离检查：通过代码审查确认 `FinishShakeRoll` 前不会因敲击、提示或配置读取消耗 `UnityEngine.Random`；如有确定性种子、`Random.state` 快照或调试钩子，再验证不同敲击次数和不同表现配置不改变结果。
- Unity 验证：有 Unity Play Mode 时运行主流程，检查投骰、出千、结算和下一次投骰。
- UI 检查：主流程核心信息不被骰盅运动、提示或轻震遮挡。
