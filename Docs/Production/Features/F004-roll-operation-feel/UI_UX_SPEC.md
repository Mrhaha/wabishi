# F004 界面体验规格

状态：已提案
功能：F004 投骰操作手感
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- `Space` 是主投骰输入。
- 投骰阶段拆为起摇、加力窗口、显示结果 / 回落、结果决策。
- 加力窗口内的连续敲击只产生表现反馈，不改变结果概率。
- 加力窗口结束后的额外敲击只产生轻量“显示结果”反馈。
- 不显示“已敲击次数”数字，避免玩家寻找隐藏最优次数。

## 体验意图

玩家应清楚知道当前能不能通过 `Space` 加力：窗口内提示鼓励爽快连打，窗口结束后提示系统正在显示结果。提示要短、明确，不使用教程墙，不暗示敲击会提高收益。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 等待投骰 | 开始本次投骰 | 当前目标、剩余出手、当前骰袋摘要 | `Space` 起摇 | 无 | 起摇 / 加力窗口 |
| 起摇 | 感到骰盅被拿起或开始晃动 | 短提示“空格起摇”或同等短句 | 无需额外操作 | 无 | 加力窗口 |
| 加力窗口 | 爽快连打制造表现强度 | 短提示“敲击空格加力” | 连续 `Space` | 无 | 显示结果 / 回落 |
| 显示结果 / 回落 | 等待骰盅停下 | 短提示“显示结果” | 无 | 超时 `Space` 只闪轻提示 | 结果决策 |
| 结果决策 | 判断结算或出千 | 当前牌型、倍率、本次分数预览 | `Space` 结算 | 出千入口 | 结算或出千 |

## 信息层级

1. 当前阶段提示：告诉玩家现在能做什么。
2. 骰盅表现：承接玩家敲击反馈。
3. 目标分、剩余出手、出千等既有战斗信息：保持可读。
4. 日志：只记录关键状态，不刷屏记录每一次敲击。

## 交互流程

1. `Ready` 中按 `Space`，进入起摇和加力窗口。
2. 加力窗口内每次有效 `Space` 触发骰盅冲量和提示脉冲。
3. 达到冲量上限后，继续敲击不再增强；是否有轻微反馈由配置控制。
4. 窗口结束后进入显示结果 / 回落阶段。
5. 回落阶段继续按 `Space` 不改变状态，只触发极轻提示。
6. 结果锁定后进入既有结果决策。

## 反馈和动画

- 有效敲击：骰盅横向偏移、轻微抬起、旋转回弹；提示条可轻微脉冲。
- 强度上限：不显示数字，不出现奖励式反馈。
- 超时敲击：只闪“显示结果”或极轻反馈。
- 回落：骰盅强度衰减到停止，然后展示结果。
- 音效和桌面轻震：首版可选，不阻塞状态机和提示验收。

## 边界情况

- 配置重载时如果正在投骰，当前提示和动画继续使用本次投骰快照。
- 配置缺失或非法时，提示仍能显示，投骰流程不中断。
- 窗口内极高频敲击不应导致提示文本溢出或日志刷屏。
- 回落阶段不能误导玩家继续加力。

## 适配 / 布局约束

- 继续遵守当前 OnGUI 16:9 安全区域和 `1920x1080` 设计目标。
- 阶段提示不得遮挡中心骰子结果、目标进度或底部主动作。
- 不新增大段说明文本。

## 实现备注

- 需要替换旧文案，例如“敲 Space 延迟停止”。
- 设置页操作说明需要同步为新流程。
- 若接入调试键 `F5` 重载配置，调试提示应放在日志或短提示中，不做玩家可见教程。

## 阻塞项

无硬阻塞。设置页按钮、音效和桌面轻震都可后置。

## 验收

- 加力窗口内，玩家能从提示和骰盅反馈理解可以连打。
- 窗口结束后，继续按 `Space` 只看到“显示结果”轻提示。
- 不显示敲击次数数字。
- 不出现暗示敲击提高概率或收益的文案。
- 主流程 UI 无文本溢出和核心信息遮挡。
