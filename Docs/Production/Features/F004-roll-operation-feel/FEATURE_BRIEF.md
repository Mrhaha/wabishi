## 功能意图

F004 解决当前投骰操作像“延长倒计时”、不像“敲动骰盅”的手感问题。首版目标是把投骰拆成起摇、有限加力窗口、自然回落、显示结果四段，让玩家可以爽快连打制造表现反馈，同时保持骰面随机完全公平。

本包不改变骰面概率、骰子效果、出千规则、计分、经济、市场或存档。所有投骰表现参数必须配置化，支持调试期运行时重载，避免后续手感迭代需要反复改代码。

## 玩家体验

- 玩家第一次按 `Space` 后，骰盅立刻开始低强度晃动。
- 在有限加力窗口内，玩家连续敲击 `Space`，每次都像给骰盅追加一次固定表现冲量。
- 连打只影响骰盅运动、提示反馈、可选音效和可选桌面轻震，不影响结果。
- 加力窗口结束后，玩家继续按 `Space` 只看到很轻的“显示结果”反馈。
- 骰盅自然回落后锁定结果，进入原有结果决策流程。

## 已确认规则

- 投骰不由点击按钮触发，仍以 `Space` 为主输入。
- 第一次 `Space` 进入起摇，骰盅先低强度晃动。
- 起摇后进入有限加力窗口。
- 加力窗口内每次 `Space` 只施加固定表现冲量。
- 加力强度有上限，不能通过连续敲击无限增强。
- 加力窗口结束后，继续按 `Space` 只给轻量“显示结果”反馈。
- 回落停止阶段结束后锁定结果并进入结果决策。
- 投骰表现配置只影响表现，不参与随机、出千、分数、结算或存档。
- 运行时重载后，新配置从下一次投骰生效；正在投骰中的本次流程继续使用快照。

## 建议默认值

| 项目 | 建议默认值 | 接受后发生什么 | 不接受后发生什么 |
|---|---:|---|---|
| 加力窗口 | `1.35` 秒 | 首版有足够爽快连打空间 | 可在配置中直接改短或改长 |
| 有效冲量上限 | `8` 次 | 玩家能快速打满强度，不必无限敲 | 可在配置中改为 `7` 或 `9` |
| 输入去抖 | `0.06` 秒 | 过滤异常重复输入，保留高速连打 | 可按实际键盘手感调整 |
| 回落时长 | `0.45` 秒 | 连打后快速进入结果 | 可按节奏调到 `0.38-0.50` 秒 |
| 起摇基础强度 | `0.45` | 第一下有低强度反馈 | 可按动画观感调整 |
| 单次冲量强度 | `0.18` | 每次敲击有明显变化 | 可按骰盅运动幅度调整 |
| 最大表现强度 | `1.75` | 限制极端连打造成画面过乱 | 可按视觉验收调整 |
| 超时提示时长 | `0.12` 秒 | 窗口后输入有轻量反馈 | 可按提示可读性调整 |
| 桌面轻震幅度 / 时长 | `2` / `0.08` 秒 | 字段可被解析和钳制；首版可不接入实际轻震 | 接入表现时直接从配置读取 |
| 有效敲击 / 超时敲击音量 | `0.65` / `0.18` | 字段可被解析和钳制；首版可不接入音效资源 | 接入音效时直接从配置读取 |

## 约束

- 不改变骰面结果生成时机以外的玩法逻辑。
- 不让敲击次数、节奏、配置值或表现强度进入随机、计分、出千或结算。
- 不把投骰表现配置写入 `PlayerPrefs` 运行存档。
- 不重构整个 `DiceKingDemo.cs` 架构。
- 不新增长教程文本。
- 音效和桌面轻震字段首版可以先解析、钳制和保留，不要求实际音频资源或额外轻震表现。
- 主流程 UI 改动后，后续实现验收需要遵守 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md`。

## 职能交接

### 美术

首版无必需新增资源。复用当前骰盅资源，通过运行时位移、旋转、缩放、阴影或轻震表达冲量。音效字段可以先作为配置项保留，不要求首版提供音效资源；若后续要接入音效或新骰盅序列动画，再另起美术切片。

### 界面体验

需要明确起摇、加力、显示结果三段提示，并处理窗口结束后的轻量反馈。不得用文案暗示敲击会提高结果。

### 程序

需要新增投骰表现配置、配置加载、运行时重载、配置快照、有限加力窗口、固定表现冲量、回落衰减和随机隔离验证。当前代码中的 `ShakeStartTime`、`ShakeTapBonus`、`ShakeMaxTime`、`StopDuration` 不应继续作为主逻辑来源。

## 验收含义

F004 验收通过意味着：投骰从“续倒计时”变成“起摇、连打加力、回落显示结果”的可读流程；表现数值可配置并可运行时重载；同一随机条件下敲击次数和表现配置不会改变骰面结果；窗口结束后的输入只给轻量“显示结果”反馈；主流程文案不再描述旧的延迟停止逻辑。
