## 功能意图

F014 为商店待购买骰子和主玩法投掷区骰子提供统一鼠标悬浮窗。它把常驻界面上的长说明收进 hover 详情层，降低市场货架和投掷区的直接信息量，同时保持玩家能快速读取骰子名称、点面、骰子效果、品质效果和卖出价格。

本包不改变骰子规则、市场价格、结算公式或存档，只补 UI 信息展示和对应运行时 UI 素材。

## 玩家体验

玩家把鼠标移到商店待购骰或主投掷区实体骰上时，出现同一套竖版紧凑悬浮窗。悬浮窗使用单层面板，不再绘制右侧“面 / 效 / 质”页签；信息以短字段呈现，不做大段解释，不遮挡主流程关键操作。

## 职能交接

### 美术

运行时 UI 素材已完成，源图保留在 `Assets/ArtSource/Production/Tooltip/runtime_ui_sources/`。后续若程序觉得面板过大或缩放不合适，只调整素材尺寸和九宫格方案，不回到纸页厚度或账册装订。

### 界面体验

初版信息量保持极简。当前不显示右侧页签，悬浮窗应贴近 hover 对象，并在 16:9 安全区内自动避边。

### 程序

程序已按 `DEV_HANDOFF.md` 静态接入。实现复用当前已有的 `DieDisplayName()`、`FaceText()`、`MaterialShortRule()`、`SellPrice()` 等运行时数据，骰子效果展示说明读取 `Assets/Resources/Data/dice_type_config.csv`；该 CSV 只服务悬浮窗说明，不参与规则结算。仍需 Unity Play Mode 与截图验收确认动态表现。

## 验收含义

F014 通过意味着：商店待购骰和主投掷区骰都能通过鼠标悬浮查看同一套竖版详情；常驻界面减少长说明；悬浮窗不会遮挡关键按钮，不显示右侧“面 / 效 / 质”页签，也不会把纸页厚度或账册翻页感带回当前 UI。
