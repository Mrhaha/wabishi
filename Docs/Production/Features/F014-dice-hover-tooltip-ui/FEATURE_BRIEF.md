## 功能意图

F014 为市场待购买骰、市场骰袋和主玩法实体骰提供统一鼠标悬浮详情。常驻骰子只负责表达类型和当前有效点数；六面分布、完整规则、家族、触发时机、成长、喂养、吞噬、化壳、来源、价格判断和必要术语解释由悬浮窗承接。

本包只优化信息组织、视觉表现和运行时展示，不改变骰子规则、市场价格公式、计分、随机、存档或玩法状态。

## 玩家体验

玩家悬浮后看到一块与当前午夜旧机厅主画面一致的横向检修终端。基础骰保持短小，复杂骰按已有个体状态增高；市场商品明确区分买入价与预计卖出价。玩家无需从长日志推断骰子当前状态，也不会在主骰面看到伪造结果或预测贡献分。

## 当前实现事实

- `DiceKingDemo.cs` 已接入统一 hover 状态机、三类场景上下文、`448 px` 横版布局、实际六面、结构化状态与上下文价格。
- 实际六面以 `die.Faces` 为基础，逐面折算喂养最低点和幸运龟最大面；长期面变化自然读取新面组，条件计分、邻位、钱包、遭遇与本次随机结果不进入该区。
- 六面按整组最大绝对值切换 `1×6` 整数、`3×2` 千分位整数或 `3×2` 科学计数；科学计数附 `万 / 亿` 快速读取，最高面只按完整整数判断，唯一最高才显示提示。
- 静态规则读取 `dice_type_config.csv`，通过 `{{kw:key}}` 使用词典显示名；可见关键词在正文中复用家族色并加下划线，底部解释通过关键词定义表和绑定表生成、去重与排序。
- `keyword_glossary_config.csv` 的 `show_explanation` 独立控制解释可见性：`market_floor / family / dual_family=0`，当前只让 `feed / devour / shell=1`。
- 可见词条的 `accent_family` 由 CSV 维护；底部不再绘制通用说明标题，每个词条只绘制一次彩色标签，解释位于标签下方。
- 面板按实际六面模式、规则、状态、价格和解释换行选择 `196 / 244 / 288 / 352 / 416 / 480 / 568` 高度档；无可见解释时不绘制分隔线和留白。
- 旧资源保留为回退路径；Unity 2019.4 Roslyn 独立编译、配表覆盖、阈值反射测试和字库生成已通过，新方向仍需 Unity 双分辨率运行验收。

## 非目标

- 不改骰子效果、家族规则、市场价格公式、评分或存档。
- 不新增骰子类型、词缀或材质玩法。
- 不显示结算预测或把单骰贡献分写进详情。
- 不重构为 UI Toolkit 或 uGUI。
- 不把接触图整张作为运行时图片。
