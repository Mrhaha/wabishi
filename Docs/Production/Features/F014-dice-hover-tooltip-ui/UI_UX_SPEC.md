# F014 界面体验规格

状态：程序已静态接入，待 Unity 运行验收
功能：骰子悬浮窗 UI
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 商店待购骰和主投掷区骰共用统一悬浮窗。
- 初版采用最小满足字段。
- 当前方向已确认：竖版紧凑简介，不显示右侧“面 / 效 / 质”页签。
- 骰子效果说明由 CSV 提供，不使用短标签作为主说明。
- 运行时不显示统计、边界标签、价格规则解释和额外状态。

## 触发范围

| 场景 | 触发对象 | 显示内容来源 | 备注 |
|---|---|---|---|
| 关间市场 / 章节市场 | 买入货架上的骰子商品 | `MarketOffer.Die` | 用商品骰面组、材质、买入后卖价 |
| 关间市场 / 章节市场 | 当前骰袋列表中的骰子 | `dice[i]` | 可显示卖出价，辅助出售判断 |
| 主玩法投掷区 | 非临时实体骰 | 当前 `Die` 和锁定结果 | 临时小骰初版不显示悬浮窗 |

## 最小字段

| 字段 | 内容 | 数据建议 |
|---|---|---|
| 骰子名称 | 材质名 + 骰子名 | `DieDisplayName(die)` |
| 点面 | 六个当前面值 | `die.Faces`，不要用本次结果替代面组 |
| 骰子效果 | 类型效果说明 | `Assets/Resources/Data/dice_type_config.csv` 的 `tooltip_effect` |
| 品质效果 | 材质短规则 | `MaterialShortRule(die.Material)` |
| 卖出价格 | 按类型回收价 | `SellPrice(die.Type)` |

## 布局规格

- 主体约 `336 x 400`，按当前 `1280x720` 虚拟坐标缩放到窗口。
- 不显示右侧“面 / 效 / 质”页签，不预留页签宽度。
- 主体从上到下为：头部、点面、骰效、质效。
- 文字不超过两行；超过时截断或缩小，不撑大面板。
- 悬浮窗应避开屏幕边缘，优先出现在目标上方或右上方，空间不足时翻到左侧或下方。

## 交互规则

- 鼠标进入目标区域并停留 `0.12s` 后显示悬浮窗。
- 鼠标离开目标区域后进入 `0.08s` 淡出隐藏。
- 悬浮窗不接管点击输入。
- 如果鼠标同时压到按钮和骰子，购买 / 出售按钮点击优先，悬浮窗只显示信息。
- 初版不需要页签点击交互，也不显示页签索引。
- 后续如果字段变多，先增加同面板内字段或再评估分页；不得在未确认前恢复右侧页签。

## 出现时机与渐变衔接

| 项目 | 规范 | 目的 |
|---|---|---|
| 显示延迟 | 鼠标稳定停留同一触发对象 `0.12s` 后出现 | 避免鼠标扫过货架或骰袋时连续弹窗 |
| 淡入 | `0.10s` 内透明度从 `0` 到 `1` | 信息出现柔和，但不拖慢阅读 |
| 入场位移 | 淡入期间从目标锚点下方 `6px` 上浮到最终位置 | 保留轻量浮窗感，不做大位移动画 |
| 入场缩放 | 可从 `0.98` 到 `1.00`，不得低于 `0.98` | 只做轻微呼吸感，避免弹窗感过强 |
| 隐藏 | 鼠标离开后立即开始 `0.08s` 淡出 | 退出干净，不保留拖尾 |
| 目标切换 | 已显示时从一个骰子移到另一个骰子，不重新走 `0.12s` 延迟和淡入；内容在 `0.06s` 内替换，锚点重新计算 | 鼠标横向查看多个骰子时保持阅读连续 |
| 位置衔接 | 面板锚点随当前目标重算，可直接切到新锚点，不做长距离追随鼠标 | 避免浮窗追鼠标造成干扰 |
| `Shaking` 阶段 | 立即隐藏，不播放完整淡出 | 投骰运动中不残留信息层 |
| `Scoring` 阶段 | 默认不显示；如果后续程序能确认结算飞字已经结束，可在收束后恢复显示 | 避免遮挡分数飞字和结算反馈 |

## 状态显示规则

| 场景 / 阶段 | 是否显示 | 说明 |
|---|---|---|
| 市场货架 | 显示 | 适用 `0.12s` 延迟和淡入淡出 |
| 市场当前骰袋列表 | 显示 | 适用 `0.12s` 延迟和淡入淡出 |
| 主玩法 `Ready` | 显示 | 可用于投骰前查看面组和效果 |
| 主玩法 `Shaking` | 不显示 | 立即隐藏，避免运动闪烁 |
| 主玩法 `Stopping` | 不显示 | 点数揭示期间不抢视线 |
| 主玩法 `ResultDecision` | 显示 | 玩家做结算 / 出千判断时可查看 |
| 主玩法 `CheatEdit` | 显示 | 出千选择时可查看面组上限和效果 |
| 主玩法 `Scoring` | 默认不显示 | 防止遮挡结算演出 |
| 主玩法 `StageClear` | 显示 | 结算收束后允许查看 |
| 主玩法 `StageFailed` | 不显示 | 失败收束期不再弹出信息层 |

## 场景差异

### 商店待购骰

- 显示商品骰的完整面组、类型效果、材质效果和卖出价。
- 不显示价格公式。
- 不解释买入价修正。
- 不显示隐藏路线倾向。

### 主投掷区骰

- 显示当前骰子的长期面组，而不是只显示本次投出的点。
- 当前短状态不作为骰子效果主来源；需要显示时，后续再新增独立状态行。
- 不把结算日志全部塞进悬浮窗。

## 适配约束

- `1280x720` 和 `1920x1080` 都不能遮挡底部 `Space` 条、市场购买按钮或右侧牌型表按钮。
- 文本不能溢出面板。
- 面板不能超过安全区；靠右货架 hover 时应向左展开。
- 投掷区六骰横排 hover 时，悬浮窗不应遮住当前鼠标下的骰子本体。

## 验收

- 玩家在市场中能少看常驻长说明，仍能读懂骰子值得买的核心信息。
- 玩家在主投掷区能通过 hover 理解当前骰子的面组和效果。
- 悬浮窗样式在商店和投掷区一致。
- 不显示右侧页签、扁平贴片或账册纸页。
- 没有纸页厚度、书脊、账册装订和翻页感。
