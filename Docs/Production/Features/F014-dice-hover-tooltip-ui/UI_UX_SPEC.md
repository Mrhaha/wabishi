# F014 界面体验规格

状态：已静态实现；待 Unity 运行验收
功能：骰子悬浮窗 UI
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md
最后更新：2026-07-23

## 核心原则

- 市场待购骰、市场骰袋和主投掷区实体骰共用同一套横向悬浮窗。
- 常驻骰子负责“类型 + 已锁定有效点数”；悬浮窗负责“六面 + 完整规则 + 个体状态 + 场景价格 + 必要术语说明”。
- 不显示伪造当前结果、预计总分或真实贡献分。
- 不做页签、滚动、空材质区或无功能按钮。
- 文字、数字和价格全部由程序根据当前数据绘制。

## 触发范围与上下文

| 上下文 | 触发对象 | 数据对象 | 经济栏 | 备注 |
|---|---|---|---|---|
| 市场商品 | 三个货架中的骰子商品 | `MarketOffer` 与 `MarketOffer.Die` | 买入价 + 预计卖出价 | 已售空货架和改造道具不触发 |
| 市场骰袋 | 当前骰袋实体骰 | `dice[i]` | 当前卖出价 | 辅助出售判断 |
| 主玩法 | 非临时实体骰 | 当前 `Die` | 不显示 | `Temporary` 骰不触发 |

## 信息层级

从上到下固定为：

1. 类型芯、骰子名称、家族标签、触发标签。
2. 六个实际生效点数格。
3. 静态规则。
4. 可选个体状态行。
5. 可选场景经济栏。
6. 可选术语说明。

### 头部

- 左侧复用现有骰子类型图标作为类型芯。
- 名称优先使用 `DieDisplayName(die)`，过长时允许缩小字号，不以省略号删除核心类型名。
- 家族标签表达中立、猪猪、恶魔、龟龟、海盗；双家族同时显示两个标签。
- 触发标签使用展示映射，例如“投掷时”“结算时”“购买时”“卖出时”“刷新时”“离场时”“常驻”；龟壳类不重复显示“化壳时”。
- 触发标签只描述现有规则入口，不创建新的触发机制。

### 六面

- 标题固定为“实际六面”。以 `die.Faces` 为基础逐面折算当前可确定的喂养最低点和幸运龟最大面；长期面变化直接读取新面组。
- 不使用 `LastValue`、当前 `EffectiveValue`、条件计分、本次贡献分、邻位、钱包或遭遇修正替代六面。
- 六格全部显示数字，不使用 pip；点数区不显示“已含喂养”或其它来源标签。
- 六面最大绝对值 `<10,000` 时使用 `1×6` 和完整整数；`10,000–99,999` 时整组使用 `3×2` 和千位分隔完整整数；`>=100,000` 时整组使用 `3×2`、3 位有效数字科学计数和中文快速读取。
- 科学计数用 `×10` 与上标指数；快速读取对大数显示“约 N 万 / 亿”，小量级保留直接整数。格式化不得改变完整整数、结算、比较或存档。
- 只在最高值唯一时用细琥珀边框和“最高”提示；并列最高不任意挑选。
- 不用颜色暗示概率、好坏或未来结果；负数可以使用克制锈红文字，但仍需清楚可读。

### 静态规则

- 读取 `Assets/Resources/Data/dice_type_config.csv` 的 `tooltip_effect`。
- 优先完整显示触发条件、目标、数值和结果，不再把动态状态拼到同一段。
- 建议两行，复杂规则最多三行；若仍装不下，应修订展示文案，不得用任意字符数截断关键条件或数值。

### 个体状态

只显示确有数据的行，空状态不留标题或占位。

| 状态 | 条件 | 推荐内容 |
|---|---|---|
| 成长 | `Growth > 0` | 当前成长层数或已影响的面值结果 |
| 喂养 | `FeedCount > 0` 或 `FeedValue > 0` | 喂养次数、最低点增量、卖价影响 |
| 最近吞噬 | `LastDevourGain > 0` | 加值、触发来源、被吞噬来源 |
| 化壳 | `TurtleAttachments.Count > 0` | 数量、主要壳类型、真实龟卖价归属 |
| 实例来源 | 已有来源数据时 | 只显示可验证的来源，不推断未记录历史 |

同类状态合并为一行；龟壳较多时显示“数量 + 关键类型摘要”，不逐项铺满。复杂版最多使用四条结构化状态行，不滚动、不把后续状态静默丢弃。

### 经济栏

- 市场商品：显示 `买入 <offer.Price>` 与 `预计卖出 <SellPrice(offer.Die)>`。
- 市场骰袋：显示 `当前卖出 <SellPrice(die)>`。
- 主玩法：不显示价格。
- 不解释价格公式；如卖价受喂养、面值、实例价或化壳来源影响，其原因放在对应状态行，不额外展开公式。

### 术语说明

- 固定在悬浮窗最底部，只解释当前骰子规则或实际动态状态关联且 `show_explanation=1` 的词条。
- 不绘制“术语说明”“关键词说明”或带问号的通用标题。每个关键词只绘制一个彩色标签，解释另起一行。
- 显示名、解释、所属家族色、默认顺序与可见开关读取 `keyword_glossary_config.csv`；骰子 / 状态关系读取 `keyword_binding_config.csv`，不得按中文正文匹配或写类型特例。
- 标签色严格复用所属家族色；规则正文中的同一可见关键词使用相同颜色并增加下划线。隐藏词条保持普通正文，不高亮。
- `market_floor / family / dual_family` 的 `show_explanation=0`：规则中的“市场最低点 / 家族 / 双阵营”和头部双家族身份仍正常显示，但不绘制底部解释；当前只解释 `feed / devour / shell`。
- 相同稳定键只解释一次，按绑定顺序、默认顺序和稳定键排序；没有可见词条时不绘制分隔线、标题或空白。

## 尺寸分档

以 `1280×720` 虚拟画布为基准，宽度约 `448`：

| 档位 | 高度 | 进入条件 |
|---|---:|---|
| 简版 | 约 `196` | 内容可在基础高度内完整容纳 |
| 中版 | 约 `244` | 一至两条状态，或只有经济栏 |
| 复杂版 | 约 `288` | 三至四条状态，或状态与经济栏同时存在 |
| 扩展小版 | 约 `352` | 规则、状态或术语说明超过复杂版容量 |
| 扩展中版 | 约 `416` | 多条可见术语解释 |
| 扩展高版 | 约 `480` | 状态较多且叠加当前最多三条可见解释 |
| 极限版 | 约 `568` | 科学计数六面、长规则、多状态与多解释同时出现 |

面板按真实换行后的内容选择最小可容纳档。切换相邻骰子导致档位变化时，锚点重新计算，内容在既有 `0.06s` 软替换中完成。

## 交互时机

| 项目 | 规则 |
|---|---|
| 显示延迟 | 同一目标稳定悬浮 `0.12s` 后显示 |
| 淡入 | `0.10s` 内完成，可从下方 `6px` 上浮 |
| 淡出 | 离开后 `0.08s` 内完成 |
| 目标切换 | 已显示时不重新等待和完整淡入，`0.06s` 内替换内容 |
| 点击 | 悬浮窗不接管点击；购买、出售和拖拽优先 |
| 拖拽 | 骰子排序拖拽开始时立即隐藏 |

## 状态显示规则

| 场景 / 阶段 | 是否显示 |
|---|---|
| 市场货架 / 市场骰袋 | 显示 |
| `Ready` | 显示 |
| `Shaking` / `Stopping` | 立即隐藏 |
| `ResultDecision` | 显示 |
| `CheatEdit` | 兼容旧模式时显示 |
| `Scoring` | 默认不显示 |
| `StageClear` | 显示 |
| `StageFailed` | 不显示 |

## 锚点与避边

- 主玩法优先放在目标骰上方并保持骰子本体可见；上方不足时放到下方或侧面。
- 市场商品优先放在货架内侧；右侧货架向左展开，不能遮住购买按钮。
- 市场骰袋优先向右展开，空间不足时向左。
- 必须避开底部 `Space` 主操作、顶部资源栏、市场购买 / 出售 / 刷新 / 离开按钮。
- 面板全体 clamp 在 `16px` 安全边距内，不追随鼠标移动。

## 字体与颜色

- 主标题和关键数值使用清晰的旧式点阵语法，但正文必须先通过无辉光硬核心清晰度。
- `1280×720` 评审使用原始像素截图，不对整图二次缩放。
- 琥珀承担名称、实际六面、快速读取和主要规则；家族色只承担家族标签、可见关键词标签及正文关键词下划线；锈红只用于负数或危险提示。
- 辉光是独立可调层，不能用辉光掩盖细字或非整数缩放。

## 验收

- 基础骰无需阅读即可在一眼内看清名称、六面和规则。
- 复杂骰能完整区分静态规则、喂养、吞噬和化壳，不靠换行拼成一段。
- 双家族与触发入口可识别。
- “市场最低点 / 家族 / 双阵营”不进入底部术语说明；“喂养 / 吞噬 / 化壳”仍按配置完整显示。
- 市场买价和预计卖价不会混淆，主玩法不出现无关价格。
- `9,999 / 10,000 / 99,999 / 100,000` 四个边界按合同切版；亿级科学计数与快速读取不溢出。
- 两种分辨率下无遮挡、无关键文案截断、无模糊点阵字。
