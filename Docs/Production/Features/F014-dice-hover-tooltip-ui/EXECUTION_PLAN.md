## 风险和处理

| 风险 | 处理 |
|---|---|
| 悬浮窗遮挡购买按钮 | 做避边和安全区 clamp |
| 投掷区结算时 hover 干扰分数飞字 | `Scoring` 阶段可暂时关闭或延迟显示 |
| 文本过长 | 初版截断或缩小，不撑大面板 |
| 素材缺失 | 回退现有程序化 UI |
| 信息又变多 | 不在 F014 扩字段；后续先评估同面板字段或分页，不默认恢复右侧页签 |
| 鼠标扫过目标时频繁弹窗 | 使用 `0.12s` hover 延迟 |
| 相邻骰子切换时反复闪烁 | 已显示状态下切换目标不重新等待和重淡入 |

## 验证计划

- 静态检查：`git diff --check`。
- 资源检查：当前运行时使用的 `Art/UI/Tooltip/...` key 可加载，`Data/dice_type_config` 可加载。
- 市场截图：三个货架 hover，当前骰袋 hover，靠右避边。
- 主流程截图：Ready、ResultDecision、CheatEdit、StageClear 四种阶段 hover。
- 动态检查：不足 `0.12s` 不显示，`0.10s` 淡入，`0.08s` 淡出，相邻骰切换不断闪。
- 负例：临时小骰、Shaking、Stopping、Scoring 和 StageFailed 阶段不显示或不闪烁。

## 交接备注

F014 是 UI 信息层，不是玩法改动。程序接入时优先复用当前 OnGUI 坐标和现有数据函数，避免把悬浮窗做成新的规则系统。
