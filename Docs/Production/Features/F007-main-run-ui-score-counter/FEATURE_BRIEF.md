## 功能意图

F007 解决主流程界面信息过载和计分爽感不足的问题。主界面不再把骰袋构成、全部上场、本关关注、长说明和进度条常驻堆在屏幕上，而是改成紧凑计分台：玩家先看骰子，再看两个核心数字，最后看 `0 / 72` 的目标推进。

本包只调整主流程 UI 信息层级、计分器表达和结算演出，不改变投骰概率、牌型表、目标曲线、金币经济、出千规则、骰子效果或真实计分公式。

## 玩家体验

玩家进入小关后，画面保留 `骰子王` 标题和关卡信息。金币、出手、出千集中在左侧，中央区域只承载骰子、`0 / 72` 和两个醒目的计分数字。投骰结果锁定后，结算不再用文字公式解释，而是让基础分数字和倍数数字随左到右结算逐步跳涨；当牌型倍率成立时，倍数数字产生明显变化，并推动 `0 / 72` 同步增长。

玩家不需要阅读“本关目标”“当前分数”“还差”“点数合计”“牌型倍率”这类常驻文字，也能通过位置、字号、颜色和动画理解：这一手由倍率前分数和当前倍数共同推高目标进度。

## 已确认规则

- 主流程画面保留 `骰子王` 标题。
- 顶部不再常驻金币、出手和出千；这些状态移到左侧。
- 左侧只保留金币、出手和出千三类资源状态。
- 删除“当前骰袋”、`6 / 6 全部上场`、构成、`基础 x6` 和“本关关注”等低价值常驻信息。
- 本关目标不显示进度条。
- 目标进度只显示类似 `0 / 72` 的数字，不显示“本关目标”“当前分数”“还差”等文字。
- 结果和结算区只突出基础分数与倍数两个核心数字，不常驻解释性文字标签。
- 结算演出按 F006 已确认的左到右槽位顺序逐颗入账。
- 基础分数字随每颗骰子的倍率前分数入账而累加。
- 倍数数字在牌型或倍率成立节点跳涨，并使 `0 / 72` 同步滚动增长。
- 底部 `Space` 提示不再横跨整屏。
- 右侧只保留本关规则和牌型表入口，不常驻大段解释。

## 建议默认值

- 左侧资源使用图标加数字表达，必要文字只作为小号辅助，不抢中央视觉。
- 中央计分器首版采用一个大基础分数字和一个较强视觉权重的 `×N` 倍数章；不在运行时显示“基础分”“倍率”标签。
- `0 / 72` 放在中央计分器上方或投掷区右上角，在接触图阶段比较后定稿。
- 内部实现的“基础分数字”表示当前已入账的倍率前分数，包含当前规则中会进入倍率前的单骰分、类型加分、材质加分、词缀加分和规则加分。
- 贿赂这类倍率后的最终补分不新增第三个常驻数字；触发时用一次性浮字或短促入账反馈，然后直接更新 `0 / 72`。
- 倍数演出可以预先读取完整结算结果，但播放时按已结算骰子的前缀状态触发跳涨：三同 / 四同 / 五同 / 六同在对应数量到达时跳，顺子在五连成立时跳，全奇 / 全偶在第六颗确认时跳。
- 首版不新增专门跳过按钮；若已有确认结算输入可自然加速，则程序实现时记录为非阻塞增强。

## 职能交接

### 美术

先服务 F007-01 接触图，不直接生产运行时切图。接触图要验证明亮账本桌游风下的极简计分器层级，重点是数字、倍率章、左侧资源状态和 `0 / 72` 之间的阅读顺序。避免把装饰、边框或文字说明重新堆满画面。

### 界面体验

主职责是定稿主流程状态下哪些信息出现、哪些信息删除、数字如何在不同阶段变化，以及结算演出如何安排焦点顺序。需要确保运行时不常驻解释性文字，同时新玩家仍能通过固定位置和动效读懂两个核心数字。

### 程序

实现时只改主流程 UI 和结算展示中间态，不改真实计分。需要从现有结算结果或账本中拆出用于动画的 `显示基础分`、`显示倍数` 和 `显示累计进度`。播放动画必须消费确定结果，不在动画过程中重新随机或重新计算会影响结果的逻辑。

## 不要改动

- 不删除 `骰子王` 标题。
- 不恢复主流程左侧骰袋构成、全部上场和本关关注。
- 不使用目标进度条替代 `0 / 72`。
- 不在双数字旁常驻写“基础分”“倍率”“本次分”等说明。
- 不把结算演出做成一次性跳最终分；必须有逐颗入账和倍数跳涨过程。

## 验收含义

F007 验收通过意味着：主流程画面删掉低价值常驻信息，保留标题、关卡、左侧资源、中央骰子、`0 / 72` 和双数字计分器；结算时基础分和倍数都能随左到右结算产生明确跳涨，最终目标进度与真实计分一致；玩家不读长说明，也能把注意力放在骰子、基础分、倍数和目标推进上。
