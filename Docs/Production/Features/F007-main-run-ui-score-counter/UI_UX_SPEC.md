状态：G 版已确认，F007-05 结算演出已实现待验证
功能：F007 主流程 UI 精简与计分器演出
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 保留 `骰子王` 标题。
- 左侧只显示金币、出手、出千。
- 删除当前骰袋、全部上场、构成、本关关注。
- 目标进度只显示 `0 / 72` 类数字，不显示进度条。
- 结果和结算区只突出基础分数字和倍数数字。
- 结算演出按左到右顺序逐颗入账，基础分和倍数都要跳涨。
- `Space` 提示改短，不做横跨整屏长条。

## 体验意图

玩家在主流程中只需要判断三件事：当前骰子结果、这一手的倍率前分数和倍数、当前累计是否达到目标。F007 通过删减常驻说明和强化数字动画，让结算像计分台过账，而不是像说明书。

## 信息层级

1. 中央骰子和当前结算高亮。
2. 左端积分塔中的当前积分、需要积分、基础分数字和倍数数字。
3. 中央结算表演区域。
4. 左侧金币、出手、出千。
5. 关卡信息。
6. 右侧本关规则和牌型表入口。
7. 短 `Space` 阶段提示。

被删除信息不得以小字、折叠标签或低透明卡片形式常驻回到主流程界面。

F007-01 用户反馈后，积分区应移动到左端；当前积分和需要积分分开显示，不用文字说明含义，而是用图标区分。中间区域应留给结算飞字、入账轨迹、倍率跳涨和其它表演反馈。

F007-01 二次反馈后，中间不应出现独立的遮罩块或单独表演面板。投掷区应整体放大，骰子展示和后续结算表演共用同一个大投掷区空间，强化“骰子结果在投掷区内继续入账表演”的连续感。

F007-01 三次反馈后，投掷区中下部不得出现静态 `+5`、散点、圆弧、角标或其它看不出功能含义的装饰符号。静态版式只能保留能解释为结构的元素：骰位、槽位基线、区域分隔、当前结算列和后续入账通道。动态加分、飞字和倍率反馈只在真实结算演出中出现，不能常驻在待投骰或静态样张中。

F007-01 四次反馈后，投掷区静态排版继续简化：不要预画结算通道框、六列空框、当前列框、垂直连接线、槽位短横线或浅色分区块。投掷区内部静态信息以骰子行和选中高亮为主，其余空间作为连续留白交给运行时动画使用。结算方向、逐颗入账轨迹和倍率反馈必须通过动态演出表达，而不是通过常驻框线预告。

F007-01 五次反馈后，F 版结构可作为方向基础，但执行前必须打磨常见 UI 错误：所有数字和中文文本必须完整落在容器内；图标与数字要使用统一中心线；左侧资源牌宽高一致；积分塔内 `42`、`72`、`21` 和倍率章不得压边、贴边或互相挤压；右侧规则文本与牌型表按钮需要居中；底部 `Space` 按钮不得偏离画面中心或贴近投掷区阴影。

F007-01 最终确认采用 G 版接触图：左侧上方显示金币、出手、出千三张资源牌；左下显示积分塔，当前积分和目标积分分两行且只用图标区分，下方显示基础分数字和倍率章；中间投掷区整体放大，静态层只保留骰子行和大面积连续留白；右侧只保留本关规则与牌型表；底部只保留短 `Space` 按钮。G 版路径为 `Assets/ArtSource/Production/F007/f007_main_run_contact_g_polished.png`。

## 交互流程

1. 进入小关，界面显示标题、关卡、左侧资源、骰位、`0 / 72` 和短 `Space` 提示。
2. 玩家按 `Space` 起摇，阶段提示进入 F004 确认的加力窗口和显示结果流程。
3. 结果锁定后，显示两个核心数字：基础分数字和倍数数字。两个数字不带常驻说明文字。
4. 玩家选择直接结算或出千；出千确认后直接进入结算演出。
5. 结算演出从左到右逐颗高亮骰子。
6. 每颗骰子的倍率前分数汇入基础分数字，数字快速跳涨。
7. 当牌型倍率成立或升级，倍数数字发生明显跳涨或盖章反馈。
8. 每次基础分或倍数变化后，`0 / 72` 滚动到当前展示累计。
9. 结算结束后，最终展示值必须和真实计分结果一致。

## 反馈和动画

- 当前入账骰子：短促高亮、缩放或描边。
- 基础分：每次入账快速滚动或弹跳，避免慢慢数太久。
- 倍数：从 `×1` 到 `×2 / ×3 / ×4 / ×6` 时使用更强反馈，例如盖章、压印、轻震或短促放大。
- `0 / 72`：在基础分和倍数变化后滚动更新，作为结果落地，不抢先跳。
- 临时小骰：继续遵守当前折叠规则；未折叠的小骰可以作为结算队列成员入账。
- 贿赂、金币、成长等非主数字事件：用骰子旁短浮字或小账本反馈，不新增常驻主数字。

## 边界情况

- 无牌型时，倍数保持 `×1`，基础分仍逐颗入账。
- 三同 / 四同 / 五同 / 六同可以在对应同点数量进入已结算前缀时跳涨。
- 顺子可以在已结算前缀形成 5 连时跳涨；若接触图或审查认为不清晰，可改为最终盖章节点。
- 全奇 / 全偶在第 6 颗实体骰确认后跳涨。
- 最终牌型由完整结果决定；动画中的倍数变化不能改变真实倍率。
- 贿赂补分不参与倍数，建议只作为一次性入账反馈并更新 `0 / 72`。
- 如果本次得分已经达标，不允许继续追分；沿用现有流程。

## 适配 / 布局约束

- 设计基准仍是 `1920x1080` 和 `16:9`。
- 当前 OnGUI 内部坐标仍按 `1280x720` 编写并映射到设计画布。
- 核心 UI 在 `1280x720` 目标窗口中不能重叠或溢出。
- 左侧资源不应压缩中央骰区。
- 短 `Space` 提示不得遮挡骰子、计分器或出千选择。
- 文本不使用随视口宽度缩放的字号；主要数字可以用固定层级字号和容器约束。
