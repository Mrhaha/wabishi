## 输入决策

- F007 只改主流程 UI 和结算展示，不改真实计分、投骰概率、出千规则、目标曲线或骰子效果。
- `骰子王` 标题保留。
- 左侧只显示金币、出手、出千。
- 删除当前骰袋、全部上场、构成、基础数量和本关关注。
- 目标进度只显示 `当前累计 / 目标`，例如 `0 / 72`，不使用进度条。
- 结算核心只显示两个数字：倍率前基础分和倍数。
- 结算动画按左到右顺序逐颗入账，并让基础分、倍数和目标进度产生中间态变化。
- UI 布局以 G 版接触图为准：左侧资源牌、左端积分塔、放大投掷区、右侧规则牌和短 `Space` 按钮；路径为 `Assets/ArtSource/Production/F007/f007_main_run_contact_g_polished.png`。

## 实现目标

在 `Assets/Scripts/DiceKingDemo.cs` 当前 OnGUI 原型中，改造主流程 UI 布局和结算演出显示：

1. 静态界面删去低价值信息。
2. 引入双数字计分器。
3. 结算动画暴露并播放基础分、倍数和目标进度的显示中间态。
4. 保证动画播放、加速或跳过都不会改变真实结算结果。

## 已确认行为

- 真实结算仍以现有得分引擎为准。
- 结果锁定、孤证重摇和出千重摇不改变槽位顺序。
- 左到右结算顺序继续使用当前骰袋槽位顺序。
- `0 / 72` 的分母来自当前关卡目标分，分子来自当前小关累计分加本次展示中的入账分。
- 基础分数字为当前演出中已入账的倍率前分数。
- 倍数数字为当前演出中已成立或已展示的倍率。
- 最终显示结果必须等于真实结算结果。

## 测试 / 验证计划

- 静态检查：确认被删除文案不再出现在主流程常驻 UI。
- 基础样例：6 颗基础骰，无牌型，确认基础分逐颗增加、倍数保持 `×1`、`0 / 72` 最终正确。
- 三同样例：确认第三颗同点进入时倍数跳到 `×2`，目标数字同步补涨。
- 四同 / 五同 / 六同样例：确认倍数逐级跳涨或按最终倍率正确呈现。
- 顺子样例：确认五连成立节点倍数跳到 `×2`。
- 全奇 / 全偶样例：确认第六颗确认时倍数跳到 `×2`。
- 龟龟临时小骰样例：确认临时小骰入账不改变主牌型倍率。
- 贿赂样例：确认最终补分不进入倍数数字，且目标数字最终正确。
- 未达标续投样例：确认结算完回到等待投骰时，基础分和倍数保留上一手结果；下一次按 `Space` 起摇后才归零。
- 窗口适配：按 `Docs/UI_ACCEPTANCE_FEEDBACK_WORKFLOW.md` 做截图验收。
