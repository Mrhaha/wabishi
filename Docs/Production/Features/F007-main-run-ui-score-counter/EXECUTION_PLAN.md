## 建议默认值

- 程序实现前，基础分数字内部定义为倍率前分数，不在 UI 上写标签。
- 贿赂等倍率后补分不新增常驻第三数字，只做一次性反馈并更新目标数字。
- 先不新增跳过按钮；如已有输入能自然加速，可作为非阻塞增强。
- 接触图优先比较两版：横向 `基础分 × 倍数` 和左大右章式。
