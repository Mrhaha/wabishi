## 本版范围

本版只实现两颗大树路线补强骰：

- `Gardener` / 园丁骰：大树自然命中时，让大树成长更快。
- `Irrigation` / 灌溉骰：灌溉骰掷中未自然命中大树的命中点数时，补一次成长。

## 已否决或后置

- 果实方向否决：会与龟龟路线过近。
- 嫁接方向否决。
- 幼苗方向否决。
- 年轮、树冠和苗圃草案不作为当前实现依据。
- 本版不新增大树图标资源，只保留运行时资源 key，缺图时使用现有程序化骰面回退。

## 确认规则

### 园丁骰

1. 园丁骰自身按有效点数正常计分。
2. 如果一棵大树自然命中自己的命中点数，该大树获得基础成长 1 次。
3. 每有 1 颗园丁骰，该自然命中的大树额外成长 1 次。
4. 多颗园丁骰叠加；多棵大树自然命中时，每棵自然命中的大树都吃到全部园丁额外成长。
5. 园丁只放大自然命中，不放大灌溉补出的成长。

### 灌溉骰

1. 灌溉骰自身按有效点数正常计分。
2. 如果灌溉骰有效点数等于某棵未自然命中大树的命中点数，则该大树成长 1 次。
3. 每颗灌溉骰每次结算最多浇中 1 棵未自然命中的大树。
4. 同一棵大树每次结算最多获得 1 次灌溉成长。
5. 出千只重摇灌溉骰面，不刷新大树命中点，因此灌溉可以通过出千追目标。

## 实现边界

- 不新增存档字段；成长仍写入大树的 `Faces` 和 `Growth`。
- 成长在二次确认结算动画结束后应用。
- 灌溉成长不是自然命中，不触发园丁。
- 不改变主牌型表、金币系统、失败规则或市场主流程。
