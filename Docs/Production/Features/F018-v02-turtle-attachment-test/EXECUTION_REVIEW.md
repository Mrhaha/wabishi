### 非阻塞后置细节

- 幸运龟建议默认作为掷骰结果改写：宿主每次出手取当前面组最大值；它不是自然掷出最大值，不额外触发“自然最大值”类效果。
- 磁力龟建议继续沿用生产包默认：宿主结算时给其它每颗非龟实体骰各吸附 1 层基础壳；若 Q5 确认为基础壳不计卖价，则该效果风险可控。

- 双倍龟若遇到多个并列最小面，建议全部并列最小面一起翻倍；这不需要单独用户确认。
- 最终七颗龟龟图标不是首测阻塞项，首测可复用现有龟龟图标和短标签。

## 已记录的用户决策

- 龟龟可以吸附在后一个位置的骰子上，并强化个别骰子。
- 吸附时把龟龟自身期望点数加到宿主各个点面。
- 宿主售出时，真实吸附龟卖出金币也一并加上。
- 金钱龟玩家侧表述规范为：结算时，获得 1 金。
- 小小龟玩家侧表述同样规范为结算时触发。
- 首版追求爽感，不设置吸附上限。
- 首版不做同名吸附不叠加限制。
- 每个骰子吸附后，悬浮页要有对应内容说明。
- 吸附扫描发生在离开市场进入下一小关前，按当前骰袋从左到右扫描。
- 双倍龟在吸附时一次性翻倍宿主当前最小面，多只双倍龟可重复叠加。
- 呼朋唤友龟按自身宿主的结算位置触发；如果它先结算，本次后续其它骰子会计算本次效果。
- 首领龟不能吸附到其它骰子上，是独立实体骰。
- 多颗首领龟可以同时响应同一真实吸附事件，但首领之间、复制壳之间不会互相触发。
- 只有真实买入并吸附进去的龟龟计入宿主卖价回收；基础壳和首领复制壳不计入卖价。

## 最终建议

F018 保留爽感方向，包审查阻塞已解除，程序静态接入已完成。下一步建议进入 Unity 编译 / Play Mode 验证，确认首领多骰不递归、无上限叠加、保存恢复、卖价拆分和龟龟测试市场的实际表现。
