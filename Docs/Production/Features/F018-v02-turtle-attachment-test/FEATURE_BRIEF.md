## 功能意图

F018 用于验证 V0.2 龟龟家族的新方向：六颗可吸附龟不再只靠当回合临时小骰滚分，而是通过吸附到后一个位置的宿主骰，压缩六槽、强化单颗关键骰；首领龟则作为独立监听骰留在场上，复制其它真实吸附事件，在结算与悬浮页中制造持续可见的爽感。

本包按用户最新确认优先保留爽感：首版不设置吸附数量上限，不禁止同名龟龟叠加。强度、结算时长和悬浮页信息膨胀都作为测试观察项，不作为前置规则限制。

## 玩家体验

玩家在市场里买到金钱龟、小小龟、双倍龟、幸运龟、磁力龟和呼朋唤友龟后，通过拖拽把它们放到想强化的骰子前一格。进入下一场前，这六类龟龟吸附到后一个位置的宿主，成为宿主的壳层，并且来源龟龟在表现上化壳消失。首领龟不吸附到其它骰子上，它保留为独立实体骰；其它骰子发生真实吸附时，场上的每颗首领龟都会给自己复制一层同类壳。

宿主之后不只是点数变大，还会携带多个可叠加的龟龟结算效果：

1. 金钱龟在宿主结算时给金币。
2. 小小龟在宿主结算时继续吸附基础骰子，让宿主越滚越厚。
3. 双倍龟强化宿主最低面，解决低面拖累。
4. 幸运龟让宿主稳定打出最大面。
5. 磁力龟把基础骰子吸附扩散到其它骰子。
6. 呼朋唤友龟读取其它骰子的龟龟吸附数量，把全队龟壳变成面组成长。
7. 首领龟作为独立骰复制其它真实吸附事件，制造大规模叠壳爆点。

核心爽点是：玩家能把一颗普通骰逐步堆成厚壳核心，并通过悬浮页看到它身上到底叠了哪些龟、每只龟在结算时会做什么、卖出时能回收多少真实吸附龟价值。

## 已确认规则

- 金钱龟、小小龟、双倍龟、幸运龟、磁力龟和呼朋唤友龟可以吸附到后一个位置的非龟实体骰上。
- 首领龟不能吸附到其它骰子上，它保留为独立实体骰。
- 吸附后，宿主获得龟龟的期望点数加值，当前全部点面增加该值。
- 吸附后的真实龟龟不再作为独立上场骰结算，但其吸附效果跟随宿主结算。
- 真实吸附龟卖价随宿主出售一起回收。
- 基础壳和首领复制壳不计入卖价回收。
- 首版不设置吸附数量上限。
- 首版不禁止同名龟龟叠加。
- 金钱龟：结算时，获得 1 金。
- 小小龟：结算时，宿主吸附 1 颗基础骰子。
- 双倍龟：吸附时，宿主当前最小面翻倍。
- 呼朋唤友龟：按自身宿主结算位置触发，早位触发会影响本次后续骰子结算。
- 每次真实吸附事件发生时，场上的每颗首领龟各给自身获得 1 层同类型复制壳；首领之间、复制壳之间不互相触发。
- 宿主 / 首领悬浮页必须展示吸附内容说明。
- 本包优先做龟龟单家族测试，不删除旧龟龟壳链相关代码和资源。

## 首版规则卡

### 通用吸附

| 项目 | 规则 |
|---|---|
| 可吸附来源 | 金钱龟、小小龟、双倍龟、幸运龟、磁力龟、呼朋唤友龟；首领龟不是可吸附来源 |
| 吸附对象 | 可吸附龟后一个位置的非龟实体骰，不跨格查找 |
| 吸附时机 | 离开市场进入下一小关前，按当前骰袋顺序从左到右扫描 |
| 没有后位宿主 | 可吸附龟保留为实体骰，不吸附 |
| 吸附后槽位 | 真实可吸附龟从骰袋实体列表移入宿主吸附列表，不占上场六槽；表现为来源龟化壳消失 |
| 吸附加值 | 使用来源龟龟当前六面期望转移值，加到宿主全部点面 |
| 取整建议 | `max(1, floor(来源骰当前六面总和 / 6))` |
| 叠加 | 不设数量上限，同名可叠加 |
| 结算顺序 | 宿主槽位结算时，按吸附顺序执行宿主身上的龟龟效果 |
| 卖出 | 宿主卖出价 = 宿主自身卖价 + 真实吸附龟固定卖价合计 |
| 首领监听 | 每次真实吸附事件发生时，当前场上每颗首领龟各给自身获得 1 层同类型复制壳 |
| 防递归 | 首领复制壳不产生新的吸附事件，不触发任何首领；首领之间也不互相触发 |

### 六颗可吸附龟和首领龟

| 龟龟 | 玩家侧短规则 | 结算 / 吸附规则 | 叠加口径 | 悬浮页重点 |
|---|---|---|---|---|
| 金钱龟 | 结算时，获得 1 金。 | 宿主结算时，每个金钱龟吸附壳让钱包 +1。 | 多只金钱龟各自触发。 | 显示 `金钱龟 xN：结算金币 +N` |
| 小小龟 | 结算时，宿主吸附 1 颗基础骰子。 | 宿主结算时，每个小小龟吸附壳给宿主新增 1 层基础骰子壳，宿主全部面 +1。 | 多只小小龟每只 +1 层。 | 显示基础骰子壳层数和累计全脸加值 |
| 双倍龟 | 吸附时，宿主最低点翻倍。 | 完成期望转移后，立即读取宿主当前最低面并翻倍该最低面；如多个最低面相同，全部最低面一起翻倍。 | 多只双倍龟按吸附顺序重复执行。 | 显示已翻倍的最低面和触发次数 |
| 幸运龟 | 宿主始终出现最大值。 | 结果锁定到宿主结算前，宿主有效点数视为当前最大面；多个幸运龟不提供额外效果，但允许存在。 | 叠加存在但效果等价。 | 显示 `幸运：本次按最大面结算` |
| 磁力龟 | 结算时，给其他骰子吸附 1 颗基础骰子。 | 宿主结算时，每个磁力龟给其它每颗非龟实体骰各增加 1 层基础骰子壳，全部面 +1。 | 多只磁力龟重复分发。 | 显示本次分发目标数量和基础壳加值 |
| 呼朋唤友龟 | 结算时，其他骰子每吸附 1 只龟，每个面 +2。 | 宿主结算时，读取其它宿主的真实吸附龟数量，让对应其它宿主全部面 `+2 * 数量`；若它先结算，本次后续骰子会计算本次成长。 | 多只呼朋唤友龟重复执行。 | 显示读取对象、吸附龟数量和总加值 |
| 首领龟 | 每次发生真实吸附，自身获得同样的吸附龟。 | 首领龟不吸附到其它骰子上，保留为独立实体骰。任意真实可吸附龟完成吸附时，当前每颗首领龟各给自身新增 1 层同类型复制壳；复制壳不计卖价，不触发新的首领监听。 | 多只首领龟可同时响应同一真实吸附事件；首领之间不互相触发。 | 显示首领自身复制壳列表、来源事件和复制壳不计卖价 |

## 已确认执行口径

- 吸附扫描放在离开市场进入下一小关前，使用市场骰袋当前排序；玩家可通过市场拖拽决定六颗可吸附龟的宿主。
- 若继续游戏恢复到小关开始，已经吸附的龟龟从存档恢复为宿主吸附列表，不重新执行吸附扫描。
- 吸附值复用 F017 吞吃的期望转移公式，降低实现分歧。
- 基础骰子壳只显示为基础壳层，不计入真实吸附龟卖价。
- 首领龟复制壳不计入真实卖价，避免复制壳变成金币套利；这不是吸附上限，只是回收口径。
- 首领龟复制壳不触发新的首领复制，避免无限循环；复制结果仍可叠加保留。

## 验收含义

F018 通过意味着：玩家可以通过排序把六颗可吸附龟放到目标宿主前一格，并在离开市场时吸附到后位宿主；首领龟作为独立实体骰监听真实吸附事件并给自身复制壳；吸附无上限且同名可叠加；金钱龟和小小龟按“结算时”触发；宿主点面、金币、基础壳、最低面翻倍、最大值控制、磁力分发、呼朋唤友成长和首领复制能在结算中可见；悬浮页能清楚展示宿主或首领当前所有吸附内容；卖出宿主时能回收真实吸附龟卖价；强度是否过高由后续 playtest 决定，而不是在本包内提前砍掉。
