# F018 程序交接

状态：静态接入完成，待 Unity 验证
功能：V0.2 龟龟家族吸附测试
关联主文档：PROJECT_CONTEXT.md、GAME_FLOW.md、DICE_ARCHETYPES.md
实现事实来源：Assets/Scripts/DiceKingDemo.cs

## 输入决策

- 金钱龟、小小龟、双倍龟、幸运龟、磁力龟和呼朋唤友龟吸附后一个位置的非龟宿主。
- 首领龟不吸附到其它骰子上，保留为独立实体骰。
- 吸附把龟龟自身期望点数加到宿主全部面。
- 金钱龟：结算时，获得 1 金。
- 小小龟：结算时，宿主吸附 1 颗基础骰子。
- 双倍龟：吸附时，宿主当前最小面翻倍。
- 呼朋唤友龟：按自身宿主结算位置触发，若先结算，本次后续骰子会计算该效果。
- 每次真实吸附事件发生时，场上每颗首领龟各给自身获得 1 层同类型复制壳；首领之间、复制壳之间不互相触发。
- 首版不设置吸附数量上限。
- 首版不禁止同名龟龟叠加。
- 宿主售出时，真实吸附龟卖价一并回收；基础壳和首领复制壳不计入卖价。
- 悬浮页必须展示宿主吸附内容。
- 本包不删除旧龟龟壳链兼容逻辑。

## 实现目标

程序实现应支持一次龟龟单家族测试：

1. 开启 F018 龟龟吸附测试市场。
2. 玩家买入六颗可吸附龟并通过拖拽排序，把可吸附龟放在目标宿主前一格。
3. 玩家买入首领龟时，首领龟保留为实体骰，不会吸附到后一个位置的骰子上。
4. 离开市场进入下一小关前，六颗可吸附龟吸附到后一个位置的宿主，并从实体骰列表移入宿主吸附列表。
5. 每次真实吸附事件完成后，当前场上每颗首领龟各给自身新增 1 层同类型复制壳。
6. 宿主或首领结算时执行自身吸附列表中的龟龟效果，允许无上限叠加。
7. 宿主和首领悬浮页显示所有吸附内容；卖出时只回收真实吸附龟卖价。
8. 保存 / 继续游戏能恢复宿主吸附状态和首领复制壳状态。

## 已确认行为

- 吸附不设数量上限。
- 同名吸附龟可叠加。
- 吸附内容必须可展示。
- 金钱龟和小小龟都用“结算时”作为玩家侧触发口径。
- 双倍龟在吸附时一次性翻倍宿主当前最小面。
- 呼朋唤友龟按自身宿主结算位置触发，早位触发可以影响本次后续骰子。
- 可吸附龟效果跟随宿主结算，不作为独立上场骰重复结算。
- 首领龟作为独立实体骰正常上场结算；它身上的复制壳跟随首领结算。
- 首领龟监听真实吸附事件；复制壳不产生新的吸附事件，不触发首领；多个首领可以同时响应同一真实吸附事件。

## 代码影响

- `DieType`：新增六颗可吸附龟类型和一颗独立首领龟类型。
- `Die` / `DiceData`：需要新增实体骰吸附列表。列表项至少记录：
  - 吸附龟类型。
  - 吸附加值。
  - 来源分类：真实吸附龟、基础壳、首领复制壳。
  - 是否计入卖价。
  - 原始卖价。
  - 来源说明或短标签。
- 市场流程：离开市场前执行吸附扫描；六颗可吸附龟吸附后移除对应实体；首领龟不吸附、不移除。
- 真实吸附事件：只有真实可吸附龟移入宿主列表时才发出；基础壳和首领复制壳不发出真实吸附事件。
- 首领监听：每次真实吸附事件完成后，当前每颗首领龟各给自身添加 1 层同类型复制壳；复制壳不触发任何首领。
- 结算流程：宿主或首领槽位结算时读取自身吸附列表，进入当前槽位局部队列。
- 经济：金钱龟金币进入左到右钱包流；卖出宿主时回收真实吸附龟卖价。
- 悬浮窗：需要动态展示宿主和首领的吸附内容，并区分真实龟、基础壳、首领复制壳。
- 日志：吸附、首领复制、结算触发、面组成长和卖出回收都需要记录。
