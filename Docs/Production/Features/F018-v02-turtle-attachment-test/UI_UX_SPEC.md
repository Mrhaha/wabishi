# F018 界面体验规格

状态：静态接入完成，待 Unity 验证
功能：V0.2 龟龟家族吸附测试
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md、Docs/Production/Features/F014-dice-hover-tooltip-ui/

## 输入决策

- 吸附数量无上限，同名可叠加。
- 每个骰子吸附后，悬浮页必须显示对应内容说明。
- 金钱龟和小小龟统一为“结算时”触发。
- 宿主卖出时回收真实吸附龟卖价。
- 首领龟不吸附到其它骰子上，作为独立实体骰显示；首领复制壳显示在首领龟自身悬浮页中。
- 基础壳和首领复制壳不计入卖价。
- 首版不重做 F014 悬浮窗骨架，只扩展内容。

## 体验意图

玩家必须能看懂“这颗骰子为什么变强”。因为吸附无上限、可叠加，悬浮页不能只写一句总效果，必须展示吸附来源、数量、叠加结果和卖出回收。首领龟需要被看成独立骰，而不是某颗宿主的壳；它身上的复制壳也必须可见。结算时也要有短反馈，让爽感不是隐藏在点面变化里。

## 信息层级

宿主 / 首领悬浮页优先级：

1. 骰子名称、当前六面和是否为首领龟。
2. 吸附摘要：`真实龟 xN`、`基础壳 xM`、`复制壳 xK`、`吸附卖价 +G`。
3. 按类型和来源折叠列表：每类龟龟数量、来源和当前效果。
4. 本次结算预期触发：金币、基础壳、最低面翻倍、幸运最大面、磁力分发、呼朋唤友、首领复制。
5. 卖出回收说明。

## 交互流程

### 吸附扫描提示

- 离开市场前，如果骰袋里有可吸附龟龟，离开按钮附近显示短提示：`离开后龟龟吸附后位`。
- 首领龟悬浮页显示 `独立首领：不吸附，监听真实吸附`。
- 如果可吸附龟后一个位置不是非龟宿主，不触发吸附表现，实体保留。
- 吸附发生后，市场反馈条显示：`金钱龟化壳消失 -> 基础骰：吸附 +3` 这类短日志。
- 如果场上有首领龟，真实吸附日志后追加短反馈，例如 `首领复制 x2：金钱龟`。

### 悬浮页展示

- 待购龟龟悬浮页显示规则短句，例如 `结算时，获得 1 金`。
- 首领龟待购悬浮页显示 `不吸附；每次真实吸附时，自身复制同类壳`。
- 宿主和首领悬浮页新增 `吸附` 区块。
- 吸附数量较少时逐条显示。
- 吸附数量较多时按类型折叠，例如 `金钱龟 x4：结算金币 +4`。
- 基础骰子壳单独显示，例如 `基础壳 x12：全部面 +12`。
- 首领复制壳单独显示，例如 `复制壳 x6：不计卖价`。
- 卖价显示为 `卖出 3 + 真实吸附 8 = 11`；基础壳和复制壳不进入卖价。

### 结算反馈

- 金钱龟：短标签 `钱龟 +N`，金币飞字进钱包。
- 小小龟：短标签 `小壳 +N`，显示宿主全部面增加。
- 双倍龟：吸附发生时显示短标签 `最低翻倍 xN`。
- 幸运龟：结果控制前显示 `幸运最大`。
- 磁力龟：短标签 `磁壳 +N`，其它目标出现小壳反馈。
- 呼朋唤友龟：短标签 `呼朋 +N`。
- 首领龟：真实吸附事件后显示短标签 `首领复制 xN`；首领自身结算时显示其复制壳的折叠触发。

## 反馈和动画

- 吸附发生时，可吸附龟图标或短标签移动到宿主下方或宿主悬浮页记录里；首版可只用市场反馈条。
- 首领复制发生时，首领龟短暂高亮或追加市场反馈条，不把首领移动到其它骰子下方。
- 结算时，吸附触发应在当前槽位局部队列中逐条或折叠播放；首领龟结算时播放其自身复制壳。
- 当触发数量很多时，前几条完整展示，后续折叠成 `金钱龟 x8 +8 金`，避免演出过长。
- 折叠是展示层折叠，不改变真实结算次数。

## 边界情况

- 吸附很多时，悬浮页必须避边并控制高度；超出高度时按类型折叠，不做滚动条首版。
- 普通宿主没有吸附时，不显示空白吸附区；首领龟没有复制壳时可显示短行 `等待真实吸附`。
- 临时小骰不显示吸附悬浮页。
- `Shaking / Stopping / Scoring` 阶段遵守 F014 抑制规则；结算吸附反馈由结算展示负责，不靠 hover。
- 卖出有吸附宿主或带复制壳首领时，必须显示总卖价，避免玩家误解真实回收；复制壳显示但不加钱。

## 适配 / 布局约束

- 继续使用 F014 竖向紧凑悬浮窗。
- `1280x720` 和 `1920x1080` 都不能遮挡市场购买按钮、卖出按钮、刷新按钮和离开按钮。
- 不新增大卡片式常驻详情面板。
- 不用长教程解释吸附，依靠市场短提示、悬浮页和结算短标签。

## 实现备注

- 优先扩展 `dice_type_config.csv` 的 `tooltip_effect` 静态说明。
- 宿主和首领动态吸附内容不能只写进 CSV，需要从运行时吸附列表生成。

## 验收

- 买入龟龟前可读懂每颗龟龟短规则。
- 宿主吸附后，悬浮页能看见吸附龟类型和数量。
- 首领龟悬浮页能看见“独立首领、不吸附、监听真实吸附”的规则。
- 真实吸附触发后，场上每颗首领龟的复制壳都能在悬浮页看到。
- 同名多只龟龟叠加时，悬浮页按数量汇总。
- 基础壳、复制壳层数和累计点面加值可见。
- 卖出宿主或首领时，真实吸附回收金币可见，复制壳不计卖价可见。
- 结算时每类吸附龟都有短标签或折叠反馈。

## 验证状态

- 静态接入完成：悬浮页会展示真实龟、基础壳、首领复制壳、吸附面值加成和真实龟卖价。
- 待 Unity 验证：极端层数下的避边、折叠高度、结算短标签节奏和不同分辨率可读性。
