# F001-A 规则卡

## 共通边界

| 项目 | 规则 |
|---|---|
| 小关开始钱包快照 | 进入小关时先记录钱包金币快照，发生在投资预算锁定之前。 |
| 可用钱包 | 可用钱包指扣除或锁定投资预算后的实时钱包金币，不包含本关账本金币。 |
| 本关账本金币 | 不参与三颗骰的战力计算，也不能被贿赂或投资消费。 |
| 有效点数 | 三颗骰都不改变有效点数。 |
| 手牌识别 | 三颗骰都不新增或修改手牌类型。 |
| 出千 | 出千只重摇骰面；金币快照、贿赂规则和投资预算不因出千刷新。 |
| 钱包扣除顺序 | 投资在小关开始锁定并视为预付扣除；贿赂在玩家确认结算后、通关收入和利息前扣除。 |
| 失败 | 任意小关失败仍直接结束本轮；投资预算不返还，金币骰不提供失败补偿。 |
| 实现状态 | 本文件是已确认规则合同，但本次不改代码、不生成资源、不写入 CSV。 |

## 国库骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 钱越厚，国库骰越稳。 |
| 路线职责 | 奖励持有本金，让金币流拥有“不花钱也能变强”的路线组件。 |
| 触发时机 | 小关开始时记录钱包金币快照；结果预览和真实结算时读取该快照。 |
| 效果公式 | `国库加分 = floor(小关开始钱包 / 10)`；本骰单骰分 = 有效点数 + 国库加分。 |
| 影响对象 | 增加本骰单骰分；不改有效点数、不改手牌、不改钱包、不改本关账本。 |
| 预览 / 结算 | 确定性效果，预览和真实结算必须一致。 |
| 出千交互 | 出千只改变本骰有效点数；国库加分仍读取本小关开始钱包快照。 |
| 持久状态 | 不需要跨小关状态；需要一个小关临时的开始钱包快照。 |
| 市场配置默认值 | 阶级 `T2`；买入价 10；卖出价 5；1-2 章权重 6；3-5 章权重 9；6-10 章权重 7。 |
| 运行界面 | 市场标签：本金越厚越稳；未投掷标签：国库；结果标签：本金 +N；预览行：国库按开局钱包 +N；结算反馈：国库 +N。 |
| 反滥用边界 | 无单颗加分上限；读取小关开始钱包，避免猪猪本关账本金币即时放大；购买国库骰会减少后续本金，形成市场取舍。 |
| 验收检查 | 小关开始钱包 18 时显示 +1；钱包 40 时显示 +4；钱包 50 时显示 +5；出千后加分不刷新；猪猪本关命中不提高本关国库加分。 |

## 贿赂骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 差一点过关时，用可用钱包金币把账本补平。 |
| 路线职责 | 提供金币流的即时救急能力，让保留现金有战术意义。 |
| 触发时机 | 结果预览阶段计算可贿赂提示；真实结算时在基础本次分计算后、通关判断前触发。 |
| 触发合同 | 先计算 `预计累计分 = 本关当前累计分 + 本次基础分`；如果已经达标，不花钱；如果未达标，计算缺口。 |
| 效果公式 | 每 1 金提供 +4 最终分；每颗贿赂骰每次结算最多花 2 金；需要金币为 `ceil(缺口 / 4)`；只有当需要金币不超过可用钱包且不超过总贿赂上限时才花钱。 |
| 影响对象 | 扣除可用钱包金币；追加本次最终分；不改单骰有效点数、不改手牌、不改本关账本。 |
| 自动触发条件 | 只有当可用贿赂能让本次结算通关时才花钱；已经通关或无法补到通关时不花钱。 |
| 预览 / 结算 | 预览显示“贿赂将花 X 金补 Y 分”；玩家确认结算后才扣钱。 |
| 出千交互 | 出千前的贿赂只是预览；出千后根据新结果重新判断。确认结算后才扣钱。 |
| 持久状态 | 不需要 per-die 持久状态；钱包扣除发生在通关收入和利息前，并随当前通关收入保存。 |
| 市场配置默认值 | 阶级 `T2`；买入价 11；卖出价 5；1-2 章权重 5；3-5 章权重 8；6-10 章权重 8。 |
| 运行界面 | 市场标签：差点过关可花钱补分；未投掷标签：贿赂；结果标签：可补；预览行：贿赂将花 X 金补 Y 分；主动作：结算（贿赂 -X 金）；结算反馈：贿赂 -X 金 +Y 分。 |
| 反滥用边界 | 不消耗本关账本金币；不能为已经通关的结果刷额外分；不能在无法通关时白白烧钱；每颗每次最多 2 金；投资锁定的钱不能再用于贿赂。 |
| 验收检查 | 差 7 分且有 1 颗贿赂骰、可用钱包 2 金时，花 2 金补 8 分并通关；差 20 分时不花钱；本关猪猪新赚金币不计入可花钱包。 |

## 投资骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 先把预算投进小关，整关都吃收益。 |
| 路线职责 | 提供金币流的提前下注组件，和贿赂骰的事后救急形成差异。 |
| 触发时机 | 小关开始时，在第一次投骰前自动计算、锁定并预付投资预算；本小关每次预览和结算都读取该预算。 |
| 效果公式 | `总投资预算 = min(投资骰数量 * 2, floor(小关开始钱包 / 3))`；按骰袋顺序分配，每颗最多 2 金；每颗投资骰的单骰加分 = 本骰预算 * 2。预算消耗不返还。 |
| 影响对象 | 小关开始从钱包锁定并扣除预算；增加本骰单骰分；不改有效点数、不改手牌、不改本关账本。 |
| 自动锁定 | 用户已确认允许每个小关开始自动锁定预算。界面必须显示锁定金额、剩余钱包和本关投资加分。 |
| 预览 / 结算 | 确定性效果，整关内预览和真实结算一致。 |
| 出千交互 | 出千只改变有效点数；投资预算和加分不刷新、不退还。 |
| 持久状态 | 需要小关临时预算分配。当前继续游戏恢复到小关开始，可在进入小关时重新计算；如果未来支持小关中途保存，需要保存每颗投资骰预算和已扣钱包。 |
| 市场配置默认值 | 阶级 `T3`；买入价 12；卖出价 5；1-2 章权重 2；3-5 章权重 5；6-10 章权重 7。 |
| 运行界面 | 市场标签：每关自动投入预算；未投掷标签：投资；结果标签：投资 +N；预览行：投资已锁 X 金，本次 +Y；结算反馈：投资 +Y。 |
| 反滥用边界 | 总预算最多为小关开始钱包三分之一；每颗最多 2 金；预算来自钱包，不来自本关账本；预算消耗不返还，避免免费持续加分。 |
| 验收检查 | 小关开始钱包 18 且 1 颗投资骰时，锁定并扣 2 金，本骰每次结算 +4 单骰分；2 颗投资骰时总预算不超过 4 金；猪猪本关收益不增加投资预算。 |

## 规则关系

| 组合 | 预期体验 | 边界 |
|---|---|---|
| 国库 + 贿赂 | 玩家保留现金，既提高国库分，也保留救急金。 | 贿赂花钱后不会改变本小关已锁定国库快照。 |
| 国库 + 投资 | 玩家需要在保本金和预付预算之间取舍。 | 实现时先记录国库快照，再扣投资预算；同一小关内国库读取原始快照。 |
| 贿赂 + 投资 | 投资提供持续分，贿赂只补最后差额。 | 投资已锁定的钱不能再用于贿赂。 |
| 多颗贿赂 | 提供更高救急上限。 | 每颗每次最多 2 金，总花费仍受可用钱包限制。 |
| 多颗投资 | 提供更高整关预算。 | 总预算不超过小关开始钱包三分之一，且每颗最多 2 金。 |

## 已确认风险和缓解

- 投资骰自动锁定预算会影响玩家钱包主权；已确认可接受，但必须在市场文案、小关开始提示、顶部钱包变化和日志中明确展示。
- 贿赂骰虽然自动补分，但只有玩家确认结算后才扣钱，且主动作必须显示扣费。
- 投资加分参与最终手牌倍率，可能在高倍率结果中放大；第一版已降为每颗最多 2 金，后续可通过调参或改为最终分加成处理。
- 新类型存档不能继续依赖固定类型序号和 `0..7` clamp；程序交接已把类型 key 存档列为默认技术方案。
