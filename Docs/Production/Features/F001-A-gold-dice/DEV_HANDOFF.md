## 实现目标

后续程序任务需要把三颗金币骰接入当前单文件原型：枚举、默认面组、市场配置、投掷预览、真实结算、钱包扣除、阶段临时状态、界面标签和文档同步。

推荐实现顺序：国库骰、贿赂骰、投资骰。

## 当前代码事实

- 当前 `DieType` 只有 8 种，新增类型需要追加到末尾，不能改变旧类型序号。
- 当前 `SaveRun` 保存 `DiceData`，字段为名称、类型 key、面组、成长层数。
- 当前 `LoadRunCollection` 已支持类型 key 存档，并兼容旧数字序号；新增类型不会再被固定 `0..7` 范围压成赌徒骰。
- 当前猪猪金币进入 `stageLedgerGold`，通关后才并入钱包。
- 当前继续游戏恢复到小关开始，不恢复小关中途状态。

## 已确认行为

- 国库骰读取小关开始钱包快照，公式为 `floor(小关开始钱包 / 10)`，无单颗加分上限。
- 贿赂骰只在能把本次结算补到通关时生效，每 1 金补 +4 最终分，每颗每次最多花 2 金。
- 投资骰在小关开始自动锁定并预付预算，公式为 `min(投资骰数量 * 2, floor(小关开始钱包 / 3))`，每颗最多 2 金，每金让本骰每次结算 +2 单骰分。
- 小关开始先记录钱包快照，再锁定投资预算；国库读取原始快照，贿赂读取投资后的可用钱包。
- 本关账本金币不参与三颗金币骰计算。
- 贿赂扣钱发生在玩家确认结算后、通关收入和利息前。

## 存档影响

- 国库骰不需要 per-die 持久字段。
- 贿赂骰不需要 per-die 持久字段，但钱包扣除需要和通关收入顺序一致。
- 投资骰需要小关阶段临时预算。当前继续游戏恢复到小关开始，可在重新进入小关时重新分配；如果未来支持小关中途保存，需要保存投资预算和已扣钱包。
- 当前实现已把 `DiceData` 从纯类型序号升级到类型 key，同时兼容旧序号。
- 进入小关时的存档保留投资扣除前的钱包；继续游戏会重新回到小关开始并重新锁定投资预算，避免读取已扣钱包后重复扣费。
