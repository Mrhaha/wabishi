## 建议改进

### 推荐生产包补丁

- 壳匠骰首版从 `每颗龟龟系临时小骰 +2 单骰分` 调整为 `每颗龟龟系临时小骰 +1 单骰分`。
- 壳匠加分仍然是本骰单骰分，会参与最终牌型倍率；这样保留“按小龟数量给分”的核心体验，但降低成型后爆发过强的风险。
- 巢穴骰保留草案：按本次有产出的龟龟链数量补 d1 临时小骰，补充数量不超过巢穴骰数量。
- 巢穴补出的 d1 计入壳匠数量，但不触发巢穴，也不触发其它实体骰效果。
- 慢龟骰保留草案：本体正常计分，临时链使用 `ceil(当前点数 / 2)` 慢衰减；当当前点数 `<= 1` 时停止，避免循环。
- 临时小骰不设置计分上限，所有真实生成的小骰完整计分；界面默认只完整展示前 8 颗，超过部分用汇总行表达，避免结算区拥挤。
- 程序实现顺序建议为：慢龟骰、壳匠骰、巢穴骰。这样先复用当前龟龟链，再接入数量收益，最后补数量来源。

### 已采纳的用户决策

1. 接受 F001-B 按审查默认值推进：壳匠 `+1 / 临时小骰`、巢穴补 d1 且不闭环、慢龟 `ceil(v / 2)` 慢衰减、临时小骰完整计分但显示超过 8 颗时折叠。

### 非阻塞后置细节

- 三颗骰的最终图标可以等规则确认后再生成，不影响当前规则审查。
- 市场权重可以先使用生产包草案，后续通过实测调整。
- 临时小骰折叠只影响显示，不影响计分、日志和验收用例。
