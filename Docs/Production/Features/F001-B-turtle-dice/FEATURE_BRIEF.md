## 功能意图

F001-B 把龟龟流第一批候选 `壳匠骰 / 巢穴骰 / 慢龟骰` 从候选名推进到已确认首版规则。

本切片目标是让龟龟路线不只依赖单颗龟龟骰的随机临时小骰，而是形成三类补强：

- `壳匠骰`：把本次生成的临时小骰数量转换成分数。
- `巢穴骰`：在有龟龟链时补充少量额外临时小骰。
- `慢龟骰`：作为新的龟龟来源，用更慢的衰减产生更长链条。

这三颗骰都必须继续遵守当前结算边界：临时小骰不进入骰袋、不参与实体骰主牌型识别、不触发其它实体骰长期效果。

## 玩家体验

玩家选择龟龟路线后，市场中会出现更明确的路线问题：

- 是买更多龟龟来源，让链条更常出现。
- 是买壳匠，把已经生成的小骰数量变成额外分数。
- 是买巢穴，补一点稳定数量。
- 是买慢龟，让单条链更长、更像龟龟流的中期核心。

玩家应能通过结算动画看到“小龟变多了、壳匠吃到了数量、慢龟链更长”，而不需要读长说明。

## 已记录的用户确认

2026-06-04，用户接受 F001-B 执行审查默认值：

- 壳匠骰：每颗龟龟系临时小骰给壳匠骰 `+1` 单骰分。
- 巢穴骰：按有产出的小龟链补 d1 临时小骰，补骰数量不超过巢穴骰数量，巢穴补骰不触发巢穴闭环。
- 慢龟骰：使用 `ceil(v / 2)` 慢衰减，`v <= 1` 时停止。
- 临时小骰：所有真实生成的小骰完整计分；显示超过 8 颗时折叠为汇总表达。

## 已确认规则

- 龟龟流第一批候选为壳匠骰、巢穴骰、慢龟骰。
- 本切片规则、美术资源、代码、市场数据和主文档同步已完成，并由用户确认验收。
- 临时小骰默认只提供单骰分，不参与实体骰主牌型识别。
- 临时小骰默认不进入骰袋、不保存、不进入市场、不触发猪猪金币、大树成长、赌徒阈值或金币骰效果。
- 出千只重摇实体骰骰面，不重摇已经生成的临时小骰；结算前出千会让龟龟链预览重新计算。
- 临时小骰完整计分；投掷区完整显示前 8 颗，超过部分使用汇总行表达，显示折叠不改变计分、日志和验收。

## 已确认首版规则

| 骰子 | 已确认行为 | 设计后果 |
|---|---|---|
| 壳匠骰 | 本次每生成 1 颗龟龟系临时小骰，本骰单骰分 +1。 | 把龟龟数量转换成可乘倍率的路线 payoff，同时降低首版失控风险。 |
| 巢穴骰 | 本次每条有产出的龟龟链可额外补 1 颗 d1 临时小骰，补充数量不超过巢穴骰数量；巢穴补骰不触发巢穴。 | 稳定补数量，同时避免多龟多巢过度膨胀。 |
| 慢龟骰 | 自身按慢衰减规则生成临时小骰：`ceil(当前点数 / 2)`，点数 1 停止。 | 提供更长龟龟链，是龟龟路线的中期来源骰。 |

## 约束

- 当前运行时只有 `Turtle` 会生成临时小骰；新增龟龟系骰前需要新增 `DieType` 和显示映射。
- 当前临时小骰使用 `scoringDice` 展示，只在结算阶段存在。
- 当前龟龟预览使用期望估算，真实结算随机生成；慢龟和巢穴也需要保持预览和真实结算的区别。
- 当前 UI 已有“小龟预估”和逐颗结算动画，新增数量不应让 1280x720 下的结算区域过载。

## 非目标 / 后置范围

- 不继续扩大 F001-B 范围。
- 不改变全局手牌识别。
- 不让临时小骰触发其它实体骰长期状态。
- 不设计龟龟以外的奇数、偶数或大树切片。
