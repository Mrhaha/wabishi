# F001-B 规则卡

状态：规则已确认
功能：F001-B 龟龟骰规则细化
规则阶段：首版确认

## 共通边界

| 项目 | 规则 |
|---|---|
| 龟龟系临时小骰 | 指由龟龟骰、慢龟骰和巢穴骰在本次结算中生成的临时小骰。 |
| 主牌型识别 | 临时小骰不参与实体骰主牌型识别。 |
| 计分 | 临时小骰默认只追加单骰分并完整计分；壳匠按临时小骰数量给自身加分。 |
| 出千 | 出千只重摇实体骰骰面；确认出千后重新计算龟龟链预览，真实临时小骰仍在结算时生成。 |
| 长期状态 | 三颗候选第一版都不需要跨小关持久状态。 |
| 闭环 | 巢穴补出的小骰不再触发巢穴；临时小骰不触发其它实体骰类型效果。 |
| 显示折叠 | 投掷区完整显示前 8 颗临时小骰，超过部分折叠为汇总行；折叠不影响完整计分。 |
| 实现状态 | 本文件规则已获用户确认；代码、数据和图标仍需后续执行。 |

## 壳匠骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 小龟越多，壳匠越会算。 |
| 路线职责 | 把龟龟路线的临时小骰数量转换成本骰分数。 |
| 触发时机 | 真实结算时，在龟龟系临时小骰数量确定后计算。 |
| 效果公式 | `壳匠加分 = 本次龟龟系临时小骰数量 * 1`；本骰单骰分 = 有效点数 + 壳匠加分。 |
| 影响对象 | 增加本骰单骰分；不改有效点数、不改手牌、不生成临时小骰。 |
| 预览 / 结算 | 预览按龟龟系临时小骰期望数量估算，并标记为预估；真实结算按实际数量给分。 |
| 出千交互 | 出千后龟龟链预览重算，壳匠预估随之更新。 |
| 持久状态 | 不需要跨小关状态。 |
| 市场配置草案 | 阶级 `T2`；买入价 10；卖出价 5；1-2 章权重 5；3-5 章权重 8；6-10 章权重 8。 |
| 运行界面 | 市场标签：小龟越多越会算；未投掷标签：壳匠；结果标签：壳 +N；预览行：壳匠按小龟预估 +N；结算反馈：壳匠 +N。 |
| 反滥用边界 | 只按本次实际临时小骰数量结算；临时小骰不参与主牌型，避免同时抬高牌型概率。 |
| 验收检查 | 本次实际生成 3 颗龟龟系临时小骰时，单颗壳匠骰 +3 单骰分；没有临时小骰时不加分。 |

## 巢穴骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 有龟开窝，巢穴添一只小龟。 |
| 路线职责 | 为龟龟路线提供少量稳定数量补充，并增强壳匠收益来源。 |
| 触发时机 | 真实结算时，在龟龟骰和慢龟骰完成自身链条后触发。 |
| 效果公式 | `巢穴补骰数量 = min(本次有产出的龟龟链数量, 巢穴骰数量)`；每颗补骰为 d1 临时小骰，分数为 1。巢穴补骰计入壳匠数量，但不触发巢穴。 |
| 影响对象 | 生成少量 d1 临时小骰；不改有效点数、不改手牌、不改变已有链条随机结果。 |
| 预览 / 结算 | 预览按有产出链条的期望数量估算；真实结算按实际有产出的链条数量补 d1。 |
| 出千交互 | 出千后龟龟链是否有产出会重算，巢穴预估随之更新。 |
| 持久状态 | 不需要跨小关状态。 |
| 市场配置草案 | 阶级 `T3`；买入价 12；卖出价 5；1-2 章权重 2；3-5 章权重 5；6-10 章权重 7。 |
| 运行界面 | 市场标签：有龟就添小骰；未投掷标签：巢穴；结果标签：巢 +N；预览行：巢穴预估 +N 小骰；结算反馈：巢穴 +N 小骰。 |
| 反滥用边界 | 补骰数量不超过巢穴骰数量；巢穴补出的 d1 不再触发巢穴，也不触发其它实体骰效果。 |
| 验收检查 | 有 2 条龟龟链产生临时小骰且有 1 颗巢穴骰时，额外生成 1 颗 d1；有 2 颗巢穴骰时最多额外生成 2 颗 d1。 |

## 慢龟骰

| 字段 | 已确认规则 |
|---|---|
| 一句话身份 | 慢一点衰减，多走一段链。 |
| 路线职责 | 作为龟龟路线的中期来源骰，让单条临时小骰链更长。 |
| 触发时机 | 真实结算时，本骰有效点数确定后生成慢龟链。 |
| 效果公式 | 慢龟骰本体正常计分；设当前链点数为 `v`，若 `v <= 1` 停止；否则生成最大点数为 `ceil(v / 2)` 的临时小骰，随机取 `1..max`，并继续按同样规则衰减。 |
| 影响对象 | 生成龟龟系临时小骰；不改有效点数、不改手牌。 |
| 预览 / 结算 | 预览显示慢龟链期望分和期望数量，并标记为预估；真实结算随机生成。 |
| 出千交互 | 出千重摇慢龟骰后，按新有效点数重新计算慢龟链。 |
| 持久状态 | 不需要跨小关状态。 |
| 市场配置草案 | 阶级 `T2`；买入价 11；卖出价 5；1-2 章权重 4；3-5 章权重 7；6-10 章权重 8。 |
| 运行界面 | 市场标签：慢衰多一段；未投掷标签：慢龟；结果标签：慢衰；预览行：慢龟预估 +N；结算反馈：慢龟 dN。 |
| 反滥用边界 | 点数 1 停止，避免 `ceil(1 / 2)` 造成循环；临时小骰不参与主牌型。 |
| 验收检查 | 慢龟骰掷出 5 时，第一颗临时小骰最大点数为 3；掷出 1 时不生成临时小骰。 |

## 规则关系

| 组合 | 预期体验 | 边界 |
|---|---|---|
| 龟龟 + 壳匠 | 基础龟龟负责生成，壳匠把数量变成额外分数。 | 壳匠只看本次临时小骰数量，不改变链条。 |
| 慢龟 + 壳匠 | 慢龟更容易产生较长链，壳匠获得更高收益。 | 壳匠加分可吃牌型倍率，需后续调参验证。 |
| 龟龟 / 慢龟 + 巢穴 | 有链条时补少量稳定 d1。 | 巢穴补骰不触发巢穴，不参与主牌型。 |
| 巢穴 + 壳匠 | 巢穴补数量，壳匠吃数量。 | 巢穴补出的 d1 计入壳匠数量，但不继续触发巢穴。 |

## 已确认后仍需验证的风险

- 壳匠加分参与最终倍率，龟龟数量成型后可能偏强；首版已降为每颗临时小骰 +1，后续通过价格和权重继续验证。
- 慢龟真实结算随机性比普通龟龟更高，预览必须清楚标记为预估。
- 巢穴和壳匠组合会把数量补充转成可乘分数，需要后续实测价格和权重。
- 临时小骰过多时可能拉长结算动画；显示超过 8 颗时折叠，但所有临时小骰仍完整计分。
