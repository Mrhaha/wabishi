## 摘要

本轮裁决推翻上一版“先攒发芽 / 成长进度”的中间层。F011 不新增“发芽”“芽点”“成长进度”或 `GrowthProgress`，也不因为该成长模型新增保存字段。任一大树候选触发效果后，直接让触发骰自身所有面整体 `+1`，并沿用现有 `Growth +1` 的长期资产表达；预览不能提前写入面组。

旧 `Tree / Gardener / Irrigation` 已从市场下架，但代码和资源保留作为兼容对象。`肥料树` 读取当前金币存量，按标准金币步长直接换算成长次数，默认绑定 `global.csv` 的 `interest_gold_step`；当前 `starting_gold = 1000` 只是测试期配置，不作为肥料步长或强度基准。

## 已写回生产包的裁决

1. 直接成长裁决：触发效果后直接真实成长，不新增发芽、芽点、成长进度或 `GrowthProgress`。
2. 成长效果裁决：真实成长等于本骰自身全部面 `+1`，现有 `Growth +1`；不影响本次已经锁定的有效点数、牌型或分数。
3. 旧树处理裁决：F011 实现时旧 `Tree / Gardener / Irrigation` 从市场生成中下架；代码兼容和旧资源暂不删除。
4. 肥料步长裁决：`肥料树` 默认使用 `interest_gold_step` 作为标准金币步长，不新增 `fertilizer_growth_gold_step`。
5. 测试金币裁决：当前 `starting_gold = 1000` 只用于测试期市场和触发验证，不用于倒推肥料步长。
6. 根系判定裁决：根系不解析 `RoundNote` 文本，应读取结构化类型触发标签；基础计分、主牌型倍率、材质触发和无条件常驻被动不计入。
7. 年轮裁决：`年轮树` 每次真实结算结束最多触发一次，不按小关限一次。
8. 牌谱裁决：`牌谱树` 首版目标池为三同、顺子、全奇、全偶；四同、五同、六同视为命中三同目标。
9. 冠层裁决：`冠层树` 首版只判断本骰掷出自身当前最高面，不要求同时是全场最高点。
10. 修枝裁决：`修枝树` 只有在确认出千且结果改善、牌型改善或帮助通关时直接成长。
