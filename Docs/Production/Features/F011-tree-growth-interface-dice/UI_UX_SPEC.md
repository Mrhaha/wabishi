# F011 界面体验规格

状态：已静态接入，待截图验收
功能：F011 大树长期成长接口骰
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 大树骰需要显示触发来源和真实成长入档。
- F011 不显示“发芽”“芽点”“成长进度”或类似资源条。
- 主流程不放长规则说明。
- 结算反馈必须能说明“为什么成长”。
- 根系树读取相邻类型触发标签，玩家侧只显示短反馈。

## 体验意图

玩家应能在三个位置读懂大树：

1. 市场卡：我买它是为了触发哪类成长。
2. 投骰结果：它当前目标或可触发条件是什么。
3. 结算反馈：本次是否触发、触发了几次真实成长、面组如何变化。

## 界面 / 状态

| 状态 | 玩家目标 | 可见信息 | 主操作 | 次操作 | 退出 / 下一状态 |
|---|---|---|---|---|---|
| 市场 | 判断是否买入大树骰 | 类型图标、短规则、价格、面组、材质 | 买入 | 查看详情、刷新、离开 | 进入骰袋或继续市场 |
| 投骰准备 | 判断槽位和触发关系 | 根系相邻位置、当前骰袋顺序 | 调整槽位 | 查看骰子详情 | 开始投骰 |
| 结果决策 | 判断是否出千或结算 | 点籽目标、牌谱目标、冠层最高面、修枝出千提示 | 结算 / 出千 | 查看牌型表 | 结算或出千 |
| 出千选择 | 判断是否追成长 | 修枝树选中态、点籽目标不刷新提示 | 选择 1-3 颗重摇 | 取消出千 | 出千后结算 |
| 结算展示 | 理解成长来源 | `籽中成长`、`谱成成长`、`冠顶成长`、`修枝成长`、`年轮成长`、`肥料成长 xN`、`根:邻触发` | 观看结算 | 后续可加速 | StageClear / Ready |
| 成长入档 | 确认长期资产变化 | `成长入档`、面组变化、成长层数变化 | 继续 | 查看骰子详情 | 关卡或市场 |

## 信息层级

1. 骰子点数和类型身份。
2. 本次短状态：目标点、目标牌型、最高面、出千提示或相邻关系。
3. 触发短标签。
4. 面组变化和详细来源日志。

## 建议短标签

| 骰子 | 常驻 / 目标标签 | 触发标签 |
|---|---|---|
| 点籽树 | `籽5` | `籽中成长` |
| 牌谱树 | `谱:顺`、`谱:三` | `谱成成长` |
| 冠层树 | `冠6` | `冠顶成长` |
| 修枝树 | `修枝` | `修枝成长` |
| 年轮树 | `年轮` | `年轮成长` |
| 肥料树 | `肥` | `肥料成长 xN` |
| 根系树 | `根` | `根:左`、`根:右`、`根:邻触发` |

## 交互流程

- 点籽和牌谱目标在出手开始固定。
- 出千不刷新点籽目标或牌谱目标。
- 修枝树只有被确认出千并实际重摇后才显示成长可能。
- 根系树在投骰准备阶段通过相邻位置产生摆位决策。
- 结算阶段只显示已确认真实触发，不把预览估算写成入档。
- 肥料树如果一次触发多次成长，主标签显示次数，详情日志列出最终面组变化。

## 反馈和动画

- 触发成长时用轻量树纹或叶片闪烁，不抢主分数结算。
- 真实成长时显示 `成长入档`，再展示面组变化。
- 肥料树多次成长时不逐次播放过长动画，可显示 `肥料成长 xN` 后一次性展示最终面组。
- 根系触发时可从相邻骰向根系树做一条短连接线或根须闪烁。
- 肥料树可以用金币小粒落入土壤，但不要表现为吐金币。

## 边界情况

- 左右都触发根系时，只显示一次，可在日志写“左右均触发，根系只成长一次”。
- 肥料树 `xN` 可能较大，标签需要容纳两位数。
- 年轮树稳定触发，不应有过长动画。
- 牌谱目标未命中时不显示失败惩罚。
- 不显示 `0/3`、`1/3`、`2/3` 这类进度条。

## 适配 / 布局约束

- 继续遵守主流程 `1280x720` 内部布局映射和 16:9 约束。
- 标签优先 2 到 6 个字；肥料多次成长可使用短数字后缀，不能遮挡骰面点数。
- 详情长说明进入骰子详情或结算日志，不进入主投掷区常驻层。

## 实现备注

- 界面不决定根系是否触发；根系读取程序侧类型触发标签。
- 结算展示事件应只读提交态，不二次调用真实计分。
- 界面不新增成长进度状态，只读取真实成长结果和触发来源。

## 剩余验证项

- 程序侧类型触发标签已落为 `TypeTriggeredThisSettle`，仍需截图和样例确认玩家能读懂根系来源。
- 肥料树需要在正式经济配置下验证多次成长标签长度。

## 验收

- 玩家不打开长说明也能读懂七颗短标签。
- 出千界面能看懂修枝和点籽追目标。
- 结算日志能说明根系读取了哪个相邻骰。
- 成长入档不会被误读成本次分数变化。
- 界面中不出现“发芽”“芽点”“成长进度”作为 F011 机制。
