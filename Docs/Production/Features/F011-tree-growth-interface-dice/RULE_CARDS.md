# F011 大树规则卡

状态：已提案
功能：F011 大树长期成长接口骰
来源：Docs/DesignDialogues/D1-future-dice-types.md 第 37-44 轮

## 统一成长模型

状态：已裁决

- F011 不新增“发芽”“芽点”“成长进度”或 `GrowthProgress` 概念。
- 大树类骰触发效果后，直接产生真实成长。
- 真实成长表现为本骰所有面整体 `+1`、`Growth +1`。
- 这里的“本骰所有面”指触发骰自身的六个骰面，不是全骰袋所有骰子。
- 真实成长只在二次确认后的真实结算中入档，不影响本次已经锁定的有效点数、牌型或分数。
- 预览只能提示可能成长，不能提前写入面组或 `Growth`。

## 点籽树

状态：已提案
触发点：指定点数
玩家短规则：命中指定点，本骰成长。

规则：

- 每次出手固定一个指定点。
- 指定点优先从本骰当前面组去重后抽取，避免出现本骰掷不到的目标。
- 本骰最终有效点等于指定点时，本骰真实成长 `1` 次。
- 出千不刷新指定点，因此玩家可以用出千追目标。
- 每次出手最多触发一次。

界面标签：

- 目标：`籽5`
- 触发：`籽中成长`
- 未触发：保留目标标签，不显示失败长文。

风险：

- 如果直接替换当前 `Tree`，需要处理旧存档和旧市场文案。
- 本包裁决为旧 `Tree / Gardener / Irrigation` 市场下架，但旧代码兼容保留到程序切片执行。

## 牌谱树

状态：已提案
触发点：指定牌型
玩家短规则：成指定牌型，本骰成长。

规则：

- 出手开始或小关开始指定一个已有牌型大类。
- 首版目标池：三同、顺子、全奇、全偶。
- 最终确认结算时命中目标牌型，本骰真实成长 `1` 次。
- 四同、五同、六同视为命中三同目标。
- 不新增全局牌型，不改变本次牌型倍率。
- 每次出手最多触发一次。

界面标签：

- 目标：`谱:顺`、`谱:三`
- 触发：`谱成成长`

风险：

- 牌型目标池过宽会让触发太稳定；首版不加入对子、葫芦、两对等牌型。
- 需要与现有全奇 / 全偶副牌型显示保持一致。

## 冠层树

状态：已提案
触发点：最高点数
玩家短规则：掷出自身最高面，本骰成长。

规则：

- 本骰最终有效点等于自身当前面组最高面时，本骰真实成长 `1` 次。
- 首版不使用“同时是本次实体骰最高点”的强版本。
- 每次出手最多触发一次。
- 成长后的高面会自然提高后续冠层触发上限和视觉资产感。

界面标签：

- 触发：`冠顶成长`

风险：

- 高面模板、成长层数和最高面材质可能叠加过强。
- 首版只读“自身最高面”，避免 UI 同时解释“自身最高”和“全场最高”。

## 修枝树

状态：已提案
触发点：出千
玩家短规则：出千变好，本骰成长。

规则：

- 本骰被确认出千并实际重摇后，才可能触发。
- 若新结果相对旧结果有明确改善，或帮助形成目标牌型，或帮助通关，则本骰真实成长 `1` 次。
- 进入出千界面、选中后取消、未选择本骰、或没有实际重摇，不触发。
- 每次出手最多触发一次。

默认改善判定：

- 新有效点大于旧有效点。
- 或出千后最终牌型倍率高于出千前预览牌型倍率。
- 或本次结果从未通关变为通关。

界面标签：

- 触发：`修枝成长`
- 选择态提示：`变好则成长`

风险：

- 如果只看点数变大，容易忽略牌型价值。
- 如果允许通关判定触发，需要保证预览和真实结算口径一致。

## 年轮树

状态：已提案
触发点：回合结束
玩家短规则：每次出手后，本骰成长。

规则：

- 每次真实结算结束时，本骰真实成长 `1` 次。
- 每次出手最多触发一次。
- 不按小关限一次。
- 不要求本次没有其它大树成长。

界面标签：

- 触发：`年轮成长`

风险：

- 稳定触发容易成为泛用必买底座。
- 需要通过市场章节、价格、权重或后置实现批次控制强度。

## 肥料树

状态：已提案
触发点：金币
玩家短规则：按当前金币，本骰成长。

规则：

- 每次真实结算结束时读取当前钱包金币存量。
- 使用标准金币步长计算成长次数，默认绑定 `global.csv` 的 `interest_gold_step`。
- 公式：`floor(当前钱包金币 / interest_gold_step)` 次真实成长。
- 不设置单次成长次数上限。
- 不创造金币，不消费金币，不读取金币变化次数。
- 每次出手最多计算一次。
- 当前 `starting_gold = 1000` 只是测试期配置，不能作为肥料步长的调参依据。

界面标签：

- 触发：`肥料成长 xN`

风险：

- 肥料树直接成长且不设单次上限，标准经济下也可能成为高本金强 payoff。
- 当前测试起手 1000 金会使该骰极端放大；F011 验证时必须使用正式经济或专门记录测试偏差。
- 如果未来调整标准金币步长，应同步影响肥料树，而不是单独设一个按测试金币倒推的特例步长。

## 根系树

状态：已提案
触发点：槽位触发
玩家短规则：相邻骰触发效果，本骰成长。

规则：

- 左邻或右邻任一实体骰在本次真实结算中触发自身类型效果时，根系树真实成长 `1` 次。
- 每次出手最多触发一次。
- 左右都触发也只成长一次。
- 边缘槽位可读取唯一相邻骰。
- 根系树自身成长，不再作为其它根系树的触发来源。

可计入类型事件：

- 金币实际入账或支付。
- 临时小骰生成。
- 自动重摇实际发生。
- 大树真实成长。
- 出千专属收益。
- 条件型单骰分替换或加分。
- 牌型借位或补全实际被采用。
- 赌徒爆发或归零。

不计入：

- 基础点数计分。
- 主牌型倍率。
- 基础骰无效果。
- 奇数或偶数骰的纯面组限制。
- 双倍这类无条件常驻修正。
- 材质触发。
- 预览估算。
- 未被采用的候选牌型。

界面标签：

- 触发：`根:左`、`根:右` 或 `根:邻触发`
- 详情日志：显示具体相邻骰来源。

程序备注：

- 不应通过解析 `RoundNote` 文本判定。
- 建议新增内部类型触发标签，先收集非根系骰标签，再结算根系。

风险：

- 若把常驻被动也计入，根系会过于稳定。
- 若事件标签过多，程序和 UI 容易变重，需要首版保持最小标签集。
