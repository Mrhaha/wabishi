## 功能意图

F011 将大树从“只服务大树路线的成长套件”整理为长期成长接口。玩家不需要只追旧 `Tree / Gardener / Irrigation` 的封闭组合，而是可以通过指定点数、指定牌型、最高点数、回合结束、金币存量、出千和槽位触发等可见事件，把不同构筑接入真实成长。

本包已整理本版本需要的大树骰子规则卡、接触图需求、界面提示和程序交接，并已按用户接受的第三版接触图生成最终运行时图标。当前已开始程序阶段，七颗新大树接口骰已静态接入代码和正式市场数据；旧三件套已从正式市场权重和测试全池随机中过滤，仍待 Unity 运行验证。

## 玩家体验

玩家在市场看到新大树骰时，应能快速判断“我这局最容易让哪种树成长”。点数控制局可以拿 `点籽树`，牌型局可以拿 `牌谱树`，高面局可以拿 `冠层树`，出千局可以拿 `修枝树`，金币本金局可以拿 `肥料树`，稳定慢养可以拿 `年轮树`，摆位连锁局可以拿 `根系树`。

大树的爽点来自长期资产真的变大，而不是先攒一层发芽资源。结算时玩家需要看到触发来源和真实成长入档；成长效果统一为触发骰自身所有面整体 `+1`。

## 已确认规则

- 本包整理七颗大树重设骰，均以七类触发点为来源。
- 大树共同收益是真实成长；不再引入发芽、芽点、成长进度或额外持久进度字段。
- 真实成长只在二次确认后的真实结算中入档。
- 真实成长效果统一为本骰所有面整体 `+1`、`Growth +1`。
- 大树成长不影响本次已经锁定的有效点数、牌型或分数。
- `肥料树` 读取当前金币存量，按标准金币步长直接换算成长次数，不加单次上限，不创造金币，不消费金币。
- `starting_gold = 1000` 是测试期配置，不作为肥料步长调参依据。
- `年轮树` 按每次出手限一次，不按小关限一次。
- `根系树` 读取相邻实体骰的可见类型触发标签，左右都触发也只成长一次。
- 根系树触发不再作为其它根系树的触发来源，避免根系互相连锁。
- 旧 `Tree / Gardener / Irrigation` 在 F011 实现后市场下架；下架不等于立刻删除历史代码。
