## 验证备注

当前已静态接入代码和数据，但尚未生成美术资源，也不能声明 Unity 验收通过。后续必须至少验证以下样例：

- 开启恶魔测试池后，普通随机市场不刷出贡品。
- 开启恶魔测试池后，随机候选只包含五颗常规恶魔骰，贡品只由深渊召唤强制生成。
- 有深渊召唤时，进入市场首批货架固定出现一个贡品。
- 有深渊召唤时，付费刷新固定出现一个贡品。
- 购买普通骰或贡品后货架留空，不自动补货，也不触发深渊召唤。
- 贡品购买后不进入骰袋，并把吞吃值加给最低点数实体骰的全部面。
- 每颗魔蝠离开市场时最多吞吃 1 颗货架骰，并触发同一套吞吃事件。
- 魔蝠离场吞吃后，目标骰详情页能看到最近一次吞吃来源和加值。
- 魔蝠离场吞吃后，下一小关 Ready 阶段能直接看到市场反馈条和目标骰 `吞+N` 短标签。
- 有魔蝠离场效果时，点击一次离开市场不会立刻进入下一小关，而是先停留在市场按当前骰袋位置顺序逐个展示离场效果；所有表现结束后自动离开，不需要再次点击。
- 吞噬骰子能读取贡品和魔蝠吞吃并给金币。
- 恶魔骰子的临时市场最低点离开市场后清空。
- 小鬼的市场最低点作为本轮持续状态保存和恢复。
