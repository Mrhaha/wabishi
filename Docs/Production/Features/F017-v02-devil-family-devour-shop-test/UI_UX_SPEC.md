# F017 界面体验规格

状态：已静态接入，待截图验证
功能：V0.2 恶魔家族吞吃与养市场测试
关联界面文档：PROJECT_CONTEXT.md、GAME_FLOW.md

## 输入决策

- 恶魔路线的关键状态在市场中发生。
- 玩家需要看清市场基础最低点、本市场临时最低点、贡品吞吃收益和离场魔蝠收益。
- 市场购买后不自动补货，买完的货架留空；玩家只能通过刷新重新补货。
- 进入市场和付费刷新都算刷新市场；有深渊召唤时固定出现 1 个贡品。
- 不重做市场主界面框架，沿用当前三货架、骰袋列表、刷新、卖出和离开按钮。
- 详细规则说明仍放在悬浮窗和日志，常驻层只放关键短反馈。

## 体验意图

玩家进入恶魔市场时，应能快速判断三件事：

1. 当前市场骰子已经被养到了什么下限。
2. 哪些动作会继续抬高本市场货架。
3. 购买贡品或离开市场会把多少价值吞吃到哪颗骰上。

界面目标不是解释每条数学规则，而是让市场动作的因果链足够短：养市场，买贡品，最低骰变强，吞吃给金币。

## 信息层级

市场顶部优先级：

1. 金币、骰袋容量、刷新费用。
2. 市场最低点：基础 + 本市场临时。
3. 恶魔测试池或贡品固定来源提示。

货架卡优先级：

1. 骰子名称和价格。
2. 当前已应用强化后的六面。
3. 若是贡品，显示“奉献 +N -> 目标名”。
4. 若购买会触发恶魔骰子，显示短提示“买后市场 +1”。
5. 购买后的空货架显示“已售出 / 刷新补货”，购买按钮禁用。

离开按钮附近优先级：

1. 是否有魔蝠离场吞吃。
2. 预计被吞吃货架和目标骰。
3. 如果没有可吞吃货架，显示无触发即可，不弹阻塞确认。

## 交互流程

### 购买贡品

1. 玩家悬浮贡品货架。
2. 货架显示贡品当前六面、吞吃值和自动目标。
3. 玩家点击购买。
4. 扣金币，贡品不进入骰袋。
5. 目标骰六面全部获得吞吃值。
6. 触发吞吃事件，吞噬骰子按槽位响应金币。
7. 该货架变为空货架，不自动补货。
8. 市场显示短反馈和日志。

### 购买普通骰

1. 玩家点击购买普通市场骰。
2. 扣金币并加入骰袋。
3. 按当前骰袋槽位从左到右扫描购买触发。
4. 每颗恶魔骰子使本市场临时最低点 +1。
5. 当前未购买货架立即应用新的临时最低点。
6. 被购买货架变为空货架，不自动补货。
7. 后续只有进入市场或付费刷新生成的新货架应用新的临时最低点。

### 进入市场 / 付费刷新

1. 玩家进入市场时，系统生成首批整批货架；该动作算一次刷新市场。
2. 玩家点击付费刷新时，扣刷新费用并重新生成整批货架，原有空货架和未购买货架都被替换。
3. 如果有深渊召唤，整批货架中固定出现一个贡品骰子。
4. 多颗深渊召唤首版不叠加贡品数量。
5. 其它货架按当前测试池或正式池生成。
6. 新货架全部应用市场基础最低点和本市场临时最低点。

### 离开市场

1. 玩家点击离开。
2. 如果没有离场效果，直接进入下一小关或完成本轮。
3. 如果有魔蝠这类离场效果，市场立即锁住购买、卖出、刷新和拖拽，离开按钮进入不可点的结算中状态。
4. 系统按当前骰袋位置从左到右扫描有离场效果的骰子。
5. 每颗魔蝠最多吞吃 1 颗市场骰，默认吞吃当前货架从左到右第一颗可吞吃市场骰；没有可吞吃货架时显示无货可吞并继续队列。
6. 被吞吃价值给当前最低点数实体骰，并触发吞噬骰子金币反馈。
7. 所有离场效果逐个展示完毕后，清空本市场临时最低点，自动进入下一小关或完成本轮。

## 反馈和动画

- 小鬼触发：结算日志和短标签显示“市场 +1”。
- 恶魔购买触发：市场反馈显示“本市场最低点 +1”。
- 贡品购买：货架飞向目标骰或用短标签表达“贡品奉献 +N”。
- 空货架：显示“已售出 / 刷新补货”，不播放补货动画。
- 魔蝠离场：点击一次离开市场后，在市场内按当前骰袋位置顺序逐颗触发离场效果，货架被吞后立即留空并显示“第 N 位魔蝠吞吃 X，Y +N”；此时购买、卖出、刷新和拖拽锁定，离场效果播放完毕后自动进入下一小关；下一小关主投掷区顶部继续显示市场反馈条，目标骰在该反馈窗口显示 `吞+N` 短标签。
- 吞噬骰子：金币飞字使用现有金币反馈，不新增大动画。
- 深渊召唤：刷新后贡品货架显示“深渊贡品”短标签。

## 边界情况

- 骰袋为空时仍不能离开市场；贡品也不能作为补空骰进入骰袋。
- 没有实体目标时，贡品购买禁用或不生成购买入口。
- 没有可吞吃货架或只剩空货架时，魔蝠显示“无货可吞”，不阻塞自动离开。
- 多只魔蝠、多个吞噬骰子按骰袋槽位从左到右显示和结算。
- 本市场临时最低点只在当前市场显示，离开后不得残留。
- 贡品不显示卖价，不进入骰袋排序，不进入卖出区域。
- 空货架没有价格、面组和购买按钮；只能通过付费刷新重新变成可购买货架。

## 适配 / 布局约束

- 不改变当前市场三货架主体结构。
- 不新增大弹窗或教程墙。
- 恶魔相关状态优先用短标签和悬浮窗，不挤占货架价格和购买按钮。
- `1280x720` 和 `1920x1080` 下，市场最低点文本不得覆盖金币、刷新费用或按钮。
- 超过 6 的点数继续使用居中数字表达，不绘制密集点阵。

## 验收

- 玩家不读长日志也能看出当前市场最低点。
- 贡品购买前能看出目标和加值。
- 购买后货架留空，且能看出需要刷新才能补货。
- 进入市场和付费刷新都能按深渊召唤规则固定显示贡品。
- 离开市场前能看出魔蝠是否会触发吞吃。
- 恶魔骰子的临时加成离开市场后不再显示。
- 深渊召唤刷出的贡品不会被误认为普通随机骰。
- 截图中市场信息不重叠，按钮文本可读。
