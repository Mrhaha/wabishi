# F017 程序交接

状态：已静态接入，待 Unity 验证
功能：V0.2 恶魔家族吞吃与养市场测试
关联主文档：PROJECT_CONTEXT.md、GAME_FLOW.md、DICE_ARCHETYPES.md
实现事实来源：Assets/Scripts/DiceKingDemo.cs

## 输入决策

- 恶魔家族核心为吞吃和养市场。
- 吞吃会把被吞吃骰子的当前期望值加到目标骰当前所有骰面。
- 吞吃和后续吸附共用同一套期望转移逻辑：来源骰当前期望转移值加到目标实体骰当前所有骰面。
- 小鬼在回合结束提高市场骰子最低点。
- 吞噬骰子在吞吃发生时给金币。
- 恶魔骰子在每次购买后提高本市场最低点，离开市场失效。
- 每颗魔蝠骰子在离开市场时最多吞吃 1 颗市场骰子，收益给最低点数实体骰。
- 深渊召唤在进入市场和付费刷新时固定刷出 1 个贡品骰子。
- 贡品骰子六面都是 1，吃市场强化，自动奉献，不能随机刷出。
- 市场购买后不自动补货，买完货架留空；只有进入市场和付费刷新会整批补货。
- 本包继续遵守 F015 的六槽、单次出手、无出千、无全局牌型计分和左到右队列。

## 实现目标

程序实现应让用户能单独验证恶魔家族市场循环：

1. 开启恶魔家族测试市场。
2. 市场强化能改变当前和后续市场骰的最低点。
3. 深渊召唤能让进入市场和付费刷新固定出现贡品。
4. 购买贡品会自动吞吃并永久提高最低点数实体骰的全部面。
5. 购买后货架留空，不自动补货；付费刷新能重新生成整批货架。
6. 离开市场时每颗魔蝠最多吞吃 1 颗剩余市场骰。
7. 吞噬骰子能读取贡品和魔蝠触发的吞吃事件并给金币。
8. 恶魔骰子的本市场临时加成离开后清空。

## 已确认行为

- 贡品骰子不能进入普通随机市场池。
- 贡品购买后不进入骰袋、不占六槽、不可卖出。
- 吞吃是市场动作确认后的真实写入，不在预览阶段改面组。
- 被吞吃骰子用于计算吞吃值时，必须已经应用当前市场最低点强化。
- 首版整数面组实现的期望转移值为 `max(1, floor(来源骰当前六面总和 / 6))`，并应封装成可供后续吸附复用的公共逻辑。
- 吞噬骰子读取吞吃事件，不要求自己是吞吃来源或目标。
- 魔蝠离场吞吃在进入下一小关前完成；每颗魔蝠每次离开市场最多吞吃 1 颗市场骰。
- 恶魔骰子的临时最低点加成离开市场后失效。
- 购买普通市场骰和贡品骰后，对应货架变为空货架，不执行单格补货。
- 进入市场首批整批货架生成和付费刷新都算刷新市场；购买不会触发深渊召唤。
- 至少一颗深渊召唤在骰袋中时，每次刷新市场固定生成 1 个贡品；多颗深渊召唤首版不叠加贡品数量。
- 恶魔测试池只随机刷小鬼、吞噬骰子、恶魔骰子、魔蝠骰子和深渊召唤；贡品随机权重保持 0。

## 代码影响

- `DieType`：新增六个恶魔相关类型，贡品可以作为市场专用类型存在，但不得进入普通随机池。
- 市场生成：新增市场最低点应用函数，确保普通货架、测试随机、进入市场整批生成、付费刷新和贡品都走同一套面组强化。
- 市场刷新：进入市场首批整批货架生成和付费刷新都算刷新市场；深渊召唤存在时，整批货架生成固定替换一个贡品位。
- 市场购买：购买普通骰、购买贡品和购买后空货架状态都需要明确顺序；购买不自动补货。
- 市场离开：点击离开市场后、进入下一小关前，按当前骰袋位置从左到右自动处理所有离场效果；播放期间锁住市场操作，队列完成后自动转场。
- 经济：吞噬骰子给金币应进入钱包，并能影响之后同一市场的购买和刷新。
- 存档：市场基础最低点如果采用本轮持续默认值，需要保存；本市场临时最低点离开后清空。
- 预览：可以显示吞吃预估，但不得提前写入面组。
- 日志：所有吞吃都应记录来源、目标和加值，便于验证。

## 存档影响

建议字段：

- `TavernBaseMinimumBonus`：小鬼造成的本轮持续市场基础最低点加成。
- `MarketTemporaryMinimumBonus`：本市场临时最低点加成，如当前架构保存市场状态则需要恢复；若市场不支持中途恢复，则至少保证离开市场清空。

不建议首版字段：

- 不做每颗骰的吞吃累计历史。目标骰的面组保存真实成长结果，并额外保留最近一次吞吃来源和加值，用于详情页验证贡品 / 魔蝠效果；有魔蝠这类离场效果时，点击离开市场后按当前骰袋位置从左到右自动结算并展示所有离场效果，锁定购买 / 卖出 / 刷新 / 拖拽，表现完毕后自动进入下一小关；下一小关继续用主投掷区市场反馈条和目标骰 `吞+N` 短标签表达本次离场吞吃。
- 不把贡品写入 `DiceData`，因为贡品不进入骰袋。

如果新增存档字段，应提升 `SaveVersion`，并按当前项目习惯清理不兼容旧运行存档，保留设置和开场状态。
