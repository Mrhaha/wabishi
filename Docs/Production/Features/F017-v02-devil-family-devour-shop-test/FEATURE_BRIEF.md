## 功能意图

F017 是 V0.2 四大家族中的恶魔家族生产包。它把恶魔家族定位为“吞吃”和“养市场”的基础路线：玩家不是只在战斗里等骰子触发，而是在市场里把市场骰子养大，再通过吞吃把市场价值转成自己骰袋的长期面组成长。

本包已完成首版规则、界面、程序交接和静态代码 / 数据接入。后续应通过 Unity 编译、Play Mode 和单家族测试模式验证恶魔循环是否能形成清楚的市场决策、可见成长和可控上限。

## 玩家体验

玩家在战斗中保留六槽构筑和左到右结算。恶魔路线的主要反馈发生在通关后的市场：

1. 小鬼让后续市场骰子的最低点逐步提高。
2. 恶魔骰子让每次购买继续抬高本市场最低点，但离开市场后失效。
3. 深渊召唤让刷新货架时固定出现贡品骰子。
4. 贡品骰子吃到市场强化后，购买时自动奉献给当前最低点数的实体骰。
5. 魔蝠骰子在离开市场时吞吃市场骰子，把价值给到最低点数的实体骰。
6. 吞噬骰子读取所有吞吃事件，把吞吃转成金币，支持继续购买和刷新。

核心爽点是：市场不再只是买东西的地方，而是可被培养、可被吞吃、可被带进下一场的成长来源。

## 已确认规则

- 恶魔家族围绕吞吃和市场培养展开。
- 吞吃是结构化触发点，后续骰子可以读取“发生吞吃”。
- 吞吃会把被吞吃骰子的当前期望值转成目标骰当前全部骰面的永久加值。
- 小鬼在回合结束后提高市场骰子的最小点数。
- 吞噬骰子在吞吃发生时提供金币。
- 恶魔骰子的购买加成只在当前市场内有效，离开市场清空。
- 魔蝠骰子在离开市场时触发吞吃。
- 深渊召唤在刷新市场时固定制造贡品骰子。
- 贡品骰子是市场来源的特殊骰，所有点数为 1，吃市场强化效果，购买后自动奉献。
- 贡品骰子不能从普通随机市场池出现。
- 吞吃和后续吸附共用同一套期望转移逻辑，来源骰当前期望转移值加到目标实体骰当前所有骰面。
- 小鬼提高的市场基础最低点在本轮持续并进入运行存档。
- 每颗魔蝠离开市场时最多吞吃 1 颗可用市场骰；多颗魔蝠按骰袋槽位左到右各自处理。
- 市场购买后不自动补货，货架留空；只有进入市场和付费刷新会整批补货。
- 进入市场也算一次刷新；有深渊召唤时，进入市场首批货架和付费刷新都固定包含 1 个贡品。
- 多颗深渊召唤首版不叠加贡品数量，避免三个货架被贡品挤满。
- 恶魔单家族测试池只随机刷小鬼、吞噬骰子、恶魔骰子、魔蝠骰子和深渊召唤；贡品只能由深渊召唤强制生成。
- 本包不改变 V0.2 已确认的六槽、单次出手、无出千和无全局牌型计分。

## 首版确认值

### 吞吃值

- 被吞吃骰子先应用所有当前市场最低点强化，再计算吞吃值。
- 吞吃和后续吸附共用同一套期望转移逻辑：计算来源骰当前期望转移值，并把该值加到目标实体骰当前六个面。
- 首版整数面组实现使用 `max(1, floor(来源骰当前六面总和 / 6))` 作为期望转移值；后续如果吸附规则重定义取整方式，必须同步替换这一公共逻辑。
- 吞吃后目标骰当前六个面全部加上期望转移值。
- 首版只直接写入目标骰面组，不复用大树 `Growth` 语义；如界面需要解释来源，可另加只读累计显示字段。

### 市场最低点

- 设两个状态：
  - 市场基础最低点：由小鬼在回合结束提高，建议本轮持续并保存。
  - 本市场临时最低点：由恶魔骰子在购买后提高，离开市场后清空。
- 市场骰子的实际最低点 = 原始面组经过市场基础最低点和本市场临时最低点共同抬高后的结果。
- 对普通骰，低于最低点的面被抬到最低点，高于最低点的面保留。
- 对贡品骰，因为六面原始都是 1，市场强化会把六面全部抬到当前最低点。

### 购买和贡品顺序

- 购买普通市场骰时，先扣金币并加入骰袋，再按当前骰袋槽位从左到右扫描购买触发。
- 购买普通市场骰后，该货架变为空货架，不自动补货。
- 购买贡品骰时，先扣金币并把贡品作为被吞吃来源结算，不进入骰袋，不占六槽；购买后该货架同样变为空货架。
- 恶魔骰子由购买事件触发的最低点 +1 影响当前未购买货架和之后进入市场或付费刷新生成的货架，不生成新货架，不追溯影响刚买走的骰子或贡品。
- 贡品自动奉献目标为当前实体骰中最低当前最小面的骰子；平局取骰袋从左到右第一颗。

### 刷新和深渊召唤

- 市场进入时的整批货架生成、手动付费刷新都视为“刷新市场”。
- 购买后没有单格补货，不触发深渊召唤，不固定刷贡品。
- 空货架只能通过付费刷新重新补货；付费刷新会重新生成整批 3 个货架。
- 至少一颗深渊召唤在骰袋中时，每次整批刷新固定把一个货架位替换为贡品骰子。
- 多颗深渊召唤首版不叠加贡品数量，避免三个货架被贡品挤满。

### 离开市场和魔蝠

- 玩家点击离开市场后，先结算魔蝠离场吞吃，再进入下一小关。
- 每只魔蝠按当前骰袋槽位左到右处理，每颗魔蝠最多吞吃 1 颗市场骰，默认吞吃当前货架从左到右第一颗可吞吃市场骰。
- 被魔蝠吞吃的货架不会补货，因为市场即将离开。
- 空货架不能被吞吃；如果没有可吞吃货架，该魔蝠不触发。
- 魔蝠吞吃目标为当前实体骰中最低当前最小面的骰子；平局取骰袋从左到右第一颗。
