## 退回来源

用户退回 V1 的 A / B：两者相比批准预览仍有明显差距，至少存在显示模糊，不能继续推进。V1 的机器 `PASS` 只覆盖灯珠尺寸、字高、装入和缺字，没有测出二次缩放、非整数采样与辉光造成的边界发虚。

## V2 修正范围

本轮只修正清晰度，不同时调整色相、整体布局或运行时代码：

- 不再生成可承担锐度验收的缩略总览图。
- 完整帧直接以目标分辨率输出：`1280×720` 与 `1920×1080`，不做二次缩小。
- 每个代表区域另外输出纵向堆叠的 1:1 原始裁切，裁切像素不缩放、不插值。
- `1280×720` 的顶部主信息、右侧终端和底部主按键另外输出 `4× NEAREST` 像素检查图，只用整数倍最近邻放大。
- 字体核心改为逐灯珠程序绘制，不再把整张低分辨率字图缩放到任意目标尺寸。
- 每种命名样式的灯珠宽高、点距和位置都是整数物理像素；同一样式内不存在混合的 `2px / 3px` 核心复制。
- 无辉光硬核心先独立输出；受限辉光只使用单独的 `1px` 外层、Alpha `34`，不使用模糊滤镜，也不改变硬核心。

## 像素合同

| 样式 | 720p 核心 / 点距 | 1080p 核心 / 点距 | 逻辑点阵 |
|---|---:|---:|---:|
| 顶部主信息 | `3×3 / 4px` | `5×5 / 7px` | `10×10` |
| 顶部次信息 | `2×2 / 3px` | `3×3 / 4px` | `8×8` |
| 积分塔读数 | `2×2 / 3px` | `3×3 / 4px` | `7×7` |
| 终端标题 | `2×2 / 3px` | `3×3 / 4px` | `9×9` |
| 终端正文 | `2×2 / 3px` | `3×3 / 4px` | `9×9` |
| 主操作键 | `3×3 / 4px` | `4×4 / 6px` | `9×9` |
| 最小辅助字 | `2×2 / 3px` | `3×3 / 4px` | `8×8` |

这些规格直接表达物理灯珠，不再使用“字号缩放后大约等于多少像素”的间接合同。
