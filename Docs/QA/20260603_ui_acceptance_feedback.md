问题列表：
1. [P1] ResultDecision / `20260603_ui_result_1920x1080.png`、`20260603_ui_result_1280x720.png` / 结果判断第三张指标卡的底部说明被卡片底边截断。
   影响：`可重摇低点`、`选低点重摇` 等出千建议是结果阶段关键判断信息，当前在 1920 和 1280 下都会贴边或被截断。
   建议：把三张指标卡的 note 行上移或移到卡片外统一提示区；必要时减少三卡内部字号或提高卡片高度。

2. [P1] CheatEdit / `20260603_ui_cheat_three_selected_1920x1080.png`、`20260603_ui_cheat_1280x720.png` / `已选 3 / 3` 计数显示在底部行动条边框下方，视觉上被压住。
   影响：选择计数是出千阶段的必读信息，当前位置像被裁切，1280 下更明显。
   建议：把计数并入确认按钮左侧或行动条标题右侧，例如 `行动条  已选 3/3`，不要放在按钮底边外。

3. [P1] StageClear / `20260603_ui_stage_clear_1920x1080.png`、`20260603_ui_stage_clear_1280x720.png` / 第三张指标卡标题为“通关收入”，但主值仍显示 `89/89 达标` 这类进度文本，收入说明只在底部小字 note 中展示且容易截断。
   影响：通关时玩家最关心账本、利息和实际入账；当前层级会把“通关收入”和“达标进度”混在一起。
   建议：StageClear 状态下第三卡主值改为 `+N 金` 或 `账本 +X / 利息 +Y`，达标进度放到标题或副文本。

4. [P2] Market / `20260603_ui_market_1920x1080.png`、`20260603_ui_market_1280x720.png` / 首次进入市场时骰袋满，三个货架按钮全是禁用的“骰袋已满”，但卖出区没有默认选中骰子，也没有明确提示“先卖一颗腾位置”。
   影响：第一市场是构筑入口，当前会让新玩家先看到三个不可买按钮，行动路径不够顺。
   建议：进入市场时默认选中第一颗骰子，卖出区直接显示卖价；或者在货架区加短提示 `骰袋已满，先卖出一颗`。

5. [P2] ResultDecision / `20260603_ui_result_1920x1080.png` / 右侧“最近反馈”文本在结果阶段承担了流程解释，但卡片面积偏小，容易形成多行拥挤。
   影响：右侧信息虽不阻断操作，但会和中央结果判断竞争注意力。
   建议：最近反馈保留最后 2 条，结果阶段只显示高价值事件；普通“摇骰盅开始/回落/开盅”可从反馈中降权或合并。

6. [P2] Scoring / `20260603_ui_scoring_1920x1080.png` / 当前正在结算的骰子有缩放和浮动分，但强调仍偏轻，第一眼不如结果指标卡明显。
   影响：结算动画的“从左到右”关系能看懂，但缺少强确认感。
   建议：当前骰增加更明显的金色描边或短暂背景高亮；浮动分避免靠近表格上边线。
