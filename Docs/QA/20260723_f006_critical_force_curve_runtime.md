# F006 第 5 档受力反差运行验证

日期：2026-07-23  
范围：`F006-V7R3-FB-005`，只调整隐藏第 5 档 `Critical`  
Unity：2019.4.33f1  
结论：程序、相位、双分辨率与降低特效自动门禁通过；正常速最终爽感待用户实机回归

## 用户反馈

> “5档的体验不够……没有那种蹲下跳起的感觉……没有火山压制之后爆发的感觉，只有那种洪水的倾泻；单调5档就行了。”

本轮不提高五档阈值、不拉长终局，也不继续堆全屏亮度。目标是让第 5 档具备明确的反向受力：

`下蹲压缩 → 硬锁小爆心 → 向上顶穿 → 上冲过冲 → 反向回落 → 余震`

## 根因

1. `SettlementFinaleWorldOffset()` 旧实现只在击穿 / 余震附近增加随机横抖和很小的向上位移，爆点前没有反向下沉或纵向压缩，因此没有“蹲下”的身体准备。
2. `DrawSettlementCriticalFinale()` 旧 `strike` 使用以 `0.66` 为中心、半宽 `0.055` 的对称脉冲。主射线、白闪、扩张波、Critical 继电与释放残留在配置击穿点之前已经开始，真空虽然变暗，视觉能量却提前外泄。
3. 主射线主要覆盖右半屏，上下放电权重接近，释放阶段还继续绘制整机负载和托盘扫光；能量总量足够，但主方向更像开闸倾泻。
4. 旧击穿音低频尾部较长，缺少短促向上的音高运动，也强化了持续洪水感。

## 修改

- 新增 Critical 专属四条单向曲线：
  - `SettlementCriticalCrouchAmount()`：真空前向下沉并纵向压缩，保持到爆点前。
  - `SettlementCriticalStrikeAmount()`：爆点前严格为零，爆点后快速起爆。
  - `SettlementCriticalLaunchAmount()`：击穿后向上拉伸并越过原位。
  - `SettlementCriticalReboundAmount()`：顶点后反向回落。
- `DrawArcadeRun()` 只在 Critical 使用以画面中心为枢轴的位移 / 纵向形变；固定背景和终局覆盖层不随动。
- 新增积分塔上下压力夹和小热核硬锁；真空末段不再保留横向流动。
- Critical 主放电改为向上分段主干，下行只留短根部；破盖、射线和碎片限制在向上扇区。
- 横向冲击长度与能量均缩减，只承担爆点瞬时的次级反馈。
- 释放时不再以主 `strike` 持续绘制整机柜负载和托盘扫光。
- 真空前增加一次低音量机械锁扣；击穿音减少长低频尾部，增加更短的瞬态和向上音高运动。

- 五档终局时长：`1.70s / 1.55s / 1.65s / 1.85s / 2.20s`。
- Critical 配置相位：真空开始 `0.46`、完全近黑 `0.54`、击穿 `0.66`、余震 `0.82`。
- 结果揭示点、结算事件、真实计分、奖励、随机、数据 CSV、存档字段与存档版本。

## 剩余风险

- 静态关键帧和数值方向能证明动作结构成立，但不能替代玩家对 `1.0x` 连续动态、扬声器声压和“压住后顶开”爽感的主观判断。
- 当前工程仍需用户重点确认三点：硬锁是否真的停住、上冲是否像顶开而不是铺开、回落是否能完成“蹲下—跳起”的受力闭环。
- 金币、龟龟、成长和贿赂的防重复专项不属于本轮，既有未关闭验收项保持不变。
