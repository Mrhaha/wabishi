# F006 普通通关异常光晕审计

日期：2026-07-23  
目标版本：F006 V7-R3  
结论：已复现。不是最高暴击特效串档，而是普通通关终局自身与上一笔入账状态叠光，导致目标图标和积分塔出现来源不清的青白 / 暖白亮区。

## 验证范围

- Unity：2019.4.33f1
- 分辨率：`1920×1080`
- 播放速度：全程真实 `1.0x`
- 档位：普通通关 `Pass`，终局时长 `1.55s`
- 场景 A：六枚基础骰均为 4 点，总分 24 / 目标 20，提前一枚骰子达标。
- 场景 B：基础骰为 `3 / 3 / 3 / 3 / 3 / 5`，总分 20 / 目标 20，最后一枚骰子刚好达标。
- 对照方法：同一终局进度下，仅清除上一笔入账留下的 `mainGameTargetCrossStartedAt` 与 `settlementTowerAfterglowStartedAt`，不修改普通通关终局本身。

两轮自动捕获状态均为 `PASS`：

- `20260723_f006_pass_glow_audit_status.txt`
- `20260723_f006_pass_glow_last_hit_audit_status.txt`

## 根因拆解

| 层 | 当前行为 | 普通通关内可见窗口 | 审计结论 |
|---|---|---:|---|
| 上一枚骰子的积分塔余辉 | 最后一笔入账在吸收点启动 `0.46s` 塔余辉；进入 `TargetSettle` 时没有清除 | 约终局前 `0.317s`，即进度 `0–0.205` | 跨阶段残留；会在积分塔内留下横向扫描与边框余亮 |
| 达标十字脉冲 | 分数首次跨过目标时启动 `0.60s` 脉冲；进入 `TargetSettle` 时没有清除 | 若最后一枚才达标，约终局前 `0.457s`，即进度 `0–0.295` | 主要异常源之一；同一 Pass 档会因“何时达标”而得到不同终局画面 |
| 普通通关目标锁存 | `DrawSettlementTargetLatch()` 在目标图标上绘制两条青白接点及一个中心圆光 | 进度 `0.20–0.56`，约 `0.31–0.87s` | 主要异常源之一；中心圆光脱离机械接点语义，单独看也像不明光晕 |
| 普通通关积分塔亮丝 | Pass 分支在塔中央绘制全高暖白竖线 | 进度 `0.24–0.44`，约 `0.37–0.68s` | 与目标锁存同拍出现，形成第二视觉焦点，进一步削弱因果 |
| 通关后积分塔残光 | 终局完成后统一绘制 `0.52s` 塔内填充与边框余光 | StageClear 前 `0.52s` | 强度较低，不是主因，但会延长“已经结束却仍在发光”的感觉 |

## 已排除项

- Pass 不调用最高暴击专用的近黑真空、全屏闪白、爆心射线、碎片、CRT 全屏切片或画面位移。
- Pass 不触发 `DrawSettlementCriticalFinale()`。
- Pass 的终局后残光只覆盖积分塔，不会像远超过 / 最高暴击那样延伸到整机边框、骰盘和终端。
- 因此当前问题不是档位分类错误，也不是最高档残留层误串档。

## 建议修改边界

1. `TargetSettle` 激活时统一收口上一笔入账的塔余辉与达标脉冲，使同一档位不再因“第几枚骰子达标”而出现不同终局。
2. 普通通关只保留一个清晰焦点：用短促、硬边、暖色机械接点表达“单拍闭环”；移除目标图标中心的漫射圆光。
3. 将积分塔全高亮丝收束为目标线附近的短接触闪，避免目标图标和积分塔同时争夺焦点。
4. Pass 的 StageClear 残光取消或缩短为不超过 `0.18s` 的边框回声；不保留整块塔内填充。
5. 保持 `1.55s` 总时长、`0.56` 结果揭示点、Latch 音效、五档阈值、计分、奖励与存档完全不变。

## 回归要求

- 提前达标、最后一枚刚好达标、贿赂补分达标三种路径进入 Pass 后，终局首帧到结果揭示的视觉层必须一致。
- Pass 全程不出现脱离接点来源的青白圆光、十字残影或全高白色亮柱。
- `Exceed / FarExceed / Critical` 的档位身份与最高档真空—击穿节奏不被削弱。
- `1.0x / 1.5x / 2.0x` 均验证；加速只能压缩时间，不能重新引入跨阶段残光。
- 连续两次普通通关及开启“减少闪烁 / 减少镜头运动”时均无残留。
