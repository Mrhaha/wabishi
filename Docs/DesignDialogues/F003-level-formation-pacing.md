## 当前摘要

F003 聚焦 F002 商店独特骰验收后暴露出的关卡体验问题：当前关卡目标、市场随机性、金币经济和内容复杂度叠加后，玩家前期生存压力偏高，构筑路线难以成型，且过早同时面对大量骰子类型、随机面组和材质。用户确认本案采用隐藏的轻度倾向槽位做软引导，不采用显式路线选择，并要求检查全部章节的目标分难度，同时让骰子种类和材质随流程渐进开放。

## 原始用户意图

用户认为当前关卡体验需要优化，主要包含两个方面：一是关卡难度本身；二是成型难度，因为当前市场纯随机，导致成型难度偏大。同时，购买骰子价格偏高或金币产出偏少，使前期异常困难。

## 问题框定

- 玩家问题：玩家还没形成路线就承受较高失败风险；进入市场后又可能连续刷不到当前路线需要的骰子，导致失败像由随机分配而非决策造成。
- 生产问题：需要同时检查目标分曲线、市场权重、刷新成本、购买价格和金币产出，避免只调一个数值后把压力转移到另一个系统。
- 当前系统张力：F002 让商品骰更独特，但也提高了单颗有效骰的价格和选择复杂度；现有经济又关闭了固定工资和剩余资源奖励，使玩家早期试错空间不足。
- 目标体验：玩家应能在前几次市场中逐步看见路线方向，前期失败率要可控，市场信息量随章节展开；中后期难度应来自构筑检验、路线适配和更复杂商品判断，而不是单纯目标分指数上升。

## 已确认决策

- 本案编号为 `F003`。
- 本案主题为关卡与成型节奏优化。
- 成型保护方向采用软引导，不采用显式路线选择作为首选。
- 软引导采用每次市场保留 1 个隐藏轻度倾向槽位的方式，不在界面上明说锁定路线。
- 难度检查范围必须覆盖所有章节，不只检查第一章。
- 骰子种类和材质需要随游戏流程渐进开放，避免前期元素过多。
- F002 在本轮讨论中视为验收通过，后续设计可基于 F002 商店独特骰已进入可用状态展开。

## 倾向方向

- 第一层：重新校准目标分曲线，尤其是第 1 章基础生存、第 2 到第 4 章成型过渡、第 5 章以后构筑检验。
- 第二层：市场增加软引导权重，例如根据骰袋内已购买类型，提高同路线补强骰、同路线基础件和经济救急件的出现概率。
- 第三层：恢复小额稳定金币收入，例如打开通关固定金币或剩余资源奖励，保证玩家每次通关后至少有一次有意义的购买或刷新选择。
- 第四层：检查价格体系，尤其是 F002 后商品价格由类型、面组和材质叠加，早期有效件可能过贵。
- 第五层：设计内容开放节奏。早期市场减少同时出现的路线数量、材质数量和强面组模板；章节推进后逐步开放补强骰、高阶骰、复杂材质和极端面组。
- 倾向的市场结构是 3 个货架中 1 个隐藏倾向槽位、2 个常规随机槽位；倾向槽位只提高相关候选池概率，不在界面文案上告诉玩家“你被分配到某路线”。

### 隐藏倾向槽位

每次市场仍展示 3 个货架，但生成逻辑分为：

| 槽位 | 生成方式 | 玩家感知 |
|---|---|---|
| 倾向槽位 | 随机挑 1 个货架位置作为隐藏倾向槽，根据当前骰袋路线提高相关候选权重 | 看起来仍是普通货架，不显示路线锁定文案 |
| 常规槽位 A | 当前章节开放池随机 | 保留探索和转路线空间 |
| 常规槽位 B | 当前章节开放池随机，尽量避开同屏重复类型 | 保留商品差异 |

倾向槽位的路线识别建议：

1. 最近购买的非基础骰路线在后续 2 次市场内获得优先权。
2. 若没有最近购买路线，则按当前骰袋内非基础骰数量最多的路线判断。
3. 若没有明显路线或并列，则使用早期入口池，优先给 `Odd / Even / Piggy / Turtle / Double / Basic` 中的可读选择。
4. 倾向槽位不是绝对保底，建议约 `70%` 从相关路线池抽取、`30%` 从通用池抽取，避免市场被锁死。
5. 如果倾向槽位生成结果和其它货架类型重复，允许重抽一次，降低同屏同质化。

路线家族建议：

| 路线家族 | 入口骰 | 补强骰 |
|---|---|---|
| 奇数路线 | `Odd` | `LoneWitness / Stamp` |
| 偶数路线 | `Even` | `HalfStep / Track` |
| 龟龟路线 | `Turtle` | `Shellsmith / SlowTurtle / Nest` |
| 大树路线 | `Tree` | `Gardener / Irrigation` |
| 金币路线 | `Piggy` | `Treasury / Bribe / Investment` |
| 通用爆发 | `Double` | `Gambler` |

### 经济补贴

首版经济建议优先走“稳定小额收入 + 早期低刷新压力”，而不是单纯提高初始金币。

| 调整点 | 建议默认值 | 理由 |
|---|---:|---|
| `starting_gold` | 保持 `18` | 不先用开局大钱包掩盖经济问题 |
| `stage_clear_base_gold` | 从 `0` 调到 `2` 或 `3` | 每次通关都有最低现金流，缓解买不起 |
| `roll_left_gold_bonus` | 首版保持 `0` 或最多 `1` | 过强会让幸运早通关滚雪球 |
| `cheat_left_gold_bonus` | 首版保持 `0` 或最多 `1` | 可以奖励保留出千，但不应成为主要收入 |
| 第一章普通市场刷新 | 从 `2` 降到 `1`，或首刷免费 | 降低早期找路线的心理成本 |
| Boss 后章节市场刷新 | 比普通市场高 `1` | Boss 后市场仍可更强，但不应过贵 |

实现风险：当前代码实际结算使用 `global.csv` 的 `stage_clear_base_gold / roll_left_gold_bonus / cheat_left_gold_bonus`。`chapter_score_table.csv` 虽有 `base_reward_gold / roll_bonus_gold / cheat_bonus_gold` 字段，但当前首要调参入口应以代码实际读取为准，除非 F003 明确要把关卡表奖励字段接回实现。

### 目标分校准口径

建议先定义通过率目标，再重算 40 关目标，而不是继续按固定指数写目标。

| 阶段 | 普通关目标体验 | Boss 目标体验 | 校准参考 |
|---|---|---|---|
| 第 1 章 | 主要保证学习和进入市场，基础骰也不应大量失败 | 检查玩家是否完成第一次路线购买 | 基础骰、1 到 2 颗入口骰 |
| 第 2-3 章 | 成型入口期，允许玩家看见路线反馈 | 检查是否有初步协同 | 2 到 3 颗同路线或混合路线骰 |
| 第 4-6 章 | 构筑检验期，普通关不应过度卡死 | Boss 开始要求明确路线收益 | 初步成型构筑 |
| 第 7-10 章 | 后期高压，普通关也可要求优化 | Boss 检查强构筑上限 | 高成长或高协同构筑 |

通过率口径建议：

- 第 1 章普通关：基础或轻度购买后通过率应明显偏高。
- 第 1 章 Boss：可以作为第一次构筑检验，但不应要求完美路线。
- 第 2-3 章普通关：围绕“路线正在成型”校准。
- 第 4 章后：允许失败更多来自路线质量、市场取舍和出千选择。

## 已拒绝 / 已后置想法

- 显式路线选择已不作为首选，因为用户希望使用软引导。
- 只修第一章已后置，因为用户明确要求检查所有章节。
- 单独提高初始金币暂不作为主方案，因为它不能解决市场纯随机导致的成型断裂。

## 待回答问题

- 软引导的识别单位是路线家族、当前最多骰子类型、最近购买骰子，还是玩家当前缺口。
- 目标分曲线要以什么通过率为基准：普通关偏宽松、Boss 偏严格，还是所有关保持接近难度。
- 金币补贴优先来自通关固定金币、剩余出手奖励、剩余出千奖励、首章折扣、免费刷新，还是价格下调。
- 渐进开放的阶段边界如何划分：按章节、按 Boss 通过、按市场次数，还是按玩家已有路线组件。
- 第一章是否允许材质出现；如果允许，是否只开放低理解成本材质。
- 强随机面组和极端同点面组是否也需要按章节后置，避免前期同时理解类型、面组和材质。

## 设计理由

本案符合当前产品方向：游戏核心幻想是把随机转化为可控计分系统。如果关卡、市场、经济和信息量共同制造不可控压力，玩家无法形成“我在构筑引擎”的感受。隐藏倾向槽位能保留 F002 的随机商品魅力，同时让玩家的购买行为影响后续机会；渐进开放能让玩家先掌握路线身份，再处理材质和面组差异；全章节目标曲线重审能确保难度增长来自构筑深度，而不是固定指数表。

还需要定义内容开放表：每个章节段开放哪些路线、哪些补强骰、哪些材质和哪些面组模板。

### 美术

当前暂无新增美术需求。若后续需要表现软引导，可考虑市场标签、路线印章或货架角标，但首版可先用文本和权重变化实现。

### 界面体验

软引导不能让玩家觉得市场被锁死。当前用户倾向是不明说，因此首版不需要明确路线锁定文案。渐进开放需要避免第一章商品卡同时出现过多机制说明。

## 风险登记

| 风险 | 重要原因 | 可能缓解方式 | 状态 |
|---|---|---|---|
| 软引导过强 | 玩家一旦买第一颗骰就被市场推向单一路线 | 设置权重上限、保留通用货架和跨路线货架 | 待评估 |
| 经济补贴过强 | 金币路线和利息机制失去身份 | 小额固定收入，金币路线仍提供额外上限 | 待评估 |
| 只看目标分不看构筑 | 目标分合理但玩家买不到核心件仍失败 | 目标曲线和市场样本一起验收 | 已识别 |
| 后期指数增长失控 | 目标表每章约 `1.47` 倍，要求构筑持续指数成长 | 按章节分段增长，设置章节喘息点 | 待评估 |
| 前期信息量过载 | 玩家第一章同时理解大量骰子类型、随机面组和材质 | 骰子类型、材质和强面组渐进开放 | 已识别 |
| 渐进开放过窄 | 前期市场重复度过高，玩家觉得内容少 | 保留少量通用骰和跨路线骰，章节后逐步打开 | 待评估 |

## 候选验收

- 生成全 40 关目标曲线报告，标记普通关、Boss、章节开局、章节收束的压力点。
- 用当前基础骰、低成长构筑、中成长构筑、高成长构筑分别估算通过率。
- 连续模拟或采样市场刷新，检查玩家已有某路线 1 到 2 颗骰后，三次市场内看到同路线补强的概率。
- 连续模拟或采样市场刷新，检查隐藏倾向槽位不会让 3 个货架都明显同质化。
- 检查第一章每次市场展示的机制说明数量，避免同时出现过多复杂骰、材质和强面组。
- 检查每个章节段新开放内容数量，确保玩家有新鲜感但不被一次性淹没。
- 检查前两章通关后平均钱包、最低钱包和可购买次数。
- 第一章普通关不应在基础骰袋下大量失败；Boss 可以作为第一轮构筑检验。
