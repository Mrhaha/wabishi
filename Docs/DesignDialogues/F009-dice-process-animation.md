## 当前摘要

F009 聚焦主流程中骰子的过程动画表现。摇骰分支采用已接入并通过双分辨率运行验收的 V4R6“固定骰体·骰面纵向滚轴停靠”：六颗立方体从按下 `Space` 到结果锁定都不旋转、不位移、不缩放、不弹跳，只有面值带在骰面裁切窗内纵向滚动、减速、越位、回勾并从左到右锁定真实结果。结算分支采用 V6B“塔芯连击”：积分塔从第一颗到最后一颗持续保留压力；左到右主游标必须先清空当前槽局部连锁队列，被再次触发且产生即时分数的骰子从自己的骰面送出短促回声积分包，只改变未来状态的目标不入塔。V6B-R3 的常驻暗框、方形孔、对开夹片与完整命中圆环因被读成额外入口而废弃；用户于 2026-07-22 确认 V6B-R4 的末段薄光、玻璃直吸收和塔内上行接管，R4 正式冻结为 `4.80s / 60fps / 288` 帧暴击导演母版。V6B-R5 随后在同一时间轴派生隐藏五档：未达到 `<100%`、过关 `100–149%`、超过 `150–199%`、远超过 `200–299%`、暴击 `≥300%`；档位只改变积分塔闭合、灯格压力、回弹、余辉、机柜负载和声音，游戏内仍只显示“本关通过 / 本关未过”。R5 同步视觉、顺序原声和五个单档已完成源区自审，待用户横向确认；确认前不接入 Unity。所有方案只调整表现，不改变随机、点数、槽位、真实结算、金币、成长或存档。

## 原始用户意图

用户先提出后续需要更好的骰子动画表现，包括进入游戏和结算计算时的表现，并询问使用 Codex 应如何实现。随后用户进一步明确：想要的是骰子的摇晃、反转、过程表现等，不是简单形变。用户认可首版目标后，要求创建 F009 设计对话记录，首版目标是 2.5D / 3D 骰子入场、翻滚、开盅定格和结算点名；当前只沉淀范围和验收标准，不写代码。

## 问题框定

- 玩家问题：当前骰子如果只作为静态 UI 图片或简单抖动，玩家难以感到“真的把骰子掷出、等待结果、看到结果落定”，投骰前后的期待感和结算归因都偏弱。
- 生产问题：需要在不破坏现有 `F004` 投骰配置、`F006` 只读结算事件队列和 `F007` 积分塔的前提下，新增一层可验证的骰子过程表现。
- 当前系统张力：真实翻滚感很有吸引力，但如果让物理模拟决定骰面，会破坏当前构筑、随机、出千和结算边界；如果继续只用 OnGUI 变形，又达不到用户想要的过程动画。
- 目标体验：玩家进入小关时看到六颗骰子按槽位落桌；按 `Space` 起摇后看到骰盅和骰子有真实摇晃/反转暗示；开盅后骰子翻滚并定格到真实结果；结算时当前骰子被点名、轻抬或短翻，和左侧积分跳涨形成同一节拍。

## 讨论时间线

### 第 1 轮：提出后续骰子动画需求

- 用户方向：希望后续无论进入游戏还是结算计算，都有更好的骰子动画表现，并询问使用 Codex 如何实现。
- Codex 回应：建议沿用 Wabish 的生产节奏，先把动画表现拆成设计记录和生产包；入场 / 投骰表现衔接 `F004`，结算表现衔接 `F006 / F007`。
- 形成的洞察：骰子动画不应孤立实现，而应服务当前投骰状态机、固定槽位顺序和只读结算事件。
- 决策状态：倾向

### 第 2 轮：明确不是简单形变

- 用户方向：追问是否可以做出想要的动画表现，例如骰子的摇晃、反转和过程表现，而不是简单形变。
- Codex 回应：确认可以做到，但不应继续只靠当前 OnGUI 图片抖动；要达到真实翻滚感，需要新增 2.5D / 3D 骰子表现层或翻滚帧动画层。
- 形成的洞察：F009 的关键生产判断是“视觉真实翻滚”和“玩法真实结果”必须分离。动画可以表现得像实体骰，但最终点数必须对齐当前随机结果。
- 决策状态：已确认方向

### 第 3 轮：确认首版推进方式

- 用户方向：认可首版目标，并询问后续如何推进流程。
- Codex 回应：建议先确认 F009 首版范围，创建设计对话记录，再生成生产包，随后按小片实现入场、投骰、开盅、结算点名和路线小爆点。
- 形成的洞察：首版不应追求最终美术和完整物理系统，而应先验证骰子是否有“实体过程感”和是否能稳定对齐真实结果。
- 决策状态：已确认流程

### 第 4 轮：创建设计对话记录

- 用户方向：要求创建 F009 骰子过程动画表现的设计对话记录，首版目标是 2.5D / 3D 骰子入场、翻滚、开盅定格和结算点名，先不写代码，先沉淀范围和验收标准。
- Codex 回应：新增本设计对话记录，明确首版范围、非范围、候选实现路径、职能备注、风险和候选验收。
- 形成的洞察：F009 已具备生成生产包的基础，但生产包前仍可补一轮技术路线选择：程序 3D 方块、2.5D 翻滚帧动画或混合方案。
- 决策状态：可生成生产包

### 第 5 轮：首版实装观感回炉

- 用户方向：反馈当前动态表现太丑、特别混乱，要求先给出整体优化思路，重点考虑美感、游戏节奏和对玩家的刺激。
- Codex 回应：判断问题不是单一动画参数，而是缺少动画导演层；下一版不应继续把骰盅、骰子、点数和结算效果同时推到前景，而应按阶段只保留一个主焦点。
- 形成的洞察：F009 当前程序 2.5D 方块只能作为技术验证，不足以承担最终美感。首版回炉应优先重建节奏和视觉层级，再决定是否继续使用程序方块、2.5D 帧动画或正式骰子资源。
- 决策状态：待用户确认

### 第 6 轮：取消骰盅阶段的方向

- 用户方向：提到云顶之弈一类更直接的骰子表现，并明确希望取消骰盅阶段。
- Codex 回应：认可这会显著降低画面混乱度，并建议把 F009 主路线从“遮挡蓄势开盅”改成“无盅直投、短滚动、落点定格、结算点名”。
- 形成的洞察：骰盅虽然能制造遮挡和悬念，但在当前主流程里会与骰子本体、点数和结算反馈争夺视觉焦点。对 `骰子王` 这种高频投骰游戏，更适合让骰子本身承担动作主角，用短促滚动和落点停顿制造刺激。
- 决策状态：倾向

### 第 7 轮：固定槽位原地立体翻转

- 用户方向：进一步明确不需要抛投概念，希望骰子固定在槽位中；玩家出手时骰子就在原地旋转，旋转要有立体翻转感觉，不是平面旋转；结束后停下并显示骰面点数，然后开始定点结算。
- Codex 回应：确认该方向更适合当前游戏，因为槽位顺序本身承载出千、结算、金币流和构筑归因；动画不应让骰子离开槽位或制造位置混乱。
- 形成的洞察：F009 的首版主表现应从“真实投掷模拟”转为“槽位内实体骰翻面”。重点不是让骰子在桌面上跑，而是在固定位置表现出厚度、转轴、侧面、阴影和最终吸附。
- 决策状态：倾向

### 第 8 轮：序列帧旋转播放器

- 用户方向：明确当前需要的是序列帧动画，而不是在骰子上继续做摇晃动作；每回合开始各槽位显示各类骰子的样子，敲击 `Space` 后每颗骰子播放旋转序列帧，随后缓慢停下，从左到右逐步停止，再依次开始跳分。
- Codex 回应：将 F009 首版实现方向从“程序方块连续形变”改为“OnGUI 序列帧播放器”：运动由离散帧号驱动，骰子不离槽，不抛投，不做横纵摇晃；加力窗口结束后锁定真实结果；`Stopping` 阶段按槽位从左到右停住并逐颗显点；结算跳分继续复用 F006 / F007 的左到右提交态。
- 形成的洞察：对当前主流程来说，最关键的是“稳定槽位 + 类型可读 + 帧动画节奏 + 左到右停转 / 跳分”。序列帧比程序摇晃更容易做出可控节奏，也更方便后续替换为正式美术帧图。
- 决策状态：已执行首版

### 第 9 轮：战力验证与累计爆发强化

- 用户方向：希望重新设计从按 `Space` 开始摇晃、到停转暂停、再到分数结算的完整表现；当前流程太平，这一段应成为构筑战力验证和累计爆发的关键时刻。
- Codex 回应：保留现有玩法状态和六槽边界，建议用“蓄压—静默—兑现—累压—越线”重建导演节奏。加力阶段只响应玩家输入；停转结果不提前庆祝；全部锁定后制造短静默并等待玩家再次按 `Space`；结算按槽位把点数、局部连锁和入账组成一个微循环，真实贡献越大或局部连锁越多，当前槽位表现等级越高；累计分越接近目标，积分塔的声光张力越强；首次越线触发局部青白破门，但后续槽位仍继续结算。
- 形成的洞察：当前问题不是缺少更多同时播放的效果，而是起转、停转、锁定、逐槽入账和目标越线都缺少能量等级差。真正的高潮应来自“玩家主动兑现构筑后，真实结算事件逐步抬高强度”，而不是从摇骰阶段就无差别爆闪。
- 决策状态：倾向，待用户确认

### 第 10 轮：分数上涨五级反馈

- 用户方向：分数上涨不能只有单一跳分反馈，需要明确区分没达到、过关、超过、远超过和暴击五个等级。
- Codex 回应：把“当前槽位单次入账力度”和“整次结算目标评级”分为两层。每颗骰仍按真实贡献决定本次跳分重量；整次结算则按最终得分与目标分的比例形成五级目标评级，并在逐槽累计过程中只播放最终能够保住的破级反馈。建议首版阈值为 `<100% / 100%–149% / 150%–199% / 200%–299% / ≥300%`，阈值配置化，暂不附带额外金币或玩法倍率。
- 形成的洞察：五级反馈应强化“构筑到底强到什么程度”，不能只把字体和颜色逐级变亮。等级差异应主要来自动作范围、节拍层数、声音密度和收束停顿；同时必须处理后续负分骰可能让累计分下降的情况，避免先播放“暴击”再退回失败。
- 决策状态：五级反馈需求已确认；具体阈值与是否附带玩法奖励待确认

### 第 11 轮：确认五级阈值与纯表现边界

- 用户方向：同意首版采用建议阈值，并同意“暴击”只作为表现评级，不附带额外玩法收益。
- Codex 回应：将五档门槛锁定为未达 `<100%`、过关 `100%–149%`、超过 `150%–199%`、远超过 `200%–299%`、暴击 `≥300%`；暴击首版不增加倍率、金币、奖励或存档字段，文案继续使用“目标的 N%”而非 `×3`，避免制造额外乘区的误解。
- 形成的洞察：五级反馈的职责已经明确为“把真实结算结果翻译成强度等级”，而不是新增规则层。后续可以独立调动作、声场和停顿，不必牵动经济与关卡数值。
- 决策状态：已确认

### 第 12 轮：取消主动结算与首版加力窗口

- 用户方向：不再保留结果锁定后的主动结算；加力时间窗口可以取消或配置化，首版时间配置为 `0`。
- Codex 回应：将流程收敛为第一次按 `Space` 后自动完成“基础旋转—左到右停转—短暂读点—逐槽结算”。加力窗口作为可选表现配置保留，首版值为 `0`，因此不出现连打加力阶段；该值只控制额外输入窗口，基础旋转序列仍正常播放。六骰锁定后保留短暂、可配置的静默读点时间，结束后自动进入结算，不显示 `SPACE 结算` 提示。
- 形成的洞察：去掉第二次确认和首版连打窗口后，节奏重点从“玩家主动兑现”转为“一键触发后的连续爆发”。静默锁定仍有必要，但它是自动流程中的断句，不再是等待输入的状态。
- 决策状态：已确认

### 第 13 轮：退回显式分档与顿挫式结算

- 用户方向：表现强化 V1 不通过。当前样片不够连续、顿挫感很强；不希望玩家形成明确的数值观感，而应主要依靠整个游戏的反馈感受本次出手强弱，不能再通过顿挫的分数表现承担高潮。
- Codex 回应：停止沿用“逐颗大数字—逐档破线—档位停顿”的导演语法，提出 V2“连续能量潮 / 整机呼吸”。五档只保留为内部强度驱动，不展示比例和评级名；旋转尾段、停转波与结算能量流连续交叠，所有真实结算事件注入同一个缓慢衰减的整机能量包络，由全局灯路、余辉、波形、机体运动和声场层数连续表达战力。
- 形成的洞察：玩家要感受到的不是“现在加了多少、跨了哪条百分比线”，而是“这台由我构筑出的机器正在被这一手推到什么负载”。五级差异仍需存在，但应成为整机响应的不同状态，而不是五张依次弹出的成绩单。
- 决策状态：V1 已拒绝；连续、弱数值化、整机反馈三项约束已确认；V2 具体方案待确认

### 第 14 轮：生成 V2 三手同机位预览图

- 用户方向：要求按新方案生成预览图。
- Codex 回应：以 F021 午夜旧街机机台为严格结构参考，生成明显未达、刚好过关、暴击三列同机位样张。三列都保持固定六槽和相同骰面，只改变连续青色能量流、机框回路、积分塔填充、右侧波形密度与整机余辉；第二轮修订移除了残留的 `SPACE 结算`、左侧目标 / 当前数字和其它明确评级信息。
- 形成的洞察：三档在不依赖比例、评级名和逐颗大分数时已能形成可见强弱梯度；下一步仍需由用户判断“暴击整机负载”是否足够有力量，以及整体青色能量覆盖是否过强。
- 决策状态：V2 方向已进入代表性样张验证；样张待用户确认

### 第 15 轮：重新聚焦摇骰本体与当前美术

- 用户方向：明确表示当前摇骰动画展示仍不满意，要求给出一版符合现有游戏流程和美术风格的新方案。
- Codex 回应：核对当前代码、固定六槽运行结构、F021 午夜旧机厅视觉基准与 F012 资源复用原则后，提出 V4“六槽机械翻面”。该方向不把整机能量潮继续放大，而是让骰子本体承担点火、惯性翻面、左到右制动、落锁、显点与自动交接；机台只提供局部琥珀槽灯、接触阴影、继电器与短促回坐。
- 形成的洞察：当前最缺的不是更大范围的光效，而是骰子作为实体物件的体积、惯性和停转触感。摇骰阶段不应根据隐藏分数提前表达强弱；构筑强弱仍在真实结算阶段兑现。
- 决策状态：倾向，待用户确认

### 第 16 轮：取消骰子间视觉关联，改为逐骰直接投向本关得分

- 用户方向：不希望表现上各个骰子相互关联；每颗骰子获得的积分应直接作用于本关得分，否则共享灯路和槽间接力会显得杂乱。
- Codex 回应：退回 `OldArcadeOverloadPreviewV4` 的骰台下共享电流与横向接力语法，提出 V5“单向投分 / 本关得分承击”。每个当前槽的点数、临时加分和局部连锁先聚合成一个贡献包，只从当前骰飞向左侧本关得分；屏幕上同一时刻最多存在一个贡献包，轨迹短暂出现并立即消失，其它骰不被沿途点亮。

- 决策状态：骰子之间不做视觉关联已确认；V5 具体节拍与静态评审板待用户确认

## 已确认决策

- 本案编号为 `F009`。
- 首版目标是 2.5D / 3D 骰子过程动画表现，不是继续只做简单 UI 形变。
- 首版关注三个玩家可见时刻：槽位内翻转、停转显点、结算点名。
- 当前阶段只创建设计对话记录，不写代码，不改数据，不改主流程文档。
- F009 首版不改变骰面随机、概率、出千、结算、目标分、金币、存档或骰子效果。
- 骰子动画最终必须对齐已经确定的真实骰面结果；动画表现不能成为新的规则来源。
- 结算点名应读取 `F006 / F007` 既有结算展示思路：真实结算先冻结，动画只播放已提交状态。
- 分数上涨和最终战力验证必须区分“未达、过关、超过、远超过、暴击”五个反馈等级。
- 五级阈值固定为：未达 `<100%`、过关 `100%–149%`、超过 `150%–199%`、远超过 `200%–299%`、暴击 `≥300%`。
- “暴击”首版只作为表现评级，不增加倍率、金币、奖励或存档字段；2026-07-21 最新约束进一步取消以“目标的 N%”或其它比例文案承担强度反馈。
- 六骰锁定后不再等待第二次按 `Space`；经过短暂的自动读点停顿后直接进入逐槽结算，也不显示 `SPACE 结算` 提示。
- 加力窗口保留为可配置表现项，首版时长配置为 `0`，等同关闭连打加力阶段；基础旋转动画使用独立时长，不能随加力窗口一起归零。
- 表现强化 V1 已被用户明确退回，不得作为 Unity 接入参考。
- 后续方案必须保持连续，不再用逐颗大数字、逐档停顿或明确比例作为玩家判断本次强弱的主要方式。
- 玩家应主要通过骰台、积分塔、终端、机框、背景、声音与机体运动组成的整机反馈感受本次出手强弱。
- 结算表现不得用共享灯路、槽间电流、连续轨道或骰子互相点亮暗示六颗骰子彼此传导。
- 每个当前物理骰的点数和本槽局部连锁应先聚合为一个贡献包，再从当前骰单向进入左侧本关得分；本关得分是唯一全局计分反馈锚点。
- 结算连续性由本关得分的承击余辉与下一骰预备动作重叠实现，不由骰子之间的视觉连接实现。

## 倾向方向

- 2026-07-21 表现强化 V2 倾向采用“一次 `Space`—连续旋转—停转波与结算流交接—整机负载累积—全局峰值收束”，详细节拍见下文“表现强化 V2 草案”；加力阶段仅在配置值大于 `0` 时插入，首版仍为 `0`。

- 首版优先验证“固定槽位内的立体翻面”是否成立，再决定是否补最终美术资源。
- 入场阶段：进入小关时，六颗实体骰可以直接出现在固定槽位，或用很短的淡入 / 轻落位表达，不再强调抛入桌面。
- `Ready` 阶段：骰子按固定槽位轻微待机，保留类型身份和槽位顺序，允许后续槽位调整表达。
- `Rolling` 阶段：骰子不离开槽位，只在自身格内做原地立体翻转；运动面不显示可读点数。
- `Stopping` 阶段：骰子按轻微错峰停止，最终正面吸附到真实点数，顶部点数和状态标签必须与真实结果一致。
- `ResultDecision` 阶段：骰子稳定停在槽位上，点数、类型色条、材质和短状态标签清晰可读。
- `Scoring` 阶段：当前结算骰轻抬、短翻或盖章点名，和左侧基础分、倍率、当前积分跳涨保持同一节拍。
- 龟龟、金币、大树和倍率等路线小爆点可继续使用 F006 的事件队列扩展，不在 F009 首版里强行一次做完所有专属特效。
- F009 后续生产包应把“程序验证层”和“美术最终资源层”拆开，避免首版卡在资产制作。

## 回炉优化方向：固定槽位、原地立体翻转、显点、点名

当前回炉方向调整为取消骰盅和抛投概念，把一次投骰拆成四个明确节拍。骰子始终留在各自槽位内，动画只改变自身朝向、厚度透视、阴影和强调状态：

1. `起转`：按下 `Space` 后，六颗骰子在原槽位同时或轻微错峰启动立体翻转，给出“这一手开始了”的明确反馈。
2. `旋转`：骰子围绕横轴、纵轴或斜轴做槽位内翻面，显示顶面、侧面和阴影变化；运动面不显示可读点数，只保留空白面、类型色或背面。
3. `停转显点`：骰子轻微错峰减速，最终正面吸附到真实点数，并给一个短暂停顿让玩家读牌型。
4. `定点结算`：结算阶段一次只强调当前入账槽位，其它骰子降噪，当前骰轻抬、描边、短闪或盖章，并和左侧计分跳涨同步。

### 美感原则

- 主视觉应更接近“明亮账本桌游风的实体棋子”，少用程序立方体硬扭曲和点数贴面拉伸。
- 点数只在可读阶段出现；运动阶段可以显示空白面、类型色、剪影或背面，不显示会乱飘的 pip。
- 每个阶段只允许一个强运动焦点：起转期看整组槽位启动，旋转期看槽位内立体翻面，结果期看点数，结算期看当前点名骰。
- 六颗骰子不应离开槽位高速乱跑；最多用错峰、翻面节拍和短吸附来制造实体感。
- 类型身份应靠小角标、色条、图标和结算标签保底，不要压在点数上方抢读数。
- 骰盅不再作为首版默认阶段；如保留，只能作为背景道具、菜单装饰或极少数特殊事件表现。
- 原地旋转必须表现出厚度：至少要有顶面、侧面、受光变化和接触阴影，不接受单张骰面图片绕中心平面旋转。

### 节奏建议

- 普通投骰从第一次按 `Space` 到结果可读，目标控制在约 `1.5` 到 `2.0` 秒。
- 基础旋转时长与加力窗口分离。首版加力窗口配置为 `0`，不等待连打输入；基础旋转仍需完整播放，后续若把加力窗口调为大于 `0`，再额外承接旋转速度、桌面反馈和输入反馈。
- 取消开盅等待和落点等待，改为停转前约 `0.10` 到 `0.15` 秒的“最后半圈”停顿，让玩家产生“马上停住”的期待。
- 结果揭示约 `0.35` 到 `0.55` 秒；六骰按两波或从左到右小错峰停转显点。
- 结果稳定后保留约 `0.20` 到 `0.28` 秒自动静默读点时间，随后直接进入结算点名，不等待玩家再次输入。
- 结算点名默认每颗约 `0.15` 到 `0.22` 秒；爆发牌型、通关、金币大入账或成长入档可以额外加短促强调，但普通手不加烟花。

### 玩家刺激来源

- 刺激不来自“所有骰子都乱动”，而来自“原地起转、速度提升、最后半圈、真实点数亮出、命中反馈”的递进。
- 玩家第一次敲 `Space` 应得到明确点火反馈；首版不要求继续连打。若未来开启大于 `0` 的加力窗口，旋转速度、桌面震动、短音效和提示脉冲可以响应追加输入，但始终不影响随机结果。
- 稀有或高价值结果才放大：四同以上、顺子、全奇 / 全偶、通关、接近失败时贿赂救场、金币大入账、大树成长入档、龟龟壳链长尾。
- 类型专属表现放在“结果 / 结算”阶段更清晰：龟龟走壳链余波，金币走金币飞向钱包，大树走成长脉冲，奇偶 / 盖章走几何校准或盖章，赌徒 / 双倍走短促爆点。
- 表现必须帮助玩家归因到构筑和槽位，而不是只提供一段与规则无关的表演。

### 技术倾向修正

- 当前程序 2.5D 方块不再建议作为最终默认表现，只保留为规则对齐和回退验证层。
- 下一版首选“动画导演层 + 2.5D 实体骰资源 / 帧动画 / 简化 3D 渲染”的混合路线。
- 表现层应显式维护阶段：`StageEnter / SlotSpinStart / SlotSpinLoop / SlotSpinStop / ResultHold / SettlementCallout`。
- 所有时长、错峰、缓动、强调等级应配置化，继续保持只读真实结果，不调用或替代 `ScoreDice()`。
- 若暂时不补最终美术资源，也应先替换掉生硬的 pip 贴面和程序方块扭曲，至少用更规整的骰面底图、阴影、空白运动面和结果吸附来保证观感。

### 回炉验收口径

- 任意截图中，玩家能立刻判断当前阶段是起转、加力、停转显点、看结果还是结算某颗骰。
- 首版默认没有骰盅阶段；如果画面出现骰盅，只能是弱背景道具，不承担主运动焦点。
- `Shaking` / `Rolling` 期间没有完整点数和高强度类型标签同时抢焦点。
- 投骰期间骰子不能离开自身槽位，不能打乱玩家对槽位顺序和后续结算顺序的理解。
- 旋转必须有立体感：能看到顶面 / 侧面 / 阴影变化，不能只是平面图片旋转。
- 结果点数在稳定后 `0.3` 秒内可读；类型标记不遮挡点数。
- 普通投骰节奏不拖慢高频游玩，玩家不需要等待一段与结果无关的长表演。
- 特效强弱和真实收益相关：普通结果安静，高价值结果才有明显刺激。
- 结算点名必须让玩家看懂“是哪颗骰、为什么加分 / 入账 / 成长 / 触发路线”。

### 不建议继续投入的方向

- 不建议继续主要调参当前 OnGUI 程序方块的摇晃和翻转；它容易继续表现为粗糙形变。
- 不建议继续把骰盅作为首版主阶段；它会延长流程并重新制造遮挡焦点。
- 不建议做抛投、位移滚动或碰撞模拟；它会削弱槽位顺序和结算归因。
- 不建议在旋转阶段显示六颗带完整点数的骰子同时翻滚；这会直接制造视觉噪声。
- 不建议让每种骰子在投骰阶段都做大幅专属旋转；类型差异应主要在结算阶段兑现。
- 不建议把普通一手做成长镜头；投骰是高频动作，刺激必须短、准、可跳过或可加速。

## 表现强化 V1 草案：起转、静默与自动兑现（已退回）

### 总体节拍

| 阶段 | 建议时长 | 玩家焦点 | 表现职责 |
|---|---:|---|---|
| 点火 | `0.08–0.12s` | 第一次 `Space` 的即时响应 | 键帽下压、六槽短促通电、骰子同时起转；建立明确起点 |
| 基础旋转 | 建议 `0.45–0.65s`，独立配置 | 六骰的实体翻面 | 正常播放旋转序列并建立速度；该时长不受加力窗口为 `0` 的影响 |
| 可选加力 | 配置值；首版 `0s` | 仅在开启时承接追加输入 | 值为 `0` 时完整跳过；未来设为大于 `0` 时，每次有效输入只提高表现强度，不暗示结果好坏 |
| 切电停转 | `0.55–0.75s` | 从左到右揭晓 | 基础旋转及可选加力结束后先做一次短促收紧，再从左到右以清楚落锁声显点；高点数不提前获得爆发庆祝 |
| 静默锁定 | `0.20–0.28s` 后自动结算 | 六个真实结果 | 全部骰子落座、环境声抽空、积分塔保持待机；短暂读点后自动进入逐槽兑现，不显示第二次 `Space` 提示 |

| 越线与收束 | 越线 `0.16–0.24s`；最终收束 `0.22–0.32s` | 是否达标与最终战力 | 首次越线局部青白破门并继续结算；六槽结束后再进入通关收入或失败分支 |

首版加力窗口为 `0` 时，从第一次 `Space` 到自动开始结算的建议目标约 `1.3–1.8s`；这不是把旋转动画压成瞬时，而是去掉等待追加输入的空窗。结算总时长不固定，应由真实局部连锁数量决定；普通六骰目标约 `1.8–2.6s`，高连锁构筑可以延长到约 `3.2–4.0s`，但不能用无事件空停顿拖时。

### 逐槽结算语法

每个物理槽使用同一套三拍语法，不增加悬空结算轨、六节点灯或新面板：

1. **点名**：当前骰与底座抬升，左右相邻骰降噪，右侧终端显示当前效果短句。
2. **触发**：当前槽的局部连锁在原位依次发生。分数使用短促琥珀脉冲，金币飞向钱包，成长或饲料落到真实目标骰，复制或响应使用一次性短电弧；所有轨迹播放后立即消失。
3. **入账**：本槽真实贡献以临时 `+N 分` 显示，一次脉冲送入左侧积分塔，当前分数字快速跳到已提交的真实累计值，并只用字框或缩放轻回弹落定，不能滚过虚假的分数；随后当前骰回座并进入下一槽。

槽位表现分三级，等级只读取已提交的真实事件：

- **普通**：只有基础点数或低强度修正，单次机械落账。
- **强化**：存在一个清楚的类型效果、金币、成长或响应，增加第二拍与局部颜色反馈。
- **爆发**：本槽包含多段局部连锁、显著真实贡献或触发目标越线，允许更强的底座冲击、低频音和局部辉光。完整目标庆祝仍留给越线与结算收束，不让每颗骰都像最终高潮。

### 累计爆发曲线

- 摇骰阶段的强弱只由玩家加力输入决定，不能根据隐藏结果偷跑庆祝。
- 停转阶段负责可信揭晓，不根据某个自然高点抢先下结论；构筑效果可能让低点骰成为真正爆发源。
- 结算阶段让积分塔成为持续累计锚点：每次真实入账都抬高数字、灯段和声音音高，接近目标时只增加紧张度，不提前改成成功色。
- 首次越过目标时，当前骰、积分塔和右侧终端同时出现一次局部青白冲击，短暂停顿后继续结算剩余槽位；不冻结后续触发，也不提前进入通关收入。
- 若最终仍未达标，累计声光在目标下方失速并局部转为锈红；若已达标，六槽结束后才锁定最终分并承接既有通关三拍收入。

### 分数上涨五级反馈草案

五级评级按 `最终已提交得分 ÷ 目标分` 判定。以下首版阈值已确认，后续若要基于实测调整，需重新进入设计确认：

| 等级 | 建议比例 | 终端短文案 | 核心反馈 |
|---|---:|---|---|
| 未达 | `<100%` | `未达目标` | 结算过程中维持琥珀蓄势；最后一槽结束才转局部锈红，积分塔在目标线下失速并显示分差 |
| 过关 | `100%–149%` | `目标达成` | 目标圈与当前骰一次青白锁定，单次机械重击，短停约 `0.16–0.20s`，随后继续结算 |
| 超过 | `150%–199%` | `超过目标` | 第二级双拍脉冲，积分数字与目标圈同步回弹，终端短显当前目标比例 |
| 远超过 | `200%–299%` | `远超目标` | 当前骰、积分塔和骰台边框形成三点同步冲击，增加低频声与局部机台回坐，短停约 `0.24–0.30s` |
| 暴击 | `≥300%` | `暴击达成` | 快速扫过此前已跨等级后，用实体数字重落、短水平信号干扰、低频冲击和约 `0.32–0.40s` 收束建立最高峰 |

`暴击达成` 下方建议显示 `目标的 320%` 这类比例，不显示 `×3` 乘号，避免玩家误解为随机暴击倍率或额外计分规则。颜色仍遵守 F021 的同级等亮约束，档位强弱主要通过作用范围、节拍数量、声音层数与停顿长度区分，不靠无限提高亮度。

#### 逐槽上涨与破级规则

- 每个槽位入账仍先显示真实 `+N 分`，其动作重量由本槽贡献与局部连锁决定；五级评级只判断整次累计分相对目标的位置，两套反馈不能混为同一个字段。
- 过关、超过、远超过和暴击各自每次结算最多触发一次。后续骰继续增加分数时只保持当前档位气氛，不重复完整破级动画。
- 单颗骰一次跨越多档时，不顺序播放四段完整庆祝。被跳过的中间档每档只用约 `0.05–0.07s` 的快速阶梯脉冲，最高新档播放完整反馈；从未达到直接跳到暴击的总附加停顿建议封顶约 `0.55s`。
- 由于真实面组和单骰贡献允许负数，破级反馈使用“已保住档位”口径：`可庆祝比例 = min(当前动画累计分 ÷ 目标分, 最终已提交得分 ÷ 目标分)`。这样早期累计分暂时越线、但后续负分导致最终失败时，不会先播放错误的过关或暴击庆祝。
- “未达”只在全部槽位结算结束后确认；结算尚未完成时即使当前低于目标，也只表现为蓄势，不提前播放失败反馈。
- 五级反馈首版默认只改变表现，不增加金币、倍率、市场奖励或存档字段。若未来希望高等级提供收益，应另开数值与经济讨论，不能从动画等级直接暗接奖励。

### 声音与震动分层

- 点火：继电器吸合与低频电机启动。
- 可选加力：首版窗口为 `0`，不播放追加输入声；未来配置大于 `0` 时，使用短机械击键和转速上扬，达到表现上限后只保留轻回馈。
- 切电：高速声骤停，保留极短空气抽空，再用六次由左到右的落锁声显点。
- 逐槽结算：基础入账使用可累计升调的干脆节拍；局部连锁插入各家族短音色，但不覆盖主节拍。
- 目标越线：只使用一次低频冲击、青白电流和积分塔锁定声；避免整屏震动、赌场爆灯或持续轰鸣。

### 当前实现诊断

- 当前老街机主游戏分支中的骰子运动主要是家族壳约 `±6°` 的轻摆、约 `1%–4%` 的缩放与扫描线，未真正呈现 F009 既有旋转序列的体积翻面，因此起转与加力视觉差距较小。
- 当前停转已具备左到右错峰，但主要是均匀减速和显点，缺少“切电—落锁—静默”的明确断句。
- 当前结算多数事件复用近似时长、轻抬、浮字和标签；槽位基础分与路线效果容易被拆成强度相近的连续卡片，未形成“当前槽局部连锁清空后一次入账”的爆发包。
- 当前目标越线已有脉冲接口，但前段累计张力不足，因此越线本身缺少对比。V1 应先建立前后明暗与声场差，再提高越线峰值。

## 表现强化 V2 草案：连续能量潮 / 整机呼吸

### 核心换法

V2 不再把六颗骰子当成六张依次报数的卡片，而把整台机器看成一个会蓄压、导流和过载的整体。真实结算仍严格按槽位左到右推进，当前槽位局部连锁仍必须清空后再进入下一槽；但在表现层中，相邻槽位的尾光、声音和机框响应允许交叠，上一槽的能量不会在下一槽开始前归零。玩家看到的是一条持续增长或衰减的流，而不是六次“点名—报数—停顿”。

### 连续节拍

| 时间 | 画面与声音 | 连续性要求 |
|---:|---|---|
| `0.00–0.55s` | 第一次 `Space` 点火，六骰在固定槽内进入同一旋转流；电机底噪和骰台暗光连续上扬 | 加力窗口为 `0`，不再等待输入；不能出现第二个启动节点 |
| `0.45–1.15s` | 停转波从左向右穿过六槽；已停骰保留轻微余震和底部流光，未停骰继续减速 | 停转与旋转重叠，不做“全体停住—画面静止”的硬切 |
| `0.95–1.20s` | 左侧第一槽下方线路先进入预充，右侧末槽仍在完成停转 | 预充只承接动势，不显示分数、比例或结果等级，也不提前庆祝 |
| `1.15–2.30s` | 结算能量潮沿六槽连续推进；每个真实事件向同一能量池注入强度，当前骰只做短促抬升、边缘压亮或家族纹理掠过 | 相邻槽位相位差约 `0.18s`，尾光与声音至少重叠 `0.05–0.08s`；不插入独立停顿 |
| `2.30–2.65s` | 能量潮汇入积分塔与右侧终端，整机按最终隐藏强度形成一次连续峰值并自然回落 | 只有一个全局收束峰，不逐档补播庆祝；强手可把余辉延长到约 `3.0s`，但不能靠停顿拉长 |

普通一手目标约 `2.5–2.7s` 完成。高连锁只增加能量密度、层数和尾迹长度，不增加“卡住等待”的节点。

### 数值去中心化

- 结算过程中取消大号 `+N`、`目标的 N%`、`过关 / 超过 / 远超过 / 暴击`逐档文案和积分数字重落。
- 左侧精确分数若玩法上必须保留，只做低关注度、无停顿的连续滚动；整段收束后再稳定显示最终总分，供玩家复盘和验证，不作为高潮主角。
- 单骰归因改由槽位被能量潮点亮、家族短纹理、影响目标之间的线路传递来表达；详细数值留给结算后信息或日志，不在主动画中抢焦点。
- 目标线只保留一次“回路接通”的功能性变化，不弹出比例、不冻结镜头。未达时回路没有闭合，达标后回路持续导通，其余强度继续由整机负载表达。

### 五档作为内部导演参数

五级阈值仍沿用已确认的 `<100% / 100%–149% / 150%–199% / 200%–299% / ≥300%`，但只用于给整机反馈设定参数区间。区间内部连续插值，跨界时使用缓动过渡，不能在 `150% / 200% / 300%` 突然跳一档。

| 内部档位 | 玩家感受到的整机状态 | 禁止做法 |
|---|---|---|
| 未达 | 能量潮在积分塔目标环前逐渐变细，机框没有闭环，电机声向下卸载，末端留下短促锈红余温 | 不弹“未达 82%”，不突然断电 |
| 过关 | 目标回路自然接通，青白电流绕机框完成一圈，机体做一次很轻的呼吸回坐 | 不停下来报“过关” |
| 超过 | 回路接通后出现第二层稳定谐波，骰台余辉更长，终端波形更密，边框有连续追光 | 不单独播放第二次破线动画 |
| 远超过 | 能量流变厚并扩展到背景与侧框，整机出现更明显但连续的低频回坐，余辉跨越结算结束保留片刻 | 不用三段重击或逐级加停顿 |
| 暴击 | 整机在最终收束时进入一次短促过载峰：线路近乎全亮、屏幕轻压缩、低频和高频同时展开，随后平滑泄压 | 不显示 `暴击 320%`，不做多次闪白或长时间冻结 |

### 预览验证方式

V2 样片不再做五张带标签的终局卡，而做同机位三手连续对照：明显未达、刚好过关、暴击。画面中不出现比例、档位名和逐颗大分数，只在最后以小号事实数字保留真实总分。评审只回答一个问题：不看数字，是否仍能从整台机器的光、运动和声场判断三手强弱。该方向通过后才拆参数接入 Unity。

## 摇骰动画 V4 草案：六槽机械翻面

### 方案判断

V4 只重做“玩家触发投骰到结果锁定”这一段，不借机改写评分、市场、家族效果或结算队列。镜头、六槽位置和 `128×128` 物理槽不动；骰子不抛出、不跨槽、不碰撞。主角始终是六颗骰子，积分塔、终端和机框在摇骰阶段保持低关注度，避免再用整屏青色或橙色能量把骰子淹没。

当前实现与最新已确认流程存在输入差异：代码仍保留 `1.35s` 加力窗口和结果锁定后的第二次 `Space`，但 F009 第 12 轮已确认首版加力窗口为 `0`，并在短暂读点后自动进入结算。V4 按已确认目标流程设计：第一次 `Space` 后连续完成点火、基础翻面、制动、锁点和自动结算；旧输入路径只作为尚未同步的实现现状，不作为本版动态样片的时序基准。

### 五段节拍

| 阶段 | 建议时长 | 玩家看到什么 | 机台响应 |
|---|---:|---|---|
| 点火压槽 | `0.08–0.12s` | 六骰同时下沉约 `2 px`、前倾约 `6°`，类型芯短暂收入中心，接触阴影压窄 | 六个槽底琥珀灯依次快速吸合，不扫整屏 |
| 槽内翻面 | `0.55–0.75s` | 每颗骰沿斜轴做共享 2.5D 翻面，能看到顶面、侧面、倒角与家族壳；运动面不显示可读点数 | 仅槽底与接触边缘亮，终端和积分塔不抢焦点 |
| 制动波 | `0.58–0.82s` | 从左到右每槽错开约 `0.07–0.09s`，完成最后半圈、轻微过冲、下压回弹和清晰落锁 | 每槽只响一次锁扣和短琥珀边光，上一槽余震与下一槽制动轻微交叠 |
| 锁点保持 | `0.20–0.30s` | 真实有效点数接管中心，类型芯缩入底部固定插槽，骰子和阴影完全稳定 | 槽灯恢复低亮；不庆祝高点，不提前表达过关或暴击 |
| 自动交接 | `0.06–0.10s` | 不等待第二次输入，第一槽底座开始预抬并进入真实左到右结算 | 摇骰电机声平滑让位给结算主节拍，不出现硬切或新提示 |

### 骰子视觉语法

- 家族壳始终保留，猪猪、恶魔、龟龟、海盗和中立只改变壳体材质与边缘色，不改变基础翻面节奏。
- 具体类型芯在待机时放大；起转后退为低对比度压印剪影，不播放待机身份动作；落锁后再缩入数字下方的既有固定插槽。
- 双阵营继续左右拼接两段家族壳，不为九颗双阵营骰制作九套独立翻面。
- 运动阶段不显示点数、长文案、饲料、吞吃、吸附或贡献分；这些信息继续由结果、悬浮详情和结算短反馈承接。
- 所有骰子共享同一体积、相机、倒角、接触点和光照骨架；差异只通过家族壳、类型芯和少量停转音色表达，符合 F012 的资源复用与表现继承原则。

### 美术与颜色

- 材质继续使用午夜旧机厅的老化塑料、烟熏玻璃、局部掉漆、暖琥珀槽灯和少量低饱和青色状态光。
- 摇骰主色使用中性实体色与暖琥珀，青色继续保留给目标越线、成功回路或明确系统状态，不能在普通摇骰阶段铺满机框。
- 不使用赌场爆灯、全屏橙色过曝、持续青色轮廓、镜头摇晃或高速拖影。强度主要来自体积翻面、帧速、惯性、落锁和声音，而不是提高全屏亮度。
- 六颗骰子的相位允许轻微不同，但启动与停转方向一致，避免各自乱转形成噪声。

### 资源与实现倾向

- 首批只制作一套共享体积翻面骨架：建议 `16` 帧循环、`8` 帧制动、`4` 帧落锁过冲，并用统一相机与灯光批量派生五种家族壳。
- 类型芯与结果数字继续作为独立运行层，不随每一帧重新生产；双阵营通过左右壳遮罩组合。
- 先用基础骰、猪猪骰和一颗双阵营骰做六槽代表样片，验证共享骨架、家族壳组合和结果锁点，不直接为全部 `44` 枚骰子生产资源。
- 表现配置至少分离基础翻面时长、循环帧速、制动错峰、制动过冲、落锁回弹、槽灯强度和自动读点保持时长；加力字段保留兼容但首版值固定为 `0`，所有字段只影响表现。
- 当前老街机运行分支必须真正消费体积翻面序列或等价的 2.5D 表现，不能继续只对静态家族壳做约 `±6°` 的平面摆动。

### 与结算表现的边界

- 摇骰阶段不读取最终得分档位，不因隐藏结果高低提前改变光效、速度或落锁力度。
- V2 / V3 的整机能量包络如继续保留，只能在真实 `Scoring` 阶段作为低面积的结算传递层使用，不再承担摇骰主体。
- 结果锁定只证明“点数可信”，构筑强弱、家族触发、金币、成长、目标越线和最终战力仍由左到右真实结算兑现。
- 这样可以同时满足两件事：摇骰本身有实体满足感，结算仍有足够表现空间，不会从第一秒就把整机推到峰值。

### V4 候选验收

- 关闭声音并隐藏文字后，玩家仍能从顶面、侧面、倒角、接触阴影和落锁过冲判断骰子正在做立体翻面，而不是平面摆动。
- 六颗骰子在所有帧中都不得离开各自 `128×128` 物理槽，也不得改变投骰前顺序。
- 运动阶段不出现可读点数；每槽停稳后显示的点数必须与真实锁定结果一致，且在 `1280×720` 下仍清楚可读。
- 首版从第一次 `Space` 后不再等待或接受加力输入；基础翻面完整播放，且不存在“连打可能提高点数”的视觉暗示。
- 普通摇骰全程不使用大面积青色 / 橙色覆盖，不遮挡家族壳、类型芯、积分塔、终端或底部操作提示。
- 基础骰、猪猪骰和双阵营骰共用同一翻面骨架时，既能保持统一节奏，也能在停转前后识别各自家族与具体类型。
- 锁点保持结束后必须自动交接到第一槽真实结算，不出现第二次 `Space` 提示；当前代码的旧等待确认路径不能作为正式验收结果。

## 结算表现 V5 草案：单向投分 / 本关得分承击

### 核心关系

V5 把结算画面收敛为唯一因果关系：`当前骰 → 本关得分`。左到右仍是主游标和真实结算顺序，但不再把顺序画成六颗骰子之间的接力。上一版骰台下的共享电流、连续轨道、槽间波纹和沿途点亮全部删除；任何时刻只允许当前骰和左侧本关得分形成一组主焦点。

每个当前槽先在后台完成真实点数、类型效果、遭遇修正与本槽局部连锁的聚合，再生成一个短命的“贡献包”。贡献包从当前骰左上缘脱离，抬高到骰列上方的空域后直接飞入左侧积分塔。它可以飞过其它骰上方，但不得触发沿途骰子的高亮、震动、变色或尾迹，也不得留下常驻线路。玩家看到的是这颗骰把本槽成果直接交给本关得分，而不是六颗骰子互相传导。

### 画面纪律

- 同一时刻最多出现一个贡献包；贡献包落入积分塔后，轨迹在 `80ms` 内消失。
- 当前骰只做一次短压缩、释放和轻回坐；其它骰保持稳定，不统一降暗，也不跟随脉冲。
- 贡献包用琥珀硬芯、温白密度和极短金橙尾迹表达力量，不显示大号 `+N`、比例、评级名或连续报数。
- 精确当前分继续保留为事实信息，但只在贡献包命中时直接更新，不计数滚动、不放大重落、不承担高潮。
- 右侧终端只允许低强度同步波形或一次短回声，不成为第二个计分目标；所有强反馈集中在左侧本关得分。
- 贡献包飞行路径不得穿过骰体中心，不画箭头、节点、共用导轨或连接线；在 `1280×720` 设计坐标中，尾迹建议只保留约 `36–48px` 的局部长度，而不是画满骰子到积分塔的整条路径。

### 连续流水节拍

| 节点 | 建议时长 | 表现 |
|---|---:|---|
| 当前骰预备 | `50–70ms` | 当前骰下压约 `1–2px`，边缘琥珀硬芯收紧；不停止后续骰的制动 |
| 贡献包释放与飞行 | `140–180ms` | 单个贡献包沿上方空域直接飞向本关得分，尾迹随飞行消失 |
| 积分塔承击 | `100–140ms` | 贡献包被吸收，积分灯段连续填入，塔身短压缩后回弹；精确分数低关注度更新 |
| 下一骰预备交叠 | 提前 `60–80ms` | 上一击积分塔余辉尚未结束，下一骰已开始轻压预备，避免每骰重新起拍 |

第一颗骰停稳后约 `60ms` 内即可开始首次投分，不等待六骰全部静止；后排骰仍可继续完成机械制动。普通六骰从第一颗锁点到最后一击完成，目标约 `1.0–1.3s`。连续性来自积分塔的余辉始终不断、下一骰预备提前进入，而不是同时飞多个贡献包。

### 贡献强弱编码

- 低贡献：贡献包小、琥珀硬芯短，积分塔只做一格柔和吸收。
- 中贡献：硬芯更密、尾迹稍长，积分塔回弹更明显，但不增加停顿。
- 高贡献或本槽多段连锁：仍只生成一个贡献包；通过包体厚度、内部双层硬芯和积分塔承击幅度表达，不拆成多个弹丸。
- 零贡献：当前骰送出一枚暗铜空包，积分塔只响一次干涩吸收声，灯段不增长。
- 负贡献：仍保持 `当前骰 → 本关得分` 的方向，送出锈铜色扣分包；命中后只在积分塔局部产生向下缺口和数值减少，不允许从积分塔反向流回骰子。当前代码展示事件会把负值钳为 `0`，正式接入前必须补齐带符号的展示字段。
- 临时骰与本槽产生的附属得分默认折叠进来源物理骰的贡献包；只有确实作为独立结算实体、且玩家能明确识别来源时，才允许单独形成贡献包。不得为临时骰恢复新的托盘或槽间连接。

### 隐藏五档只作用于本关得分

五档继续沿用已确认的内部阈值，但玩家界面不显示档位名、比例或阈值。强弱升级只发生在积分塔和极小范围机柜响应中：

- 未达：每次贡献正常吸收，目标锁扣不闭合；全部结束后自然卸载，最终失败状态才使用局部锈红。
- 过关：首次跨过目标时，积分塔目标锁扣以少量青白光闭合一次；流程不停，后续骰继续直接投分。
- 超过：后续承击增加一层琥珀回弹和更长塔身余辉，不增加第二次破线动画。
- 远超过：积分塔硬芯逐渐转为温白，塔身与局部机框出现可感知回坐，右侧终端只做低强度回声。
- 暴击：决定性贡献包在接近积分塔时进入白热，所有直接冲击从本关得分区域爆发；骰台不形成共享网络，六颗骰子不同时闪烁。

### 暴击承击四拍

- 预压 `50–70ms`：积分塔内部供电短降，飞行中的贡献包硬芯转为温白；塔体表面不出现入口夹紧或门板开合，画面不断流。
- 主击 `80–110ms`：贡献包撞入当前分与目标灯段，积分塔向上重推并产生一次局部温白冲击环；整机仅 `4–6px` 快速回坐，冲击不横扫骰列。
- 余震 `180–240ms`：积分塔灯段保留双层残像，当前分做一次低幅脉冲，右侧终端出现一次弱同步撕裂；骰台保持安静可读。
- 恢复 `300–450ms`：积分塔回到真实最终填充，温白退回琥珀，流程直接承接通关或失败收束，不额外展示“暴击”成绩单。

### 机制归因边界

真实规则允许当前槽影响其它骰，但这不等于需要画连接线。被影响骰可以在效果真实发生时做一次局部、无轨迹的状态反应，例如表面短闪或类型芯轻点；所有属于当前槽的真实计分仍聚合进当前槽的贡献包并进入本关得分。玩家需要理解“谁产生了这次结算、这一包让本关得分发生了什么”，不需要看到一张骰子关系网。

### V5 预览门禁

1. 先做一张四帧“交互语法板”：当前骰预备、贡献包飞行、积分塔吸收、下一骰预备交叠。必须复用 F021 母版，画面中不得出现任何骰子间连接。
2. 再做一张“积分塔五档响应板”：未达、过关、超过、远超过、暴击只裁取左侧本关得分及必要上下文；档位文字只放在游戏画面外的评审区。
3. 两张静态板通过后，再制作带临时声音的连续动态样片，验证六次单向投分是否持续、清楚且不顿挫。
4. 动态通过后才允许调整展示事件、生成覆盖层或接入 Unity；当前方案确认不等于实现授权。

## 结算分支文字方案：单一压力包络（讨论中）

### 连续导演关系

- 停转到结算不设置新的启动点。最后几颗骰仍在完成锁点时，积分塔只做低幅待吸收准备；第一枚积分印记在最后一颗锁点附近被吸收，使停转声场直接交给结算声场。
- 六颗骰严格按左到右主游标结算，同屏仍最多一个积分印记，但积分塔的亮度、压力、回弹余势和声音底层从第一击开始不归零。
- 上一击处于回弹或余辉阶段时，下一骰已开始局部来源识别；下一枚积分印记形成时，上一击只退到背景而非完全结束。连续性由两个阶段交叠形成，不由多发弹丸或槽间连线形成。
- 来源骰、其它五骰和骰台均不承担累计反馈。其它骰不降暗、不跟随、不震动；积分塔是唯一持续变化的主体。

### 积分印记的职责

- 积分印记不是独立角色，也不是完整飞行弹丸，只需让玩家读到“从当前骰提取、被积分塔捕获”。玩家重点看起点与吸收结果，中间路径只提供方向，不按屏幕距离展示完整长途飞行。
- 因此不同槽位到积分塔的距离不会改变整体节拍：右侧骰不会因为更远而拥有更长、更抢眼的飞行表演。
- 印记保留模糊的积分 / 点数语义，但不可计数，不显示具体 `+N`，也不让玩家把大小直接换算成数值。

### 单骰重量与整手等级分层

- 单骰重量只改变当前一次吸收的凝实程度、压入感、局部回弹和短余韵，所有单骰仍使用同一节拍，不通过额外停顿表现强弱。
- 单骰局部峰值设有上限。即使某颗贡献很大，也只会被读成“这一击很重”，不会出现整机白热、完整目标锁定或暴击收束。
- 整手等级只改变积分塔持续累计后的工作状态：未达时无法闭合；过关时目标锁扣成立；超过时锁扣后出现稳定溢出；远超过时局部机框加入过载响应；暴击时才允许一次短促、决定性的整机主峰。
- 等级变化发生在同一条压力曲线上，不逐档停顿或补播。为避免后续负贡献造成错误庆祝，只允许表现最终提交结果能够保住的等级；失败反馈只在全部吸收结束后成立。

### 数字与声音位置

- 精确当前分仍保留为事实信息，在每次吸收时直接更新，但不滚动、不缩放、不闪烁，不承担高潮。
- 从第一击到最后一击保持一层连续、低存在感的机器底音；每次吸收只向底音中加入不同重量的瞬态。等级提升增加声场层次而不是插入新的庆祝曲。
- 最后一击结束后，积分塔直接落入最终工作状态，并交给既有通关或失败收束，不再弹出独立评级卡。

## 关联结算与回环效果文字方案（讨论中）

### 主游标不被跨骰触发夺走

- 左到右主游标表示“当前是谁发起这一组结算”，不是“当前画面中只有谁能响应”。主游标到达某槽后，该槽保持主触发态，直到它引发的局部连锁队列完全清空。
- 被当前规则点名的其它骰进入次级“回声触发态”：只做短促的类型芯、骰面内沿或状态确认，不抬升、不离槽、不获得与主游标相同的完整聚焦。
- 回声处理结束后，视觉焦点回到仍未释放的主来源槽；只有局部队列为空，主游标才正式移到下一槽。

### 是否入塔取决于事件是否当场产生分数

- 如果被触发骰当场产生新的真实分数、给已结算骰追加分数，或复制效果直接形成带符号贡献，则由该实际贡献骰发送一枚次级回声积分印记进入积分塔。主来源槽仍保留主触发标识，因此玩家能同时理解“是谁发动”和“是谁产生这份分”。
- 如果事件只给尚未结算骰增加饲料、成长、面值或其它未来状态，目标骰只播放状态确认，不在此刻向积分塔发送积分印记；等主游标以后到达它时，新状态才并入它的正式贡献。
- 如果事件只产生金币、吸附或其它非积分收益，则沿用对应真实目标反馈，不伪造积分塔承击。积分塔只响应实际得分变化。

### 二重唱式再次触发示例

1. 主游标到达第二槽，第二槽保持主触发态。
2. 第二槽的“再次触发左邻”成立，第一槽出现一次次级回声确认；不重新播放第一槽的完整主结算，也不把主游标移回第一槽。
3. 第一槽被复制的效果按真实规则执行。它影响到的目标按局部队列顺序逐个短响应；任何即时分数由实际贡献骰依次直接进入积分塔。
4. 复制事件明确不允许触发的后续响应保持关闭；表现层读取已经结清的事件来源和递归标记，不能自行推演或重复触发。
5. 局部队列清空后，第二槽完成自身剩余贡献并释放主触发态，主游标再进入第三槽。

### 多目标与连续回环的压缩

- 同一时点点名多个目标时，按真实队列的左到右顺序快速确认，不同时点亮整排骰子；同屏仍只允许一个积分印记进入积分塔。
- 第一层跨骰关系完整展示一次来源与目标；后续重复的同类回环缩短为更密的回声节拍。玩家感到连锁在加速累积，但不会被迫观看多次完整聚焦、停顿和回座。
- 主动画不显示 `连锁 ×N` 或逐次大分数；完整触发次数和事件详情留给日志或结算后复盘。强构筑通过积分塔吸收频率、压力连续上升和声场密度体现爆发。
- 真实规则层必须继续负责防递归和队列终止。表现层只消费已提交事件树，需要至少保留主来源、直接触发者、实际贡献者、目标骰、事件深度、是否复制、是否即时入账和所属局部事件组；不得根据动画再次执行规则。

### 对“骰子互不关联”原则的修订边界

- 普通基础分、单骰自身效果和常规逐骰入账仍不得绘制骰子之间的关系。
- 只有规则文本真实点名其它骰、复制其它骰效果或产生跨骰响应时，才允许出现短促的来源—目标回声语法。
- 回声语法不使用常驻连线、贯穿灯路、沿途点亮或整排传播；未被事件点名的骰子始终保持安静。
- 因此玩家看到的是“规则真的让这颗骰响应了”，而不是系统默认把六颗骰子连成网络。

## 设计理由

`骰子王` 的核心不是单纯赌博，而是玩家把随机结果转化为可控计分系统。更强的骰子过程动画应放大这个核心：玩家先安排槽位，按下 `Space` 触发投骰，看见骰子作为真实桌面物件运动，再看到结果稳定落入既定槽位，最后由结算点名把高分归因到构筑、槽位和路线触发。表现层越强，越需要保持玩法边界干净；否则动画会从“放大决策”滑向“替代决策”。

F009 因此不应把重心放在炫技物理模拟，而应先解决三件事：骰子像实体物件、结果可读且可信、结算归因更清楚。只要这三点成立，后续美术、音效和专属特效都能在同一表现层上迭代。

## 职能备注

### 策划

F009 不新增机制收益，不改变玩家策略。策划重点是确认动画服务哪些真实决策：槽位安排、出千选择、路线触发、倍率成立和目标达成。验收时应关注玩家能否更清楚地感知“我为什么得到这个结果”，而不是只评价动画是否热闹。

### 美术

首版可接受程序临时骰子，但方向应贴合明亮账本桌游风：干净轮廓、清楚点数、柔和阴影、轻微弹跳和桌面物件质感。正式资产后续再决定是否需要倒角骰子模型、类型图标贴花、材质层、桌面投影或极少数特殊事件装饰。不要用过度写实、赌场感或厚重暗色风格。

### 界面体验

动画不能遮挡目标分、当前分、剩余出手、出千和结算主提示。主焦点应按阶段转移：入场看槽位，投骰看固定槽位内立体翻转，停转看最终点数，结算看当前点名骰和左侧积分跳涨。首版应验证 `1280x720` 和 `1920x1080` 两个尺度下没有重要文字或骰面被遮住。

### 程序

建议把 F009 做成只读表现层。它可以读取 `rollPhase`、当前实体骰列表、最终骰面、槽位顺序、`scoringDice` 和 `SettlementDisplayEvent`，但不能调用或替代 `ScoreDice()`，不能重复随机骰面，不能重复写钱包，不能重复应用大树成长或临时小骰。若 3D 表现层加载失败或关闭，游戏应回退到当前 2D 结果展示，保证原型仍可玩。

### 跨职能

F009 的第一轮验收必须包含录屏或 Play Mode 实看。静态检查只能证明文件和结构没坏，不能证明翻滚节奏、遮挡、最终点数一致性和结算点名是否有效。

| 动画结果和真实结果不一致 | 玩家会失去对随机和结算的信任 | 最终帧强制吸附到真实骰面；动画只读已确定结果 | 已识别 |
| 物理系统侵入玩法 | 让物理决定点数会破坏构筑、出千和可复现调试 | 物理只作为视觉模拟，不参与规则 | 已识别 |
| 实现范围失控 | 3D 骰子、相机、UI 叠加、材质和音效容易一起膨胀 | 首版只做程序验证层和四个关键时刻 | 已识别 |
| OnGUI 与 3D 叠加冲突 | 当前主流程仍是 OnGUI，3D 表现可能遮挡或坐标不一致 | 生产包中先确认相机和坐标方案，必要时保留 2D 回退 | 已识别 |
| 动画拖慢高频操作 | 投骰是高频动作，过长动画会消耗耐心 | 普通投骰控制短节奏，爆发手才允许更长停顿；后续考虑加速 | 已识别 |
| 骰子类型身份丢失 | 3D 方块只显示点数会削弱类型可读性 | 使用类型色条、角标或结算标签保底 | 已识别 |
| 表现变成纯表演 | 动画好看但不强化决策，可能掩盖玩法密度问题 | 动画节点绑定槽位、出千、路线触发和结算归因 | 已识别 |
| 爆发等级与真实收益脱节 | 普通骰也持续大爆闪会让玩家无法判断真正战力来源 | 表现等级只读取已提交的贡献、局部连锁和目标越线事件 | 已识别 |
| 统一事件时长拖慢高连锁构筑 | 每个标签都完整停留会让强构筑变成冗长字幕片 | 按槽位组织局部爆发包；普通事件短，关键事件才延长 | 已识别 |
| 停转阶段提前庆祝高自然点 | 低自然点也可能通过构筑成为高贡献源，提前强调会造成错误归因 | 停转只可信揭晓，真实强弱在 `Scoring` 中兑现 | 已识别 |
| 后续负分使评级倒退 | 当前累计分可能先越线，再被较晚槽位的负贡献拉回；提前庆祝会给出错误结果 | 庆祝档位同时受当前动画累计分与最终提交档位约束；未达只在结算结束确认 | 已识别 |
| 暴击被误解为随机倍率 | `暴击 ×3` 容易让玩家以为存在额外乘区或随机规则 | 使用 `暴击达成 / 目标的 300%+`，不显示乘号，不暗接奖励 | 已识别 |
| 多档连跳拖慢结算 | 一颗爆发骰若逐档播放完整动画会把高潮变成长等待 | 中间档使用快速阶梯脉冲，只完整播放最高新档，总附加停顿封顶 | 已识别 |
| 自动结算吞掉读点 | 取消第二次确认后，如果停转立刻跳分，玩家来不及读取六骰结果 | 将静默锁定保留为独立可配置停顿；结束后再自动结算 | 已识别 |
| 去数值后失去结算归因 | 如果只剩全屏灯光，玩家会感到热闹却不知道哪颗骰子发动 | 能量潮仍严格沿槽位推进，用槽位压亮、家族纹理和线路传递表达来源 | V2 重点验证 |
| 五档重新变成硬切 | 即使不写文字，参数在阈值处骤变仍会产生顿挫 | 档位只定义参数区间，最终强度在区间内连续插值并统一缓动 | V2 重点验证 |
| 精确分数抢回主焦点 | 常驻大数字滚动会继续让玩家盯着报数 | 过程数字降级、无停顿滚动，最终稳定后才恢复事实可读性 | V2 重点验证 |
| 单向轨迹跨过其它骰产生误读 | 从右侧骰飞向左侧积分塔时会经过其它槽位上方，若沿途响应就会再次像骰子接力 | 轨迹走骰列上方空域、尾迹短且立即消失；沿途骰子完全不响应 | V5 重点验证 |
| 局部连锁无法聚合到物理骰 | 当前展示事件含临时骰与附属事件，未必保存明确来源；若逐事件发包会重新变杂 | 在展示层增加“物理槽贡献包”聚合与来源字段；临时骰默认折入来源包 | V5 实现前阻塞 |
| 负分被展示层吞掉 | 当前 `SlotScore` 的 `ValueDelta` 会把负数钳为 `0`，无法表现真实扣分 | V5 接入前保留带符号贡献，负分仍单向送入积分塔并局部回撤 | V5 实现前阻塞 |
| 跨骰连锁被单包聚合吞没 | 二重唱、触发器、贪吃等真实来源与目标会被压成一个当前槽光晕，玩家无法理解构筑为何发动 | 展示事件保留局部事件组、主来源、直接触发者、实际贡献者、目标与复制深度；主触发和回声触发分层 | 新增重点验证 |
| 回环演出无限拉长 | 强构筑可能连续复制或响应，若每次都重播完整聚焦与入账会再次产生顿挫 | 首层因果完整展示，重复同类事件逐步压缩；规则层继续负责递归终止，表现层不得重新执行规则 | 新增重点验证 |

## 候选验收

- 进入小关时，六颗实体骰按当前槽位落桌，玩家能看出它们不是静态弹出。
- `Ready` 阶段骰子槽位稳定，后续槽位调整或结算顺序不会被动画打乱。
- 第一次按 `Space` 后进入投骰动画，骰子在固定槽位内有可见立体翻转过程。
- 首版加力窗口配置为 `0`，第一次按 `Space` 后不再等待或要求连打；基础旋转序列仍完整播放。
- 后续若把加力窗口配置为大于 `0`，追加输入也只能增强表现，不能改变骰面概率、出手次数、出千次数、金币或分数。
- 停转后骰子必须定格到真实结果；顶部点数、短状态标签和结果预览一致。
- 结果锁定后仍保持投骰前槽位顺序，不按点数重排。
- 结算阶段当前骰子有明确点名动作，例如轻抬、短翻、发光边框或盖章，并和左侧计分跳涨同步。
- 出千重摇如果纳入首版，重摇骰只播放被选实体骰的翻滚，未选骰保持槽位和结果。
- 禁用或加载失败 F009 表现层时，当前主流程仍能用现有 2D 展示完成投骰、出千和结算。
- Play Mode 录屏中普通一手不显著拖慢节奏；玩家能说清楚骰子何时入场、何时停转显点、何时结算点名。
- 录屏或截图验收覆盖至少 `1280x720` 和 `1920x1080`，确认骰面、目标分、当前分和主提示不互相遮挡。
- 从第一次 `Space` 到六骰全部锁定，玩家能够清楚感知点火、蓄压、切电、逐颗落锁与静默五个断句，而不是一段匀速动画。
- 结果锁定后不提前显示本次预计总分，也不出现第二次 `Space` 提示；静默读点时长结束后必须自动进入逐槽结算。
- 普通槽、强化槽和爆发槽在不看文字时也能通过动作幅度、声音密度和局部光色区分；表现等级必须与已提交事件一致。
- 当前槽的局部连锁清空后才进入下一槽，玩家可以指出本槽“为什么加分、影响了谁、累计分到了多少”。
- 首次目标越线后后续槽位继续结算，越线表现不提前发放金币、不切市场，也不覆盖后续真实事件。
- 最终得分落入五个建议区间时，终端文案、积分塔动作、声音层数与收束节奏能明显区分，且不会只靠文字解释等级。
- 一颗骰从未达直接推到暴击时，中间等级以快速阶梯反馈出现，完整破级附加时长不超过约 `0.55s`。
- 构造“前槽越线、后槽负分、最终未达”的测试时，不得提前播放过关、远超或暴击庆祝；最终必须显示真实未达和分差。
- `暴击` 反馈不得改变最终得分、倍率、金币、奖励、市场或存档，也不得使用乘号暗示额外乘区。
- V2 从旋转尾段到最终收束必须是一条可感知的连续动势，不出现全体静止、逐槽停住等待或逐档冻结。
- 隐去比例、档位名和逐颗大号分数后，玩家仍能只凭整机反馈正确排序“明显未达 / 刚好过关 / 暴击”三手强弱。
- 五档参数跨界不得产生硬跳；同一档内部也应随真实最终强度连续变化。
- 精确总分只在收束后作为事实信息恢复可读，不承担峰值演出。
- V5 画面中任意时刻都不能出现骰子到骰子、槽位到槽位的可见连接；当前骰与本关得分必须是唯一因果焦点。
- 六个贡献包必须依次直接进入积分塔，同屏最多一个；上一击积分塔余辉与下一骰预备需有 `60–80ms` 交叠，流程不能出现六次独立停顿。
- 零分、负分、临时骰聚合和一槽多段连锁都必须保持同一单向语法，不得因边界情况恢复槽间线路或额外托盘。

## 2026-07-21 V4R2 通用变速反转方案

### 核心判断

V4R2 不再为不同骰子设计不同运动，也不把家族壳、类型芯或特性反馈带入摇骰过程。六个槽位都直接播放同一套中性骰快速反转序列；真实骰子的家族、类型与最终数字只负责投骰前识别和停稳后结果展示。摇骰过程只回答“骰子正在越来越快地反转、随后逐渐刹停”，不回答“这是什么骰、这颗骰有多强、最后会得多少分”。

### 动画资源结构

- `通用反转循环`：建议 `12` 帧无缝循环，包含正面、倾斜、侧面、背面和回到正面的完整一周；运动帧不出现可读点数、类型芯、家族图标或特性标记。
- `侧面盲换帧`：循环中必须有一张正面宽度只剩约 `8%–12%` 的最窄侧面帧；该帧不显示任何可读内容，并用短促接触暗影遮住外壳轮廓变化。
- `归正揭示段`：建议 `5` 帧非循环，从最窄侧面依次展开到约 `35% / 65% / 100% / 104% / 100%` 宽度。进入该段前已经在盲区切换为真实结果组合，因此展开时自然看到真实家族壳、点数和类型芯，而不是在正面突然换贴图。
- 首版六槽共用完全相同的两段资源，不按 `DieType`、家族、品质、成长、饲料、吸附、效果、最终点数或贡献分选择其它帧图。
- 为避免六颗骰子像同一面墙同步翻页，只按槽位编号给予固定相位偏移；相位偏移不读取骰子 ID 或属性，拖拽换位后动画相位仍属于槽位，不跟随骰子个体移动。

### 三档速度与整段时序

| 阶段 | 建议时段 | 循环速度 | 玩家感受 |
|---|---:|---:|---|
| 缓速起转 | `0.00–0.10s` | `10→18 帧/秒` | 骰子从静止被带动，先有重量再进入连续反转 |
| 中速加力 | `0.10–0.26s` | `18→30 帧/秒` | 反转明显加快，形成正在用力摇晃的上升感 |
| 快速强摇 | `0.26–0.72s` | `30 帧/秒` | 全段最高速，六槽只看见连续快速反转，不出现属性提示 |
| 中速回落 | `0.72–0.88s` | `30→18 帧/秒` | 从高强度自然撤力，不突然切到停转序列 |
| 缓速刹停 | `0.88–1.22s` | `18→5 帧/秒` | 帧间隔逐渐拉长，并按槽位从左到右依次进入归正锁点段 |
| 锁点读数 | `1.22–1.42s` | 停止 | 六骰显示真实结果，短暂稳定后自动交给第一槽结算 |

速度切换不能重置帧号。播放器维护一条连续相位，速度只改变相位推进量；缓速、快速不是三套不同画面，也不能在切档时跳回第一帧。缓动使用连续曲线，避免速度阈值产生肉眼可见的顿帧。

### 六槽错相与逐渐停止

- 起转时六颗骰子同时进入播放器，但使用固定槽位相位表，例如 `0 / 4 / 8 / 2 / 6 / 10` 帧，保证画面有层次又不会随机乱转。
- 进入缓速刹停后，第一槽先捕获下一次最窄侧面相位，之后每槽间隔约 `0.05–0.06s` 完成盲换并进入相同的 `5` 帧归正揭示段。
- 已经停稳的槽位立即显示真实结果；尚未停稳的槽位继续播放同一个通用循环。顺序固定从左到右，不按点数大小或骰子类型改变。
- 最后一槽锁定后保持约 `0.20s`，随后第一槽轻抬并进入现有左到右真实结算。玩家不需要再次按 `Space`。

### 身份与结果边界

- `Ready`：继续显示真实家族壳与放大的类型芯，玩家在投骰前完成识别和排序。
- `Shaking / Stopping`：切换到通用中性反转骰；隐藏类型芯、数字、饲料、吸附、成长和所有属性特效，不画扫描线、家族色条或类型角标。
- `锁点`：每槽在侧面盲区先切换为真实结果组合；归正展开到约 `65%` 时显示家族壳，约 `75%` 时让真实有效数字渐显，最后一帧再把类型芯扣入数字下方固定位置。
- `Scoring`：继续使用当前槽轻抬、局部反馈与左侧积分跳涨；V4R2 不接管任何骰子效果或分数演出。

### 从无信息旋转到真实信息的过渡

推荐使用“侧面盲换 + 归正揭示”，不使用两个完整正面之间的交叉淡化：

1. 当前槽继续播放中性反转循环并减速，直到进入最窄侧面帧。
2. 在约 `20–35ms` 的侧面盲区内，关闭中性骰绘制，把底层数据源换成该槽的真实家族壳、已锁定有效数字和具体类型芯；因为正面宽度接近零，玩家看不到贴图替换。
3. 用统一槽内裁切遮罩限制真实结果组合，只允许它从中心窄缝逐步展开；猪耳、恶魔角等外轮廓也随遮罩一起展开，不得在盲换帧外突然弹出。
4. 归正揭示约 `90–130ms`：先出现家族壳轮廓，再在正面宽度超过约 `75%` 时显示数字，最后 `25–40ms` 扣入缩小类型芯。
5. 最后一帧做约 `2 px` 轻压和回弹，接触阴影收紧，随后保持真实结果并等待自动结算。

点数和类型芯不烘进通用序列帧。它们仍由运行时绘制，并继承同一个槽内裁切、横向展开和透明度曲线；一至四位数字在正面宽度不足时完全隐藏，避免被横向压成乱码。全部骰子使用相同盲换时机、展开宽度和信息出现顺序，只有被揭示的静态内容不同，因此不构成类型专属动画。

不推荐全正面交叉淡化：中性骰和类型骰会在短时间内重影，数字像浮到贴图上，家族外轮廓也会明显“长出来”。也不推荐用整屏闪白遮挡切换；投骰是高频动作，重复闪光会疲劳，并把简单的信息交接误做成奖励演出。

- 动画播放器只读取当前阶段、槽位编号、统一速度曲线、统一循环帧和统一锁点帧；不得读取真实骰子效果来选择动画。

- 速度切档处不得出现帧号复位、画面倒跳或额外停顿；从第一帧到最终锁点必须是一条连续运动。

## 2026-07-21 V4R4 整体旋转重构候选

### 共同设计边界

- `Ready` 和最终落稳必须清楚读作圆角立方体，不能退回卡片、面板或按键帽。
- `Shaking / Stopping` 仍完全不读取 `DieType`、家族、类型芯、饲料、成长、吸附、最终点数或贡献大小；六槽共用同一套运动资源与速度曲线，只保留固定槽位相位差和左到右落稳顺序。
- 不再追求“连续物理旋转的每一帧都正确”。动画采用 2D 手工关键姿势、速度残影和接触遮挡组成的表演连续性，以静态轮廓好看、整排节奏清楚为优先。
- 慢 / 中 / 快分类继续存在，但用于控制英雄姿势停留、残影密度和节拍，而不是把任意难看立方体朝向放慢给玩家看。
- 真实结果早已由规则层锁定；表现层只决定何时显示。运动段保持中性，家族壳、真实点数和类型芯只能在当前槽落地时恢复。

### 方向 A：清晰两端 + 立方体动势核（推荐）

- 核心：画面只在起转和落稳展示清晰立方体；中高速阶段进入一个屏幕占地稳定的“立方体动势核”。动势核不是圆球，也不是模糊贴图，而是由圆角方形体积、交替出现的三条角线、斜向面光和短残影组成，让玩家读到“立方体正在高速翻滚”，却看不到某个笨重的背面姿态。
- 动画帧只保留 `3–4` 个经过美术挑选的英雄姿势，全部为轻微俯视的 `3/4` 立方体，禁止正侧面、正背面和单角直指镜头；姿势之间使用 `1–2` 张手绘压缩 / 拉伸残影帧跨越，不补齐真实旋转的所有中间角度。
- 屏幕包络保持稳定：主体中心位移不超过约 `2 px`，投影面积变化控制在约 `8%` 内，最窄可见面仍不低于主体宽度约 `35%`，任一帧都不能把尖角伸进相邻槽。
- 缓速段停留在英雄姿势；中速段以“英雄姿势—残影—英雄姿势”推进；快速段主要播放动势核与角线节拍。减速时残影逐步缩短、英雄姿势停留拉长，最后直接进入统一落地姿势，不经过连续背面慢转。
- 建议首轮节奏：`0–80ms` 轻压起势，`80–180ms` 英雄姿势进入动势核，`180–610ms` 快速强摇，`610–760ms` 残影收紧；之后六槽以约 `50–60ms` 间隔从左到右各用 `110–140ms` 完成落地、轻压与一次短回弹。第一槽落稳可与后排收紧重叠。
- 结果交接发生在每槽落地接触的 `25–40ms` 局部遮挡内：中性动势核压进槽底暗影，暗影最紧的一帧切换为已经完整组合好的真实家族壳、有效数字和底部类型芯，下一帧直接以统一漂亮的结果立方体回弹。无需背面换面、横向展开、交叉淡化或整屏闪白。
- 优点：六骰同屏最整洁；最能保留“快速反转”和速度分档；类型过渡因果简单；符合 F021 旧街机中克制、局部、实体化的动效语言。
- 风险 / 成本：需要专门绘制动势核和残影帧，不能直接用真实 3D 旋转导出；必须用动态样片验证快段是否仍能被读成骰子而不是旋转灯或圆球。资源成本中等，代码仍只是统一序列播放器。

### 方向 B：短弹起 + 一次强翻 + 重落显点

- 核心：把刺激从“持续旋转”转移到“起跳—强翻—落地”。骰子只在槽内上浮约 `8–12 px`，用一次半到两次半高速对角翻转完成摇骰，随后重落、压槽、短回弹；停止感主要由落地冲击完成，不拖长慢速旋转。
- 起跳前的压缩和落地后的两拍衰减承担缓速感，中间翻转承担快速感。真实结果仍在接触暗帧内整体切换，落稳时直接出现完整结果立方体。
- 优点：实体感、重量和音画打击最强，起落结构天然掩盖难看的减速姿态；单颗动作因果非常容易理解。
- 风险 / 成本：竖向位移会改变当前“固定槽内原地旋转”的既有观感；六颗同时弹起容易形成整排跳动，需严格错相和裁切；渐停感弱于方向 A，更像一次机械掷骰。资源成本中等。

### 方向 C：烟熏视窗半遮挡旋转

- 核心：保留较真实的立方体翻转，但让骰子在槽内烟熏玻璃 / 上下窄遮罩后运动，只露出中央约 `55%–65%` 的体积带；减速到漂亮落地姿势后遮罩退开并显示结果。
- 优点：最容易保留完整立方体和连续转动，同时用物理环境遮掉难看的尖角与背面；资源复用率最高。
- 风险 / 成本：玩家会感到系统在故意遮住骰子，六槽并排也容易接近老虎机滚轴；与 F021 明确规避赌场 / 滚轴暗示的边界冲突最大。实现成本低到中，但方向适配度最低。

### 当前推荐

- 推荐方向 A，并吸收方向 B 的“落地接触 + 短回弹”，形成一句话方案：`清晰起势 → 中性立方体动势核高速翻滚 → 残影收紧 → 左到右重落显点`。
- 这套方案把“立方体”保留在玩家真正需要识别的开始与结果，把“力量”交给动势核，把“逐渐停止”交给残影密度和落地节拍，主动删除最丑的连续慢速转姿。
- 不建议继续迭代方向 C，也不建议再从 V4R3 调整摄像机角度或多画几张中间面；这些做法不能消除六槽小尺寸下的根本轮廓问题。
- 当前为设计倾向，尚未获得用户确认；不更新 `GAME_FLOW.md`、`PROJECT_CONTEXT.md`、`ART_ASSETS.md`，不进入运行资源或 Unity 代码。

## 2026-07-21 V4R5 固定骰体·骰面跳点锁定方案

### 核心表现

- 六颗骰子始终保持 `Ready` 时的立方体姿态、屏幕尺寸、坐标、家族壳、边框、投影和接触阴影；从开始到停止都不旋转、不位移、不缩放、不压扁、不弹跳。
- 唯一持续变化的主体是正面中央面值层。按下 `Space` 后，中央面值快速切换多种通用点数；播放频率由快到慢，最后按槽位从左到右停止并显示真实有效点数。
- 骰子类型身份不参与换值节奏。家族壳保持静止；具体类型芯在摇骰开始时进入结果态既有的底部固定插槽并保持不动，中央区域只服务面值。该状态切换不做移动演出，只在第一次面值出现时完成一次内容切换。
- 不使用数字上下滚动、卡片翻页、横向滑入、循环转轮或连续形变。每次变化都是同一位置上的直接换面，避免六槽画面接近老虎机滚轴。

### 通用动画面值

- 摇骰过程只使用固定通用池 `1 / 2 / 3 / 4 / 5 / 6`，不展示类型专属面组、成长后的高点、饲料下限、负值或真实贡献。

- 相位属于槽位而非骰子个体；玩家在 `Ready` 拖拽换位后，投骰动画仍按新槽位相位播放，不从骰子类型读取任何动画分支。
- 动画值只承担“正在摇骰”的视觉占位，不是连续抽样、候选结果或差一点命中的近失值。不得根据最终结果安排最后两个动画值，也不得用动画值参与随机、计分或存档。
