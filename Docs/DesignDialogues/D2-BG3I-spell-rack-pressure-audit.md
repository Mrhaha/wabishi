# BG3-I 法术架压力、严格存牌与替换规则审计

审计日期：2026-07-23  
对应设计对话：`D2` 第 `56` 轮  
审计对象：十二张候选法术 × 空架 / 两张占用 / 满架，购买店后继续两个市场；通用零金丢弃关闭 / 开启镜像  
状态：纸面定向窗口审计完成；`BG3-I` 仍为候选，未进入代码、配置、存档版本或正式玩法文档

## 一、结论

三格法术架可以作为首版默认容量继续推进，但不建议提供一个随时可按的通用零金丢弃按钮。更合适的逃生口是：

> 法术架已满时，只有在购买一张当前展示的法术时，才可选择销毁一张已有法术完成原子替换；旧牌不退款、不视为施放，也不触发任何玩法事件。

本轮结果：

- 建立 `12 × 3 = 36` 个基础窗口，每个窗口从购买店 `T0` 继续到 `T+1 / T+2`；再镜像运行通用零金丢弃关闭 / 开启，共 `72` 条规则轨迹。
- 按 `BG3-H` 的持久输入、两店内窗口、未来全成本可支付、不被持币支配四项证明，只有 `7 / 12` 张牌在本轮指定状态中属于“可明确保存”；其余 `5` 张仍是纯诱惑，不得仅因法术架有空位就乘 `0.70`。
- 严格合格牌在三档容量中共出现 `21` 次。关闭通用丢弃时购买保存 `14` 次；开启时为 `17` 次。两套规则只有 `3 / 36 = 8.3%` 的配对窗口改变首选，全部发生在满架且有一张旧计划已经失效的状态。
- 满架切片中，关闭通用丢弃只取得 `2 / 12` 张新牌；开启后为 `5 / 12`。新增的三次取得全部可以由“购买时替换”复现，不需要开放随时丢牌。
- 通用丢弃还额外开放 `2` 个“只清容量、当前展示法术仍不买”的动作机会。它们没有形成新的资源方向，只会弱化法术架机会成本，并可能让玩家通过主动清架影响空转保护与供给判断。
- 本轮实际购买的严格存牌在关闭 / 开启规则下分别为 `14 / 14`、`17 / 17` 于 `T+1` 或 `T+2` 兑现。这个 `100%` 是刻意构造的规则合同通过率，不是自然购买率或真人行为结论。
- 三格容量仍保留真实压力：两张占用时，`7` 个严格合格机会中有 `2` 个因“最后一格”机会成本被放弃；满架且三张旧计划仍有效时，又有 `2` 个新机会被合理放弃。购买时替换不会把容量代价抹掉。

因此形成候选 `BG3-I`：三格架、严格存牌四项证明、额外容量判断、无通用丢弃、满架购买时原子替换。

## 二、窗口与判定口径

### 1. 时间口径

每个基础窗口包含：

- `T0`：看到目标法术并决定购买、保存或跳过。
- `T+1`：完成一场结算后进入下一市场。
- `T+2`：再完成一场结算后进入第二个后续市场。

钱包沿用当前候选经济：

`下一市场钱包 = 离店钱包 + 3 + min(4, floor((离店钱包 + 3) / 6))`

同店刷新递增，进入下一市场后恢复章节基础刷新价；托盘仍必须在离店前清空，不携带实体。

### 2. 三档架压

同一张法术的未来资源、钱包、骰袋和章节节点保持不变，只改变法术架：

| 架压 | 初始状态 | 本轮要回答的问题 |
|---|---|---|
| 空架 | `0 / 3` | 四项证明通过时，法术本身是否值得锁定 |
| 两张占用 | `2 / 3` | 同一计划是否值得占用最后一格 |
| 满架 | `3 / 3` | 应先施放旧牌、放弃新牌，还是牺牲一张旧牌 |

满架再覆盖三种旧牌状态：

- `F-A 到期可施放`：一张旧牌本店已有完整执行链，可以先施放腾位。
- `F-B 计划失效`：一张旧牌购买时曾有计划，但因路线或资源改变已不再满足四项证明。
- `F-C 三张仍有效`：三张旧牌都有不超过两店的明确窗口，没有免费清理对象。

### 3. 两套主规则

- `OFF`：没有零金丢弃；架满时只能先合法施放旧牌，否则不能购买。
- `ON`：市场内可随时零金丢弃任意旧法术；不退款、不视为施放。

审计完成后，再把 `ON` 中所有实际“丢旧并买新”动作套入“仅购买时替换”的约束，检查战略结果是否变化。

## 三、十二张法术的严格存牌证明

这里判断的是法术在**当前不能形成完整立即执行链**时，能否进入 `0.70` 的明确保存档。判为纯诱惑不代表卡牌不能立即施放，也不禁止玩家自行下注。

| 编号 | 法术 | 持久输入 | 两店内窗口 | 未来全成本 | 不被持币支配 | 分类 |
|---|---|---|---|---|---|---|
| `S01` | 单格换货 | 没有可跨店保留的具体货架目标 | 下一店商品与应保留格未知 | `2` 金可支付 | 保留金币可用于完整刷新或采购 | 纯诱惑 |
| `S02` | 瑕疵折价 | 没有已命名的未来采购目标 | 下一店是否出现适合承受永久减面的商品未知 | `2 + 折后买价` 可估但目标未知 | 持币等待真实目标更完整 | 纯诱惑 |
| `S03` | 饲料袋 | 若已有好目标，现在就能施放；若没有，未来目标未知 | 等更好目标或更高质量不是确定窗口 | `3` 金可支付 | 有目标时被立即施放支配，无目标时被持币支配 | 纯诱惑 |
| `S04` | 拆解许可 | 人口海盗与两枚已指定的低价值待售资产持续存在 | `T+1` 同店卖出两枚，确定生成基础材料到托盘 | 买牌后 `14` 金；下一店材料无需额外采购 | 把待售资产转成核心面组成长，持币不能完成该转换 | 明确保存 |
| `S05` | 加急清算 | 待退出资产持续存在 | 未来额外现金要打开的具体商品或刷新阈值未知 | `4` 金可支付 | 正常出售与保留现金仍保有更多响应 | 纯诱惑 |
| `S06` | 外壳转运 | 真实壳、旧宿主与新宿主三者都已存在 | 旧宿主还需完成两场已知用途，`T+2` 退出前转运 | 买牌后不掉利息档，施放无追加支付 | 提前移动会损失两场用途，持币会失去锁定的转运机会 | 明确保存 |
| `S07` | 货架施肥 | 至少两个可处理市场商品的吞吃 / 化壳消费者持续存在 | 当前只余一格，`T+1` 固定重新生成三格 | 买牌后 `12` 金，下一店入口 `17` 金 | 下一店至少处理两件，明显高于本店单件使用 | 明确保存 |
| `S08` | 定向招募 | 已承诺家族与极高价结果的吞吃兜底持续存在 | `T+1` 章节切换，目标家族中阶池确定开放 | 买牌后下一店 `17` 金，可买最高 `14` 金结果 | 提前锁定家族搜索权，持币不能保证下一店仍见到法术 | 明确保存 |
| `S09` | 黑市分账 | 三次市场来源吞吃能力持续存在 | 当前货架已空，`T+1` 固定重新生成三件食物 | 买牌后下一店 `17` 金；吞吃不需追加采购 | 三次分账返 `6` 金并制造金币事件，超过 `5` 金牌价 | 明确保存 |
| `S10` | 金币饲料 | 高质量饲料、肉猪 / 贪吃猪与长期目标持续存在 | `T+2` 已知 Boss 前需要六次集中喂养 | `17→13` 买牌；两次收入后足够追加 `6` 金 | 现在没有完整追加预算，持币又不能锁定这张高容量牌 | 明确保存 |
| `S11` | 清仓祭典 | 两颗吞噬骰子与黑市折扣消费者持续存在 | 当前只余一格，`T+1` 固定重新生成三格 | `32→24` 买牌，仍在利息上限；无追加支付 | 三格批量吞吃改变成长、折扣和停止点，持币不能复制 | 明确保存 |
| `S12` | 换班凭证 | 旧资产持续存在，但未来托盘新骰不能跨店预先命名 | 下一店是否出现值得特殊回收的新宿主未知 | `6 + 新骰买价` 可估但目标未知 | 普通换代仍被基础卖旧、装新支配 | 纯诱惑 |

严格分类后的结构是：

- 可明确保存：拆解许可、外壳转运、货架施肥、定向招募、黑市分账、金币饲料、清仓祭典，共 `7` 张。
- 在本轮状态中只能算纯诱惑：单格换货、瑕疵折价、饲料袋、加急清算、换班凭证，共 `5` 张。

这五张牌在其它市场仍可进入 `1.00` 的立即可执行档。例如当前确有一格坏货、折后整链可支付、已有成熟喂养目标、清算能立刻打开采购、或换班能立刻把实体龟回收到托盘时，都不受本节保存判断影响。

- `存→1 / 存→2`：在 `T0` 购买保存，于 `T+1 / T+2` 施放。

## 五、镜像结果

| 指标 | 通用丢弃 `OFF` | 通用丢弃 `ON` | 仅购买时替换 |
|---|---:|---:|---:|
| 基础窗口 | `36` | `36` | 复用 `ON` 的实际换牌节点 |
| 严格合格机会 | `21` | `21` | `21` |
| 空架取得 | `7 / 12` | `7 / 12` | `7 / 12` |
| 两张占用取得 | `5 / 12` | `5 / 12` | `5 / 12` |
| 满架取得 | `2 / 12` | `5 / 12` | `5 / 12` |
| 合计取得严格存牌 | `14 / 21` | `17 / 21` | `17 / 21` |
| 两店内兑现 | `14 / 14` | `17 / 17` | `17 / 17` |
| 真正改变方向的丢弃 / 替换 | `0` | `3` | `3` |
| 只清容量、不买当前牌的额外机会 | `0` | `2` | `0` |

容量造成的合理放弃并没有被替换规则消除：

- 两张占用时，定向招募与金币饲料因为要占最后一格被放弃，共 `2` 次。
- 满架三张旧计划仍有效时，外壳转运与金币饲料被放弃，共 `2` 次。
- 满架旧计划已经失效时，拆解许可、定向招募、清仓祭典通过牺牲旧牌取得，共 `3` 次。

因此“满架购买时替换”解决的是计划失效后的软锁，不是让玩家无损保留所有未来选项。

## 六、为什么首版仍是三格

三格在本轮形成了清楚的非强制心理预算：

1. 一格承载近期确定兑现。
2. 一格承载组合或高阶爆发。
3. 一格承担机动、转向或赌注。

这不是界面上的硬分类，只是容量设计的目标。

对照判断：

- 若只有两格，本轮所有“两张占用”状态都会直接变成满架；当前 `5` 次不需牺牲旧牌的合理取得都要改成破坏性替换，保存惩罚过重。
- 若有四格，本轮所有满架状态都会先变成 `3 / 4`；三次真正的旧计划替换和两次“旧计划仍优于新牌”的容量取舍都消失，纯诱惑牌更容易长期堆积。
- 三格让“最后一格”已经能反转 `2 / 7` 个合格存牌决定，同时仍给一张短期计划、一张组合件和一张机动牌共存的空间。

三格因此通过纸面结构审计，但仍需真人连续市场验证操作负担、记忆负担和平均持牌时长。

## 七、候选丢弃与替换合同

### 1. 不提供通用丢弃

- 法术架上不放常驻“丢弃”按钮。
- 玩家不能为了清 UI、影响供给倍率或预先腾位而无目标销毁旧牌。
- 系统不会自动清理、过期或替玩家判断某张牌已经无用。

### 2. 满架购买时替换

玩家在法术架满时点击购买一张当前展示且可支付的法术：

1. 打开替换确认，列出三张已有法术与“取消”。
2. 玩家选择一张旧法术；预览明确显示旧牌销毁、无退款，以及购买后的钱包和利息档。
3. 提交前再次验证商店法术实例、旧牌实例、钱包与市场队列。
4. 原子提交：扣除新牌价格、销毁选定旧牌、把新牌加入同一空位、清空商店法术位。
5. 新牌只发布一次正常“购买法术”；旧牌不发布“施放法术”、出售、金币或任何可被骰子读取的事件。
6. 旧牌销毁只写审计原因 `RackReplacement`，首版不开放“丢弃法术”玩法端口。
7. 任一验证失败或玩家取消，钱包、旧牌、商店位全部保持提交前状态。

替换后商店法术位仍然留空，直到下一次付费刷新或下一市场；不会因牺牲旧牌免费补出第二张法术。

### 3. 存档边界

替换是一个无中间响应的单次事务。稳定存档只能位于：

- 完整提交前；或
- 已扣款、旧牌已销毁、新牌已入架、商店位已清空的完整提交后。

不能恢复为扣了钱但没有新牌、旧牌和新牌同时存在、或商店法术仍可重复购买的中间状态。

## 八、严格存牌与供给导演的接口

`BG3-H` 四项证明继续负责判断一张当前不能执行的牌是否属于明确保存；`BG3-I` 再增加一个独立的容量层：

1. 先判断持久输入、两店窗口、未来全成本与持币非支配。
2. 再判断当前架位是否允许取得，以及占最后一格是否仍值得。
3. 四项证明通过但因容量放弃的牌，仍是好牌，不应被系统继续追发。
4. 缺少四项证明的牌，即使架空也只算纯诱惑。

法术架满时的供给控制：

- 独立法术位仍按正常章节池生成和随付费刷新更换，玩家可以看到新牌并决定是否替换。
- 连续空转保护在满架时暂停，不因为玩家选择存满三张牌而追发“更适合”的第四张。
- 满架法术只有在“先施放某张旧牌”或“明确替换某张计划已失效的旧牌”形成完整链时，才可计入当前两方向底线。
- 硬导演不能替玩家销毁旧牌，也不能把“三张仍有效但新牌也不错”虚报成免费第四方向。
- 若满架且法术无法形成容量处理链，法术纠偏跳过；市场仍不足两方向时，最多改一颗非高阶保底骰，不扩大纠偏上限。

后续自然种子验收建议新增：

- 被系统判为“可明确保存”且由玩家购买的法术，两店内实际施放率目标至少 `60%`；低于 `50%` 时停止调价，先继续收紧分类器。
- 满架替换必须记录被牺牲牌、原计划、失效原因、新牌完整链与未替换对照。
- 只清容量而不关联当前购买的动作不计入有效决策数。

## 九、候选 BG3-I

1. 三格法术架作为首版默认容量。
2. `1.00 / 0.70 / 0.35` 供给倍率不变；`0.70` 必须通过 `BG3-H` 四项证明。
3. 四项证明之后增加容量判断，架有空不等于值得占用最后一格。
4. 不提供市场内随时零金丢弃。
