# BG3-J 资源边界与弱跨族循环审计

审计日期：2026-07-23  
对应设计对话：`D2` 第 `57` 轮  
审计对象：贸易猪现金 / 转栏、托盘龟实体 / 化壳、贡品估值、批量目标、外壳转运，以及猪猪 / 恶魔、龟龟 / 海盗两条弱环  
状态：纸面规则合同完成；`BG3-J` 仍为候选，未进入代码、配置、存档版本或正式玩法文档

## 一、结论

五项待定边界可以收敛为一套互不重复结算的规则：

1. 贸易猪出售前选择“现金兑现”或“饲料转栏”。现金拿完整饲料卖价但销毁饲料；转栏只拿结构回收值，把同一份饲料账本移到另一枚骰。两项不能同时获得。
2. 贸易猪转栏目标扩展为骰袋或交易托盘中的其它合法骰，贡品也可以接收饲料。转栏只是状态迁移，不重新发布喂养。
3. 吞吃材料的转移值读取“饲料下限折算后的长期六面”，但不读取龟壳、临时点数或条件加分。这样成熟猪资产可以成为恶魔食物，同时仍会随吞吃一起永久消失。
4. 托盘龟可以装入为实体，或直接化壳到骰袋 / 另一托盘中的合法非龟实体。直接化壳不经过首次装入，不需要空骰袋位，只触发一次真实化壳。
5. 贡品按生成时面组使用标准估值，基础显示价最低 `2` 金并锁定；后续市场最低点成长改变其吞吃价值，不回算价格。
6. 黑帆魔蝠、魔蝠骰子和清仓祭典共用一个批量目标命令，但各自保留不同约束。黑帆 / 魔蝠的食物不能重复；清仓的来源每格只处理一次，但允许玩家把多件食物集中给同一宿主。
7. 外壳转运移动一层真实壳的实例、持续效果、内部状态和回收值。已经写入旧宿主的面组成长不移动、不扣除，也不会在新宿主重放。
8. 猪猪 / 恶魔通过“饲料账本 → 可吞吃面组 → 吞吃成长与返金”形成普通弱桥；龟龟 / 海盗通过“化壳压缩 → 金钱龟金币事件 / 实体位 → 海盗成长与市场动作”形成普通弱桥。两条都不要求指定双阵营骰。
9. 审计同时发现一条真递归：若多颗人口海盗完整叠加，且它们生成的基础骰出售又继续累计人口海盗，就能自我补回出售数量。候选修正为人口招募来源骰可以正常出售，但该出售不再推进人口海盗计数。

这轮没有改十二张法术价格，也没有推翻 `BG3-I` 的三格法术架、无常驻丢弃和满架购买替换。

## 二、统一资源口径

### 1. 长期面组、饲料与吞吃面组

设来源骰长期六面为 `F1...F6`，长期面组最小值为 `Fmin`，饲料账本累计最低点价值为 `V`：

`饲料下限 = Fmin + V`

`可吞吃面 i = max(Fi, 饲料下限)`

`吞吃转移值 = max(1, floor(sum(可吞吃面 i) / 6))`

这套口径只为吞吃、奉献和明确写“读取吞吃转移值”的转换动作服务：

- 读取长期面组已经存在的永久成长。
- 读取饲料账本带来的稳定下限。
- 不读取幸运龟等持续壳效果。
- 不读取本手随机结果、临时分、邻位条件、钱包条件或遭遇修正。
- 来源被吞吃后，面组、饲料和附着状态一起销毁，不留下第二份饲料。

普通龟龟化壳继续读取当前既有的长期面组期望转移值，不自动读取饲料下限。只有驮粮龟等明确规则把饲料转为化壳收益，避免猪猪 / 龟龟原有专属桥被通用规则取代。

### 2. 饲料账本

饲料不是一个只有总数的无来源数值。候选状态至少保留：

- 饲料次数 `C`：用于贸易猪现金卖价等按次数读取的效果。
- 饲料最低点价值 `V`：用于实际六面下限与可吞吃面组。
- 已结算质量：每次饲料进入账本时就记录当时提供的值；批发商不追溯修改旧条目。

饲料转栏移动完整账本，不重新计算质量，也不让肉猪、贪吃猪或其它“获得饲料”监听再次响应。

### 3. 结构回收值

`结构回收值 R` 包含：

- 骰子实例基础卖价。
- 真实龟壳的可回收卖价。
- 明确写入实例的其它永久回收修正。

它不包含贸易猪的饲料次数卖价，也不包含首领龟复制壳、基础壳或未兑现的未来收益。

组合宿主无论包含多少真实壳，仍只是一项出售：

- 一次主动出售。
- 一笔合并金币入账。
- 一次人口海盗计数资格。
- 其它财宝海盗只按这一笔金币事件响应一次。

### 4. 实例与来源标签

进入候选生产包时，以下对象都需要稳定实例标识：

- 骰子实例。
- 货架商品实例。
- 法术实例。
- 真实龟壳实例。
- 批量命令实例。

骰子还需保留来源标签，至少区分：

- 初始 / 正常资产。
- 真实采购。
- 人口海盗招募。
- 抢劫 / 系统生成。
- 特殊回收。
- 贡品。

来源标签只在明确规则读取时影响结果，不改变骰子的正常卖价、面组或家族。

## 三、贸易猪：现金兑现与饲料转栏

### 1. 提交前选择

贸易猪有饲料时，玩家点击主动出售，先进入一个原子选择：

- `现金兑现`：获得 `R + C` 金，贸易猪及全部饲料消失。
- `饲料转栏`：获得 `R` 金，选择另一枚合法骰并把完整饲料账本移给它。
- `取消`：不出售，不改变任何状态。

没有饲料时不显示伪二选一，直接按普通资产出售。

### 2. 转栏目标

合法目标可以位于：

- 骰袋。
- 交易托盘。

目标不能是正在出售的贸易猪，也不能是已经进入销毁队列或不允许承载长期状态的临时表现对象。贡品是合法目标，因为它是有稳定实例、可以接收饲料并随后被奉献的托盘材料。

托盘目标离店前仍必须被装入、出售、化壳、吞吃、奉献或其它合法方式处理；转栏不会让托盘跨市场保存。

### 3. 事件顺序

现金兑现：

1. 锁定贸易猪、结构回收值、饲料次数与钱包前态。
2. 移除贸易猪。
3. 发布一次主动出售。
4. 入账 `R + C`，发布一笔金币事件。
5. 清空该实例饲料账本。

饲料转栏：

1. 锁定贸易猪、目标、结构回收值和完整饲料账本。
2. 再次验证目标仍合法。
3. 从来源移除饲料账本并原样挂到目标。
4. 移除贸易猪。
5. 发布一次主动出售。
6. 入账 `R`，发布一笔金币事件。

转栏不发布“获得饲料”，也不逐条重放历史喂养。若目标验证失败，整项出售取消，不能出现猪已卖掉但饲料丢失的半状态。

### 4. 加急清算

加急清算不是出售，不打开贸易猪二选一。它先计算贸易猪当前的现金估值：

`X = R + C`

随后获得：

`X + min(X, 6)`

并发布一次资产清算与一笔合并金币事件。它不触发：

- 贸易猪转栏。
- 人口海盗。
- 其它主动出售监听。

这使“正常现金出售”仍保留出售响应，“加急清算”则用法术和响应损失换更高即时现金。

### 5. 三个对照状态

以下只用于说明资源去向，不是最终平衡结论。贸易猪基础卖价沿用当前 `2` 金。

| 状态 | 首个承诺 | 结果 |
|---|---|---|
| `C=4 / V=4`，钱包紧、没有吞吃入口 | 现金兑现 | 获得 `6` 金；饲料销毁，打开采购或刷新 |
| `C=4 / V=4`，托盘已有基础贡品，骰袋有吞噬骰子 | 转栏给贡品后奉献 | 先得 `2` 金；基础全 `1` 的贡品可吞吃面变为全 `5`，奉献给目标后吞噬骰子返 `1` 金；放弃了 `4` 金现金换长期成长 |
| `C=2 / V=6`，生命压力高、恶魔实体稳定性差 | 转栏给恶魔实体并保留 | 只得 `2` 金，但高质量饲料保留在恶魔身体上；不立刻吞吃，也不把两次高质量饲料按普通次数低价出售 |

这三个状态分别偏向现金、即时转换和保留状态。次数 `C` 与价值 `V` 的不同估值使贸易猪不再只有一个固定答案。

## 四、托盘龟：实体与直接化壳

### 1. 采购后的四类合法出口

托盘龟进入托盘时仍是一枚完整实体。它可以：

1. 装入骰袋为实体。
2. 直接化壳到合法宿主。
3. 主动出售。
4. 被拆解、吞吃或其它明确转换消耗。

“实体 / 化壳”是保留该龟价值的两种互斥方式，不取消出售和材料出口。

### 2. 装入为实体

- 需要空骰袋位，或先完成换班凭证等合法回收交换。
- 发布一次首次装入。
- 保留独立身体、龟龟身份、独立卖价、继续喂养和以后调整位置的能力。
- 离店时仍使用当前位置规则：只检查后一槽非龟实体；不满足时继续保持实体。

### 3. 直接化壳

合法宿主可以是：

- 骰袋中的非龟实体。
- 另一托盘格中的普通非龟实体。

首版不允许把真实壳挂到贡品等“只能作为材料、不能进入结算”的专用对象上；这类目标无法兑现持续壳效果，只会制造无意义合法按钮。

直接化壳：

1. 锁定来源龟、目标宿主和来源长期面组。
2. 将来源期望转移值一次性写入宿主长期六面。
3. 创建一层带独立实例 ID 的真实壳并挂到宿主末尾。
4. 移除来源龟。
5. 发布一次真实化壳。
6. 每颗首领龟各获得一次同类型复制壳。

它不发布首次装入、第二次采购或主动出售。来源托盘格清空；若目标在另一托盘格，目标仍留在原格。

### 4. 实体与化壳的真实差异

| 维度 | 装入为实体 | 托盘直接化壳 |
|---|---|---|
| 骰袋位 | 占用一格 | 不新增实体位 |
| 家族身份 | 保留龟龟身份 | 来源龟身份消失 |
| 独立投骰 | 有 | 无 |
| 继续喂养 | 可以 | 壳本身不能继续吃饲料 |
| 初始面组转移 | 暂不发生 | 立即写入宿主 |
| 持续效果 | 以后化壳才有 | 立即成为真实壳 |
| 退出灵活性 | 可单独出售或以后选择宿主 | 与宿主绑定，出售时作为组合体 |
| 首领龟 | 当前不触发 | 触发一次复制 |

至少保留一个“继续当龟”的合理窗口，是避免托盘龟退化为无槽装备的关键。

## 五、贡品：最低二金、价格锁定与零金退出

### 1. 生成与估值

- 贡品仍只由深渊召唤等明确效果生成，不进入普通随机市场池。
- 生成时先确定长期六面，再使用与普通商品一致的面组估值。
- 基础显示买价为 `max(2, 标准估值)`。
- 生成完成后价格锁定。

恶魔骰子、货架施肥和其它市场最低点效果可以继续提高贡品长期六面与吞吃价值，但不会回算已锁价格。

瑕疵折价等显式价格修改仍可把最终支付价降到 `1`；这不违反“生成价最低二金”，因为玩家额外支付了法术且贡品长期六面同时承受损耗。

### 2. 采购后的去向

贡品采购后进入托盘，不能装入骰袋。玩家必须选择：

- 奉献给一枚合法骰袋目标。
- 以 `0` 金主动出售。

奉献：

- 使用当前可吞吃面组计算转移值。
- 发布一次市场来源真实吞吃。
- 可以触发吞噬骰子、黑市小鬼、共食魔龟和黑市分账。
- 贡品及饲料账本随后销毁。

零金出售：

- 计一次主动出售，可以推进人口海盗。
- 不产生金币入账事件。
- 不触发吞吃、黑市小鬼或分账。

这使“把贡品用于成长”和“把已付费贡品改作出售节奏”成为有代价的改道，而不是隐藏弃置。

### 3. 成本下限

基础贡品在没有显式折价时至少支付 `2` 金：

- 一颗吞噬骰子只返 `1` 金，净现金成本仍至少为 `1`。
- 两颗以上吞噬骰子可以把贡品推到零成本或净收益，作为多组件高光。
- 贡品仍占一个货架位，吞吃后不补货；继续获得贡品必须支付递增刷新。

因此最低二金解决的是“一只吞噬骰子就让贡品成为免费固定手续”，而不是禁止高阶恶魔引擎短时返正。

## 六、统一批量目标命令

### 1. 为什么必须统一

多颗黑帆魔蝠、多颗魔蝠骰子和清仓祭典都可能在一次根动作中处理多件货架商品。逐颗弹窗会把高阶引擎变成重复确认；全自动目标又会拿走最重要的食物与成长落点选择。

候选统一为一次 `BatchMarketCommand`：

1. 准备并冻结候选对象。
2. 在同一面板完成全部分配。
3. 玩家统一提交或取消。
4. 提交后按确定顺序自动结算，期间不再请求输入。

### 2. 三类命令

| 触发 | 食物来源 | 吞吃目标 | 可跳过 | 重复规则 | 提交后顺序 |
|---|---|---|---|---|---|
| 黑帆魔蝠：付费刷新 | 当前非空旧货架 | 各黑帆自身固定为目标 | 每颗黑帆可跳过 | 同一货架商品不能被两颗黑帆选择 | 骰袋顺序逐颗吞吃，随后清除余货并生成新货 |
| 魔蝠骰子：离开市场 | 当前非空货架 | 各魔蝠自身固定为目标 | 每颗魔蝠可跳过 | 同一货架商品不能被两颗魔蝠选择 | 骰袋顺序逐颗吞吃，随后继续其它离场队列 |
| 清仓祭典：施放法术 | 所有非空货架，全部必须处理 | 玩家为每件选择合法骰袋目标 | 不可跳过单件 | 每件来源只处理一次；目标允许重复 | 货架从左到右逐件吞吃 |

清仓允许目标重复，才能形成“把三件食物集中养一枚核心”与“分散给多个宿主”两条不同路线。黑帆和魔蝠限制的是食物来源不能重复，不是禁止多个事件最终成长同一类构筑。

### 3. 付费刷新顺序

玩家点击付费刷新时：

1. 按点击时状态锁定本次原始价格、刷新海盗减费、既有黑市折扣与最终净价。
2. 打开黑帆批量分配。
3. 取消时不扣款、不吞吃、不增加刷新价。
4. 提交时先结清已锁净价。
5. 按骰袋顺序处理黑帆；每次吞吃完整清空局部响应队列。
6. 黑市小鬼在本批吞吃中产生的折扣只进入下一次刷新。
7. 清除未被吞吃的旧货，增加原始刷新价，生成新骰货架与法术位。

负刷新费返金也在提交时按已锁金额处理，不能被本次黑帆刚产生的折扣倒改。

### 4. 离店顺序

玩家点击离店时：

1. 先验证托盘已经清空。
2. 若存在魔蝠与非空货架，打开一次批量分配。
3. 取消时返回可交互市场。
4. 提交后锁定市场交互。
5. 按骰袋顺序处理魔蝠，再继续抢劫、清仓猪、龟龟位置化壳等既有离场队列。

离店提交后不再生成需要玩家处理的托盘物品，也不重新打开市场。

### 5. 建议与事务

- 面板可以显示默认建议与一键采用建议，但不能自动提交。
- 提交前若任一来源、目标、钱包或市场版本失效，整项命令取消并回到准备态。
- 提交后使用冻结的实例 ID、顺序和随机数状态，不接受出售、刷新、改目标、施法或离店输入。
- 稳定存档只允许完整前态或完整后态；不能恢复到“吃了一半货架”的中间状态。

## 七、外壳转运

### 1. 合法来源与目标

来源：

- 骰袋组合宿主上的一层真实龟壳。
- 通过换班凭证回到托盘的组合宿主上的一层真实龟壳。

目标：

- 与来源宿主不同的非龟实体。
- 可以位于骰袋或交易托盘。
- 必须能进入结算或作为正常长期资产存在；贡品等专用一次性材料不是合法宿主。

首领龟复制壳、基础壳和其它无真实来源的壳不能移动。

### 2. 移动内容

移动同一个 `ShellInstanceId`，包括：

- 龟壳类型与持续效果。
- 原真实龟的可回收卖价。
- 壳自身计数、层数和来源信息。
- 已使用 / 未使用等内部状态。
- 与该壳绑定的每手、每关或每市场标记。

状态跟随实例，移动不能把“本轮已经触发过”重置成未触发。

### 3. 不移动内容

- 初次真实化壳时已经写入旧宿主的期望转移成长。
- 双倍龟等已经结清的一次性面组变化。
- 该壳过去在旧宿主上产生的永久成长、基础壳或金币。
- 旧宿主的饲料、其它龟壳与个体状态。

旧宿主保留全部历史成长；新宿主不重放初次化壳。

### 4. 顺序与经济

- 壳从旧宿主列表移除，并追加到新宿主壳列表末尾。
- 旧宿主组合回收值减少该真实壳回收值，新宿主增加同额回收值。
- 总回收值守恒，不产生金币事件。
- 移动只发布一次法术移壳，不算真实化壳、采购、首次装入或出售。
- 不触发首领龟，也不让真实化壳计数器增加。

如果新宿主在托盘，后续装入、出售、拆解或特殊处理都会携带这层壳。玩家仍需在离店前清空托盘。

## 八、猪猪 / 恶魔弱环

### 1. 公共资源链

`猪猪制造饲料 → 贸易猪选择转栏 → 托盘材料获得可吞吃面组 → 奉献 / 拆解发布真实吞吃 → 目标成长 + 吞噬返金 / 黑市折扣 → 搜索下一份猪猪资产`

这条链不要求贡猪或其它双阵营骰：

- 早中期可由贸易猪、任意托盘材料、拆解许可和吞噬响应完成。
- 中后期可由贸易猪、深渊召唤生成的贡品和普通奉献完成。
- 贡猪仍是加速器：它把“资产退出 + 吞吃”压到自身规则里，但不再是两家唯一接口。

### 2. 三个首选分叉

| 局面 | 合理首选 | 放弃 |
|---|---|---|
| 钱包不足以购买关键商品，饲料次数高但质量普通 | 贸易猪现金兑现 | 放弃把成熟饲料写进恶魔目标 |
| 钱包尚可，贡品或可拆材料已在托盘，吞噬 / 黑市响应齐全 | 转栏到材料后吞吃 | 放弃饲料现金与材料本身，换指定永久成长、返金和下刷折扣 |
| 生命只剩 `1`，恶魔实体面组过低，高质量饲料次数不多 | 转栏给恶魔实体并保留 | 不立刻获得高额现金，也不消费成吞吃爆发，先提高下一战稳定性 |

现金按次数 `C` 估值，吞吃按饲料价值 `V` 估值，因此批发商提高质量后可能把首选从现金推向吞吃；大量低质量喂养则可能继续偏现金。这个反转比简单给“猪恶魔同时在场 +N”更有构筑意义。

### 3. 自然停止

- 贸易猪会被卖掉。
- 饲料账本只能位于一个实例上，吞吃后销毁。
- 托盘材料或贡品会被消耗。
- 拆解许可是一次性法术。
- 深渊贡品占货架且不自动补货。
- 吞噬返金固定，原始刷新价继续递增。

链条可以在高组件状态下延长，但不会凭一次饲料同时获得完整现金与完整吞吃成长。

## 九、龟龟 / 海盗弱环

### 1. 公共资源链

`龟龟实体 → 直接化壳避免占位 / 位置化壳释放实体位 → 金钱龟壳产生金币事件 → 财宝海盗把金币事件转成成长 → 空位承接人口招募、抢劫或新采购 → 保留组合体复利或出售组合体换行动资金`

这条链只使用单家族普通组件：

- 金钱龟提供持续金币事件。
- 财宝海盗读取金币事件。
- 人口海盗、抢劫海盗和普通采购读取实体位 / 托盘处理能力。
- 不要求驮粮龟、黑帆魔蝠或其它双阵营骰。

### 2. 化壳的两种容量价值

- 托盘直接化壳：让一件新龟商品进入既有宿主，不增加骰袋实体数。
- 骰袋位置化壳：来源龟从实体列表移除，真实释放一个骰袋位。

释放出的实体位可以在下一市场装入人口海盗招募的基础骰、抢劫获得的商品或正常采购；直接化壳则让满袋玩家仍能购买一层新引擎，而不用先卖宿主。

### 3. 保留、出售与转运

以基础卖价 `3` 的海盗宿主和回收值 `2` 的金钱龟真实壳为例：

| 方向 | 结果 | 放弃 |
|---|---|---|
| 保留组合体 | 每次宿主结算时金钱龟入账 `1` 金并发布一笔金币事件；财宝海盗可响应成长 | 当前拿不到组合回收 `5` 金，可能错过强购 |
| 出售组合体 | 一次主动出售获得 `5` 金，只发布一笔金币事件并腾出实体位 | 永久失去海盗身体、金钱龟持续收入和已接好的壳顺序 |
| 先用外壳转运再卖旧宿主 | 支付法术机会成本，把金钱龟壳留给新宿主，再只回收旧宿主本体 | 当期净现金较少，但保存长期金币端口 |

当钱包正好差 `5` 金才能购买关键商品时，出售可以合理胜过复利；离 Boss 还有多场且没有紧急购买时，持续金钱事件更优；旧宿主必须退出但壳未来价值很高时，转运才值得支付 `4` 金牌价。

### 4. 人口海盗来源递归修正

人口海盗招募出的基础骰：

- 正常进入第一个空托盘。
- 可以装入、出售或作为材料。
- 出售仍获得其真实卖价，并发布普通主动出售与金币事件。
- 但来源为 `CrewRecruit` 的骰子出售不推进任何人口海盗招募计数。

原因不是限制强度，而是阻断真递归。若两颗人口海盗各自完整响应每两次出售，两颗招募基础骰继续反哺计数，就能用“卖两颗招募骰 → 再生两颗招募骰”保持同一状态并无限获得出售响应。来源排除只切断自举；玩家出售正常资产、贡品、采购骰或组合宿主仍能完整触发人口海盗。

### 5. 自然停止

- 金钱龟收入每次结算触发，不在同一市场自我重复。
- 组合体出售会摧毁持续收入端口。
- 人口招募来源不能反向招募。
- 抢劫受货架、托盘和实体位限制。
- 外壳转运需要法术与明确新宿主。
- 采购、刷新和强购继续消耗金币。

因此这条环能增加市场动作，但不会把一枚龟壳变成同市场无限金币。

## 十一、首轮行为验收

规则合同通过后，仍需用自然固定种子和真人试玩检查：

1. 贸易猪有饲料且存在合法转栏目标的样本中，现金兑现与转栏都应在不同压力下成为首选；任一长期超过 `80%` 说明估值或目标窗口失衡。
2. 猪猪 / 恶魔固定状态至少覆盖一次现金、一次转栏保留、一次转栏后吞吃的首选反转。
3. 吞吃高质量饲料资产的收益需与同阶段 `6–12` 金长期骰比较；若贸易猪转贡品稳定支配所有购买，优先降低饲料进入吞吃面组的效率，而不是取消公共桥。
4. 托盘龟合法直接化壳样本中，至少 `20%` 仍因身份、独立得分、继续喂养或等待宿主而选择实体。
5. 含一颗吞噬骰子的可购买贡品样本中，至少 `25%` 因利息、托盘顺序、当前商品或强化时机而延后 / 放弃。
6. 多黑帆、多魔蝠和清仓都只出现一次批量面板；提交后不得再次弹目标选择。
7. 外壳转运提交瞬间，新旧宿主长期面组总变化必须为 `0`，真实壳回收值总和也必须守恒。
8. 龟龟 / 海盗样本至少出现保留组合体、出售组合体、保存壳后退出宿主三种合理方向；外壳转运不要求常见，但不能永远被出售支配。
9. 满袋样本中，至少有一次化壳造成的“避免占位 / 释放实体位”实际改变人口招募、抢劫或采购处理顺序。
10. 任何数量的人口海盗都不能让 `CrewRecruit` 来源骰回到相同或更富的招募状态。
11. 双阵营骰移除后两条弱环仍能运行；加入贡猪、黑帆魔蝠、驮粮龟等桥梁后应明显提速、改道或提高上限。
12. 所有事务固定种子恢复只能得到完整前态或完整后态，不出现重复出售、重复吞吃、复制饲料、复制龟壳或半清货架。
