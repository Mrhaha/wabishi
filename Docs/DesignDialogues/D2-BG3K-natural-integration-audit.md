# BG3-K 自然连续市场融合审计

审计日期：2026-07-23
对应设计对话：`D2` 第 `58` 轮
审计对象：带 `BG3-I + BG3-J` 的三市场连续序列、严格存牌、满架替换、贸易猪三方向、贡品、托盘龟、两条弱跨族环、批量目标与人口招募递归
状态：纸面自然连续审计完成；`BG3-K` 为生产包前候选，未进入代码、配置、存档版本或正式玩法文档

## 一、结论

`BG3-I` 与 `BG3-J` 没有被自然连续序列推翻，可以作为同一生产包的规则底稿继续推进。

- 主样本沿用第 `55` 轮的 `30` 组三市场序列，每组分别运行保守、成长、破局三种策略，共 `90` 条策略序列、`270` 个连续市场决策。没有为了命中某条跨族环而额外塞入指定骰子或指定法术。
- 正常资源状态的 `213 / 213` 个市场决策在有限纠偏后都保有至少两个后果不同的可辩护方向；玩家前店主动花空利息后的 `57 / 57` 个后续市场都至少保有一个可支付且有结果的动作。真正无事可做为 `0 / 270`。
- `239 / 270 = 88.5%` 个市场完全不需要硬纠偏。发生纠偏的 `31` 个市场中，`24` 个只改法术、`4` 个只改一颗非保底骰、`3` 个两者都用；没有扩大既有纠偏权限。
- 严格保存且拥有完整两店窗口的法术共购买 `19` 张，其中 `13` 张在窗口内施放，兑现率 `68.4%`，高于 `60%` 首测目标。
- 法术架满共出现 `23` 个决策，其中 `10` 个新法术形成完整取得链：`4` 次替换失效旧计划，`3` 次先施放旧牌再购买，`3` 次保留三张更重要的旧计划并跳过新牌。没有出现为了清容量而裸丢法术。
- 贸易猪在 `15` 个完整合法窗口中，现金兑现、转栏保留、转栏后吞吃各成为首选 `5` 次；按饲料次数与饲料价值分开估值确实能反转答案。
- `13` 个贡品处理窗口中，奉献 `7`、零金出售 `2`、延后或放弃 `4`。只有一颗吞噬骰子的 `8` 个窗口里有 `3 / 8 = 37.5%` 没有立刻奉献，达到至少 `25%` 的拒绝目标。
- `23` 个托盘龟且存在合法直接化壳目标的窗口中，装入实体 `7`、直接化壳 `14`、出售或作材料 `2`；实体率 `30.4%`，高于 `20%` 下限。
- 移除双阵营骰后，猪猪 / 恶魔公共弱环在 `8 / 90` 条策略序列中完成，龟龟 / 海盗公共弱环在 `15 / 90` 条策略序列中完成；两条都覆盖早、中、晚期，不再只是定向合同。
- `14` 次批量根动作共分配 `34` 个目标，只打开 `14` 次面板。平均每次 `2.43` 个分配，加一次提交为 `3.43` 个必要输入，最大 `4` 个；没有逐颗弹窗或提交后再次索取目标。
- 额外的 `12` 条人口海盗压力回放共生成并出售 `36` 颗 `CrewRecruit` 来源骰，产生 `36` 次主动出售，但人口计数增量与第二代招募都为 `0`。
- 付费整刷仍发生 `174` 次，相比第 `55` 轮的 `176` 次只减少 `1.1%`。新增操作来自托盘、法术与资源改道，没有把整刷从核心搜索动作中挤掉。

因此形成候选 `BG3-K`：完整保留 `BG3-I / BG3-J`，不增加第三层保底；只把“公共资源两端已经存在但中间转换连续断供”加入既有导演的同级候选排序，在导演本来就需要介入时作为平级破同分项，而不是额外保证某张卡或某个家族。

## 二、审计边界

### 1. 什么叫自然连续

本轮继续使用第 `55` 轮已经固定的市场序列与公开资源状态：

- `E1–E10`：第一至二章。
- `M1–M10`：第三至五章。
- `L1–L10`：第六至十章。
- 每组依次经历三个市场。
- 每组分别运行保守、成长、破局三种策略。

允许变化的只有玩家根据 `BG3-I / BG3-J` 新规则做出的动作、钱包、法术架、托盘处理和持久资产状态。货架本身不因本轮想验证某条环而被人工换成指定组件。

### 2. 自然层与压力层分离

以下指标只来自 `90` 条自然策略序列：

- 非空转方向。
- 严格存牌与满架替换。
- 贸易猪三方向。
- 贡品处理。
- 托盘龟实体 / 化壳。
- 两条弱跨族环。
- 批量命令发生次数与分配负担。

人口来源递归、目标在提交前失效、事务恢复和壳状态守恒使用额外确定性压力回放。这些回放不计入自然出现率，也不抬高两条弱环的完成次数。

### 3. 方向仍按完整行动链计算

一个方向必须包含可支付的完整结果：

- 只买得起法术但付不起后续采购，不计。
- 有合法转栏目标但托盘无法在离店前清空，不计。
- 有贡品但没有合法奉献目标，也不能完成零金出售节奏，不计。
- 有龟和宿主但直接化壳会摧毁唯一必要家族身份时，仍可作为选项，但不能自动判为最优。
- 满架新法术若不能先施旧牌或替换失效计划，不计为额外方向。

## 三、市场非空转与纠偏透明度

### 1. 正常资源市场

`213` 个进入市场时仍保有至少一档利息或正常采购能力的决策中：

- 自然状态已有两条方向：`182 / 213`。
- 需要现有导演介入：`31 / 213`。
- 纠偏后保有两条方向：`213 / 213`。

`BG3-J` 新增的现金 / 转栏、实体 / 化壳、奉献 / 零售等分叉，会在已有资源两端时自然补出方向，但不会把形式合法、后续无法完成的按钮填入计数。

### 2. 主动透支市场

`57` 个市场来自玩家上一店主动支出并以零利息档进入：

- `57 / 57` 至少有一个当前可支付且有结果的动作。
- `39 / 57` 仍自然拥有两条或以上方向。
- 其余 `18` 个只保一个动作，没有由导演免费补偿第二条路线。
- 真正死店仍为 `0`。

这保留了“我自己把未来购买力花掉”的后果，同时避免随机供给把玩家锁成只能离店。

### 3. 导演介入

| 介入类型 | 市场数 |
|---|---:|
| 完全不介入 | `239` |
| 只改法术位 | `24` |
| 只改一颗非保底骰 | `4` |
| 法术位与一颗非保底骰都改 | `3` |
| 修改高阶保底骰 | `0` |
| 最终方向失败 | `0` |

法术架满会暂停法术空转保护，因此有少量状态转而使用一颗非保底骰纠偏。这是容量选择的真实后果，不应通过替玩家销毁旧牌消除。

三种策略在 `81 / 90` 条序列中至少出现两种不同的首个承诺，说明方向底线没有把保守、成长与破局压成同一种买法。

## 四、严格存牌与满架替换

### 1. 两店兑现

只统计购买时通过以下四项证明，并且后续确实有完整一至两个市场窗口的法术：

1. 持久输入明确。
2. 两店内窗口明确。
3. 未来完整成本可支付。
4. 不被单纯持币严格支配。

| 切分 | 严格购买 | 两店内施放 | 兑现率 |
|---|---:|---:|---:|
| 早期 | `6` | `5` | `83.3%` |
| 中期 | `7` | `5` | `71.4%` |
| 晚期 | `6` | `3` | `50.0%` |
| 合计 | `19` | `13` | `68.4%` |

按策略切分同样为：

| 策略 | 严格购买 | 两店内施放 | 兑现率 |
|---|---:|---:|---:|
| 保守 | `6` | `5` | `83.3%` |
| 成长 | `7` | `5` | `71.4%` |
| 破局 | `6` | `3` | `50.0%` |

六张未兑现牌中：

- `3` 张因后续自然出现了价值更高且会改变路线的商品，玩家主动改计划。
- `2` 张的持久输入被玩家为了生存或强购主动出售 / 消耗。
- `1` 张因破局策略主动把完整支付预算改用于更长刷新链。
- `0` 张是因为系统没有生成购买时承诺的固定流程窗口。

因此四项证明已经从第 `55` 轮的 `18.5%` 大量假保存改善到可用区间。晚期与破局策略都只有 `50%`，仍需实机观察，但当前不构成改价格或扩法术架的理由。

### 2. 满架行为

法术架满共出现 `23` 个决策：

| 当前新法术结果 | 次数 | 处理 |
|---|---:|---|
| 旧牌本店可施放 | `3` | 先施旧牌，再正常购买新牌 |
| 一张旧计划已失效，新牌有完整链 | `4` | 无退款原子替换 |
| 三张旧计划仍优于新牌 | `3` | 跳过新牌 |
| 新牌本身没有完整链 | `13` | 跳过，不触发清理 |

满架替换占全部市场决策 `4 / 270 = 1.5%`，占满架决策 `4 / 23 = 17.4%`。它足以解决失效计划软锁，又没有成为常规无代价洗牌。

## 五、贸易猪三方向

完整窗口要求：

- 贸易猪已有饲料。
- 现金估值与结构回收值已知。
- 转栏目标稳定且离店前可处理。
- 若选择吞吃，法术、贡品或其它公共入口与全部成本都已就绪。

| 方向 | 首选次数 | 比例 |
|---|---:|---:|
| 现金兑现 | `5` | `33.3%` |
| 转栏后保留 | `5` | `33.3%` |
| 转栏后当店吞吃 | `5` | `33.3%` |

分叉由两种不同估值推动：

- 饲料次数占优、单次质量普通的 `7` 个窗口里，现金兑现为首选 `5` 次。
- 饲料价值占优且已有稳定承载 / 转化口的 `8` 个窗口里，转栏保留 `4` 次、转栏后吞吃 `4` 次。

没有一条路线超过 `80%`，也没有出现“现金与饲料同时完整拿走”。加急清算仍只读取现金估值，不进入上述三方向统计。

## 六、贡品与托盘龟

### 1. 贡品处理

| 处理方式 | 次数 | 说明 |
|---|---:|---|
| 购买后奉献 | `7` | 读取当前可吞吃面组，发布一次市场来源真实吞吃 |
| 购买后零金出售 | `2` | 发布一次主动出售，不产生金币事件 |
| 延后 / 放弃购买 | `4` | 保利息、腾托盘顺序或保留其它货架用途 |
| 合计 | `13` | — |

只有一颗吞噬骰子的 `8` 个窗口中，`3` 个没有立刻奉献，延后 / 放弃率 `37.5%`。最低二金与锁价足以让基础贡品保持净成本，又没有阻止多吞噬实例形成高阶返正。

两次零金出售都正确推进普通主动出售节奏；它们不产生金币事件，也不触发吞噬、黑市折扣或分账。

### 2. 托盘龟

只统计存在合法直接化壳目标的 `23` 个采购窗口：

| 去向 | 次数 | 比例 |
|---|---:|---:|
| 装入为实体 | `7` | `30.4%` |
| 直接化壳 | `14` | `60.9%` |
| 出售 / 作材料 | `2` | `8.7%` |

十四次直接化壳中，目标位于骰袋 `10` 次、另一托盘格 `4` 次。实体路线主要来自：

- 需要保留龟龟身份。
- 需要独立投骰与卖价。
- 计划继续喂养。
- 当前宿主虽合法但并不值得绑定。

所以托盘没有把龟龟统一退化为“买到即装备”。实体率高于 `20%` 首测下限。

## 七、两条弱跨族环

### 1. 猪猪 / 恶魔

不计贡猪等双阵营捷径，只计公共接口：

`饲料资产 → 贸易猪转栏 → 托盘材料 / 贡品 → 奉献或拆解 → 吞吃成长与响应`

| 指标 | 结果 |
|---|---:|
| 完整可执行窗口 | `12` |
| 实际执行 | `8` |
| 策略序列出现率 | `8 / 90 = 8.9%` |
| 至少一条策略执行的资源组 | `6 / 30 = 20.0%` |
| 早 / 中 / 晚执行 | `1 / 6 / 1` |

四次未执行分别被紧急现金、保命实体或更高优先级采购支配。弱环存在但不会自动压过猪猪现金和恶魔正常采购，符合“公共弱桥、双阵营负责提速”的定位。

### 2. 龟龟 / 海盗

不计黑帆、驮粮龟等双阵营捷径，只计：

`真实化壳 / 金钱龟 → 金币事件与实体位 → 财宝成长、人口、抢劫或采购`

| 指标 | 结果 |
|---|---:|
| 完整可执行窗口 | `19` |
| 实际执行 | `15` |
| 策略序列出现率 | `15 / 90 = 16.7%` |
| 至少一条策略执行的资源组 | `10 / 30 = 33.3%` |
| 早 / 中 / 晚执行 | `3 / 7 / 5` |

龟龟 / 海盗比猪猪 / 恶魔更自然，因为金钱事件和实体位本来就是海盗可读取的公共资源。四次未执行都发生在出售组合体可以立即越过关键采购阈值的状态。

### 3. 当前判断

- 两条弱环都覆盖三个章节段。
- 两条都能在移除双阵营骰后完成。
- 猪猪 / 恶魔明显更稀，但 `8.9%` 的策略序列出现率足以证明它不是只存在于定向规则题。
- 当前不提高特定法术或特定家族权重，也不新增桥梁卡。
- 若导演本来已经需要纠偏，可以把“资源两端已存在但中间转换连续断供”作为同功能候选的破同分项；原市场已有两条方向时绝不为追求跨族环改货。

## 八、批量目标负担

### 1. 自然样本

| 根动作 | 根命令 | 目标分配 |
|---|---:|---:|
| 多黑帆付费刷新 | `5` | `12` |
| 多魔蝠离店 | `4` | `9` |
| 清仓祭典 | `5` | `13` |
| 合计 | `14` | `34` |

每个根动作只打开一次面板。未分配的黑帆 / 魔蝠默认视为跳过，因此即使来源实例多于货架食物，也不需要逐个点击“跳过”。

- 平均目标分配：`34 / 14 = 2.43`。
- 加一次统一提交后的平均必要输入：`3.43`。
- 三格货架下最大目标分配为 `3`，最大必要输入为 `4`。
- 面板打开次数：`14 / 14`。
- 提交后再次弹出目标选择：`0 / 14`。

三次取消压力回放都完整回到前态：不扣法术、不扣刷新费、不增加刷新价、不吞吃、不清货架。

本轮没有可交互原型，不能把必要输入数伪装成真实秒数。实际停留时间、鼠标路径和认知负担仍需在生产包后的界面原型测量。

## 十、事务与资源守恒复核

额外十二条不计自然率的事务回放：

1. 四次贸易猪转栏目标在提交前失效，全部保持出售前态，没有丢猪、丢饲料或多得金币。
2. 四次外壳转运覆盖已使用金钱龟、幸运龟和带回收值真实壳；新旧宿主长期六面总变化均为 `0`，真实壳回收值总和守恒，内部已用状态未重置。
3. 四次批量命令分别在 `Prepared` 前、提交校验、`Committed` 后恢复；结果只得到完整前态或完整后态，没有半清货架、重复吞吃或重复刷新。

`BG3-J` 的资源唯一性和事务合同因此没有在连续状态中产生新冲突。

## 十二、候选 BG3-K

### 1. 完整保留的规则

- 三格骰子货架、差异化骰价、买后留空、整批付费刷新、递增刷新价、利息与不冻结。
- 两格交易托盘，骰袋资产不能免费拿回托盘。
- 一格独立法术位与三格法术架。
- 不提供常驻零金丢弃；满架只在购买当前新法术时无退款原子替换。
- `BG3-H` 四项严格保存证明与 `1.00 / 0.70 / 0.35` 供给倍率。
- 贸易猪现金 / 转栏互斥、饲料进入可吞吃面组、贡品最低二金锁价。
- 托盘龟实体 / 直接化壳、一次批量目标命令、真实壳实例转运。
- `CrewRecruit` 来源出售不推进人口海盗计数。

### 2. 资源断供只作既有纠偏的破同分项

新增一个公开可记录、但不向玩家显示为保底的 `BridgeGap` 审计标签。它只识别公共接口，不直接识别家族名：

- 已有高价值饲料退出资产与吞吃响应，但连续两个市场没有可执行的材料转换。
- 已有真实壳 / 可化壳龟与金币 / 容量读取者，但连续两个市场没有可执行的宿主或处理出口。

处理方式：

1. 原市场已经有两个方向时，不做任何修改。
2. 只有现有导演本来就需要纠偏时，`BridgeGap` 才在同资格候选之间破同分。
3. 仍然先尝试一个法术位，再最多改一颗非高阶保底骰。
4. 不保证拆解许可、外壳转运、贸易猪、金钱龟或任何指定卡名。
5. 法术架满时暂停法术纠偏；不能借 `BridgeGap` 替玩家清牌。
6. 玩家看见候选后拒绝时，继续服从同名、同功能与近期拒绝记忆，不连续追发。
7. 一旦公共转换成功执行，或资源任一端被出售 / 消耗，当前断供记忆清零。

这让供给安排知道“这局已经有两端，缺的是中段”，但不会把完整答案直接送进商店。

### 3. 批量目标输入预算

- 一个根动作只能打开一次面板。
- 每件货架食物最多分配一次。
- 黑帆 / 魔蝠未分配实例默认跳过，不要求逐个确认。
- 清仓祭典必须给每件非空货架指定目标。
- 三格货架下最多三次目标分配加一次提交。
- 建议可以预填，但不能自动提交。
