## 原始用户意图

用户认为当前主流程 UI 有太多无用信息，重点信息没有层次。左下角“本关关注”、`6 / 6 全部上场`、构成、当前骰袋都应去掉；本关目标不需要进度条式展示；底部 `Space 开始摇骰` 太长；顶部 HUD 文字太多。用户要求先出优化方案，确认后再生成美术接触图。随后进一步明确：`骰子王` 标题不需要删除；分数由基础骰子点数分和倍数构成，表达必须极致明确；最终只需要两个数字，一个是基础分数，另一个是倍数，不需要文字说明，也不需要“本关分数 / 当前分数 / 还差”这类表述，只保留 `0 / 72` 这种当前进度表达。结算演出时，本局分数要随着从左到右的骰子结算逐步累加，倍数也要同步变化并带来爽感。

## 问题框定

- 玩家问题：玩家需要在投骰和结算阶段一眼看见真正重要的信息，而不是被骰袋构成、说明文字和重复标签分散注意力。
- 生产问题：主流程 UI 需要先确立信息层级和计分器表现，再进入美术接触图和生产包，否则美术容易强化错误的信息结构。
- 当前系统张力：游戏已有金币、出手、出千、关卡规则、目标、骰子类型、牌型和结算反馈；如果都常驻展示，会稀释最核心的投骰与涨分体验。
- 目标体验：画面保留明亮账本桌游风，但信息像紧凑计分台；玩家先看骰子，再看 `基础分 × 倍数`，最后看 `0 / 72` 的实时推进。

## 讨论时间线

### 第 1 轮：删除低价值常驻信息

- 用户方向：左下角“本关关注”、`6 / 6 全部上场`、构成、当前骰袋、本关目标进度条、过长底部 Space 条和过多 HUD 文字都需要调整或去掉。
- Codex 回应：提出把金币、出手、出千放到左侧；中央保留骰子和目标；右侧只保留规则和牌型表；底部长条动作改短。
- 形成的洞察：当前问题不是缺少信息，而是无关信息太多，导致玩家不知道主流程阶段到底该看哪里。
- 决策状态：倾向

### 第 2 轮：保留标题并强调分数构成

- 用户方向：`骰子王` 标题不需要删除；分数主要由基础骰子点数分和倍数构成，必须极致明确。
- Codex 回应：修正顶部方案，保留标题；分数从说明型文案改成算式型表达。
- 形成的洞察：标题属于品牌和场景锚点，可以保留；真正需要压缩的是重复解释、进度条和低价值状态块。
- 决策状态：已确认

### 第 3 轮：去文字化的两个核心数字

- 用户方向：废信息仍然太多；只需要两个数字，一个是基础分数，另一个是倍数；不需要文字，不需要“本关分数”“当前分数”“还差”，只需要 `0 / 72`。
- Codex 回应：将分数区改为目标进度 `0 / 72` 加核心计分数字 `基础分` 和 `倍数`，去掉解释性标签。
- 形成的洞察：分数区应该靠位置、字号、颜色和动画建立语义，而不是靠文字说明。主界面应让玩家像看计分器一样读信息。
- 决策状态：已确认

### 第 4 轮：结算时数字逐步跳涨

- 用户方向：结算演出时，本局分数要随着从左到右的骰子结算逐步变化累加；倍数分数也一样，需要类似小丑牌的涨分爽感。
- Codex 回应：将分数区定义为结算计分器，而不是静态结果说明；基础分和倍数都应随逐颗骰子结算跳涨，倍数成立时触发明显跳变，并同步推高 `0 / 72`。
- 形成的洞察：本功能的重点不是“把结果显示得更简洁”，而是“用极少数字承载逐颗入账、倍率成立和目标推进的爽感”。
- 决策状态：已确认

### 第 5 轮：整理为 F007

- 用户方向：将本轮 UI 方向整理成 `F007`，后续由用户安排生产包。
- Codex 回应：创建本设计对话日志，作为后续 `$wabish-production-pipeline` 的来源。
- 形成的洞察：F007 已具备生产包输入价值，后续应拆给界面体验和美术先做接触图，再进入程序实现。
- 决策状态：已确认

- 形成的洞察：G 版解决了信息删减和数字层级；7 月 6 日后的屏幕级方向进一步要求主界面成为一台可演出的皇家账本计分机。下一步应比较“计分机主导”“账桌轻量升级”“市场柜台一体化”“状态剧场化”几种方向。
- 决策状态：倾向探索

### 第 7 轮：融合参考图后的三块细化

- 用户方向：认可融合版主场景参考图的大方向，但指出左侧积分塔灯饰抢注意力，右侧关卡说明和中间下方 UI 需要进一步细化。
- Codex 回应：把细化拆为三块：左侧积分塔从“灯泡装饰”改为“低噪声仪表 / 刻度 / 指针 / 爆分时临时发光”；右侧关卡说明从图标清单升级为关卡档案板；中下方 UI 从装饰槽位改为资源、阶段提示和可交互状态的功能条。
- 形成的洞察：融合版的核心价值是骰子区的滚动停格节奏和左侧爆分反馈；常驻态不能同时保留强灯饰，否则会让玩家在投骰前就被计分塔抢走视线。右侧与底部应承担清晰信息和交互，不应只是装饰。
- 决策状态：倾向探索

### 第 8 轮：主场景动效拆件执行计划

- 用户方向：需要可执行的动效计划，重点包括爆分时各个灯泡亮灯节奏、分数快速显示、骰子转动效果、金币吞吐感，并询问后续是否应先把一个个素材做好再拼接。
- Codex 回应：建议不制作整屏视频作为最终资源，而是拆成可复用模块：骰子固定槽位旋转、积分塔爆分灯光 / 指针 / 刻度、分数字体滚动、金币吸入 / 吐出、底部功能条微动效。运行时由 `Ready / Shaking / Stopping / ResultDecision / Scoring / StageClear` 状态机和 F006 展示事件驱动拼接。
- 形成的洞察：当前 F009 已有固定六槽序列帧骰子底座，F006 / F007 已有只读结算事件和分数跳涨底座；新工作应补正式视觉素材、节奏表和组合预制件，而不是重做规则或让动画重新计算分数。
- 决策状态：倾向计划

## 已确认决策

- `骰子王` 标题保留。
- 顶部 HUD 不再堆金币、出手和出千；这些资源全部放到左侧。
- 左侧删除“当前骰袋”、`6 / 6 全部上场`、构成、`基础 x6` 和左下角“本关关注”。
- 本关目标不使用进度条。
- 目标与当前累计只保留 `0 / 72` 这种表达，不显示“本关分数”“当前分数”“还差”等文字。
- 分数区只突出两个核心数字：基础分数和倍数。
- 基础分数和倍数不加解释性文字标签，依靠位置、字号、颜色、图形和动画建立语义。
- 结算演出中，骰子按从左到右的结算顺序逐颗入账，基础分数实时累加。
- 倍数也要在结算过程中发生变化；牌型或倍率成立时，倍数数字跳涨，并同步推动 `0 / 72` 增长。
- 底部 `Space 开始摇骰` 不再做成长条，改为更短、更轻的阶段提示。
- 后续先按本方向生成美术接触图，再进入生产包和实现。

## 倾向方向

- 主画面优先级建议为：骰子结果最高，基础分与倍数第二，`0 / 72` 第三，左侧资源第四，右侧规则入口最低。
- 基础分数字应是厚重、可跳涨的大号数字；倍数数字应像印章或倍率牌，触发时有更明确的放大、盖章或震动反馈。
- `0 / 72` 可以放在中央计分器上方或投掷区边角，平时安静，结算时随每次入账快速滚动。
- 倍数变化不应只在最后结算时出现；三同、四同、五同、顺子、全奇、全偶等成立时，应在对应结算节点显著跳变。
- 没有触发牌型时，倍数保持 `×1`，但仍作为固定计分器数字存在。
- 右侧关卡信息只保留本关规则和牌型表入口，不常驻解释段落。
- 融合版主场景后续应保留中央骰子滚动停格节奏和左侧爆分反馈，但常驻态左侧积分塔要降低噪声，去掉一圈灯泡式装饰。
- 右侧关卡说明建议做成“关卡档案板”：上方显示关卡目标和规则图标，中部显示本关规则的短句，底部显示奖励 / 风险 / 进度的小章，不做长教程。
- 中间下方 UI 建议改成功能条：左段资源与状态，中央为短操作提示，右段为阶段反馈或本次预览；装饰性空格和重复小方块应减少。
- 主场景动效后续按独立素材生产，不把分数、目标、关卡文案、金币数量烘焙进图片或视频。
- 骰子转动继续沿用 F009 的固定槽位序列帧思路，可在视觉上加强“滚动停格的机器节奏”，但不能引入老虎机拉杆、假大奖或赌场符号。
- 积分塔爆分只在 `Scoring` 达标或大额跳涨时临时增强；`Ready` 和 `ResultDecision` 常驻态保持低噪声，不保留一圈常亮灯饰。
- 金币吞吐拆为金币粒子、金币口开合、路径高光和资源牌数字跳动四层，金币真实增减仍只读取已提交的钱包事件。

## 已拒绝 / 已后置想法

- 拒绝保留左侧骰袋构成和“全部上场”说明，因为初始 6 颗全上场是默认事实，不值得常驻占用主流程空间。
- 拒绝常驻“本关关注”，因为它与关卡规则和目标信息重复，且容易变成低价值提示。
- 拒绝进度条式本关目标展示，当前需求更偏向数字计分器。
- 拒绝用长文字公式解释基础分和倍数，后续应把解释放到牌型表或教程外层，而不是主结算区。
- 后置完整教程化说明；F007 首版主界面不承担讲解所有计分规则的任务。

- 中间下方 UI 是否保留 `Space` 作为唯一主操作视觉锚点，还是额外显示少量状态按钮或资源入口。

## 设计理由

`骰子王` 的主流程应该服务“把骰子结果转化为可控分数”的核心幻想。当前界面把大量静态说明堆在主屏上，会削弱玩家对骰子、基础分、倍率和目标推进的关注。F007 的方向把主界面从说明面板改成计分台：平时只保留必要状态，结算时让数字自己表演。这样既符合项目“紧凑桌面工具”的 UI 目标，也能让后续结算演出承接 F006 的左到右槽位顺序规则。

## 职能备注

### 策划

首版应确认基础分和倍数的定义口径。基础分可以理解为本次已结算的单骰分合计，倍数为当前已成立的牌型倍率或最终倍率。非基础骰、材质、词缀和关卡规则如果改变单骰分，应默认汇入基础分数字，而不是在主界面增加新常驻数字。

### 美术

后续美术接触图重点不是画更多面板，而是验证极简计分器是否有层级。需要保留 `骰子王` 标题和明亮账本桌游风；左侧资源可做成小账牌或竖向印记；基础分数字应有“入账上涨”的质感；倍数可借用印章、王冠、牌型牌或账本倍率章，但不要生成大量文字。

### 界面体验

主流程 UI 应去掉解释性堆叠。中央投掷区、计分器和 `0 / 72` 是核心；左侧资源只显示金币、出手和出千；右侧只保留本关规则与牌型表入口。交互提示 `Space` 应短而轻，不再横跨底部。

结算期的焦点顺序应为：当前结算骰子高亮，基础分数字跳涨，必要时倍数数字跳涨，随后 `0 / 72` 滚动更新。同一时刻不要让多处大数字同时争抢注意力。

### 程序

实现时可复用 F006 的左到右结算顺序。结算动画需要暴露中间态：当前已入账基础分、当前显示倍数、当前累计分。动画播放不应改变真实结算结果；跳过或加速时应落到同一最终值。

如果完整牌型需要全局识别，程序仍可预先算出最终结果，但前端播放时按照骰子顺序在合适节点触发倍率数字跳变。需要避免“显示中的临时倍数”和最终结算倍率不一致导致玩家误解。

## 风险登记

| 风险 | 重要原因 | 可能缓解方式 | 状态 |
|---|---|---|---|
| 过度去文字后语义不清 | 新玩家可能不知道两个数字分别代表什么 | 用固定位置、颜色、图形和首次轻提示建立语义，常驻不写说明 | 已识别 |
| 倍数中间态误导 | 部分牌型需要完整结果才能判断，提前跳倍可能与最终识别冲突 | 后台预判最终牌型，按确定节点播放，不临时猜测 | 已识别 |
| 数字抢焦点 | `0 / 72`、基础分和倍数都在跳，可能视觉混乱 | 设定主次节奏：基础分先跳，倍数重击，目标进度随后滚动 | 已识别 |
| 低价值信息被重新加回 | 生产中容易把骰袋构成、说明文字和提示重新堆回界面 | 生产包明确禁止常驻这些信息，必要信息只在详情或牌型表出现 | 已识别 |
| 美术装饰压过计分器 | 明亮账本风可能引入过多边框、纹样和标签 | 接触图优先检查阅读路径和数字层级，再评价装饰完成度 | 已识别 |
| 积分塔常驻灯饰抢焦点 | 融合版左侧灯泡过亮，Ready 阶段就会分散玩家对骰子区的注意力 | 常驻态去掉灯泡圈，只保留刻度、指针和少量金属结构；爆分时再临时点亮 | 已识别 |
| 右侧说明停留在装饰图标 | 仅有图标清单不能承担关卡说明，玩家仍不知道本关规则重点 | 改为关卡档案板，使用标题区、规则短句区和少量状态章 | 已识别 |
| 底部 UI 变成无意义占位 | 中下方装饰槽太多会像未实现控件，削弱正式感 | 减少空槽，明确资源、操作提示和阶段反馈的三段职责 | 已识别 |
| 整屏动效资源难以维护 | 如果把分数、金币、骰子和 UI 一起烘成视频，后续数值和布局变化都会重做 | 只把通用视觉部件做成序列帧、遮罩、粒子和 UI 贴图，运行时拼接 | 已识别 |
| 动画暗示错误规则 | 过强的投骰加力、老虎机符号或金币吐口可能让玩家误以为能操控概率或提前获得金币 | 动画只读 `RollPhase` 和已提交展示事件；投骰加力只影响表现强度，金币只在真实事件后播放 | 已识别 |
| 爆分灯光过密 | 灯泡逐个亮过多会拖慢结算，并抢过分数数字 | 灯光只做 3 档节奏：普通入账短闪、大额跳涨追光、达标爆亮；默认不逐个长播 | 已识别 |

## 候选验收

- 主流程画面保留 `骰子王` 标题。
- 左侧只显示金币、出手和出千三类资源状态。
- 画面中不再常驻显示“当前骰袋”、`6 / 6 全部上场`、构成、`基础 x6` 和“本关关注”。
- 本关目标区域不出现进度条。
- 目标进度只显示类似 `0 / 72` 的数字。
- 结果或结算区只突出基础分数字和倍数数字，不常驻解释性文字标签。
- 底部 `Space` 提示不再横跨整屏。
- 结算时骰子从左到右逐颗高亮并入账，基础分数字随之累加。
- 倍数成立时，倍数数字发生明显跳涨或盖章反馈。
- `0 / 72` 随入账和倍率变化滚动更新，能产生连续涨分爽感。
- 玩家在不阅读长说明的情况下，能把主要注意力放在骰子、基础分、倍数和目标推进上。
