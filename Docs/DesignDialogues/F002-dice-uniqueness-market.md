## 当前摘要

用户要求废除旧的“体验爆点”提案，改以 `F002` 开启新的内容开发方向。新方向的核心是：关间商店不只刷新固定类型骰，而是刷新具有个体差异的骰子；每颗骰子的独特性主要来自两个维度：第一是面组分布不再固定为常规 `1 / 2 / 3 / 4 / 5 / 6`，第二是材质带来额外身份和单一机制修饰。最新收敛是：词缀系统后置，本版本只做商店生成骰子的随机点面和材质逻辑；面组差异开放给所有骰子类型；首版材质池暂定为 `官铁 / 鎏印 / 明釉 / 铅封 / 铜骨`。本对话已转为生产包 `Docs/Production/Features/F002-dice-uniqueness-market/`，后续执行以该生产包为准。

## 原始用户意图

用户提出，`F002` 的主要功能应废除之前关于“爆点”的提案，开始新的内容开发。商店刷新应提供独特的各类型骰子。独特性来自两个维度：一是点面不再固定为常规 `1-6`，而是有随机和偏置，例如某种奇数骰可能六个面都是 `3`；二是骰子的材质，例如黄金、钢铁、玻璃等，但最好不要直接和 `小丑牌` 使用同名，以免被认为是抄袭。

## 问题框定

- 玩家问题：当前市场主要卖“类型”，同一类型骰之间差异较小，玩家容易把购买决策简化为按流派买固定答案。
- 生产问题：需要让商店生成更多可比较的独特骰，但不能形成难以维护的无限词缀矩阵。
- 当前系统张力：现有文档曾强调首章不以“材质 + 性格”作为核心表达，而是用清晰的骰子类型。`F002` 需要重新评估这个限制，并决定“骰质”应是弱修饰、强机制，还是只服务稀有版本。
- 目标体验：玩家在商店看到的不只是“买一颗奇数骰”，而是“这颗奇数骰的面组和骰质刚好适合我当前路线”，从而形成更强的路线判断和局内记忆。

## 已确认决策

- 旧 `D2` 体验爆点提案被 `F002` 新方向替代，后续不以抽骰手牌循环作为当前生产依据。
- `F002` 的探索重点是商店刷新具有个体差异的骰子，而不是继续扩张大量全新路线类型。
- 独特骰至少围绕两个维度讨论：面组分布和骰质。
- 玩家可见命名应避免直接复用 `小丑牌` 中高度相同的材质命名。
- 面组差异开放给所有骰子类型，而不是只限制在基础、奇数、偶数、双倍或赌徒等少数简单骰上。
- 骰质不再只是价格或稀有度标签，可以从多个机制维度影响骰子，已明确提到的候选包括初始积分 `+1` 和倍率额外 `+0.1`。
- `POE2` 式装备构建思路可作为 F002 的重要参考方向，用于制作独一无二的具体骰子。
- 词缀系统后置，不进入 F002 当前首版。
- F002 当前首版只做商店生成骰子的随机点面和单一材质逻辑。
- F002 首版材质池暂定为五种：`官铁 / 鎏印 / 明釉 / 铅封 / 铜骨`。
- F002 首版包含随机点面生成；随机点面开放给所有骰子类型。

## 倾向方向

- 保留现有“骰子类型”作为第一识别层，例如基础、奇数、偶数、龟龟、大树、猪猪等。
- 将“面组”定义为概率和牌型维度，例如重复 `3`、偏向奇数、高均值、低点密集、顺子补洞、对子专精。
- 将“骰质”定义为比类型更轻的个体修饰，优先服务一句话规则、价格、稀有度或局内反馈，不应抢走类型身份。
- 可把玩家口语中的“材质”改成更有项目身份的“骰质”或“铸质”，让它听起来像骰子工坊和账本官署体系，而不是直接照搬其它牌组游戏的增强名。
- `黄金 / 钢铁 / 玻璃` 可以先作为功能参考，不宜直接定为最终 UI 名称。候选方向可用 `鎏金`、`官铁`、`明釉`、`账蜡`、`铅封`、`木纹`、`骨白`、`铜印`、`墨痕` 等更贴近 `骰子王` 语气的词。
- 面组强度不能只看平均点数，还必须看它对牌型概率的贡献。例如六面全是 `3` 的骰子，单颗平均不高，但会极大提高三同、四同、五同、六同路线稳定性。
- 商店卡片需要把三层信息压成可读结构：类型名、面组条、骰质短标签、价格和一句购买建议。
- 装备化骰子的三层底座可暂定为：`骰子类型` 是底材，`面组` 是基础数值和概率倾向，`骰质 / 词缀` 是显性修饰。
- 首版骰质维度不宜全部同时开放。更稳的路线是先做 4 到 6 个清晰维度，每颗骰最多带 1 到 2 条骰质效果，避免市场卡片变成说明书。
- 倍率额外 `+0.1` 需要谨慎定义。它可以作为最终倍率的小修正层，但不应轻易参与主牌型倍率叠乘，否则会和当前整数倍率表冲突。
- 首版材质建议采用“每颗骰 1 个材质、1 句规则、1 个价格修正”的结构。
- 材质效果应尽量通用于所有骰子类型，避免出现大量类型专属例外。
- 首版材质不做多词缀、不做主动改造、不做复杂负面词缀池；这些保留给后续装备化系统。
- 首版五种材质的初始设计方向：
  - `官铁`：本骰单骰分 `+1`，承担稳定提分材质。
  - `鎏印`：本骰掷出自己面组最高点时，本关账本金币 `+1`，承担经济材质。
  - `明釉`：本骰掷出自己面组最高点时，单骰分额外 `+2`，承担高点爆发材质。
  - `铅封`：本骰被出千重摇后，本次单骰分 `+2`，承担出千互动材质。
  - `铜骨`：本骰与任意实体骰同点时，单骰分 `+1`，承担追同点材质。

## 设计理由

该方向能直接回应当前市场的重复性问题。现有市场按 `die_type` 刷新时，同一类型骰子的价值大多固定，玩家一旦理解流派，就容易把商店选择变成“看到核心件就买，看不到就刷新”。如果商店里的每颗骰子都有面组差异，玩家会开始比较这颗骰子的具体概率和自己的当前路线是否相合。

面组是最适合 `骰子王` 的个体差异来源，因为它直接属于骰子本体。它不会像额外遗物系统那样脱离投骰，也不会像大规模新类型那样要求每次新增完整结算规则。玩家看到 `3 / 3 / 3 / 3 / 3 / 3`，立刻能理解这颗骰稳定，但也能推断它对三同、四同和全奇有价值。

骰质的价值在于提供命名、视觉和少量机制记忆点。它不应取代骰子类型，而应让玩家记住“这颗具体骰子”而不是“又一颗奇数骰”。但骰质如果过强，会和现有骰子类型争夺主身份，也会让商店组合数量过快爆炸。因此首版更适合把骰质控制在少量、强可读、可定价的修饰层。

用户提出的装备化思路可以成立，因为 `骰子王` 当前最需要的是“同一类型骰子的个体价值差异”。装备化不是照搬复杂词缀海，而是借用它的结构：玩家先看底材是否属于当前路线，再看数值基础是否合适，最后看词缀是否产生稀有协同。对本项目来说，底材就是骰子类型，数值基础就是六个面，词缀就是骰质或工艺修饰。这样市场商品能从“模板货架”变成“这颗具体骰子我想要”。

但装备化也会放大平衡风险。当前计分公式已经有骰面分、类型效果、关卡规则、临时小骰和牌型倍率，如果骰质再直接新增多个乘区，会让首章目标分和路线强度快速失控。因此倍率类骰质应比加分类、经济类、出千类更稀有，也应该优先用小幅、条件化、可预览的方式出现。

## 职能备注

### 策划

需要先建立面组强度评估。平均点数只是一个维度，重复点数、奇偶纯度、顺子补洞、极端稳定性和与现有骰子类型的联动都应参与定价。首版可先做少量面组家族，而不是完全随机生成任意六面。

装备化首版建议把词缀分成若干清晰维度：单骰分修正、倍率修正、经济修正、出千修正、触发条件修正、成长修正、市场价值修正和负面代价。每颗骰最多出现少量词缀，并由价格公式反映强度。

最新首版范围收窄后，策划重点改为定义少量材质。每种材质应该能用一句话表达，并且不依赖额外 UI 面板解释。材质可以参考装备化的“底层手感”，但当前不生成多条词缀。

### 美术

骰质需要可视化，但不应盖过骰子类型图标。更适合在骰子本体上做边角、表面纹理、封章、釉面、铆钉、金箔线、墨痕等局部符号。避免把骰子放进大型外部场景说明。

### 界面体验

商店卡片需要清楚展示六个面，不要只显示一句“奇数骰”。面组可以用小骰面条或六格点阵表示，骰质用短标签和一个小图标表示。购买建议应解释为什么它适合当前路线，例如“高同点，适合追三同”和“纯奇，适合全奇与盖章”。

如果采用装备化表达，商店卡片需要严格控制层级：第一行是类型和稀有度，第二行展示六面，第三行展示骰质词缀，第四行给价格和一句路线提示。不要让每张卡超过两条主要规则文本。

首版商店卡片更适合显示为：类型名、材质名、六面展示、材质一句话、价格。不要出现多条装备词缀列表。

### 程序

如果进入实现，需要在 `DiceData` 中持久化面组和骰质 key。市场配置可能需要从当前按类型权重，扩展为类型权重、面组池权重、骰质池权重和价格修正。结算层应继续以每颗骰子的 `Faces` 为事实来源，出千只在本骰面组内重摇。

装备化实现需要把市场生成拆成可测试步骤：选择骰子类型、选择面组模板或随机面组、选择骰质词缀、计算价格、生成显示名、写入存档。必须先建立禁止组合表，例如某些倍率骰质不能叠在高重复同点面组上，或特定骰质不能与特定类型同时出现。

首版实现可以拆成更小步骤：选择骰子类型、生成或选择面组、选择单一材质、按材质修正价格、生成显示名、写入存档。暂不需要词缀数组，只需要一个材质 key。

推荐后续工作流：

1. 设计规格收敛：确认五种材质的最终规则、随机点面模板池、价格修正、出现权重和禁止组合。
2. 生成生产包：用本 F002 日志生成 `FEATURE_BRIEF.md`、`UI_UX_SPEC.md`、`DEV_HANDOFF.md`、`ACCEPTANCE.md`。
3. 数据结构设计：为每颗骰增加材质 key、面组来源、显示名和价格修正字段，确认旧存档兼容。
4. 市场生成实现：把当前按 `die_type` 刷新，扩展为类型选择、面组生成、材质选择、价格计算和购买建议生成。
5. 结算接入：把五种材质效果接入单骰分、账本金币、出千重摇和同点判断，不改变牌型表。
6. 商店界面接入：市场卡片展示类型、材质、六面、价格和一句材质规则。
7. 验证与调参：用多次刷新和小关结算检查随机面组强度、材质触发、价格合理性和 UI 可读性。
8. 文档同步：实现后同步 `PROJECT_CONTEXT.md`、`GAME_FLOW.md`、`DICE_ARCHETYPES.md` 和生产包验收记录。

## 风险登记

| 风险 | 重要原因 | 可能缓解方式 | 状态 |
|---|---|---|---|
| 强面组破坏牌型倍率 | 六面全同或高重复面会显著提高同点牌型稳定性 | 用阶级、价格、章节权重和稀有度控制 | 开放 |
| 骰质抢走类型身份 | 玩家可能分不清类型效果和骰质效果哪个更重要 | 类型保持主标题，骰质只做短标签和轻规则 | 开放 |
| 命名被认为照搬 | `黄金 / 钢铁 / 玻璃` 与同类游戏强化名高度相似 | 改用更贴合骰子工坊和账本语气的命名 | 开放 |
| 商店信息过载 | 类型、六面、骰质、价格、建议同时展示 | 卡片分层：标题、六面、短标签、价格、建议 | 开放 |
| 组合数量失控 | 类型乘以面组乘以骰质会快速膨胀 | 首版限制面组家族和骰质数量，只开放部分类型 | 开放 |
| 存档兼容复杂 | 旧存档只有类型和面组，没有骰质 | 读取缺失骰质时回退为无骰质或普通骰质 | 开放 |
| 装备词缀过多 | 玩家在商店中需要同时理解类型、面组、骰质、价格和路线协同 | 每颗骰最多 1 到 2 条词缀，卡片只显示一句路线判断 | 开放 |
| 小倍率失控 | `+0.1` 如果作为独立乘区反复叠加，会让目标分很难调 | 限制倍率词缀数量、条件和稀有度，优先做最终倍率小修正 | 开放 |
| 负面词缀体验差 | 若强骰靠负面词缀平衡，玩家可能觉得买到陷阱 | 负面必须换来明显强面组或强词缀，并在商店醒目展示 | 开放 |
| 首版材质过多 | 材质数量过多会接近词缀系统，破坏本轮收窄目标 | 首版控制在 4 到 6 种核心材质 | 开放 |
| 材质效果太像类型 | 如果材质也做复杂路线效果，会抢走骰子类型身份 | 材质只做通用轻规则，不做完整路线 | 开放 |

## 候选验收

- 玩家能在商店中区分两颗同类型骰子的价值差异。
- 玩家能通过六面展示判断这颗骰子适合追同点、全奇、全偶、顺子、成长或爆发。
- 每颗骰子的卡片说明不超过一句关键购买理由。
- 骰质命名与视觉能让玩家记住具体骰子，但不会掩盖类型机制。
- 首版独特骰不会要求重写现有结算公式，只扩展市场生成和单骰数据。
- 同一类型骰因为面组和骰质不同，能在商店中形成至少两种明显不同的购买理由。
- 装备化骰子的文本长度可控：玩家在一张卡上能在数秒内理解类型、面组和最多两条骰质效果。
- 倍率类骰质在预览阶段必须清楚显示作用位置，不能让玩家只看到最终分数变化而不知道原因。
- F002 首版商店卡片中，每颗骰只展示一个材质效果，不出现多词缀列表。
- 玩家能用一句话记住材质差异，例如“官铁稳分”“鎏印赚钱”“明釉赌高点”。
- 五种首版材质都能在结算或商店预览中明确显示触发原因。
- 随机点面不会生成无法解释、明显破坏首章平衡或与骰子类型互相矛盾的商品。
