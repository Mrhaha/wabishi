# BG3-F 十二法术配对支配与原子交易审计

审计日期：2026-07-23
对应设计对话：`D2` 第 `53` 轮
审计对象：`BG3-D` 十二张候选法术的立即施放、跨店保存与明确跳过；黑市分账 × 清仓祭典；换班凭证的实体、事件与恢复边界
状态：纸面配对审计完成；卡表、资格与原子事务规则仍为候选，未进入代码、配置或正式玩法文档

## 一、结论

十二张候选法术都能保留，但不能继续沿用“只要存在合法目标，就能替导演补一条路”的宽松口径。

- 本轮为每张法术建立 `3` 个只改变关键资源条件的固定状态，共 `36` 个配对状态：`12 / 12` 均存在合理的立即买、跨店保存和明确跳过场景。
- 这证明十二张牌在逻辑上都不是无条件必买，也不是只能现抓现用；但样本是刻意配对的存在性证明，不能把三类各 `1 / 3` 解释为真实选择率。
- 当前不改十二张牌的价格。真正需要修改的是导演资格、预览信息和两张高阶牌的事务边界。
- `黑市分账` 不能因为“有一次吞吃入口”就成为导演方向。基础现金盈亏为 `2n - 5`，没有其它金币响应时至少需要 `3` 次本市场来源吞吃才转正。
- `清仓祭典` 不能因为货架有两格就成为导演方向。常规资格应收紧为三格非空、至少两个有效吞吃目标，并且成长、现金或下一刷折扣中至少一项形成明确后续；两格只允许极高质量食物和多重响应的例外。
- `金币饲料` 的最低档总成本为 `6` 金，只做两次真实喂养时单位效率明显低于 `3` 金饲料袋。导演只应在需要至少四次喂养容量，或高饲料质量与多响应同时成立时把它计作方向。
- `换班凭证` 被基础托盘流程揭示出一条重要支配关系：普通“卖旧骰、装新骰”不需要支付 `6` 金，玩家本来就能购买到托盘、直接卖旧、再装入。它的独特价值必须限定为“把袋内资产回收到托盘后接托盘专属处理”，或“让新骰首次装入先于旧资产退出”；普通换代不进入导演候选。
- `清仓祭典` 与已施放的多张黑市分账可以形成高额返金，最极端的“三张分账、两颗吞噬骰子、三格清仓”在同一市场把全部法术成本也计入后仍可净赚 `1` 金并获得三次吞吃成长。这是高阶破局而非孤立真无限：三张分账和一张清仓会被消耗，单法术位一次刷新只补一张牌，原始刷新价格继续递增。

因此建议形成 `BG3-F`：保留卡表与价格，修正五张牌的导演资格，并为清仓祭典和换班凭证采用可恢复的原子事务。

## 二、判定方法

本轮的三种选择固定为：

- **立即买**：本市场购买并施放，整条“买牌—追加支付—选目标—后续处理”当下可完成。
- **跨店保存**：本市场购买但不施放；必须有空法术架、明确的下一阶段用途，并且保存成本没有被当前更强动作严格支配。“以后也许有用”不算。
- **明确跳过**：不购买该牌；原因可以是现金 / 利息、法术架容量、不可逆资产损失、缺少完整后续，或存在更便宜且同结果的基础动作。

每个判断同时计算五种机会成本：

1. 法术标价和追加支付。
2. 支付后是否仍能完成下一动作。
3. 离店钱包所在的利息档。
4. 三格法术架占用。
5. 被改造、清算、吞吃或回收资产的不可逆损失。

配对状态不追求模拟自然出现率，而是刻意只改变一至两个决定变量，让同一张牌的首选发生反转。例如单格换货的三态只主要改变“整刷价格”和“是否有值得保留的货架”，加急清算只主要改变资产回收值、出售响应与额外现金能否打开采购。

## 四、卡表与导演资格修正版

这轮没有证据支持立即改价。建议先采用下列 `BG3-F` 资格，再用真实选择轨迹观察出现频率。

| 法术 | 价格判断 | 导演可计数的最低资格 |
|---|---|---|
| 单格换货 | 保留 `2` | 至少保留一件有用商品，或当前整刷价 `> 2` |
| 瑕疵折价 | 保留 `2` | 折后采购整链可支付，且目标短期价值或现金缺口足以补偿六面各 `-1` |
| 饲料袋 | 保留 `3` | 有长期目标，并且饲料质量、肉猪、贪吃或其它读取中至少一项让两次真实喂养改变当前状态 |
| 拆解许可 | 保留 `3` | 已有材料，或能支付“材料买价 + 法术”；材料不是当前唯一更高价值装入件 |
| 加急清算 | 保留 `4` | 回收值 `>= 5`，且相对正常出售的额外现金打开采购、刷新或必要退出 |
| 外壳转运 | 保留 `4` | 目标是真实壳，存在更合适宿主或旧宿主退出；不能把历史面组误计为可搬运价值 |
| 货架施肥 | 保留 `5` | 通常至少两件非空商品会在本市场被采购、化壳或吞吃；只抬数不处理不计数 |
| 定向招募 | 保留 `5` | 支付后能买常规结果，或已有不依赖价格的市场消费者 |
| 黑市分账 | 保留 `5` | 预计 `n >= 3` 次合格吞吃；`n = 2` 只有在金币事件响应或现金阈值打开独特后续时例外 |
| 金币饲料 | 保留 `4 + 追加` | 计划至少四次喂养，或两次喂养已被高质量与多重响应放大；最低两次不能作为通用保险 |
| 清仓祭典 | 保留 `8` | 常规要求三格非空、至少两个有效目标，并有成长 / 现金 / 下一刷中至少一条完整后续；两格只作极端例外 |
| 换班凭证 | 保留 `6` | 回收旧资产后能接托盘专属化壳、拆解或有价值的首次装入顺序；普通卖旧换新不计数 |

导演只评估“立即买”资格。跨店保存是玩家自由度的一部分，但不能用来虚报当前市场已经提供第二条可执行路线。

## 五、黑市分账 × 清仓祭典净资产

设：

- `F`：清仓时非空货架数，范围 `1–3`。
- `D`：骰袋中的吞噬骰子数量；每颗对每次真实吞吃入账 `1` 金。
- `S`：已经施放并仍生效的黑市分账数量；同一次吞吃合并入账 `2S` 金。
- `I`：黑市小鬼数量；每次市场来源吞吃给下一次刷新累计 `I` 点折扣。

清仓祭典结算当下的直接现金为：

`现金入账 = F × (D + 2S)`

若分账已经施放、只比较现在是否再买清仓祭典，则现金净额为：

`祭典现金净额 = F × (D + 2S) - 8`

| `D / S` | 一格清仓 | 两格清仓 | 三格清仓 |
|---|---:|---:|---:|
| `0 / 0` | `-8` | `-8` | `-8` |
| `1 / 0` | `-7` | `-6` | `-5` |
| `2 / 0` | `-6` | `-4` | `-2` |
| `0 / 1` | `-6` | `-4` | `-2` |
| `1 / 1` | `-5` | `-2` | `+1` |
| `2 / 1` | `-4` | `0` | `+4` |
| `1 / 2` | `-3` | `+2` | `+7` |
| `2 / 2` | `-2` | `+4` | `+10` |

上表没有把吞吃写入的永久面组成长折算成金币，也没有把黑市折扣当成当前现金。它回答的是“返金本身是否覆盖八金祭典”，不是整张牌的总价值。

若分账也在同一市场刚购买，需要再减去每张 `5` 金：

`整组法术现金净额 = F × (D + 2S) - 8 - 5S`

三格清仓时：

- 一张分账加一颗吞噬：`9 - 13 = -4`。
- 一张分账加两颗吞噬：`12 - 13 = -1`。
- 两张分账加两颗吞噬：`18 - 18 = 0`。
- 三张分账加两颗吞噬：`24 - 23 = +1`。

因此确实存在“全部法术成本都覆盖后仍净正”的后期破局点，但它需要三张跨店保存的同名五金法术、两颗吞噬骰子、八金祭典和三格食物。四张法术在结算后全部消耗或失效，单法术位每次刷新只产生一张新牌，不能从结果中自动复制自己。

黑市小鬼只提供下一次刷新的延迟价值：

`下一刷新增折扣 = F × I`

刷新海盗与黑市折扣允许让净刷新费为零或负数并返金，但不能倒过来降低已经点击的本次刷新。递增原始刷新价仍继续上涨。专项重复状态的停止条件为：保存法术耗尽、法术位一次只出一张、货架清空不补货、玩家主动停止或递增刷新重新超过每轮固定返金。当前组合没有孤立真无限，不需要为了压掉 `+1` 金极端高光而加硬上限。

### 事件顺序

清仓祭典提交后锁定全部货架实例与目标实例，按货架从左到右执行：

1. 将该货架骰的吞吃面组成长写入预分配目标。
2. 发布一次保留市场来源的真实吞吃。
3. 按稳定骰袋顺序结算吞噬骰子、黑市小鬼、共食等正常响应。
4. 所有生效的黑市分账对本次吞吃只发布一笔金额为 `2S` 的合并金币事件。
5. 当前货架格保持为空，局部队列清空后再处理下一格。

财宝海盗按真实金币入账事件读取：不同吞噬骰子的正常入账仍按各自响应进入队列；多张黑市分账只有一笔合并入账，不能用复制法术数量把同一次吞吃拆成多笔财宝触发。

## 六、清仓祭典的批量事务与恢复

批量目标不能只靠界面锁定，必须成为可恢复命令。建议的事务记录至少包含：

- 本次法术实例 ID。
- 左到右三个货架实例 ID；空格明确记为空。
- 每个非空货架对应的目标实例 ID。
- 提交前金币、法术架、货架、骰袋、托盘和相关实例状态快照。
- 提交时随机数状态。
- 事务状态：`Prepared / Committed`。

执行口径：

1. 提交前一次性验证所有货架与目标仍存在、目标合法、法术仍在架上。
2. 先写入 `Prepared` 意图与提交前快照，再锁定输入并完整执行左到右队列。
3. 队列完成后只保存一个稳定的 `Committed` 游戏状态，再清除事务记录。
4. 若加载时发现 `Prepared` 而没有稳定提交，恢复提交前快照，并用已保存的同一命令与随机数状态自动重放；不能加载到只吞了一至两格的半成品。
5. 当前卡池没有会在清仓局部队列中删除后续目标的响应。若未来效果导致预分配目标失效，应中止并回滚整笔事务、记录错误，不能静默改投系统推荐目标。

玩家从按下统一提交到队列完成之间不能出售、刷新、改目标、离店或手动存档。长动画可以加速或跳过，但逻辑事务仍一次完成。

## 七、换班凭证的支配关系与原子性

### 1. 四类资产结果

| 旧骰 / 新骰 | 原子交换后的状态 | 审计判断 |
|---|---|---|
| 普通旧骰 / 普通新骰 | 新骰进入原槽，旧骰进入同一托盘格 | 若旧骰只准备出售，基础“卖旧—装新”已得到同结果，通常跳过 |
| 实体龟 / 非龟新宿主 | 新宿主进入原槽，实体龟回到托盘，可直接化壳到新宿主或另一宿主 | 形成换班凭证最清楚的独特用途 |
| 组合宿主 / 普通新骰 | 组合体连同全部真实壳、复制壳、内部状态和回收值整体进入托盘 | 可以整项出售、先转运真实壳、或作为拆解材料；交换本身不拆壳、不复制、不逐层发事件 |
| 普通旧骰 / 带首次装入的新骰 | 新骰首次装入恰好触发一次，旧骰只发布特殊回收 | 只有首次装入先于旧资产托盘处理能改变后续时，六金才有独特价值 |

组合宿主被拆解时只读取其当前面组作为材料，全部壳状态随材料销毁；被出售时整项只发布一次主动出售和一笔合并金币入账。若玩家要保留真实壳，必须先施放外壳转运，不能在换班时自动拆分。

### 2. 原子步骤

换班凭证选择的是一个托盘格和一个骰袋实例，而不是两个可暂存指针：

1. 预检法术仍在架上、所选托盘格仍是新骰、所选骰袋实例仍在原槽，且没有未完成市场队列。
2. 消耗法术；在同一状态变更中把托盘新骰放入旧骰原槽，并把旧骰放进刚空出的同一托盘格。另一托盘格不受影响。
3. 交换完成后先发布旧骰“特殊回收”，再发布新骰“首次装入”。
4. 新骰的首次装入标记永久写入该实例；以后被回收再归位不重放。
5. 旧骰保留原实例 ID、历史成长、壳状态和已经使用的主动次数；它回袋只算归位。
6. 玩家仍必须在离店前通过出售、吞吃、化壳、拆解或归位清空两个托盘格。

交换前后骰袋实体数不变，所选托盘格始终从一个新骰变成一个旧骰，不存在中间第七个袋内槽位，也不建立隐藏待处理区。

### 3. 恢复边界

换班凭证复用清仓事务模型，但命令只需记录法术、托盘实例、骰袋实例、原槽位、首次装入标记和提交前快照。

- 崩溃或强退后只能恢复到完整交换前，或完整交换并完成回收 / 首次装入队列后。
- 不允许出现“新骰已进袋、旧骰仍在袋”“旧骰消失”“托盘同时保留新旧两骰”或首次装入重复触发。
- 若首次装入响应带随机目标，事务必须保存提交时随机数状态；自动重放得到同一目标。
- 动画表现可以先移新骰再移旧骰，但逻辑层必须是一次交换，不能让中间画面成为可保存状态。

### 4. 卡面与预览建议

卡面主句保持原子交换，补一行用途提示：

> 把袋内旧骰回收到该托盘格，以便化壳、拆解或重新处理。

确认预览同时显示：

- 新骰将进入的准确槽位与会触发的首次装入效果。
- 旧骰进入托盘后的完整组合状态和可用处理方式。
- “普通卖旧换新不需要此法术”的灰色比较提示。
- 交换后仍有几格托盘未清，以及当前是否可以合法离店。

## 八、未通过项

本轮通过的是十二张牌的逻辑可拒绝性与两张高阶牌的静态事务设计，不是行为平衡：

- `36` 个状态是人为配对，不能证明真人会让任何单张法术长期低于 `60%` 首选，也不能证明自然样本中合法法术至少 `25%` 被跳过。
- 没有对比加入单格换货、定向招募前后的真实整刷次数，“局部搜索不让整刷下降超过 `50%`”仍未通过。
- 没有把永久面组、饲料质量、壳持续效果与当前过关压力统一折算成一个分值，因此这不是总战力平衡表。
- 清仓祭典的事务日志、崩溃重放和固定随机数仍未进入代码验证。
- 换班凭证的组合宿主、首次装入和恢复边界只是生产合同，仍需单元状态和 Play Mode 恢复测试。
