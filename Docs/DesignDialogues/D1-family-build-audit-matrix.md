# D1 四族构筑体检与混合市场纸面矩阵

状态：初版完成，待用户审阅  
日期：2026-07-15  
来源对话：`D1`  
权威实现：`Assets/Scripts/DiceKingDemo.cs`  
本阶段边界：只做纸面审查，不启动 Unity，不修改程序、数据、界面或资源。

## 一、审查目标

本轮不判断“还要补多少颗骰子”，而是先回答四个问题：

1. 每个家族是否已经有不依赖双阵营骰的最小成型循环。
2. 玩家是否会在购买、刷新、卖出、摆位和离场之间做出不同选择。
3. 双阵营与中立骰是否承担桥接职责，而不是成为所有构筑的固定答案。
4. 当前内容缺口究竟需要新骰子、规则调整、反馈增强，还是只需要数值修正。

## 四、单家族最小构筑样本

这里的“最小构筑”不是最终推荐阵容，而是能让核心循环第一次成立的最小纸面样本。后续 Unity 验证应先按这些固定样本执行，不使用随机市场结果代替规则验证。

### P1：猪猪的投喂—放大—兑现

**样本组件**

- 肉猪
- 养猪
- 三只小猪
- 批发商
- 1 颗基础骰作为投喂对象

**推荐顺序**

`肉猪 -> 养猪 -> 基础骰 -> 三只小猪 -> 批发商`

**纸面过程**

1. 批发商在此前市场买入，提高后续饲料质量。
2. 养猪结算时给左侧肉猪和右侧基础骰各 2 份饲料。
3. 肉猪把自身获得的饲料效果计数翻倍，但不制造第二次饲料事件。
4. 三只小猪在后位读取其它骰已经结清的 `FeedValue`，转成本次临时分。

**玩家决策**

- 养猪摆在谁中间。
- 高质量饲料优先喂肉猪、普通宿主还是准备出售的资产。
- 三只小猪放在前位获得更早分数，还是放在后位读取更多已结清饲料。

**通过信号**

- 没有三只小猪时，饲料仍通过最低点和卖价产生价值。
- 有三只小猪时，玩家能看懂“先喂养、后兑现”的链条。
- 肉猪不是所有饲料都必须投喂的唯一正确目标。

**警报信号**

- 批发商越早买越永远正确，没有任何放弃理由。
- 所有投喂最终都只服务三只小猪，最低点和卖价变成无关收益。
- 玩家不需要调整顺序，任何摆位结果都接近相同。

### P2：猪猪的养成资产退出

**样本组件**

- 母猪
- 贪吃猪
- 已累计饲料的贸易猪
- 1 颗剩余实体骰

**纸面过程**

1. 母猪给全队投喂，贪吃猪响应其它骰获得饲料。
2. 主动卖出贸易猪，先结算自身卖价，再把累计饲料转移给剩余实体骰。
3. 确认饲料没有复制、丢失或再次触发贪吃猪递归。

**核心问题**

- 贸易猪的“卖价 + 饲料转移”是否让卖出永远没有代价。
- 随机转移是否足以形成决策，还是只形成结果波动。

### D1：恶魔的养市场—造来源—吞吃

**样本组件**

- 深渊召唤
- 恶魔骰子
- 魔蝠骰子
- 吞噬骰子
- 1 颗基础骰作为最低点成长目标

**纸面过程**

1. 进入市场时，深渊召唤固定制造 1 个贡品。
2. 玩家购买其它货架骰，让恶魔骰子提高本市场最低点。
3. 玩家决定购买哪些货架、保留哪些货架给魔蝠离场吞吃。
4. 离开市场时，魔蝠吞吃可用货架来源并强化当前最低点实体骰。
5. 吞噬骰子按吞吃事件获得金币。

**玩家决策**

- 货架骰是购买、留给魔蝠，还是刷新掉。
- 是否继续购买来提高本市场最低点。
- 是否为了强化当前最低点实体骰而保留低点目标。

**通过信号**

- 至少存在一次“想买，但留给魔蝠更好”的真实取舍。
- 玩家离场前能预判吞吃来源、目标和加值。
- 吞噬骰子提供继续操作的金币，但不能单独支付无限刷新。

**警报信号**

- 最优操作始终是买光便宜货，然后自动吞掉剩余货架。
- 吞吃目标永远是自动最低点，导致玩家不关心骰袋排序和目标规划。
- 小鬼或恶魔骰子把所有后续普通市场骰都永久抬成高质量必买品。

### D2：恶魔的长期市场成长

**样本组件**

- 小鬼
- 深渊召唤
- 魔蝠骰子

**纸面过程**

连续观察两个市场周期：小鬼在回合结束提高市场基础最低点，下一次深渊贡品和普通货架继承成长，再由魔蝠吞吃。

**核心问题**

- 长期市场成长是否仍服务恶魔决策，还是直接提升所有路线的普通市场质量。
- 玩家是否能感受到“先养市场，后吃来源”，而不是只看到所有骰面整体膨胀。

### T1：龟龟的吸附—压槽—时序放大

**样本顺序**

`呼朋唤友龟 -> 基础宿主 A -> 小小龟 -> 基础宿主 B`

**纸面过程**

1. 离开市场时，两颗龟分别吸附到紧邻后位非龟宿主。
2. 实体数量由 4 颗压缩为 2 颗宿主。
3. 宿主 A 先结算，呼朋唤友龟读取宿主 B 的真实吸附数量并强化宿主 B。
4. 宿主 B 后结算，小小龟再生成基础壳并获得长期成长。

**玩家决策**

- 哪颗龟放到哪个宿主前一格。
- 呼朋唤友宿主应该放在目标之前还是之后。
- 是集中强化一颗核心宿主，还是分散吸附制造更多读取对象。

**通过信号**

- 玩家明确感受到吸附腾出了槽位，但仍付出了购买、宿主选择和顺序成本。
- 呼朋唤友的早位与晚位产生可理解的不同结果。
- 宿主悬浮信息能说明本次成长来自哪只龟。

**警报信号**

- 可吸附龟因为最终不占槽而成为任何构筑的无脑购买项。
- 呼朋唤友龟越多、吸附越多就只做乘法成长，没有新的决策。
- 多层吸附后玩家无法从宿主界面判断点面、卖价和结算效果来源。

### T2：龟龟的首领复制

**样本顺序**

`首领龟 -> 金钱龟 -> 基础宿主 A -> 磁力龟 -> 基础宿主 B`

**纸面过程**

1. 金钱龟和磁力龟分别完成真实吸附。
2. 首领龟各复制一层同类壳，但复制壳不计卖价、不再触发首领监听。
3. 结算时分别观察金币、基础壳分发和首领自身复制壳效果。

**核心问题**

- 首领龟是否把所有后续龟龟都变成无脑高收益购买。
- 复制壳折叠后是否仍能看懂来源与数量。
- 槽位从 5 压到 3 后，六槽压力是否仍然成立。

### R1：海盗的刷新—交易—金币成长

**样本组件**

- 刷新海盗
- 海盗头领
- 练腿海盗
- 财宝海盗
- 海贼王

**纸面过程**

1. 连续执行两次付费刷新，记录实际费用和金币入账事件。
2. 购买一颗海盗骰，记录头领折价和练腿随机成长。
3. 若刷新净费用为负，记录返金币是否只产生一次财宝海盗事件。
4. 结算海贼王，读取当时钱包并转成自身长期成长。

**玩家决策**

- 金币用于继续刷新、购买海盗还是保留给海贼王。
- 是否为了财宝海盗的成长刻意制造更多小额金币事件。
- 海盗头领占一个槽位，是否值得换取后续交易折价。

**通过信号**

- 连续刷新越往后仍然越来越昂贵，不能在状态不变时自我供给。
- 买入、保留金币和追海贼王成长之间存在不同选择。
- 财宝海盗按金币事件而不是金币数量触发，反馈可追踪。

**警报信号**

- 刷新返金同时触发财宝成长，使继续刷新既赚钱又永久成长。
- 海盗头领折价后的买卖可以不消耗货架或其它有限资源而无限套利。
- 海贼王让“花钱构筑”退化为“永远不花钱”。

### R2：海盗的牺牲与离场抢货

**样本组件**

- 劫掠海盗
- 人口海盗
- 抢劫海盗
- 2 颗普通实体骰

**纸面过程**

1. 分别把劫掠海盗放在中间和边缘，比较牺牲邻居换金币的结果。
2. 主动卖出两颗骰，观察人口海盗补充基础骰和随机售价。
3. 离开市场时由抢劫海盗拿走货架骰，检查骰袋容量和新增离场队列。

**核心问题**

- 劫掠海盗是否真的产生“牺牲谁”的摆位决策。
- 人口海盗与抢劫海盗是否能让卖出—补充—再卖出持续自循环。
- 抢劫海盗是否在任何市场都稳定获得高于自身成本的额外资产。

## 五、双阵营与中立混合样本

### M1：黑帆—黑市连续刷新引擎

**样本组件**

- 黑帆魔蝠
- 黑市小鬼
- 刷新海盗
- 吞噬骰子
- 财宝海盗

**连续记录**

- 每次刷新前金币。
- 本次刷新基础费用、刷新海盗减费、黑市折扣和最终净费用。
- 黑帆吞吃来源数量与永久成长总量。
- 吞噬骰子入账次数。
- 负费用返金触发财宝海盗的次数。

**失败条件**

- 连续刷新在没有消耗骰袋资产、章节次数或其它不可恢复资源时，可以长期维持钱包不下降。
- 同一个刷新动作同时稳定提供净金币、永久面组成长和三张新货架，形成无停止理由的自供给循环。

### M2：血契船长满袋购买

**样本状态**

- 骰袋已满。
- 场上有血契船长、批发商、恶魔骰子、练腿海盗。
- 货架有一颗可购买实体骰。

**观察顺序**

1. 允许购买并支付真实买价。
2. 完整发布一次真实购买事件。
3. 批发商、恶魔骰子和练腿海盗各响应一次。
4. 新骰移出临时购买队列，货架清空，作为市场来源吞吃。

**失败条件**

- 同一颗被吞买入骰被记录为多次购买。
- 临时超出六槽的实体进入存档、排序或离场队列。
- 买入监听的总收益长期高于买价和吞吃价值，使满袋购买没有筛选成本。

### M3：补给货架与清仓兑现

**样本组件**

- 补给猪
- 清仓猪
- 抢劫海盗或黑帆魔蝠

**纸面选择**

- 买下已获得饲料的货架骰。
- 用抢劫海盗在离场时拿走它。
- 把它留给黑帆吞吃，但货架饲料不提高吞吃期望值。
- 清空货架，让清仓猪按空槽获得饲料。

**通过信号**

- 玩家能清楚知道货架饲料只对购入或抢到后的实体有价值，不会偷偷提高吞吃值。
- “买高价值货架”和“留空货架喂清仓猪”之间存在真实取舍。

### M4：饲料转壳

**样本组件**

- 驮粮龟
- 托底龟
- 养猪或母猪
- 2 颗非龟宿主

**观察重点**

- 驮粮龟吸附时按自身 `FeedValue` 每 2 点生成基础壳，来源实体消失后饲料不重复保留。
- 托底龟只在宿主自然点确实被饲料最低点托高时生壳。
- 新壳只改变长期面组，不回改本次锁定有效点数。

**失败条件**

- 同一份饲料既作为驮粮来源保留，又在宿主上再次计算。
- 托底龟只要宿主有饲料就触发，而没有检查是否真的托高。

### M5：共食魔龟的站位分流

**样本组件**

- 已吸附到宿主的共食魔龟
- 1 个恶魔吞吃来源
- 宿主右邻非龟实体骰

**观察重点**

- 宿主获得吞吃全脸成长。
- 每层共食壳向当前右邻分流一半加值，向下取整且最低 1。
- 分流不记录为吞吃，不再次触发共食或吞噬骰子。

**通过信号**

- 玩家可以通过选择宿主和右邻明确决定第二个成长目标。
- 没有右邻或右邻是龟时不触发，并能从反馈中看懂原因。

### M6：双标签与议会终局门槛

**样本组件**

- 两颗能够覆盖四个主体家族标签的双阵营实体骰。
- 光牙、王冠、盟约各一颗。

**并行比较口径**

1. 当前实现口径：双阵营实体同时计入两个家族的存在性、光牙候选和王冠最高点；盟约仍使用单一主家族代表值。
2. 待比较口径：同一实体在议会类终局统计中最多占一个家族席位，但普通家族监听仍同时识别双标签。

**核心问题**

- 两颗双阵营实体是否过早满足四族收束，让中立终局从高难度路线变成常规过渡路线。
- 同一颗骰同时作为两个家族最高点时，王冠是否获得了重复价值。
- 光牙、王冠和盟约使用不同身份口径时，玩家是否能够理解。

本矩阵不在纸面阶段直接修改当前实现；该样本用于后续决定 F020 是否需要“一颗实体最多占一个议会席位”的独立算法。

## 六、统一通过标准

| 维度 | 通过标准 | 警报信号 |
|---|---|---|
| 家族识别 | 玩家能用一句话复述家族在做什么 | 只能复述为“买到后加更多分” |
| 最小成型 | 3 至 5 颗核心组件即可第一次形成循环 | 必须凑齐唯一高阶骰才开始工作 |
| 市场决策 | 每个样本至少出现一次合理购买和一次合理放弃 | 看到家族骰就无脑购买 |
| 摆位决策 | 摆位能改变目标、时序或收益，而不是只影响动画顺序 | 任意排序收益接近相同 |
| 资源转换 | 家族资源至少能进入成长、分数、金币、卖价或槽位中的明确用途 | 资源只累计，没有可见兑现 |
| 经济安全 | 不存在状态可重复且钱包不下降的无限市场循环 | 刷新、买卖或吞吃可以自我支付并持续成长 |
| 槽位安全 | 龟龟压槽仍需要宿主、排序和购买成本 | 可吸附龟成为所有构筑无代价扩容 |
| 桥接必要性 | 双阵营骰至少需要一侧家族前置才能发挥主要价值 | 双阵营骰在任何阵容都是同档最优购买 |
| 反馈可读 | 一次主要链条能指出来源、目标和结果，重复项可折叠 | 玩家只能通过最终面组反推发生了什么 |
| 终局门槛 | 中立王冠仍需要真实多家族投入和槽位成本 | 两颗双阵营骰即可低成本替代四族构筑 |

## 七、首轮纸面结论

### 猪猪

内部结构最完整，已经具备入口、放大、临时兑现和退出。当前不缺新骰，优先验证批发商、肉猪和三只小猪是否把路线压成固定答案，以及贸易猪双收益是否让卖出没有代价。

### 恶魔

资源循环完整，但主要动作依赖自动最低点目标和自动货架来源。当前最可能的缺口不是收益不足，而是玩家对“吞谁、喂谁”的控制感不足。先验证控制感，再决定是否需要目标控制型骰子。

### 龟龟

类型身份最独特，吸附和六槽压缩足以独立成族。当前不应继续增加成长型龟龟；优先验证无上限吸附、首领复制、呼朋唤友和悬浮信息是否把强度与理解负担同时推高。

### 海盗

市场行为覆盖最广，已经拥有刷新、买入、卖出、牺牲、抢货、随机成长和金币终局。当前最需要验证的是经济安全，而不是再增加市场收益骰。

### 双阵营与中立

最高优先级风险是黑帆—黑市—刷新海盗的连续刷新引擎，其次是龟龟压槽与双标签降低王冠门槛。只要这三个问题没有验证，不进入第二波双阵营或中立骰设计。

## 八、后续执行顺序

当前纸面阶段结束后，建议按以下顺序使用现有原型验证；这不是本阶段的 Unity 工作范围：

1. `M1` 黑帆—黑市连续刷新引擎。
2. `M2` 血契船长满袋购买事件链。
3. `T1 / T2` 龟龟压槽、时序放大和首领复制。
4. `P1 / P2` 猪猪线性度与贸易猪退出。
5. `D1 / D2` 恶魔控制感与长期市场成长。
6. `R1 / R2` 海盗经济循环与离场队列。
7. `M3 / M4 / M5` 其它双阵营桥接。
8. `M6` 双标签与议会席位算法。

在这些样本得到实玩证据前，不新增家族骰子，不改当前数值，不提前决定 F020 席位算法。
