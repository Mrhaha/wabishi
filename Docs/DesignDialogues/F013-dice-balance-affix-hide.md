## 当前摘要

当前骰子方案已经经过 F008 金币收益族、F010 奇偶短规则骰、F011 大树长期成长接口骰等一轮扩张和改写，原有数值环境需要重新校准。同时，F005 词缀和改造道具已经接入原型但会增加平衡变量和界面负担，本阶段先临时隐藏相关设定、道具内容和界面显示，等核心骰子平衡稳定后再决定是否恢复。

## 原始用户意图

用户要求整理该问题，并准备 `F013` 生产包：因为整体骰子方案调整过一版，数值想要重新做平衡，同时暂时隐藏词缀设定、相关道具内容和 UI 显示。

## 问题框定

- 玩家问题：当前市场和骰子池信息量已经很高，如果词缀与改造道具继续暴露，玩家会同时面对类型、材质、面组、词缀和道具，早期决策负担过重。
- 生产问题：F008、F010、F011 接入后，关卡目标、市场价格、刷新权重、经济参数和骰子强度需要在同一基线上重新平衡。
- 当前系统张力：F005 词缀如果只隐藏 UI 但仍然生效，会产生不可见的分数和金币影响，破坏重平衡样本。
- 目标体验：玩家先只围绕骰子类型、面组、材质、金币和槽位做决策；生产侧先得到清晰的无词缀平衡基线。

## 讨论时间线

### 第 1 轮：F013 范围整理

- 用户方向：准备 `F013` 生产包，处理骰子方案调整后的数值重平衡，并暂时隐藏词缀设定、改造道具内容和 UI 显示。
- Codex 回应：将问题拆成“词缀系统软关闭”和“骰子/市场/关卡重平衡”两个主轴，先产出生产包，不直接改代码。
- 形成的洞察：隐藏词缀不能只做视觉隐藏，否则旧存档或已有词缀可能继续影响分数和金币；应以配置开关软关闭运行时效果，同时保留数据读取和存档兼容。
- 决策状态：可生成生产包。

### 第 2 轮：软关闭验证后推进无词缀基线

- 用户方向：已经验证软关闭，开始做整体数值平衡，包括经济系统和初始金币。
- Codex 执行：以 F013 生产包为范围，接入 F013-03 首轮无词缀数值候选，不恢复词缀，不改 F005 资源和存档结构。
- 已接入候选：`starting_gold = 8`，`stage_clear_base_gold = 3`，基础利息每 `6` 金 `+1` 且封顶 `4`，`market_test_random_refresh = 0`。
- 后续按用户要求把前三章从“过平”改成更明显梯度：第一章目标 `60 / 74 / 70 / 70`，第二章目标 `78 / 70 / 86 / 82`，第三章目标 `100 / 88 / 82 / 94`；按“6 基础公平骰 + 3 次出手 + 不出千”的单关口径，通过率从约 `95.43%` 降到 `51.62%`，每个小关仍高于 `50%`。第三章后目标改为平滑递增，第四章 `104 / 115 / 127 / 140`，第十章 `1141 / 1261 / 1393 / 1539`，不再从第三章 Boss 直接跳到 `180`。
- 验证状态：CSV 静态解析通过；Unity Play Mode、截图、旧存档构造验证和前 3 章实机平衡样本仍待执行。

### 第 3 轮：市场刷新可见度二次调整

- 用户方向：市场刷新中去除基础骰子类型，提高本次修改骰子的刷新概率，并且每次刷新尽量和上一次刷新不同。
- Codex 执行：把 `Basic` 从正式市场候选和 CSV 权重中双重下架，保留为初始骰袋和兜底兼容；提高 F008 金币收益族、F010 奇偶短规则骰和 F011 新大树接口骰在各自开放后的章节段权重；付费刷新时先用“当前同屏排除 + 上一批货架类型排除”抽取，候选不足时回退到正常池，避免刷空或破坏高阶保底。
- 设计理由：基础骰在当前 F013 起手和经济下不再承担市场填充价值，继续刷出会稀释玩家接触新路线的机会；近期接入的三组骰需要更高可见度才能进入实玩样本；相邻刷新避重可以减少“花钱刷新但看起来没变”的挫败感。
- 验证状态：代码、CSV 和主文档已静态同步；仍需 Unity Play Mode 中做正式市场连续刷新样本，确认 `Basic` 不出现、F008/F010/F011 更容易刷到、相邻刷新有可观察差异。

## 探索过的方案

| 方案 | 核心思路 | 优点 | 风险 | 状态 |
|---|---|---|---|---|
| 只隐藏 UI | 不显示词缀和道具，但保留结算效果 | 改动少 | 玩家看不到却仍受数值影响，不适合重平衡 | 已拒绝 |
| 删除 F005 内容 | 移除词缀、道具、资源和存档字段 | 平衡最干净 | 破坏兼容，后续恢复成本高 | 已后置 |
| 配置软关闭 | 用全局配置关闭市场入口、UI 和结算效果，保留存档与资源 | 风险低，便于恢复，适合重平衡 | 需要清楚验证所有入口都被关掉 | 倾向 |

## 已确认决策

- `F013` 生产包处理当前一轮骰子方案调整后的数值重平衡。
- `F013` 生产包同时处理词缀设定、改造道具内容和 UI 显示的临时隐藏。
- F013 首轮执行已经完成软关闭静态接入，并在用户验证软关闭后接入无词缀数值基线首轮候选。
- F013 市场刷新二次调整已静态接入：基础骰退出随机货架，F008/F010/F011 开放后权重提高，付费刷新优先避开上一批货架类型。

## 倾向方向

- 建议使用“软关闭”而不是删除：保留 `DiceData` 中已有前缀/后缀字段、三类改造道具持有数量、F005 资源和 CSV 文件，但在关闭状态下不显示、不出售、不使用、不触发。
- 建议用 `global.csv` 增加或等价实现一个运行时开关，例如 `affix_feature_enabled = 0`，作为隐藏期间的主控。
- 建议重平衡样本默认在词缀关闭状态下采集，避免不可见变量干扰。

## 已拒绝 / 已后置想法

- 不在 F013 内扩展新骰子。
- 不在 F013 内重新设计词缀系统。
- 不在 F013 内做定向打造、锁词缀、负面词缀或稀有度系统。
- 不删除 F005 资源、CSV 和存档字段。

## 待回答问题

- 词缀恢复时是否要保留旧存档中的词缀和道具数量，还是清空后重新开放。当前建议默认保留。
- 本轮重平衡已以关闭 `market_test_random_refresh` 后的正式市场规则为准。
- 市场刷新样本需要单独验证，不应只看 CSV 权重。

## 设计理由

当前最需要的是稳定的骰子核心体验和可读的市场决策。F005 词缀可以让长期经营更丰富，但它会增加独立变量和界面信息量；在 F008、F010、F011 刚接入后继续暴露，会让平衡判断和玩家反馈变脏。先软关闭词缀，再做无词缀基线重平衡，符合“先定范围、再产出生产包、再实现”的项目节奏。

## 职能备注

### 策划

需要冻结本轮平衡对象：关卡目标、经济参数、市场价格、刷新权重、材质权重、骰子强度观察点。不要把词缀强度纳入本轮核心平衡。

### 美术

不需要新增资源。三类改造道具图标保留为历史资源和后续恢复候选，不从工程中删除。

### 界面体验

需要定义隐藏后的市场和骰袋显示：不出现“改造道具”“加印石”“剥印石”“换印石”“词缀”“前 0/2 后 0/2”等内容，骰子信息回到类型、材质、面组和短规则。

### 程序

需要在 `Assets/Scripts/DiceKingDemo.cs` 中增加或等价实现词缀功能开关，并保护市场生成、购买、使用、显示、预览、真实结算、钱包收益和保存恢复。

## 风险登记

| 风险 | 重要原因 | 可能缓解方式 | 状态 |
|---|---|---|---|
| 只隐藏 UI 但词缀继续生效 | 会产生玩家不可见的分数和金币 | 软关闭结算效果和钱包收益 | 已记录 |
| `market_test_random_refresh = 1` 掩盖正式市场问题 | 当前测试随机本来就不刷改造道具，也可能掩盖正式市场权重和避重体验 | F013 验证必须覆盖正式市场关闭状态和连续刷新样本 | 已处理，当前正式基线为 `0` |
| 1000 金起手污染平衡 | `starting_gold = 1000` 是旧测试配置 | F013 首轮候选改为 `starting_gold = 8` | 已处理，待实机验收 |
| 旧存档带词缀 | 可能恢复后突然生效 | 隐藏期保留但不生效，并在恢复包中另行决定 | 已记录 |

## 候选验收

- 关闭状态下，市场不会刷出改造道具。
- 关闭状态下，市场不显示改造道具栏和目标选择流程。
- 关闭状态下，骰袋、商品卡、结算短标签和日志不显示词缀内容。
- 关闭状态下，已有词缀不提供分数或金币。
- 关闭状态下，保存和读取仍不报错，旧词缀字段被保留。
- 重平衡输出至少覆盖关卡目标、起手金币、市场价格、刷新费用、权重和关键骰子观察点。
- 市场连续刷新样本中不出现基础骰，并能更频繁看到 F008 / F010 / F011；相邻付费刷新优先不同，候选不足回退时需记录。
