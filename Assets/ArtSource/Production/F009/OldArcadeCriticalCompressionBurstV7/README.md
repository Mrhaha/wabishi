# F009 V7-R1 / V7-R2 五档正常速结算动态样片

## 目的

V7 直接回应 V6B-R6 最高档被退回的问题：不继续给画面叠加常亮特效，而是先主动压缩亮度、运动和声场，再用一次局部击穿把能量释放到整台机柜。V7-R1 把该结构校准到游戏 `1.0x`，并已由用户确认冻结为最高暴击母版。

V7-R2 在不修改 V7-R1 的前提下派生“未达到 / 过关 / 超过 / 远超过”四档。五档不做同一爆炸的亮度缩放，而是分别使用反向泄压、单拍闭环、双拍继电、可控整机过载和真空击穿五种动作语义。

## V7-R1 游戏正常速校准

V7 的 MP4 没有在编码阶段倍速，但 7 次贡献事件平均约 `0.30 秒/击`；当前游戏 `1.0x` 的普通结算事件为 `0.55 秒/击`，所以 V7 的结算主体观感接近游戏 `1.8x`。

V7-R1 采用分段时间映射：

- `0–1.42s` 投骰与停轮保持 V7 原速。
- 从结算开始到爆发释放点按 `2×` 拉长，对齐游戏 `1.0x` 的逐击节拍。
- 爆炸、整机后坐、CRT 击穿和余震保持 V7 原速，不做慢动作。
- 总长从 `5.2s` 增加到 `7.9s`，爆发点从约 `4.17s` 移到约 `6.86s`。
- 完整声场下潜约 `0.62s`，其中近静默核心约 `0.30s`。

用户已确认 V7-R1“这个版本没问题”，因此 `f009_v7_r1_critical_normal_speed_preview_20260722.mp4` 直接作为第 05 档引用，不在 V7-R2 中重新渲染或替换。

## V7-R2 五档派生

五档共用 `7.9 秒 / 60 fps / 1.0x` 的评审标尺，逐击贡献节奏一致；差异集中在最后一笔入账后的动作方向、拍数、响应范围、声音形态和释放规模。

01–04 档的继电追灯与 CRT 声音包络按时间映射逐个换算起止点，不只移动第一个声音事件；因此正常速下的听觉扫动与画面扫光保持同一展开速度。第 05 档音轨继续使用已通过的 V7-R1 文件。

| 档位 | 终局语义 | 允许的主要响应 | 明确禁用 |
|---|---|---|---|
| 01 未达到 | 反向泄压 | 积分塔回落、灯路逆向回收、低频下坠、失败稳定态 | 正向庆祝、CRT 撕裂、整机爆发 |
| 02 过关 | 单拍闭环 | 目标锁扣、一次正向暖脉冲、局部稳定亮起 | 第二拍、欠压、整机回坐 |
| 03 超过 | 双拍继电 | 正反两次灯路接力、终端响应、一次轻微 CRT 抖动 | 持续过载、黑场、击穿 |
| 04 远超过 | 可控整机过载 | 持续机柜负载、双向扫光、明显回坐、受控 CRT 扰动 | 近黑真空、全屏白洗、爆心射线、碎片外抛 |
| 05 最高暴击 | 真空击穿 | V7-R1 的锁死、吸光、近静默、单次全屏击穿、强后坐与余震 | 不下放给前四档 |

## 五阶段导演节奏

| 阶段 | 时间窗口 | 画面动作 | 声音动作 |
| --- | --- | --- | --- |
| 01 锁死 | 约 `3.42–3.62s` | 最后一笔贡献落塔后，整机短暂钳住，停止继续抬亮 | 继电器闭锁，连续节奏开始减速 |
| 02 吸光 | 约 `3.62–3.86s` | 非核心区域逐层熄灭，亮度和视线被拉回积分塔 | 低频压力持续堆高，高频逐渐收窄 |
| 03 真空 | 约 `3.86–4.14s` | 画面进入近黑，只保留积分塔高温核心和收束环 | 主声场压低，保留极弱引线，避免误读成卡死 |
| 04 击穿 | 约 `4.14–4.28s` | 核心先闪断，再以水平冲击带、主射线、整机后坐和二段复电贯穿全屏 | 低频冲击、机柜金属撞击和 CRT 破裂声同步落下 |
| 05 余震 | 约 `4.28–4.48s` | 两层冲击波扩散，亮度快速回落，整机停止后坐 | 爆点进入长尾衰减，补一次较弱余震 |

结果文字从约 `4.48s` 才开始出现，避免“本关通过”抢在爆发之前泄气；`4.66s` 后进入稳定结果态。

## 最高档冻结约束

- V7 原版总长为 `5.2s`；已通过的 V7-R1 正常速母版总长为 `7.9s`。两者均为 `1280×720`、`60 fps`。
- 六颗骰子始终留在原槽位；后坐只作用于整机父节点。
- 积分塔仍是爆发起点，但冲击会扩散到顶部 HUD、骰盘、左右机柜、右侧终端和整屏边框。
- 峰值只保留一个主爆点，不用多个同权特效互相争抢。
- 结果文案保持二值，不显示档位名、百分比、倍率或新增奖励。
- 当前目录是仅源文件方向样片；最高档通过不等于批准运行时接入。五档运行方案未获单独授权前，不复制到 `Assets/Resources/Art/`，不修改运行时代码。

## 构建

在 PowerShell 中执行：

```powershell
& .\build_preview.ps1
& .\build_normal_speed_preview.ps1
& .\build_five_tier_normal_speed_previews.ps1
```

首个脚本重建 312 帧 V7；第二个脚本重建 474 帧 V7-R1，并生成同屏节奏对照、顺序带声对照、关键帧和审查板；第三个脚本重建 01–04 四条 474 帧正常速样片，直接复用已通过的 V7-R1 作为 05 档，再生成五档同步 / 顺序对比和两张总览板。三个脚本都会逐支完整解码并清理临时 PNG 帧。

## 本轮只评审四件事

1. 01 未达到的反向泄压与 02 过关的正向闭环，能否在不看外部档位标题时被感知为两种结果。
2. 03 超过的双拍继电是否显著强于 02，又没有提前变成整机事故。
3. 04 远超过是否足够兴奋，同时仍与 05 最高暴击保留“可控过载 → 真空击穿”的质变。
4. 前四档在游戏 `1.0x` 下是否耐看；统一 7.9 秒只用于本轮同步比较，后续运行时可在不改变逐击节拍的前提下收短普通档终局停留。
