# F009 V6B-R5 五档结算样片自审

日期：`2026-07-22`

状态：`R4 暴击母版已获用户确认；R5 五档源区自审通过，待用户横向视觉 / 听觉验收，未接入 Unity`

## 自审结论

五档均继承 V6B-R4 的完整链路与玻璃直吸收语法，只在整手结果层调整积分塔闭合、灯格压力、回弹、余辉、机柜负载和声音。视觉峰值形成 `未闭合残余 → 琥珀稳定 → 白热点双回弹 → 持续暖白硬核 → 欠压后整塔过载` 的递进；游戏内结果仍只有“通过 / 未过”，没有把隐藏档位或阈值暴露给玩家。

结论：`通过源区五档自审，建议提交用户横向验收；用户确认前不接入 Unity。`

## 连续性检查

- 五个单档和同步总览均为 `60fps / 4.80s / 288` 帧；顺序原声总览为 `60fps / 24.00s / 1440` 帧。
- 所有视频均已完整解码；五个 `_frames_*` 缓存目录内正式帧已清理为 `0`。
- `-80dB / 50ms` 精确级重复检测中的较长片段均对应计划内的停骰标点、结果驻留或顺序总览切片边界；单档没有异常掉帧或编码造成的长停顿。
- 抽查结算后半段六个时点，确认积分包、直接玻璃接触、塔内上行与结果切换连续；没有入口轮廓、空白断层、骰子跳位或突然重置。
- 同步总览只用于视觉对齐，故意不混合五条音轨；顺序总览保留每档原始临时声轨。

## 仍需用户判断

1. 五档是否在不读取数字和外部评审标签时仍能自然感到逐级变强。
2. “过关”与“超过”的单次稳定 / 二次回弹差异是否足够，但没有形成新的顿挫。
3. “远超过”是否已足够兴奋，同时仍给“暴击”的欠压重击保留明确上限。
4. 五档全部确认后，是否授权创建 Unity 接入交接；该确认不自动批准当前临时声音作为正式音频。
