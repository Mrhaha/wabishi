# F009 V6 在线参考结算动画方向评审

状态：V6B 结构已获用户认可；V6B-R1 老街机特效评审稿待确认，未接入 Unity。

## 评审目标

在不移动骰体、不绘制骰子互联、不依赖逐颗精确加分数字的前提下，让“左到右主结算 + 跨骰再次触发 + 回环压缩 + 五级整手结果”形成一条连续、可归因且有高潮的结算演出。

三张评审板使用同一压力案例：第 2 槽是主来源，触发第 1 槽再次生效，随后第 4 槽产生一笔更重的即时贡献，局部队列结清。只有实际产生即时分数的骰子向左侧积分塔发送积分光晕；只改变未来状态的响应不入塔。

## 共同表现语法

- 六颗骰子全程固定在原槽；允许骰面与外框做低幅聚焦，不允许骰体跳、飞、放大或旋转。
- 主来源：紧贴骰面的实心暖琥珀聚焦封印。
- 次级触发：空心双环或短促暖白回声；被点名骰不抢主游标身份。
- 即时分数：由实际贡献骰析出简化积分光晕，直接进入左侧积分塔。
- 贡献大小：只由光晕直径、环宽、核心亮度、尾迹厚度和入塔深度表达。
- 回环：首次因果完整显示；同类重复触发压缩成塔内密集环、刻痕或短音阶，不重播完整路径。
- 整段结算共用一个不归零的积分塔压力包络；上一击余辉尚未结束时，下一事件已经预备。
- 精确分数只在低关注层更新，最终稳定后恢复事实可读性；过程不出现 `+N`、倍率、档位名或比例。

## V6A：点名回声 / Echo Callout

评审图：`f009_v6a_echo_callout_review_20260721.png`

- 目标：最清楚地回答“谁是主来源、谁被再次触发、谁真正贡献了分数”。
- 节拍：主来源点名约 `70ms`；每次次级回声约 `90–110ms`；积分光晕入塔约 `80–100ms`；局部队列结清回弹约 `150–190ms`。
- 亮点：因果清晰，学习成本最低；小、中、重贡献一眼可排序。
- 风险：如果终局塔击不再增加额外声画层，惊艳上限低于 V6B。
- 适用：新手期、规则复杂骰较多、需要优先教会玩家读连锁的版本。

## V6B：塔芯连击 / Tower Core Combo（推荐）

评审图：`f009_v6b_tower_core_combo_review_20260721.png`

- 目标：把所有爽点聚焦到积分塔，形成“不断吸收—共振加压—局部过载—一次释放”的持续上扬曲线。
- 节拍：单次入塔约 `80–100ms`，相邻事件交叠 `50–70ms`；三节点局部链约 `0.55–0.70s`；队列闭合过载约 `180–240ms`。
- 亮点：三版中最有冲击力，同时没有让骰子本体抢戏；塔芯亮度、波形密度和机壳应力能够承担隐藏五级结果。
- 回环处理：重复触发只在塔芯增加更密的压缩环和更高的音高，不再逐次拉长光晕飞行。
- 风险：塔芯常态不能过亮；如果每手都接近白芯，会迅速失去等级差。
- 适用：作为主方案进入短动效样片，优先验证强手与暴击。

## V6B-R1：老街机塔芯连击 / Old Arcade Tower Core Combo（当前评审稿）

评审图：`f009_v6b_r1_old_arcade_tower_core_review_20260721.png`

- 保留内容：V6B 的固定六骰、主 / 次触发点名、所有即时贡献持续压入左侧积分塔、局部队列闭合后只释放一次主峰。
- 回炉目标：去除平滑能量涡、彗星光球、等离子光柱和泛用科幻 Bloom，让每个亮点都能对应旧机台内部真实存在的灯泡、灯排、继电器或 CRT 现象。
- 来源骰：实心 / 空心聚焦环改为紧贴骰框的离散追灯；主来源是一圈连续追灯，次级回声是更薄、更断续的双层追灯。
- 贡献载体：积分光晕改为 `6–16` 枚方形点阵灯珠组成的紧凑积分包；重量只通过灯珠数量、密度和极短温白灯丝核心连续变化，不做球体或长尾迹。
- 积分塔：平滑压缩环改为烟熏玻璃后的矩形步进灯排；每次吸收让灯排机械式爬升，并保留两至三条衰减扫描余辉作为局部队列记忆。
- 暴击：先局部欠压一帧，再由继电器吸合触发整列灯排琥珀 / 温白过驱、顶部目标灯机械咬合、塔内单条 CRT 同步撕裂与少量触点火花；冲击只限积分塔，不横扫骰列。
- 审核结果：旧街机材质贴合、六骰原位、无骰子互联、无逐颗数值、积分塔主焦点和四段强弱递进均通过静态检查；动态中的追灯方向、灯排爬升连续性、欠压帧时长、声音和双分辨率仍未验证。

## V6C：连锁刻印压机 / Chain Imprint Press

评审图：`f009_v6c_chain_imprint_press_review_20260721.png`

- 目标：让复杂构筑留下可读但短暂的“链深记忆”，并用老式机械压印形成独特收束。
- 节拍：每枚刻印约 `70–90ms`；塔内最多保留三层；同类循环合并为相邻刻痕；最终对齐压入约 `210–260ms`。
- 亮点：最有系统辨识度，能让玩家感到这一手不是单次大分，而是多层响应共同成立。
- 风险：信息密度最高；若刻印数量、符号细节或停留时间失控，会重新变成读图标而不是感受爆发。
- 适用：高阶骰、Boss 规则或需要重点展示连锁深度的场合；不建议所有普通手都完整使用。

## 隐藏五级结果的共同落点

五档仍只作为内部导演参数，并在档内连续插值，不出现硬切成绩卡。

| 内部等级 | 积分塔状态 | 收束方式 |
|---|---|---|
| 未达到 | 压力不足、低位余辉快速衰减 | 无庆祝，短促熄落 |
| 过关 | 塔芯稳定、目标位机械咬合 | 一次干净锁定 |
| 超过 | 环宽、填充速度与低频承击提升 | 一次明显回弹 |
| 远超过 | 塔内压缩环叠加、机壳应力和余响更长 | 局部过载但不见白芯 |
| 暴击 | 极短预压后出现暖白硬核、厚冲击环与局部 CRT 扭曲 | 只在积分塔附近猛烈释放一次 |

## 当前推荐与下一门禁

优先采用 V6B-R1。V6B 的结构已经获得用户认可，当前只评审老街机特效材质是否足够贴合；不再回退到骰子互联、实体骰飞行或逐颗数值弹跳。

用户确认 V6B-R1 后再制作 `1.2–1.5s` 的同机位短动效样片，至少覆盖：普通三节点链、重复同类回环、目标越线、最终暴击四种压力段。动态通过前不生产运行时切图、不登记加载 key、不修改 `Assets/Resources/Art/` 或 Unity 代码。
