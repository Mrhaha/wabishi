# F009 V6B-R6 整机回路 + CRT 五档动态样片

## 目的

本轮不是替换积分塔，而是在 R5 积分塔反馈之上增加第二层、第三层冲击：

- 约 75%：整机负载过载。让顶部 HUD、中央骰盘框、底部操作区、左右机柜和右侧终端共同响应。
- 约 25%：CRT 屏幕击穿。只在后三档逐级加入同步撕裂、边缘色散、欠压和余辉。

样片继续使用当前旧街机主界面、同一机位、同一组骰子、同一结算顺序和同一 4.8 秒时长，方便直接与 R5 对比。

## 五档导演差异

档位名称只用于评审文件，不在游戏结果界面展示。

| 评审档位 | 整机反馈 | CRT 反馈 | 峰值观感 |
| --- | --- | --- | --- |
| 01 未达到 | 回路从右向左卸载，暖光衰减 | 无 | 机器泄压、结果偏冷 |
| 02 过关 | 单次骰盘底灯扫过，机框稳定闭合 | 无 | 明确过关，但不抢后续档位 |
| 03 超过 | 两次继电波传遍机框，终端轻微回弹 | 轻微单条同步偏移 | 整机开始有负载感 |
| 04 远超过 | 顶栏、骰盘、左右机柜和终端持续高负载 | 两段同步撕裂与短余辉 | 整台机器被推高一档 |
| 05 暴击 | 全机短暂欠压，二段复电，继电灯追逐和 3–4 px 整机后坐 | 多条撕裂、边缘色散、磷光残留 | 先吸气再爆发，峰值最强 |

## 冻结约束

- 六颗骰子始终留在原槽位；只有整机后坐，骰子不单独飞出。
- 贡献事件仍是 7 次，包含第 1 槽的二次贡献。
- 积分能量仍直接进入左侧积分塔，不增加骰子之间的连线。
- 不显示档位名、百分比、倍率、奖励或每颗骰子的巨大数字。
- 结果文案保持二值：`本关未过` / `本关通过`。
- 本目录是源文件评审包；未获确认前不复制到 `Assets/Resources/Art/`，也不修改玩法代码。

## 时间轴

- `0.00–1.42s`：共享的投掷、停轮和锁定。
- `1.42–3.55s`：7 次顺序贡献；每次撞塔同时让整机回路积累负载。
- `3.55–4.08s`：五档峰值差异窗口。
- `4.08–4.80s`：结果文案出现，高档位保留短暂余辉后冷却。

重点评审帧：

- 峰值：第 229 帧，约 `3.817s`。
- 结果：第 264 帧，约 `4.400s`。
- 暴击欠压发生在约 `3.61–3.76s`，必须通过视频判断，不能只看峰值静帧。

## 构建

在 PowerShell 中执行：

```powershell
& .\build_preview.ps1
```

脚本会以 60 fps 渲染每档 288 帧，生成临时 WAV、编码 H.264/AAC MP4、抽取审查帧、生成对比视频和审查板，并逐个执行解码验证。

## 本轮希望确认

1. 第 02 档是否足够明确，又没有提前透支冲击。
2. 第 03、04 档是否能在不看积分塔时也被区分。
3. 第 05 档“欠压 → 复电 → CRT 撕裂”的呼吸是否比持续加亮更有重量。
4. CRT 撕裂是否仍保持在旧街机世界观内，没有变成现代霓虹故障风。
5. 峰值是否影响骰面、当前分数和右侧结果区的可读性。
