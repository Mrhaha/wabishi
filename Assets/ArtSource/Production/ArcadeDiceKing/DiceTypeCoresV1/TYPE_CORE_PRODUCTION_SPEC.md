# 骰子类型芯 V1 生产规格

状态：可批量生产  
确认日期：2026-07-17  
适用范围：基础骰、四个主体家族、中立骰，共 37 枚；双家族 9 枚另立后续批次。

## 一、目标

- 家族外壳只表达中立、猪猪、恶魔、龟龟、海盗。
- 中央类型芯只表达具体骰子类型，静态 `Ready` 必须在没有点数、文字和悬浮详情时可辨认。
- 结果锁定后，数字接管中心；同一类型芯缩入数字下方固定小槽，不另做结果专用图案。
- 低频身份动作只帮助记忆，不承担首次识别；玩家关闭动态后仍能依靠静态剪影区分。

## 二、统一资源合同

- 源图尺寸：`512×512`、正交正视、透明背景、`RGBA PNG`。
- 运行显示：映射到 `128×128` 物理槽内的中央内容框，约 `80×76` 像素。
- 构图：主体占源图宽高约 `66%–76%`，四周保留安全区；只允许 2–3 个主形体。
- 材质：深海军蓝或炭黑压印主体、旧化塑料与少量黄铜边；琥珀为默认活动灯，青色只用于触发、传导或复制。
- 禁止：完整透视骰子、点数、伪结果、文字、字母、乘号、箭头、角色脸、外置道具、微缩场景、赌场、王室珠宝、军工武器和复杂背景。
- 静态资源：`arcade_type_core_<名称>.png`。
- 活动遮罩：`arcade_type_core_<名称>_activity.png`，只保留需要发光或轻微位移的零件；不制作 37 套逐帧序列。
- 结果状态：复用静态资源，等比缩小后置于数字下方；活动遮罩停止播放。
- 候选运行目录：`Assets/Resources/Art/DiceTypes/`。
- 本阶段生产目录：`Assets/ArtSource/Production/ArcadeDiceKing/DiceTypeCoresV1/`。

## 三、共用动作词

| 动作词 | 视觉行为 | 建议时长 |
|---|---|---:|
| 稳态 | 单灯缓慢呼吸，不发生位移 | `0.45s` |
| 双脉冲 | 两个端点按先后或同步点亮 | `0.40s` |
| 传导 | 灯沿既有机械通路移动一次 | `0.45s` |
| 咔哒 | 零件移动一个机械刻度并回位 | `0.32s` |
| 吸入 | 外缘信号向中心收束 | `0.42s` |
| 外送 | 中心信号向外缘分发 | `0.42s` |
| 回声 | 主体先亮，副本延迟出现或点亮 | `0.48s` |

常态每 `3.5–5.5s` 才允许一次身份动作；同屏最多 2 枚同时活动。鼠标输入、拖拽、投骰、停转、结算和结果阶段均暂停身份待机动作。

## 五、首轮生产与验收

1. 先生成五张分组生产板：基础＋中立、猪猪、恶魔＋贡品、龟龟、海盗。
2. 每张生产板同时显示大号静态类型芯和小号结果芯；活动只表现一个峰值，不生成完整动画。
3. 分组生产板通过后，再逐枚生成透明静态资源和活动遮罩。
4. 逐枚资源必须在 `80×76` 与结果小槽两档做最近邻或等比缩小检查；不能只看 `512×512` 原图。
5. 双家族批次复用同一视觉语法，但外壳由两个家族壳拼接，类型芯不得简单叠加两个现有单家族图标。
