## 当前推荐版表达

- 卖出键固定显示“卖出”，保留浅米色通电材质、深墨实心字和凸起状态。
- 三枚购买键固定显示“购买”，整键压暗、降饱和、取消暖色亮边并轻微内陷。
- 骰袋补满为 `6 / 6`，普通商品处于受阻状态。
- 点击受阻购买键时，在该键附近短暂显示“骰袋已满”，不把原因写进按钮。
- 提示使用烟熏黑玻璃、克制锈橙状态边和琥珀点阵字；体积小、局部、非模态。

## 动效补充

- 提示由触发按钮向上飞入，不从屏幕边缘进入；飞入 `0.14–0.18s`，在 `1280×720` 下移动约 `24–32 px`。
- 飞入期间透明度由零到完全可见，缩放由 `94%` 到 `103%` 后收至 `100%`，只做一次轻微到位回弹。
- 完整停留 `0.45–0.55s`，随后用 `0.22–0.28s` 再上浮 `8–12 px` 并淡出；总时长约 `0.85–1.00s`。
- 动画不拦截输入；同一提示重复触发时替换并从飞入段重新播放，不堆叠。

## 按钮统一规则

- 同一按钮家族的同一状态必须使用相同表现；位置、商品和受阻原因不得改变状态外观。
- 浅色实体动作键统一使用 `可用常态 / 悬停或焦点 / 按下 / 受阻常态 / 受阻点击` 五种表现。
- 可用态统一保持浅米色原纹理、凸起和深墨实心字；悬停只增加统一暖亮外沿；按下统一下移并短促压暗。
- 受阻态统一整键压暗、降饱和、取消边光并轻微内陷；点击只做受阻回弹和提示，不执行动作。
- “购买”“卖出”“离开市场”等动作名固定，成功结果与受阻原因只进入统一短时提示，不替换按钮文案。
- 三枚购买键的状态只读取各自真实购买资格；血契船长等特殊规则仍可在满袋时呈现可用态。
