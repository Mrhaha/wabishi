## 按钮统一合同

- 浅色实体动作键统一覆盖“卖出”“购买”“离开市场”；相同家族、相同状态必须使用相同亮度、饱和度、凸凹、边光、文字色、位移和反馈节奏。
- `可用`：原浅米色纹理、正常饱和度、凸起、深墨实心字。
- `悬停 / 焦点`：文字与底色不变，只增加克制暖亮外沿。
- `按下`：键帽与文字同步下移 `1–2 px` 并短促压暗。
- `受阻`：整键压暗、降饱和、取消通电边光并轻微内陷。
- `受阻点击`：不执行交易；只播放一次短促拒绝回弹，并在全画面中间偏上重播原因 Tip。
- 动作文字固定：卖出键只显示“卖出”，商品键只显示“购买”，离场键只显示“离开市场”。价格、结果和阻断原因不进入按钮文字。

## Tip 统一合同

- 市场动作 Tip 固定出现在全画面中间偏上，`1280×720` 虚拟画布最终面板中心约为 `(640, 200)`；从终点下方飞入，不再跟随触发按钮。成功和受阻共用同一运动合同，只切换文案与语义色。
- 飞入 `0.14–0.18s`：上移 `24–32 px`，Alpha `0→1`，缩放 `94%→103%→100%`。
- 完整停留 `0.45–0.55s`；淡出 `0.22–0.28s`，继续上浮 `8–12 px` 并 Alpha `1→0`；总长 `0.85–1.00s`。
- 不拦截输入；同一替换键重复触发时替换并从飞入段重播，不堆叠。

## 混合资格门禁

- 视觉状态只能读取真实动作资格，不能由“骰袋已满”自行推导全部商品受阻。
- 六槽已满时，普通实体骰商品进入统一受阻态；血契船长若仍满足现有购买与吞吃规则，则保持统一可用态。

- 生成方式：内置图像生成编辑，以现有市场截图和既有午夜旧机厅补给柜视觉为基准。
