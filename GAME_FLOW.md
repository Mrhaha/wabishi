# 游戏流程文档

本文档记录 `骰子王` 当前确认的游戏流程。后续如果流程规则被确认或修改，应同步更新本文档，保证新对话开始前可以先读懂完整游戏循环。

## 新会话读取规则

同项目的新会话开始时应先读取：

1. `AGENTS.md`
2. `PROJECT_CONTEXT.md`
3. `GAME_FLOW.md`
4. 涉及骰子、计分、市场或骰子效果时，再读取 `DICE_ARCHETYPES.md`

当前代码事实以 `Assets/Scripts/DiceKingDemo.cs` 为准；如果文档和代码不一致，先核对代码，再同步文档。

## 流程总览

1. 启动游戏，进入通用主界面。
2. 玩家选择开始新游戏、继续游戏或设置。
3. 开始新游戏或继续游戏确认后，先播放短促的“六节点启动—CRT 全屏信号覆盖”过渡；全覆盖帧负责隐藏开始界面到独立主游戏场景的切换。首次新游戏如果尚未看过开场过场，可在全覆盖后插入原有黑幕睁眼段；继续游戏和已看过开场的新游戏直接进入竞技台组装段。
4. 过渡及首次开场结束后直接进入默认初始骰袋，不再出现开局流派选择；主游戏使用独立完整 `16:9` 画布，不保留开始界面的窗雨、吊灯或完整机壳。
5. 进入小关后，玩家可在 `Ready` 阶段拖拽调整六槽顺序，然后按 `Space` 开始投骰。
6. 每个小关初始有 1 次出手机会，没有出千机会；生命续投只在本手累计结算未达标并完成扣命后产生。
7. 第一次 `Space` 后开始基础旋转。加力窗口作为可选表现配置保留，首版目标值为 `0`，因此不要求或提示继续敲击；未来配置大于 `0` 时，追加输入也只影响表现，不影响骰面概率。
8. 骰子在原槽位内进入左到右停转显点；结果锁定后不等待第二次输入，自动交接真实结算。
9. 结算按当前槽位左到右推进。每个当前槽的点数与局部连锁聚合后，当前骰保持原槽并析出一枚小型积分光晕进入左侧本关得分；不提供出千改点，也不绘制骰子之间的传导关系。
10. 骰子类型效果按 `DICE_ARCHETYPES.md` 的结算边界处理；V0.2 首版按“左到右主游标 + 当前槽位局部连锁队列”推进。
11. 完成目标分后进入关卡结算。
12. 小关通关时，出手内金币来源已经直接进入钱包；再发放通关固定金币，并按当前钱包金币结算基础利息和复利骰追加金币。

## 小关状态机

小关建议按以下状态组织：

1. `EnterStage`：进入小关，初始化目标分、本关累计分、一次初始出手和本关临时状态；沿用整轮剩余生命，不建立用于失败回滚的小关开始快照。
2. `ReadyToRoll`：等待玩家按 `Space`。
3. `Rolling`：玩家第一次按 `Space` 起转；加力窗口保留为可选表现配置，首版目标值为 `0`，不等待追加输入。
4. `RevealResult`：基础旋转结束后从左到右停转并锁定真实结果；首颗锁点可以与后排制动重叠进入结算表现。
5. `ResultDecision`：若代码继续保留该阶段，只作为短暂、不可交互的结果交接态；不等待确认结算，不提供出千选项，自动进入 `Settle`。
6. `Settle`：根据最终骰面、骰子类型效果、饲料下限、临时小骰和关卡规则直计得分。
7. `StageClear`：达成目标分，发放金币奖励。
8. `StageFailed`：本手已经完整结算且本关累计分仍未达目标；当前实现立即扣除生命，扣后仍大于 `0` 则写入稳定续投检查点，保留累计分与已提交效果，经水平扫描清理每手状态并回到可排序的 `ReadyToRoll`；扣到 `0` 则清档并进入本轮结束。
9. `InterStageMarket`：普通小关之间进入骰子市场，玩家可以买骰、卖骰、刷新货架或离开；F013 默认软关闭时改造道具和词缀入口隐藏。
10. `ChapterShop`：Boss 后进入章节市场；当前原型复用普通市场 UI、货架权重和买卖规则，刷新费用可单独配置；离开后如果配置表还有后续关卡则进入下一小关。

## 结算规则

确认规则：

- 每小关初始只有一次真实出手。每手按当前骰袋槽位从左到右完整结算，并把本次总分加入本关累计分后与 `target_score` 比较；只有未达标扣命且扣后仍有生命时才追加下一手。
- 基础单骰分等于最终有效点数；1 点等于 1 分，0 与负点数按真实值计分，不使用旧倍率尺度。
- 当前不计算全局牌型倍率，也不生成牌型规则分。四家族、中立、双阵营效果只修改有效点数、单骰分、长期面组、钱包或市场状态。
- 结算使用“左到右主游标 + 当前槽位局部连锁队列”：当前槽自身效果先执行，其引发的响应全部清空后才进入下一槽；同一时点的多个响应按骰袋槽位从左到右入队。
- 双阵营实体同时提供两个主体家族身份；同一次光牙结算中，同一双阵营实体最多被选中一次。龟骰吸附成为壳后不再提供家族身份。
- 长期面组允许低于 1。最高面、最低面、指定当前面、饲料最低点和吞吃来源均读取真实面组；吞吃只对最终平均转移量使用最低 1 的保底。
- 饲料最低点为 `真实最低面 + FeedValue`。自然点低于该下限时才被托高；托底龟只在确实发生托高时生成基础壳。
- 真实吸附把来源骰的期望转移量加到宿主全部面，并按吸附顺序保存壳效果。双倍龟按字面把当前最小面乘 2，负最小面会变得更低。
- 金币事件按当前结算顺序直接进入钱包；较早入账可被后续槽位读取，后发生的金币不回溯修改已结算效果。
- 完整逐骰效果、双阵营身份和递归边界统一见 `DICE_ARCHETYPES.md`，本文件不维护第二份逐骰规则。

当前遭遇规则：

- `None`：不追加规则修正。
- `OddLedger`：最终有效点数为奇数时单骰分 `+1`；负奇数按绝对值判奇数。
- `LowFog`：最终有效点数恰为 1 或 2 时单骰分 `-1`；0 与负点不受该规则重复惩罚。
- `DoubleJudge`：偶数单骰分 `+1`，奇数单骰分 `-1`；不附带顺子或牌型奖励。
- `HandAudit` 不在当前章节表中使用。

基础积分公式：

```text
本骰贡献 = 最终有效点数 + 当前骰类型效果 + 当前遭遇规则修正
本次得分 = 六颗实体骰及其当前壳效果的贡献总和
小关累计分 = 旧小关累计分 + 本次得分
小关累计分 >= target_score 时通关
```

通关经济：

- 新游戏初始金币 `starting_gold = 8`。
- 通关先发放 `stage_clear_base_gold = 3`，再按当时钱包计算基础利息。
- 每持有 `interest_gold_step = 6` 金产生 1 份利息，每份 `interest_gold_per_step = 1` 金，每关最多 `interest_gold_cap = 4` 金。
- 当前不按剩余出手或出千发金币，两个奖励参数均为 0。
- 超出目标分不提供额外金币；未达标也不发安慰金币。

```text
通关固定金币 = stage_clear_base_gold
计息前金币 = 当前钱包金币 + 通关固定金币
基础利息 = floor(计息前金币 / interest_gold_step) * interest_gold_per_step
利息金币 = interest_gold_cap > 0 ? min(基础利息, interest_gold_cap) : 基础利息
钱包金币 += 通关固定金币 + 利息金币
```

## 失败规则

确认规则：

- 生命是本次游戏整轮共享的失败次数；新游戏从 `global.csv` 的 `starting_lives` 读取初始值，首版为 `3`，不会自动恢复。

- 扣除后生命仍大于 `0` 时，不推进 `stageIndex`、不发通关固定收入、也不进入市场；保留本关累计分、已入钱包金币、长期面组、饲料、吸附和全部已提交长期效果，清理每手状态后回到可排序的 `Ready`。

- 扣除后生命等于 `0` 时清除本轮存档并进入 `GameOver`，不再给予额外出手机会。满生命开始时，同一关最多承受三次未达标出手；此前已消耗生命时上限相应减少。
- 累计首次达标后只发放一次通关收入，并只进入一次市场。
