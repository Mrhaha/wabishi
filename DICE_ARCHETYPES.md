# 骰子王当前骰子规则

本文档是当前骰子名单、效果边界、市场资格与关卡积分曲线的唯一设计依据。实现以 `Assets/Scripts/DiceKingDemo.cs` 为准，类型提示以 `Assets/Resources/Data/dice_type_config.csv` 为准，市场数值以 `Assets/Resources/Data/dice_market_config.csv` 为准。

## 当前内容边界

- 正式随机市场只允许 `44` 枚骰子：猪猪 `7`、恶魔 `5`、龟龟 `7`、海盗 `8`、中立 `8`、双阵营 `9`。
- `Basic`（基础骰）是系统骰：用于初始六骰、人口海盗招募和异常兜底，不进入随机市场。
- `Tribute`（贡品骰）是深渊专用骰：只由深渊召唤固定生成，不进入随机市场。
- 历史版本骰子不属于当前产品内容，不得出现在市场、配置、图鉴、提示、资源目录或有效运行存档中。
- 运行存档版本为 `12`。旧版本运行记录直接清除，避免历史骰子回流；菜单设置与开场观看状态不受影响。版本 `12` 同时保存剩余生命、本关累计分、稳定续投状态、投资锁金和本关一次性触发标记。
- 单家族、中立与双阵营市场开关仅用于隔离测试。显式开启时覆盖正式混合池，关闭后恢复 44 枚正式池。

## 家族与双阵营身份

四个主体家族是猪猪、恶魔、龟龟、海盗。中立不算主体家族。双阵营骰同时为两个标注家族提供身份：

| 双阵营骰 | 家族 |
|---|---|
| 黑帆魔蝠 | 恶魔 / 海盗 |
| 驮粮龟 | 猪猪 / 龟龟 |
| 贡猪 | 猪猪 / 恶魔 |
| 黑市小鬼 | 恶魔 / 海盗 |
| 血契船长 | 恶魔 / 海盗 |
| 清仓猪 | 猪猪 / 海盗 |
| 补给猪 | 猪猪 / 海盗 |
| 共食魔龟 | 恶魔 / 龟龟 |
| 托底龟 | 猪猪 / 龟龟 |

双阵营实体骰可同时参与两个家族的存在性、光牙候选、王冠最高点与海盗头领价格判断。龟类骰真实吸附成为壳后不再是实体，不再提供任何家族身份，也不会把家族身份传给宿主。

## 通用结算边界

- 每小关初始一次出手，无出千；当前骰袋中的实体骰全部上场，容量上限为 6。本手累计结算未达标时扣除整轮共享生命，扣后仍有生命才在同一关追加下一手；续投不重置已提交长期状态。
- 骰子按槽位从左到右结算。当前槽位引发的局部连锁全部清空后，主游标才进入下一槽。
- 长期面组允许出现 `0` 和负数。单骰分读取真实最终有效点数，负点可直接扣分。
- 饲料最低点为 `真实最低面 + FeedValue`；它在结算时托高自然点，不直接重写自然面组。
- 悬浮窗“实际六面”是逐面可确定点数：先读取当前长期面组，再应用饲料最低点；宿主存在幸运龟壳时再与当前最高面取大。该展示不纳入条件计分、邻位、钱包、遭遇修正、本次随机结果或最终贡献分，也不写回面组与存档。
- 吞吃与真实吸附读取来源当前长期面组，转移量为 `max(1, floor(各面总和 / 面数))`，只在最终转移量处保底。
- 真实吸附龟会从骰袋移除，把转移量加入宿主全部面，并保留类型、效果与可回收卖价；基础壳和首领复制壳不计卖价。
- 购买、卖出、刷新、货架生成与离开市场都是结构化市场事件。每个市场动作只扫描一轮响应队列，禁止无限递归。
- 多颗同名骰默认分别响应；规则文本明确排除递归时，以排除规则为准。
- 最高面、最低面、指定当前面和奇偶均读取真实面组；负奇数按绝对值判为奇数。

## 猪猪家族（7）

猪猪围绕饲料、卖价与饲料质量构筑长期成长。

| 类型 | 效果 |
|---|---|
| 养猪 | 本槽结算时，给左右相邻实体骰各喂 2 饲料。 |
| 肉猪 | 自己获得饲料时，饲料次数和最低点增量都翻倍；翻倍部分不再次广播获得饲料。 |
| 贸易猪 | 卖出时先按自身饲料次数提高卖价，再把已有饲料随机转给 1 颗剩余骰。 |
| 母猪 | 本槽结算时，给所有实体骰各喂 1 饲料，包含自己。 |
| 三只小猪 | 本槽结算时，获得其它实体骰当前饲料最低点增量总和的临时单骰分。 |
| 贪吃猪 | 其它实体骰获得饲料时，自己获得等量饲料；贪吃响应不继续触发贪吃响应。 |
| 批发商 | 买入时让饲料质量 +1，只影响之后新获得的饲料。 |

## 恶魔家族（5）

恶魔围绕市场最低点、吞吃、贡品与离场转移构筑成长。

| 类型 | 效果 |
|---|---|
| 小鬼 | 过关进入市场前，让市场基础最低点 +1；本轮持续并影响之后刷出的市场骰。 |
| 吞噬骰子 | 每发生 1 次吞吃事件时，钱包 +1 金；多颗可分别响应。 |
| 恶魔骰子 | 每购买 1 次市场货架，让本市场临时市场最低点 +1；离开市场后失效。 |
| 魔蝠骰子 | 离开市场时按骰袋顺序吞吃当前货架中 1 颗市场骰；每颗最多吞 1 次。 |
| 深渊召唤 | 进入市场和付费刷新时固定刷出 1 个贡品骰；多颗首版不增加贡品数量。 |

贡品骰六面基础都是 1，接受市场最低点强化；购买后直接奉献给当前最低点实体骰，不进入骰袋。

## 龟龟家族（7）

除首领龟外，龟龟在离开市场时只检查后一个槽位：若其为非龟实体骰则真实吸附，否则保留实体且不跨格寻找。

| 类型 | 吸附或结算效果 |
|---|---|
| 金钱龟 | 宿主结算时获得 1 金。 |
| 小小龟 | 宿主结算时，宿主吸附 1 颗基础骰壳。 |
| 双倍龟 | 吸附时把宿主当前最小面乘 2；最小面为负数时会变得更低。 |
| 幸运龟 | 宿主结算时始终按当前最大面结算。 |
| 磁力龟 | 宿主结算时，给其它非龟实体骰各吸附 1 颗基础骰壳。 |
| 呼朋唤友龟 | 宿主结算时，其它骰每有 1 只真实吸附龟，全脸永久 +2。 |
| 首领龟 | 不吸附；每次发生真实吸附，自身获得同类复制壳，复制壳不递归。 |

## 海盗家族（8）

海盗围绕刷新费用、买卖、货架与高风险长期成长构筑经济循环。

| 类型 | 效果 |
|---|---|
| 刷新海盗 | 每颗让本次刷新净费用 -1；净费用可为 0 或负数，负数返还金币。 |
| 劫掠海盗 | 本槽结算时，左右相邻实体骰全脸永久 -1；每个实际目标让钱包 +3 金。 |
| 人口海盗 | 本市场每主动卖出 2 颗骰，招募 1 颗基础骰；实例售价随机为 0–3。 |
| 海盗头领 | 在骰袋中时，购买海盗家族骰的实际买价半价；不改变卖价。 |
| 练腿海盗 | 买入任意骰后，每颗随机选择其它己方骰，全脸永久 +1。 |
| 财宝海盗 | 每次真实金币入账事件后，随机己方骰全脸永久 +1；按事件触发，不按金币数触发。 |
| 抢劫海盗 | 离开市场时，从非空货架随机获得 1 颗骰；抢到的新离场骰加入当前队列末尾。 |
| 海贼王 | 真实结算时读取当前钱包，每 3 金让自身全脸永久 +1；不消耗金币且无上限。 |

## 中立骰（8）

中立连接多家族，不作为第五个主体家族。

| 类型 | 效果 |
|---|---|
| 光牙 | 结算到本骰时，每个已存在主体家族随机选择 1 颗该家族实体骰，全脸永久 +2；同一双阵营实体在本次光牙结算中最多被选一次。 |
| 二重唱 | 让左邻实体骰最近一次可复制的主体家族结算效果额外触发 1 次；复制事件不再触发触发器。 |
| 触发器 | 每次真实且非复制的主体家族结算效果发生时，使事件来源骰本次单骰分 +2；多颗叠加。 |
| 王冠 | 常规结算结束后，读取每个已存在主体家族的最高最终有效点，向下取整求均值，作为本骰额外单骰分。 |
| 接济 | 结算到本骰时，每有 1 个不同主体家族，钱包 +1 金。 |
| 空袭 | 至少存在 1 个主体家族时，每缺少 1 个主体家族，本骰本次单骰分 +2；没有主体家族时为 0。 |
| 盟约 | 左右邻均属于主体家族且家族不同时，左右两骰本次单骰分各 +2。 |
| 缝合 | 每有 1 个不同主体家族，本骰本次单骰分 +2，最多 +8。 |

## 双阵营骰（9）

| 类型 | 效果 |
|---|---|
| 黑帆魔蝠 | 付费刷新替换旧货架前，按骰袋顺序吞吃 1 颗不同的非空旧货架骰；首次生成货架不触发。 |
| 驮粮龟 | 真实吸附时，每 2 点自身饲料最低点价值，额外给宿主 1 层基础壳。 |
| 贡猪 | 主动卖出只获得类型基础卖价，随后把自身作为非市场来源奉献给当前最低点实体骰。 |
| 黑市小鬼 | 每吞吃 1 颗市场骰，使本市场下一次付费刷新费用 -1；多颗分别叠加，可产生返利。 |
| 血契船长 | 骰袋满时仍可购买实体骰；完整触发购买监听后不入袋，直接作为市场来源被吞吃。 |
| 清仓猪 | 离场队列结算到自身时，每个当前空货架使自身获得 1 份当前质量饲料。 |
| 补给猪 | 每次货架生成或付费刷新完成后，给当前最低价非空货架骰 1 份当前质量饲料；同价取最左。 |
| 共食魔龟 | 真实吸附后，宿主每次获得吞吃面组加值时，右邻非龟实体获得其一半，向下取整且每层最低 1；不递归。 |
| 托底龟 | 真实吸附后，宿主自然点被饲料最低点实际托高时生成 1 层基础壳；每层每次结算最多一次，不回改本次点数。 |

## 正式市场数值边界

- 44 枚随机骰必须在 `dice_market_config.csv` 各有且仅有一行，并在至少一个章节区间拥有正权重。
- 基础骰与贡品骰可以保留价格行供系统逻辑使用，但三个随机权重必须全部为 0。
- 正式混合池过滤必须同时作用于普通候选、高阶保底、路线倾向与最终可选判断；任何 CSV、旧存档或数值枚举都不能绕过过滤。
- `dice_type_config.csv` 与 `dice_market_config.csv` 当前都应为 46 行有效数据：44 正式随机骰 + 基础骰 + 贡品骰。

## 关卡积分曲线

当前曲线按六颗公平基础骰平均基础分 `21`、单手直接比较目标、无出千、无牌型倍率，以及新版饲料 / 吞吃 / 吸附 / 海盗 / 中立 / 双阵营长期成长能力设定。第二轮依据 10 次固定种子离线自动流程样本降低首章随机低点淘汰与中期 Boss 卡点；双阵营虽然提高跨家族成型率，但也把正式随机池从 35 扩到 44 并稀释单家族密度，因此不以少数极端成长路线反向抬高全局门槛。2026-07-22 已确认目标流程改为整轮共享初始 `3` 命、失败手分数与效果保留的本关累计续投，因此本表在实现前只是现有单手基线。

| 2 | 20 | 22 | 27 | 32 |

校准原则：

- 首关目标 `14` 使六颗公平基础骰的理论通过率由旧目标 `15` 的 `93.92%` 提升至 `96.41%`；首章后半开始要求使用市场。
- 每章内部持续加压，下一章第 1 关相对前章 Boss 留出短暂喘息。
- 后期目标必须迫使玩家建立至少一条可持续长期成长链，不能只靠基础面组。
- 终章 Boss `616` 是第二轮候选，不是冻结值。Play Mode 需记录低成长、中位成长、高成长构筑在第 12、24、36、40 关的真实得分。
- 当前曲线原按“单关失败即流程结束、单手直接比较目标”校准；生命累计续投现已接入，必须记录每关耗命、续投前差值、续投得分与每点生命带来的长期收益并重新模拟，不可直接沿用旧通过率判断。
- 播种 + 多只三只小猪、小鬼 + 龟龟等少数极端爆发应单独评估效果上限，不以它们抬高所有普通构筑的过关门槛。
- 若中位构筑连续两章无法达到目标，优先下调该章第 3 关与 Boss；若高成长构筑连续两章超过目标 50%，再提高下一章整段斜率。

## 验收要求

1. 默认配置下连续刷新至少 100 批，货架只能出现 44 枚正式随机骰；深渊召唤生成的贡品除外。
2. 单家族、中立、双阵营隔离测试各自只能出现对应名单。
3. 旧版本存档不能恢复历史骰子；无效类型输入必须被拒绝或安全降级为基础骰，且不能进入市场。
4. 骰袋详情、悬浮提示、类型图标加载与市场配置不得枚举历史骰子。
5. 覆盖负面 + 饲料、负面 + 双倍龟、双阵营 + 光牙、双阵营 + 王冠、刷新海盗负费用、深渊贡品六组边界。
6. 完成上述运行样本前，积分曲线标记为“静态接入、待 Play Mode 数值复核”。
