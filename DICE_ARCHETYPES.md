# 骰子流派设计定稿

本文档记录 `骰子王` 当前正式骰子类型、流派方向和关间事件规则。旧版 `金币流派 / 动物园流派` 临时配置不再作为后续实现依据。

数值是首章原型初值，可继续调参；机制边界默认视为已确认设计，除非明确写在“待确认”中。

## 维护规则

- 新增或修改骰子类型时，优先更新本文档。
- 如果骰子类型改变小关、结算、失败、商店等主流程，同步更新 `GAME_FLOW.md`。
- 如果骰子类型改变长期产品方向、实现边界或跨系统注意事项，同步更新 `PROJECT_CONTEXT.md`。
- 临时灵感不要直接写成定稿；先放入“后续扩展”或“待确认”。

## 通用术语

- `实体骰`：玩家骰袋中的骰子，会保留类型、面组、成长层数和其他长期状态。
- `临时小骰`：由龟龟骰等效果在一次结算中生成，只参与本次结算，默认不进入骰袋、不继承长期成长、不进入关间买卖。
- `上场骰`：本次出手使用的实体骰。首章骰袋容量上限为 6，当前骰袋中的实体骰默认全部上场。
- `回合`：当前文档中指一次出手/投骰，从按下 `Space` 开始摇骰到本次结果二次确认结算结束。
- `有效点数`：骰子最终参与计分的点数，可能因出千改点、类型效果或关卡规则临时改变。
- `基础单骰分`：骰子没有特殊效果时按有效点数得到的分数；1 点就是 1 分。具体倍率和数值由 `GAME_FLOW.md` 的结算公式维护。
- `材质`：F002 商店生成骰子的第二维独特性。材质不取代骰子类型，不改变有效点数或牌型识别；首版每颗商店骰最多一个材质。
- `钱包金币`：玩家长期持有的金币，用于关间买卖骰子。
- `流式钱包金币`：猪猪、鎏印、F008 金币收益族等出手内金币来源按当前骰袋槽位从左到右直接写入钱包，较早来源可被较晚对象读取或消耗。F005 金币后缀在 F013 默认软关闭期间保留在存档中但不生效。
- `命中点数`：大树骰当前追踪的目标点数。每次出手随机生成，掷中后触发自身成长。

## 总体设计方向

当前正式设计不再用“材质 + 性格”作为首章核心表达，而是使用清晰的 `骰子类型`。F002 重新引入的材质只作为商店独特骰的辅轴，服务具体商品差异，不作为首要流派身份：

1. 基础骰子：无特效，承担稳定底座。
2. 猪猪骰子：每次出手随机目标点数，命中后获得金币。
3. 龟龟骰子：按点数启动逐级衰减的壳链，形成迭代收益。
4. 双倍骰子：基础点数收益翻倍。
5. 奇数骰子：只出现奇数点数，服务全奇和奇数牌型。
6. 偶数骰子：只出现偶数点数，服务全偶和偶数牌型。
7. 孤证骰子：自然开盅后落单时自动重摇自己一次，服务奇数追同稳定。
8. 盖章骰子：有同点伙伴时固定高分兑现，服务奇数同点收益。
9. 半步骰子：顺子判定时可低一格借位，服务偶数顺子辅助。
10. 轨道骰子：`2 / 4 / 6` 齐备时固定高分兑现，服务偶数目标收益。
11. 旧大树骰子：每次出手随机命中点数，命中后自身骰面整体成长；保留兼容但已从市场新刷内容下架。
12. 旧园丁骰子：大树自然命中时，让该大树额外成长；保留兼容但已从市场新刷内容下架。
13. 旧灌溉骰子：掷中未自然命中大树的命中点数时，补一次成长；保留兼容但已从市场新刷内容下架。
14. 赌徒骰子：每次出手随机阈值，低于阈值不计分，高于阈值按期望守恒倍率爆发。
15. 国库骰子：读取小关开始钱包本金，按本金给本骰稳定单骰加分。
16. 贿赂骰子：结果差一点达标且可补齐时，确认结算后花可用钱包补最终分。
17. 投资骰子：小关开始自动锁定并预付预算，本关每次结算给本骰单骰加分。
18. 悬赏骰子：每次出手从自身当前面组去重抽一个悬赏点，命中后获得高额金币。
19. 顶金骰子：掷出自身当前最大面时获得少量稳定金币。
20. 牌税骰子：触发三同、顺子、全奇、全偶或更高同点牌型时获得金币。
21. 收账骰子：每个小关第一次真实结算时获得金币。
22. 复利骰子：通关结算时基于已封顶基础利息追加金币。
23. 铅票骰子：本骰被确认出千后获得金币。
24. 壳税骰子：本次真实生成临时小骰数量达标时获得金币。
25. 柜台骰子：左侧相邻骰本次有正向钱包入账时获得金币。
26. 伐木骰子：削减左侧相邻骰本次单骰分，换取金币。
27. 异邻骰子：左邻奇偶不同时，本骰固定 6 分。
28. 同邻骰子：左邻奇偶相同时，本骰固定 6 分。
29. 补全骰子：只差本骰时，补全全奇 / 全偶副牌型。
30. 复核骰子：自然结果破坏全奇 / 全偶时，本骰重摇一次。
31. 翻号骰子：本骰出千后奇偶变了，固定 8 分。
32. 守号骰子：本骰出千后奇偶没变，固定 6 分。
33. 转号骰子：本骰出千改点时读取全局 `+1` 封顶结果，未封顶时通常改变奇偶。
34. 点籽树：命中本次指定点数后，本骰直接真实成长。
35. 牌谱树：命中本次指定牌型后，本骰直接真实成长。
36. 冠层树：掷出自身当前最高面后，本骰直接真实成长。
37. 年轮树：每次真实结算后，本骰直接真实成长一次。
38. 肥料树：按当前金币和标准金币步长，本骰直接真实成长多次。
39. 修枝树：本骰出千后结果改善、倍率改善或帮助通关时，本骰直接真实成长。
40. 根系树：左右相邻实体骰本次触发自身类型效果时，本骰直接真实成长一次。

这些类型可以自然形成几条构筑路线：

- `金币经营`：猪猪骰子命中随机目标点数后直接提供钱包金币，国库骰奖励持有本金，贿赂骰用可用钱包救急补分，投资骰提前锁定预算换取本关持续加分；F008 九颗金币收益族骰子把点数、牌型、出千、小骰数量、槽位和利息都接入钱包收益。
- `迭代扩张`：龟龟骰子启动壳链生成临时小骰，用数量和递归补足分数。
- `奇偶牌型`：奇数骰子和偶数骰子提高全奇、全偶、三同和顺子等简化牌型的稳定性；F010 七颗短规则骰把左邻摆位、全奇 / 全偶补救和出千奇偶判断接入短决策。
- `长期成长接口`：F011 新大树骰通过指定点数、指定牌型、最高点数、回合结束、金币、出千和槽位触发让自身面组真实成长；旧 `Tree / Gardener / Irrigation` 只保留兼容，不再作为市场新刷路线。
- `爆发风险`：双倍骰子和赌徒骰子承担高分上限；赌徒骰用期望守恒控制长期强度。
- `稳定过渡`：基础骰子作为便宜、可卖、可替换的构筑底座。

后续骰子扩展的确认方向：

- 第一批后续扩展已先补齐现有基础流派内部结构；后续新增或重做骰子的上位方向改为打破封闭 build 壁垒，把骰子设计成更通用的机制提供者。
- 新框架下，每颗候选骰都应同时说明 `触发条件 / 收益类型 / 结算接口`：它读取什么事件，提供什么机制，在左到右结算、出千、预览、临时小骰、钱包流和市场定位中如何工作。类型名仍服务阅读和美术身份，但不再代表只能服务单一 build。
- 后续骰子设计的通用触发点白名单已确认：指定点数、指定牌型、最高点数、回合结束、金币、出千和槽位触发。后续设计应优先从这些触发点中选择，不再随意添加一次性触发条件；白名单外触发必须先进入设计对话或生产包评审。
- 金币流第一批已实现国库骰、贿赂骰、投资骰；F008 金币收益族泛用化已接入原型代码、数据和最终透明图标，仍需 Unity Play Mode 与截图验收。
- 龟龟流第一批补强骰已接入当前原型并由用户确认验收，包含壳匠骰、巢穴骰、慢龟骰。
- 龟龟后续优化方向已确认：龟龟不再主要作为多龟龟自闭环套装，而应提供“临时小骰余波”和“结算连锁表演”机制，读取非龟骰、相邻槽位、牌型成立、金币支付、出千改点、高面命中或其它 build 成功事件，把它们转成可展示的滚分尾音。当前原型已先把 `Turtle / SlowTurtle` 的底层生成改为“壳链”：实体骰只决定起始 `dN` 种子，后续按 `dN -> d(N-1) -> ... -> d1` 展开；具体新候选和壳匠读取范围仍待生产包细化。
- 奇偶流第一批补强骰已接入当前原型，包含孤证骰、盖章骰、半步骰、轨道骰；F010 奇偶短规则骰已接入当前原型代码、数据和最终透明图标，包含异邻、同邻、补全、复核、翻号、守号、转号，并于 2026-06-16 通过 Unity Play Mode 验证。后续调参优先从市场价格、刷新权重和 UI 清晰度入手，不先扩张新全局牌型。
- F011 大树长期成长接口骰已静态接入当前原型代码、市场数据和最终透明图标，待 Unity 运行验证；七颗分别为 `点籽树 / 牌谱树 / 冠层树 / 年轮树 / 肥料树 / 修枝树 / 根系树`。
- F011 共同收益为触发后直接真实成长，表现为触发骰自身全部面 `+1` 和现有 `Growth +1`；不新增发芽、芽点、成长进度或 `GrowthProgress`。大树不以临时小骰作为主收益，也不直接做“成长一次给金币”的硬接口。
- 旧 `Tree / Gardener / Irrigation` 的代码、旧资源和旧存档读取保留，但正式市场权重和测试全池随机都已下架旧三件套。后续大树设计默认围绕 F011 七触发接口继续迭代。
- F002 商店独特骰首版已接入原型：商店生成商品骰时先抽类型，再按类型家族生成随机面组，最后抽单一材质。奇数族只生成 `1 / 3 / 5` 面，偶数族只生成 `2 / 4 / 6` 面，其它类型使用通用模板；带词缀成品骰、重铸、多材质和负面词缀后置。F005 已接入词缀数据、存档基础和三类市场改造道具，但 F013 默认 `affix_feature_enabled = 0` 时这些入口保留兼容、不显示、不生效。

## 结算边界

- 首章骰袋容量上限为 6，默认每次出手使用当前骰袋中的全部实体骰。
- 当前实体骰先形成主牌型倍率；临时小骰默认只提供额外单骰分，不参与实体骰主牌型识别。
- 临时小骰完整计分；运行展示超过 8 颗时折叠为汇总表达，折叠不改变计分。
- 类型效果如果改变点数，默认发生在主牌型识别前；如果只改变分数，默认发生在主牌型识别后。
- 出千选择 1 到 3 颗实体骰点数 `+1`，最高不超过该骰当前面组最大点数；未选实体骰保持原结果，临时小骰默认不能被出千选择。
- 出千选择界面可以在确认前取消；取消不消耗出千机会，也不改变实体骰结果、猪猪目标、旧大树命中点、点籽目标、牌谱目标或赌徒阈值。
- 出千只改动被选实体骰的最终有效点数，不刷新本回合已经生成的猪猪目标、旧大树命中点、点籽目标、牌谱目标或赌徒阈值。
- 猪猪、鎏印和 F008 金币收益族等金币来源按当前骰袋槽位从左到右直接进入钱包；较早金币可以被较晚贿赂或后续钱包读取对象使用，较晚金币不会回头影响较早对象。F005 金币后缀在 F013 默认软关闭期间不进入钱包流。
- F008 出手内金币来源也按当前骰袋槽位从左到右直接进入钱包；柜台骰只读左侧相邻骰本次槽位收入，伐木骰只削减左侧相邻骰本次单骰分，不改变有效点数、牌型或长期面组。
- F010 邻位类读取当前骰袋槽位左邻实体骰，最左槽不触发；补全只影响全奇 / 全偶副牌型识别；复核只在自然结果锁定后触发；翻号、守号和转号只读取本骰被确认出千改点的前后奇偶，不写入长期存档。
- 收账骰每颗每小关最多触发 1 次；出手预览不得写入收账标记，只有真实二次确认结算才写入。
- 复利骰只基于通关基础利息追加金币，不递归计算，不提高利息上限。
- 国库骰读取小关开始钱包快照，不读取本小关后续新增钱包；投资预算在小关开始锁定，不因后续新增钱包刷新。
- 贿赂骰不改变有效点数或主牌型识别；其可花金币上限按当前骰袋槽位的左到右顺序模拟，避免后置金币回流给前置贿赂。
- F011 新大树接口骰只在真实二次确认结算后入档成长，不在预览写入面组；成长不影响本次已经锁定的有效点数、牌型或分数。
- 肥料树读取真实结算中当前钱包金币，按 `floor(chapterGold / interest_gold_step)` 计算成长次数，不创造金币、不消费金币、不设置单次成长上限；当前 F013 正式基线为 `starting_gold = 8`。
- 根系树只读取左右相邻实体骰本次结构化类型触发标签；基础计分、主牌型倍率、无条件常驻修正、材质触发和其它根系成长不计入。
- 投骰前的 `Ready` 阶段允许玩家调整骰子槽位；结果锁定、孤证重摇和出千改点都不改变槽位顺序，也不按点数重排展示。
- 小关开始先记录钱包快照，再锁定投资预算；国库读取原始快照，贿赂读取投资扣除后的实时可用钱包和顺序内已产生金币。
- 能长期成长的骰子只在完成一次二次确认结算后记录成长结果，避免进入投骰界面后取消或失败读取异常。
- 巢穴骰补出的 d1 临时小骰计入壳匠数量，但不触发巢穴，也不触发其它实体骰效果。
- 临时小骰不携带材质，不触发材质，也不能作为铜骨的同点伙伴。

## 骰子类型

### 基础骰子

核心定位：

- 没有特殊效果。
- 价格最低，主要用于补齐 6 骰阵容、过渡、出售换钱或作为后续改造对象。

首章默认面组：

```text
1 / 2 / 3 / 4 / 5 / 6
```

结算规则：

- 按基础单骰分正常计分。
- 没有额外金币、成长、召唤或倍率。

市场定位：

- 买入便宜。
- 卖出价值低。
- 允许玩家在缺骰时快速补齐阵容。

### 猪猪骰子

核心定位：

- 经济骰。
- 每次出手随机生成一个指定点数，摇到该点数就获得金币。
- 主要服务关间买卖骰子的资金循环。

首章规则：

1. 每次出手开始时，本颗猪猪骰随机生成一个 `1-6` 的目标点数。
2. 猪猪骰投出结果后，如果有效点数等于目标点数，获得 `piggy_gold_per_hit` 钱包金币；当前 `global.csv` 配置为 `+1`。
3. 猪猪骰自身仍按基础单骰分计分，金币收益不直接提高本次分数。
4. 如果有多颗猪猪骰，每颗猪猪骰各自生成并显示自己的目标点数。

防刷边界：

- 猪猪收益在本次出手结算时直接进入钱包。
- 猪猪收益如果发生在较早结算位置，可以被较晚贿赂或后续钱包读取对象使用；如果发生在达标出手，也会进入本次通关利息基数。
- 同一颗猪猪骰每次出手最多触发 1 次金币收益。
- 任意小关失败直接结束本轮，不保留猪猪本关收益、骰子状态或其他本轮信息。

市场定位：

- 买入价格中等。
- 早买收益更高，后期仍可通过出售换成强分数骰。

### 国库骰子

核心定位：

- 金币路线的本金奖励骰。
- 玩家保留的钱包本金越厚，国库骰越稳定。

首章规则：

1. 小关开始时记录钱包金币快照，发生在投资预算扣除之前。
2. 国库加分为 `floor(小关开始钱包 / treasury_gold_step)`，当前 `global.csv` 配置为每 10 金 +1。
3. `treasury_score_cap = 6` 表示单颗国库骰本次最多获得 6 点国库加分；`0` 才表示不封顶。
4. 本骰单骰分为有效点数加国库加分。

边界：

- 国库骰不花钱，不改变有效点数，不改变主牌型识别。
- 国库骰只读取小关开始钱包快照，猪猪、鎏印或词缀后续新增钱包不会提高本小关国库加分。
- 出千只改变有效点数，不刷新小关开始钱包快照。

市场定位：

- 阶级 `T2`，买入价 9，卖出价 4。
- 适合保留本金、减少市场乱花钱的金币路线。

### 贿赂骰子

核心定位：

- 金币路线的救急骰。
- 结果差一点通关时，用可用钱包把最终分补到目标。

首章规则：

1. 先计算不含贿赂的本次基础分和预计累计分。
2. 如果预计累计分已经达标，不花钱。
3. 如果未达标，计算缺口；每 1 金补 `bribe_score_per_gold` 最终分，当前为 +3。
4. 每颗贿赂骰每次结算最多花 `bribe_gold_cap_per_die` 金，当前为 2 金。
5. 只有需要金币不超过可用钱包且不超过总贿赂上限，并且能补到通关时，确认结算后才扣钱补分。

边界：

- 贿赂追加的是最终分，不吃主牌型倍率。
- 贿赂读取投资扣除后的实时钱包，并可读取在当前骰袋槽位左到右顺序中已经发生的较早金币来源。
- 贿赂扣钱发生在玩家确认结算后、通关收入和利息结算前；可花上限按顺序模拟，较晚金币来源不能回头供给较早贿赂。
- 出千前只是预览；出千后按新结果重新判断是否可补。

市场定位：

- 阶级 `T2`，买入价 9，卖出价 3。
- 适合保留应急金的金币路线，不能在无法通关时白白烧钱。

### 投资骰子

核心定位：

- 金币路线的提前下注骰。
- 小关开始自动锁定预算，换取本关每次结算的持续单骰加分。

首章规则：

1. 小关开始记录钱包快照后，自动计算总投资预算。
2. 总预算为 `min(投资骰数量 * investment_gold_cap_per_die, floor(小关开始钱包 / investment_wallet_divisor))`；当前每颗最多 2 金，钱包四分之一封顶。
3. 预算按骰袋顺序分配给投资骰，每颗最多 2 金。
4. 投资预算立即从钱包扣除，失败或通关都不返还。
5. 每颗投资骰的单骰加分为本骰预算乘以 `investment_score_per_gold`，当前每金 +2。

边界：

- 投资骰不改变有效点数，不改变主牌型识别。
- 投资预算整关固定，不因出千、猪猪 / 鎏印 / 词缀金币或后续结算刷新。
- 当前继续游戏恢复到小关开始，会按保存的预投资钱包重新计算预算；小关中途投资状态不是持久存档字段。

市场定位：

- 阶级 `T3`，买入价 10，卖出价 4。
- 适合愿意提前付费换取稳定战力的金币路线。

### 龟龟骰子

核心定位：

- 迭代生成骰。
- 根据当前点数启动一段壳链，壳链按最大点数逐级衰减生成临时小骰，形成更稳定的滚分尾音。

结算规则：

1. 龟龟骰先按自身有效点数正常计分。
2. 设龟龟骰有效点数为 `v`，壳链种子规格为 `floor(v / 2)`。
3. 如果种子规格小于 `1`，不生成壳链；`1` 是最小有效小骰点数。
4. 如果种子规格为 `N`，启动 `dN` 壳链。
5. 壳链每一节从 `1..当前最大点数` 随机取值并正常提供基础单骰分。
6. 下一节不读取上一节实际掷出的点数，而是让当前最大点数固定 `-1`。
7. 壳链按 `dN -> d(N-1) -> ... -> d1` 展开，完成 d1 后停止。

示例：

```text
龟龟骰掷出 6
启动 d3 壳链
生成 d3 小骰，实际掷出 1 到 3
继续生成 d2 小骰，实际掷出 1 到 2
继续生成 d1 小骰，实际掷出 1
壳链结束
```

首章边界：

- 临时小骰默认不参与实体骰主牌型倍率。
- 临时小骰只追加基础单骰分，不触发猪猪金币、大树成长、赌徒倍率等实体骰类型效果。
- 壳链不按实际掷点提前死亡；设计上依靠起始种子规格逐级降到 d1 后自然停止。
- 结算展示时，每一层实际生成的临时小骰都应显示在投掷区，并随结算动画逐颗计分。

市场定位：

- 买入价格较高。
- 分数稳定性取决于首个龟龟骰点数，高点收益明显。

### 壳匠骰子（已接入原型，已验收）

核心定位：

- 龟龟路线的数量结算骰。
- 本次龟龟系临时小骰越多，壳匠骰自身分数越高。

首版规则：

1. 壳匠骰自身按有效点数正常计分。
2. 真实结算时，在龟龟骰、慢龟骰和巢穴骰完成临时小骰生成后，统计本次龟龟系临时小骰总数。
3. 壳匠加分为 `本次龟龟系临时小骰数量 * 1`。
4. 壳匠加分是本骰单骰分加成，会进入基础本次分并参与最终牌型倍率。
5. 没有临时小骰时，壳匠不获得额外加分。

边界：

- 壳匠不生成临时小骰，不改变有效点数，不改变主牌型识别。
- 壳匠只按真实结算中的实际临时小骰数量给分；预览只能标记为预估。
- 巢穴补出的 d1 临时小骰计入壳匠数量。

市场定位：

- 阶级 `T2`，买入价 10，卖出价 5。
- 适合已经有龟龟来源的路线，把数量转换成可被倍率放大的单骰分。

### 巢穴骰子（已接入原型，已验收）

核心定位：

- 龟龟路线的少量补骰骰。
- 有龟龟链产出时，额外补充稳定的 d1 临时小骰。

首版规则：

1. 巢穴骰自身按有效点数正常计分。
2. 真实结算时，在龟龟骰和慢龟骰完成自身链条后触发。
3. `巢穴补骰数量 = min(本次有产出的龟龟链数量, 巢穴骰数量)`。
4. 每颗巢穴补骰都是 d1 临时小骰，分数为 1，并完整计入本次结算。

边界：

- 巢穴补骰不参与主牌型识别。
- 巢穴补骰不触发巢穴，避免闭环。
- 巢穴补骰不触发猪猪金币、大树成长、赌徒倍率、国库、贿赂或投资等实体骰效果。
- 巢穴补骰计入壳匠的临时小骰数量。

市场定位：

- 阶级 `T3`，买入价 12，卖出价 5。
- 适合已经有龟龟来源和壳匠收益的路线，提供少量稳定数量。

### 慢龟骰子（已接入原型，已验收）

核心定位：

- 龟龟路线的中期来源骰。
- 使用更高的起始种子规格生成较长壳链。

首版规则：

1. 慢龟骰本体按自身有效点数正常计分。
2. 设慢龟骰有效点数为 `v`。
3. 如果 `v <= 1`，不生成壳链。
4. 如果 `v > 1`，壳链种子规格为 `ceil(v / 2)`。
5. 如果种子规格为 `N`，启动 `dN` 壳链。
6. 壳链每一节从 `1..当前最大点数` 随机取值并正常提供基础单骰分，下一节最大点数固定 `-1`，直到完成 d1 后停止。

- 慢龟临时小骰不参与主牌型识别。
- 慢龟链真实结算随机生成；结果预览必须保留“预估”提示。
- 慢龟与普通龟龟的差异只在起始种子规格：普通龟龟用 `floor(v / 2)`，慢龟用 `ceil(v / 2)`；壳链展开后都按最大点数逐级 -1。

市场定位：

- 阶级 `T2`，买入价 11，卖出价 5。

### 旧大树骰子（兼容保留，市场下架）

核心定位：

- 长期成长骰。
- 每次出手随机生成命中点数；命中后只成长自己，不影响其他骰子。

长期状态：

- `成长层数`：本骰命中后积累的成长次数。
- `命中点数`：当前需要掷中的目标点数，每次出手随机生成。

首章默认面组：

```text
1 / 2 / 3 / 4 / 5 / 6
```

结算规则：

1. 每次出手开始时，本颗大树骰随机生成一个 `1-6` 的命中点数。
2. 如果本骰有效点数等于当前命中点数，本骰成长。
3. 成长效果是把本颗大树骰自己的所有骰面整体 `+1`，并长期保留。
4. 例如初始面组 `1 / 2 / 3 / 4 / 5 / 6`，命中成长后变成 `2 / 3 / 4 / 5 / 6 / 7`。
5. 多颗大树骰各自独立生成命中点数、独立成长。

边界：

- 大树骰只改变自己的骰面，不提高其他骰子的点数。
- 成长后可能出现 `7` 或更高点数；这些点数按原值正常参与 6 骰主牌型识别。
- 为避免 UI 混乱，大树命中应在结算预览中明确展示“命中点数”和“自身骰面 +1”。

市场定位：

- 旧兼容对象，当前正式市场和测试全池随机都不再新刷。
- `dice_market_config.csv` 保留行但权重为 `0 / 0 / 0`。
- 旧存档或历史生成对象仍可按旧逻辑继续结算。

### 赌徒骰子

核心定位：

- 高风险爆发骰。
- 每次出手随机一个阈值。
- 小于阈值的点数不计分，大于阈值的点数按相应倍率计分。
- 长期期望应与基础骰子保持一致，风险来自更大的波动而不是免费提高平均收益。

长期状态：

- `赌徒阈值`：每次出手随机生成，用于判断低点归零和高点倍率。

首章规则：

1. 每次出手开始时，本颗赌徒骰随机生成一个阈值 `t`。
2. 如果本骰有效点数 `v < t`，本骰自身分数为 `0`。
3. 如果 `v = t`，本骰按正常基础单骰分计分。
4. 如果 `v > t`，本骰自身分数乘以期望守恒倍率 `m(t)`。
5. 倍率只作用于本骰自身，不改变有效点数，不影响主牌型识别。

期望守恒公式：

```text
基础公平骰点数总和 = 1 + 2 + 3 + 4 + 5 + 6 = 21
m(t) = (21 - t) / sum(t+1 ... 6)
```

说明：

- 上式按 `v = t` 正常计分、`v > t` 乘倍率计算。
- 为了避免 `t = 6` 时没有高于阈值的点数，首章阈值只在 `1-5` 中随机。
- 如果出千改点赌徒骰，按改点后的有效点数和本回合既有阈值触发对应结果。

市场定位：

- 买入价格中高。
- 适合和出千、奇偶骰、大树自身成长、关间买卖筛选配合。

关间事件的主体改为 `骰子市场`，不再以旧版随机遗物、改材质、刻面奖励为主。

- 卖骰提供纠错和转路线空间，但不能让玩家通过频繁倒卖套利。

### F011 大树设计备案（已静态接入）

目标：

- 把大树从“只服务大树路线的成长套件”改成长期成长接口，让不同构筑可以通过已确认触发点接入成长。
- 保留大树的身份：长期养成、面组变大、资产感。不要把大树主收益做成临时小骰，也不要直接写成金币收益。
- 当前 `PointSeedTree / PatternTree / CanopyTree / RingTree / FertilizerTree / PruningTree / RootTree` 已静态接入代码、市场数据和最终透明图标，待 Unity 运行验证。旧 `Tree / Gardener / Irrigation` 仍保留兼容，但已市场下架。

统一收益裁决：

- F011 不新增“发芽”“芽点”“成长进度”或 `GrowthProgress`。
- F011 大树类骰触发后直接真实成长：触发骰自身所有面整体 `+1`，现有 `Growth +1`。
- 真实成长只在二次确认后的真实结算中入档，避免预览、取消出千或失败状态提前写入长期面组。
- 除 `肥料树` 按金币存量可产生多次成长外，每颗大树类候选每次真实结算默认最多触发 1 次。
- F011 静态接入后旧 `Tree / Gardener / Irrigation` 已从市场生成中下架；代码和旧资源保留作为兼容对象。

七类触发对应的大树候选：

| 触发点 | 候选名 | 玩家侧短规则 | 设计作用 |
|---|---|---|---|
| 指定点数 | 点籽树 | 命中指定点，本骰成长 | 保留当前大树的随机命中核心，可作为基础大树重做入口 |
| 指定牌型 | 牌谱树 | 命中指定牌型，本骰成长 | 让同点、顺子、全奇、全偶等构筑接入长期成长 |
| 最高点数 | 冠层树 | 掷出自身最高面，本骰成长 | 连接高面模板、已成长大树、赌徒高点和顶金类触发 |
| 回合结束 | 年轮树 | 每次出手结束，本骰成长 | 稳定的长期资产底座，每次出手最多触发一次，不按小关限一次 |
| 金币 | 肥料树 | 按当前金币，本骰成长多次 | 让持有金币像肥料一样滋养成长，不直接产金币，也不设置成长次数上限 |
| 出千 | 修枝树 | 出千变好，本骰成长 | 把一次出千机会变成成长取舍，适合要求结果变好或通关才触发 |
| 槽位触发 | 根系树 | 相邻骰触发，本骰成长 | 让玩家把根系树放在会触发效果的骰子旁边，保留摆位决策但去掉方向判断 |

实现范围记录：

- 本次按用户要求一次性静态接入七颗 F011 新大树接口骰。
- `点籽树 / 牌谱树 / 冠层树 / 年轮树 / 肥料树 / 修枝树 / 根系树` 均进入 `DieType`、市场配置、图标加载、短标签和成长队列。
- 仍需 Unity Play Mode 验证预览、真实结算、出千、市场刷新和保存恢复。

强度观察点：

- `年轮树` 和 `肥料树` 稳定性较高，已通过章节、价格和权重控制首版投放；后续需用正式经济参数验证强度。
- `根系树` 已简化为读取任一相邻实体骰的类型触发，每次出手最多触发一次；后续重点验证标签覆盖是否过宽或过窄。

实现边界摘要：

- 统一成长模型：
  - 大树类骰的共同收益统一为直接真实成长，不设发芽、芽点或成长进度。
  - 真实成长表现为本骰所有面 `+1`、`成长层数 +1`，并且只在二次确认后的真实结算中入档。
  - 成长入档不影响本次已经锁定的有效点数、牌型或分数；它只影响后续出手和后续市场资产判断。
  - 不新增 `GrowthProgress` 或其它进度存档字段；继续复用现有 `Faces` 和 `Growth`。
- 首批规则卡建议：
  - `点籽树`：每次出手固定一个指定点，优先从本骰当前面组中抽取；本骰最终有效点等于指定点时直接成长一次。出千不刷新指定点，因此玩家可以选择用出千追目标。
  - `牌谱树`：出手或小关开始指定一个已有牌型大类，例如三同、顺子、全奇、全偶；最终牌型命中时直接成长一次。四同、五同、六同可视为命中三同目标。
  - `冠层树`：本骰掷出自身当前面组最高面时直接成长一次；不使用“同时也是本次实体骰最高有效点”的强版本。
  - `修枝树`：本骰被确认出千改点后，如果新结果相对旧结果有明确改善，或帮助形成目标牌型、完成通关，则直接成长一次。
- 后置规则卡边界：
  - `年轮树`：每次真实结算结束时直接成长一次；每次出手最多触发一次，不再按小关限一次。因为稳定性高，生产包应通过章节、价格或低权重控制强度。
  - `肥料树`：每次真实结算结束时读取当前钱包金币存量，按标准金币步长直接换算成长次数，默认公式为 `floor(当前钱包金币 / interest_gold_step)`；不设置单次成长次数上限。它不创造金币、不消费金币，也不读取“金币变化次数”，而是奖励玩家保留本金。当前 F013 正式基线为 `starting_gold = 8`。
  - `根系树`：左邻或右邻任一实体骰在本次真实结算中触发自身类型效果时，根系树直接成长一次，每次出手最多触发一次。它不再按本骰奇偶决定方向，也不要求玩家判断左读还是右读。首版内部只判断相邻骰本次是否产生了明确的类型触发标签；基础骰、纯面组限制、纯被动概率分布和材质触发不计入。
  - 根系触发标签判定：只有相邻实体骰在真实结算中实际产生了可见类型事件，才算“效果触发”。可计入事件包括金币实际入账或支付、临时小骰生成、自动重摇实际发生、成长入档、出千专属收益、条件型单骰分替换或加分、牌型借位或补全实际被采用、赌徒爆发或归零等。不可计入事件包括基础点数计分、主牌型倍率、基础骰无效果、奇数/偶数骰的纯面组限制、双倍这类无条件常驻修正、材质触发、预览估算和未被采用的候选牌型。
  - 根系结算顺序：先收集本次所有非根系骰的类型触发标签，再让根系树读取相邻标签并直接成长。根系树由此产生的成长不再作为其它根系树的触发来源，避免根系互相连锁。
- UI 表达建议：
  - 触发短标签优先使用 `籽中成长`、`谱成成长`、`冠顶成长`、`修枝成长`、`年轮成长`、`肥料成长 xN`、`根:邻触发` 这类短语。
  - 不显示 `芽 X/3` 或成长进度条；真实成长时显示 `成长入档` 并展示面组变化。
  - 运行时主界面不放长解释，详细原因放在骰子详情或结算日志。
